# UI 原型调整沟通工作规则

**版本:** 1.0 | **生效日期:** 2026-06-17 | **适用范围:** `_ui-prototype/` 目录下全部 `.html` 原型文件

---

## 一、核心原则

> 仅对 UI 原型（`_ui-prototype/` 文件夹）进行展示层修改，**不触及任何项目系统文件**。

### 修改边界

| 允许修改 | 禁止修改 |
|:---------|:---------|
| `_ui-prototype/*.html` — 原型页面 HTML/CSS/JS | `backend/` — 后端 Python 代码、路由、数据库 |
| `_ui-prototype/assets/*.css` — 原型公共样式 | `frontend/vue-project/` — Vue 前端工程化代码 |
| — | `app/` `models/` `services/` 等后端模块 |
| — | `database/` 数据库迁移/脚本 |
| — | `requirements.txt` `package.json` 等依赖配置 |
| — | `.env` `config.py` 等环境/配置文件 |

## 二、沟通与记录标准

### 2.1 每一轮沟通必须执行的步骤

```
用户提出需求
    ↓
第①步：明确修改范围 → 仅 _ui-prototype/ 内的哪些文件
第②步：确认修改内容 → 功能变更 / 布局调整 / 样式优化
第③步：实施修改       → 仅操作允许修改的文件
第④步：浏览器验证     → 截图/检查/确认结果
第⑤步：更新 166 号文档 → 记录 UI 功能变更和展示形式
第⑥步：更新 171 号文档 → 同步前端功能对后端 API 的依赖变化（如有）
第⑦步：提交 git       → 确保可恢复
```

### 2.2 沟通记录格式

每一回合的沟通记录（`001-沟通记录/YYYY-MM-DD.md`）必须包含：

```markdown
### ### NNN - 简要标题

**用户输入：** （原文或精准摘要）

**AI执行内容：**
（分步描述修改了什么、为什么修改、怎么修改的）

**涉及文件：** `_ui-prototype/xxx.html`
```

- 标注状态变更（待办 → 已完成 / 已废弃）

**禁止修改：** 与本次调整无关的任何章节和内容。

## 三、技术约束

### 3.1 原型文件技术栈

| 技术 | 用途 | 备注 |
|:-----|:------|:------|
| 纯 HTML5 | 页面结构 | 无框架，直接浏览器打开 |
| CSS3 | 样式布局 | 含 `assets/base.css` 公共样式 |
| 原生 JavaScript (ES5/ES6) | 交互逻辑 | 含 ECharts 图表 |
| ECharts 5.4.3 | K线/副图图表 | CDN 加载 |

### 3.2 ECharts 架构约束（2026-06-17 v3）

当前 `indicator-ide.html` 采用 **单实例 + 多grid + graphic + DOM标签** 架构：

- 1 个 ECharts 实例，N 个 grid（grid[0]=主图，grid[1..N]=副图）
- 十字光标：graphic line（cv竖线/ch横线），z:9998
- 标签文字：DOM div（#cpl-dom/#cdl-dom），z:10000，绝对定位
- 副图数值：`updateSubHeaders()` DOM 标题栏更新
- dataZoom：单一 slider，`xAxisIndex` 动态关联所有 grid

### 3.3 数据模拟

- 原型使用模拟数据（硬编码 JS 数组），不依赖真实后端 API
- 模拟数据需保证展示完整性（含边界情况：空心、空值、极值）
- 数据量控制在合理范围（如 750 根 K 线）

### 3.4 兼容性

- 视口宽度 ≥ 1200px（原型为 1440px × 900px 设计）
- 浏览器：Chrome / Edge 最新版
- 无需考虑 IE / 移动端适配

## 四、Git 提交规范

### 4.1 提交时机

每完成一**轮**沟通的修改和文档更新后，立即提交。

### 4.2 提交粒度

| 内容 | 提交策略 |
|:-----|:---------|
| 原型文件修改（.html/.css）| 必须纳入提交 |
| 166 号文档更新 | 必须纳入提交 |
| 171 号文档更新 | 如有变更则纳入提交 |
| 沟通记录更新 | 必须纳入提交 |
| 其他无关文件 | **禁止**纳入提交 |

### 4.3 提交信息格式

```
# 前导说明：UI原型调整 仅限 _ui-prototype/ 和 002-方案存档/

修改内容: （简要描述本轮做了什么）
- 功能/布局变更说明
- 涉及文件列表

Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>
```

### 4.4 可恢复性保障

- 每次提交前确认 `git status` 只包含应修改的文件
- 若有需要保留但当前不用的原始版本，创建 `.crosshair-backup` 等后缀备份文件（已在 `.gitignore` 中保留）

## 六、验证标准

每次修改后必须验证以下内容：

1. **功能正确性** — 修改的功能按预期工作
2. **视觉完整性** — 无遮挡、无错位、无残留
3. **布局响应性** — 在当前视口下布局正确
4. **交互流畅性** — 鼠标/键盘操作无异常
5. **文档一致性** — 166/171 文档描述与当前展示一致
