### 002 — 复盘报告框架设计 + 三文档更新

**沟通主题：** 报告生成框架完整设计——输入输出规范、五层分层诊断模型、防AI幻觉设计、深度分析与报告联动、166/171同步更新

**关键设计决策确认：**
1. ✅ 系统不自修正 → 报告可下载（JSON）→ 开发平台大模型审阅 → 人工决定修订
2. ✅ 深度分析应聚合报告的共性诊断结论 → 新增"报告诊断汇总"第6项
3. ✅ 报告只输出诊断事实，不输出系统修订指令 → 防AI幻觉
4. ✅ 基于策略组合五层运行方式设计分析维度（非通用心理评估）

**核心设计要点：**

**五层诊断模型：**
```
Layer 1 — 多维度现状识别（缠论/量价/筹码/情绪/多因子/主力——6个维度并行判断）
Layer 2 — 交叉验证链（4条：主力行为/结构位置/外部环境/共振）
Layer 3 — AI仲裁（DeepSeek V4 Layer 2：仲裁方向 + 情景推演A/B/C）
Layer 4 — 执行条件层（入场条件/出场条件/止损位/目标位）
Layer 5 — 资金与风控执行层（仓位大小/止损距离/止盈层级/手续费）
```

**九维分析：** 方向判断/维度间冲突/交叉验证有效性/仲裁方向准确度/情景推演匹配度/入场条件/退出条件/止损止盈/资金管理

**防AI幻觉五项设计：** ① 分层限制不跨层推测 ② 只诊断不开方 ③ 数据完整性标注 ④ 诊断置信度标注 ⑤ 结构化枚举值优先

**报告与深度分析的关联：** 每次报告生成时将 diagnostic_summary.root_layer 等关键诊断字段写入 report_diagnosis_collection（localStorage 全局数组），深度分析视图从该集合聚合：按root_layer分布柱状 + 按诊断事实频率排序 + 按exit_reason交叉表

**下载规范：** JSON格式，文件名 RR-{日期}-{股票代码}-{序号}.json，含诊断数据 + ReviewUnit原始输入数据完整副本

**涉及文件更新：**
- `002-方案存档/203-复盘中心原型重构方案v2-量化实盘标准重设计.md` — 新增§4.9完整框架（9个子节）+ §4.7第6项诊断汇总
