### Docker部署评估 + 无Docker替代方案完整实施

**沟通主题：** Docker 构建障碍根因分析 + Windows 无 Docker 部署方案完整实施

**关键发现：**

1. **Docker 构建的真正障碍**不是之前以为的 pypi.org，而是 **Docker Hub (registry-1.docker.io) 在中国大陆不可达**。`python:3.11-slim` 和 `node:20-alpine` 基础镜像拉取超时。配置的镜像加速器（docker.1ms.run、docker.xuanyuan.me）也返回 429 限流或超时。
2. **系统对 Docker 无硬依赖**：SQLite 替代 PostgreSQL（已有），DuckDB 替代 Redis（已有），Flask 可直接托管前端 SPA。
3. **设备迁移场景下 Docker 反而增加复杂度**：用户需要在新机器上装 Docker Desktop + 解决 Docker Hub 连通性。

**实施内容：**

**Flask 新增前端 SPA 托管：**
- `backend/app/__init__.py`：新增静态文件服务 + SPA fallback 路由
- 自动检测 `frontend/vue-project/dist/` 目录，存在即启用
- 非 API 路由全部回退到 `index.html`，保持 Vue Router 正常工作
- 不影响已有 Nginx/Vite 模式（检测到 dist 才启用）

**Windows 启动脚本重写：**
- `start.bat`：无 Docker 版本，自动处理 Python 检查、虚拟环境创建、依赖安装、前端构建、服务启动
- `stop.bat`：无 Docker 版本，优雅停止后端进程

**运维手册重写 §5：**
- 废弃纯 Docker 部署说明
- 新增"三方案评估对比表"（裸跑 ⭐⭐⭐⭐⭐ / Docker ⭐⭐⭐ / WSL ⭐⭐）
- 无 Docker 模式完整启动/停止/迁移步骤
- Docker 保留为备选方案 B

**方案存档 129 修订：**
- 原方案追加 §7 修订说明
- 裸跑方案完整替代实施步骤
- 迁移时只需复制目录 + 安装 Python

**结论：**
```
推荐方案：裸跑（无 Docker）
迁移成本：复制目录 + 安装 Python 3.10+
日常使用：双击 start.bat
访问地址：http://localhost:5001
```
