# DarwinRiskFilter — 第一层策略排布合理性评估与优化方案

## 评估基准

基于知识库《我从达尔文那里学到的投资知识》提出的四层递进剔除方案，对照当前系统实现进行逐层盘点。

---

## 知识库标准架构

| 层级 | 名称 | 内容 | 否决规则 |
|------|------|------|----------|
| L1 | 硬性否决项（六大禁区） | 骗子企业 / 扭亏困境 / 高杠杆 / 并购瘾 / 快速迭代行业 / 利益错位所有者 | 一票否决 |
| L2 | 基本面量化剔除 | ROCE<15% / 利润不稳定 / 稳健性检查5项 | 量化阈值否决 |
| L3 | 技术面与筹码风险剔除 | 主力出货 / 趋势衰竭 / 情绪过热 / 资金外流 / 假突破 | 2个及以上信号排除 |
| L4 | 行为金融与情绪陷阱剔除 | 这次不一样 / 群体狂热 / 锚定效应 / 信息茧房 | 补充判断 |

操作纪律：一票否决制（L1） → 冰箱里没有糖果（剔除后不可重新纳入）

## 关键缺口评估

### 严重缺失（6项 — 知识库L1+L2核心）

1. **骗子企业检测**（L1-①）：无财报造假识别、无审计机构频繁变更检测、无大股东资金占用检测
2. **扭亏困境检测**（L1-②）：`_filter_continuous_loss` 是占位符
3. **高杠杆检测**（L1-③）：无资产负债率>70%检测、无有息负债/EBITDA>5倍检测
4. **并购瘾检测**（L1-④）：无商誉/总资产比率检测、无并购频率检测
5. **ROCE计算与筛选**（L2核心）：完全缺失，这是达尔文方法论的第一筛选标准
6. **稳健性检查**（L2配套）：完全缺失（客户集中度、管理层稳定性、现金流覆盖等5项）

### 中度缺失（2项 — 知识库L3）

7. **技术面风险信号集成**（L3）：主力出货、趋势衰竭、假突破等检测未接入L1层
8. **情绪与资金流向**（L3）：龙虎榜、DDE大单净额等未接入

### 轻度缺失（2项 — 知识库L4）

9. **行为金融检查**：完全缺失，但优先级低于L1-L3
10. **操作纪律**：无"冰箱里没有糖果"规则的代码级约束

### 架构冗余（2项）

11. **双重DarwinRiskFilter**：两个类同名不同实现，`chanlun_strategy.py`中的存根会造成误导
12. **重复的流动性过滤**：ChipPreFilter和DarwinRiskFilter各自实现了低流动性过滤，规则接近但逻辑独立

## 优化方案（分三档优先级）

### 🔴 P0 — 必须修复（L1否决项占位符）

| 项目 | 工作量 | 依赖 | 说明 |
|------|--------|------|------|
| `_filter_continuous_loss` 实现 | 1~2d | 需接入 Tushare `fina_indicator` | 获取近5年净利润数据，连续2年亏损或净利润<0即剔除 |
| `_filter_high_valuation` 重构为 ROCE筛选 | 2~3d | 需 Tushare 财报数据（EBIT、净营运资本、净固定资产） | 按知识库公式：ROCE = EBIT/(净营运资本+净固定资产)，阈值15% |
| 高杠杆检测（新规则） | 1d | 需 Tushare `fina_indicator` 资产负债率 | 资产负债率>70%（非金融）+ 有息负债/EBITDA>5倍 |
| 利益错位所有者（新规则） | 1d | 需东方财富/大智慧 API | 大股东减持比例、股权质押比例 |
| 合并/移除 chanlun_strategy.py 中的存根 | 0.5d | 无 | 删除或重定向到 screener.DarwinRiskFilter |

## 推荐实施路径

**Phase 1（2~3天） — 止住L1漏洞**
1. 实现 `_filter_continuous_loss`（接入净利润数据）
2. 将 `_filter_high_valuation` 替换为 ROCE 初步筛选
3. 删除 `chanlun_strategy.py` 中的存根
4. 统一流动性过滤规则（去重）

**Phase 2（3~5天） — 覆盖知识库L1全部六大禁区**
5. 实现高杠杆检测规则
6. 实现利益错位所有者检测（减持+质押）
7. 实现并购瘾检测（商誉+并购频率）
8. 实现稳健性检查5项

**Phase 3（滚动） — L3+L4 进阶**
9. 对接技术面信号（筹码/缠论）
10. 行为金融检查
11. 操作纪律系统化

## 架构建议

当前 ChipPreFilter（大盘环境+新股+流动性）和 DarwinRiskFilter（六大禁区+ROCE）定位有重叠：

```
建议统一架构：

Layer 0: 大盘环境过滤  (ChipPreFilter → market+circuit_breaker)
Layer 1: 硬性否决      (DarwinRiskFilter — 六大禁区+ROCE)
Layer 2: 流动性+新股   (ChipPreFilter — eligibility+liquidity)
Layer 3: 技术面风险    (筹码分布+缠论信号)
Layer 4: 行为金融      (情绪检测)
```

建议将 ChipPreFilter 和 DarwinRiskFilter 整合到一份统一的过滤链中，按逻辑顺序排列而非按两个独立类管理。

---

## 结论

当前第一层（DarwinRiskFilter）的 **实际过滤能力约为知识库标准的15~20%**。4个过滤器中2个是占位符，存活的`_filter_st`和`_filter_low_liquidity`也只能覆盖六大禁区的极小部分。系统架构上还存在两个同名的DarwinRiskFilter产生混淆。

建议优先止住L1的占位符漏洞（Phase 1），再用2周时间将六大禁区和ROCE筛选全覆盖（Phase 2），届时知识库的"先避雷，再求胜"策略才能真正落地。
