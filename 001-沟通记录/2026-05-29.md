### 已完成变更

#### 1. 重写 screener.py 路由（核心变更）
- **文件**: `backend/app/routes/screener.py` ✅ 重写
- **变更**: 6 个端点从 Mock 数据改为真实计算引擎
- 新增 `/api/v3/screener/stats` 统计状态端点

| 端点 | 之前 | 之后 |
|------|------|------|
| `POST /run` | 返回7只固定股票 | 加载真实股票列表 → L1风险剔除 → L2 ChipScorer评分 → L3验证 |
| `POST /layer1` | 固定 3500/1500 | DarwinRiskFilter 真实过滤（ST/流动性/数据天数） |
| `POST /layer2` | 固定3只股票 | ChipScorer 真实评分，按分数排序 |
| `POST /layer3` | 返回空列表 | 真实验证 + 提取 ASR/集中度/量比/RSI 指标 |
| `GET /fusion-config` | 固定权重 | 可读写配置（持久化预留） |
| `GET /params` | 固定参数 | 含筛选器详细参数说明 |

**数据流程**:
```
DataManager.get_stock_list() → Stock 列表 (来自 PostgreSQL/Tushare)
  → load_stock_data_batch() → DuckDB 缓存日线数据
    → DarwinRiskFilter.filter()  L1 过滤
      → ChipScorer.score()  L2 评分 + 排序
        → 策略验证 + 指标提取  L3 输出
          → 结果缓存 1 小时
```

- **文件**: `backend/app/services/signal_computation_service.py` ✅ 新建
- **功能**: 当数据库无缓存策略输出时，从真实 K 线数据实时计算

| 信号维度 | 计算逻辑 | 输出字段 |
|---------|---------|---------|
| L2 筹码主力分析 | ChipScorer 评分 + 价格位置 + 量比分析 | 信号方向/置信度/入场区间/止损/目标/证据列表 |
| L3 缠论信号 | 分型判断 + MACD 背离检测 | 买卖点分类/背驰确认/置信度 |
| L3 因子评分 | 动量/量价/波动率/RSI 多因子加权 | 因子明细/综合评分 |

#### 3. 更新 strategy.py 路由
- **文件**: `backend/app/routes/strategy.py` ✅ 更新
- **变更**: `/api/v2/strategy/outputs` 增加实时计算 fallback

  - 无缓存 → 调用 SignalComputationService 实时计算
  - 返回标记 `computed: true` 区分实时计算与缓存数据

## 005 - A股实时行情数据源对比研究

**沟通主题**：调研A股实时行情数据获取渠道并对比分析

**产出物**：`002-方案存档/127-A股实时行情数据源对比研究报告.md`

**调研结论**：

**核心矛盾**：当前系统使用的 Tushare Pro 不支持实时行情，AkShare 懒加载+ HTTP轮询延迟3-5s

**推荐方案**（两步走）： 
1. 短期（1天）：接入东方财富 WebSocket → 1-3s延迟，免费，替代AkShare轮询
2. 中期（2-3天）：启用已预留的 QMT (`qmt_provider.py`) → Tick级，开户免费，需Windows

**11个数据源对比矩阵**见报告第7章

**不需要** Wind/Choice — 个人项目成本过高(3000-8万/年)
