### 019 — 通知保存策略（B+C组合）+ 周期MD报告方案确认

**沟通主题：** 对通知记录保存策略进行评估比较方案A（滚动窗口）/B（时间分层）/C（用户自配置），确认B+C组合方案，增加周期MD报告和历史归档管理

**决策确认：**

| 决策项 | 结论 |
|:-------|:------|
| 保留策略 | ✅ **方案B（时间分层）+ 方案C（用户自配置）组合** — L1全量30天→L2摘要31~90天→L3聚合90天+→L4收藏永久，所有参数用户可调 |
| 报告推送时机 | ✅ 每周/每月最后一个交易日**闭市后**（15:00-15:30，随日终评估一起执行），不设独立定时器，解决用户周末/月初非开机场景 |
| 报告格式 | ✅ MD格式，不提供CSV/JSON下载，扩展为周期性报告而非仅"导出"功能 |
| 报告维度 | ✅ 周报：概览+规则排行TOP5+股票排行TOP5+状态变化+时段分析；月报额外增加月度趋势+规则有效性环比+系统健康+建议 |
| 历史归档 | ✅ 设独立归档页（报告列表浏览 + _reports/目录MD原文件 + SQLite元数据表）|

**三种方案对比：**

| 方案 | 核心逻辑 | 实现复杂度 | 评语 |
|:-----|:---------|:----------:|:-----|
| A — 滚动窗口 | 保留最近N条，超出自动删最旧 | ⭐ 极简 | 功能单薄，不考虑 |
| B — 时间分层 | L1全量→L2摘要→L3聚合逐级降级 | ⭐⭐⭐ 中等 | 骨架优良，历史趋势可追溯 |
| C — 用户自配置 | 默认90天+手动清理+导出兜底 | ⭐ 极简 | 骨架偏薄，但哲学一致 |

**最终方案：B+C组合（B的分层结构为默认模板 + C的配置弹性）+ 周期报告**

### 020 — 通知交互与规则联动全量确认（5种规则类型闭环 + 隐式确认 + 操作热度 + 复盘联动）

**沟通主题：** 确定通知交互完整闭环：从用户价值出发的规则分类（5类）→ 每类的通知卡片操作设计 → 隐式确认机制 → 操作热度算法替代点赞 → 复盘联动 → 新标签页跳转策略

**关键讨论路径：**

1. **规则分类方式**：从技术领域（条件库的买卖信号/缠论/筹码等）→ 改为**从用户价值出发**分类，用户确认5种类型
2. **用户视角模拟**：对每种类型的通知模拟用户接收后的可能操作，设计上下文敏感的操作按钮
3. **系统能力整合**：回顾166号文档的系统能力（indicator-ide/策略分析/多角色投研/复盘中心/加自选），确定各类型通知可引导的功能及按钮
4. **三问确认**：（1）规则点赞→改为操作热度算法；（2）内联编辑可行性→纪律执行采用内联展开表单；（3）异动追踪❓→悬浮tooltip

**决策确认：**

| 决策项 | 结论 |
|:-------|:------|
| 规则分类基准 | ✅ **从用户价值出发**，5种类型：📈发现机会("我想抓住上涨") / 🛡️规避风险("帮我盯着别出事") / ⚡异动追踪("这只股票不对劲") / 📋状态监控("我的股票现在什么情况") / 🎯纪律执行("帮我管住手") |
| 创建流程 | ✅ 类型选择作为规则创建第1步 → 系统自动配置推送策略/冷静期/推荐条件 |
| 📈发现机会 | ✅ 按钮：看行情(新标签) + 策略分析(AI判读→改名) + 多角色投研 + 加自选 + 误报。**隐式确认生效**，不设独立"信号有效"按钮 |
| 🛡️规避风险 | ✅ **特殊微信推送**：1次→15min未处理→2次→停止。按钮：看行情 + 已处理 + 加自选。✅已处理含提前处理场景 |
| ⚡异动追踪 | ✅ ❓→悬浮tooltip（来自条件库user_description）。按钮：看行情 + 策略分析 + 多角色 + 加自选 |
| 📋状态监控 | ✅ 最轻量。按钮：看行情 + 已处理 |
| 🎯纪律执行 | ✅ 按钮：看行情 + 已处理 + ✏️修改参数(内联展开)。简单参数直接内联改，复杂参数链接完整编辑器 |
| 隐式确认 | ✅ **任何有意义操作=对触发事件的认可**。看行情/策略分析/多角色/加自选/已处理→认可，误报→负反馈 |

| "已处理"统一 | ✅ 将分散的已收悉/已处理/已执行/已知悉统一为"已处理" |
| 看行情跳转 | ✅ **新标签页 window.open()**，保留用户当前操作进度 |
