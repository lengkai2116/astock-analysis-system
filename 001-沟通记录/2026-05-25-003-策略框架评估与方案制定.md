# 2026-05-25 沟通纪要

## 003-策略框架评估与方案制定

### 沟通背景
用户提出了完整的策略框架需求，分为两大层次：
1. 股票筛选层：剔除风险股，识别主力资金关注的股票
2. 策略验证层：多策略验证、AI分析、仓位管理

核心问题：将筹码分布策略调整为选股策略是否可行？

### 讨论内容

#### 1. 策略框架架构设计

**两层架构**：
- **第一层：股票筛选层**
  - 风险剔除（达尔文策略）
  - 基本面过滤
  - 主力资金识别（筹码分布策略改造）
  
- **第二层：策略验证层**
  - 多策略验证（缠论、量化因子、用户自建因子）
  - AI专项分析
  - 买卖机会、仓位管理、风险管理

**投资者交易风格分类**：
| 风格 | 持仓周期 | 特点 |
|------|---------|------|
| 长线投资 | >1年 | 价值投资 |
| 中线投资 | 3-12个月 | 趋势投资 |
| 短线交易 | 1-30天 | 技术面驱动 |
| 日内交易 | 1天内 | 高频交易 |
| 隔日交易 | 1-2天 | 超短线 |

#### 2. 筹码分布策略改造可行性评估

**结论**：✅ 完全可行

**改造思路**：
- 从单一股票分析 → 股票池批量扫描
- 从阶段识别 → 筛选特定阶段股票
- 从信号生成 → 选股评分排序

**主力阶段选股逻辑**：
| 阶段 | 选股价值 | 说明 |
|------|---------|------|
| 建仓期 | ⭐⭐⭐⭐⭐ | 最高优先级，主力低位吸筹 |
| 洗盘末期 | ⭐⭐⭐⭐ | 高优先级，即将拉升 |
| 拉升初期 | ⭐⭐⭐ | 中优先级 |
| 出货期 | ⭐ | 剔除 |
| 下跌期 | ⭐ | 剔除 |

**选股评分体系**：
- ASR指标（30%）
- 筹码集中度（20%）
- 获利率（20%）
- 成交量（15%）
- RSI指标（15%）

#### 3. 与达尔文策略的协同

**分工明确**：
- 达尔文策略：守门员，剔除风险股和垃圾股
- 筹码分布策略：伯乐，选出有主力关注的股票

**筛选流程**：
```
全市场股票（5000只）
        ↓
达尔文策略风险剔除（3000只）
        ↓
基本面过滤（1000只）
        ↓
筹码分布主力识别（50-200只）
        ↓
精选股票池
```

#### 4. 实施路线图

| 阶段 | 时间 | 任务 |
|------|------|------|
| Phase 1 | 1-2周 | 筹码分布策略选股改造 |
| Phase 2 | 1周 | 整合到达尔文策略框架 |
| Phase 3 | 2周 | 策略验证层开发 |
| Phase 4 | 1周 | AI分析模块开发 |
| Phase 5 | 1周 | 回测验证 |

### 关键结论

1. **筹码分布策略改造可行**：技术上、数据上、算法上均具备可行性
2. **评分体系设计合理**：综合考虑ASR、集中度、获利率等指标
3. **阶段优先级明确**：建仓期和洗盘末期为主要选股目标
4. **与达尔文策略协同**：形成完整的两层筛选体系

### 下一步建议
1. 优先实现主力资金选股功能
2. 建立科学的选股评分模型
3. 区分阶段优先级，剔除出货期股票
4. 考虑性能优化（批量处理、缓存）
