# A股股票分析系统前端 UI 设计优化详细方案

- 编制日期：2026-05-24
- 适用项目：A股股票分析决策支持系统
- 关联报告：`QuantDinger对标能力升级优化建议报告.md`
- 方案定位：面向“策略输出与投研决策支持”的前端体验升级，不涉及自动交易实施。

## 一、设计目标

当前系统已经具备 Vue 前端、指标 IDE、回测页面、自选监控、AI 分析、因子管理等模块雏形。下一阶段前端优化不应追求“交易终端化”，而应围绕以下目标展开：

1. 提升策略研究效率：用户能快速从行情、指标、信号、回测中形成策略观点。
2. 提升决策可信度：每个策略建议都应有数据依据、触发原因、风险说明和回测验证。
3. 明确系统边界：系统只输出策略建议、风险提示和研究报告，不提供自动下单或实盘执行。
4. 建立统一设计语言：颜色、图表、表格、标签、风险等级、策略信号保持一致。
5. 强化 A 股语义：红涨绿跌、涨跌停、T+1、行业板块、资金流、政策事件等应成为界面的一等信息。

## 二、产品定位与 UI 原则

### 2.1 产品定位

本系统应定位为：

> A 股策略研究、信号解释、回测验证和投研报告生成平台。

不是：

- 自动交易系统
- 券商下单终端
- 实盘托管平台
- AI 自动调仓系统

### 2.2 文案边界

前端应统一避免交易执行语义。

| 不建议文案 | 推荐文案 |
|------------|----------|
| 买入 | 看多信号 / 关注信号 |
| 卖出 | 风险退出信号 / 减弱信号 |
| 下单 | 生成策略建议 |
| 自动交易 | 策略验证 |
| 实盘运行 | 模拟验证 |
| 交易记录 | 回测事件 / 模拟事件 |
| 持仓 | 模拟组合 / 策略组合 |

所有策略结果页面建议固定展示声明：

> 本系统输出为策略研究与决策支持信息，不连接券商交易接口，不自动下单，不执行真实资金调仓。所有结论仅供研究参考。

## 三、整体信息架构优化

建议将系统主导航重组为 6 个一级模块：

| 模块 | 核心作用 | 主要页面 |
|------|----------|----------|
| 市场总览 | 快速判断市场环境 | 指数、行业、涨跌分布、成交额、风险雷达 |
| 自选监控 | 跟踪重点股票变化 | 自选列表、信号变化、风险提示、标签管理 |
| 指标 IDE | 策略研究主工作台 | K线、指标、代码、信号解释、策略建议 |
| 策略验证 | 验证策略可靠性 | 回测、参数优化、样本外验证、策略对比 |
| 因子研究 | 因子筛选与组合 | 因子库、因子评估、因子组合、IC/IR |
| AI 投研报告 | 生成结构化报告 | 单股报告、策略报告、多角色分析、导出 |

建议导航结构：

```text
首页 / 市场总览
自选监控
指标IDE
策略验证
因子研究
AI投研报告
系统设置
```

## 四、视觉设计规范

### 4.1 视觉方向

建议采用“专业投研终端”风格：

- 信息密度高，但层级清晰。
- 深色为主，但避免纯黑大面积铺底。
- 图表和表格优先，不做营销型大标题。
- 风险信息要醒目，辅助信息要克制。
- A 股上涨使用红色，下跌使用绿色。

### 4.2 色彩建议

| 用途 | 建议颜色 |
|------|----------|
| 页面背景 | `#0f172a` 或 `#111827` |
| 主面板背景 | `#161b22` |
| 次级面板背景 | `#1f2937` |
| 边框 | `#2f3a4a` |
| 主文字 | `#e5e7eb` |
| 次文字 | `#9ca3af` |
| A股上涨 | `#ef4444` |
| A股下跌 | `#22c55e` |
| 中性/观察 | `#f59e0b` |
| 风险提示 | `#fb7185` |
| 主操作色 | `#2563eb` |

### 4.3 信号标签规范

| 信号 | 含义 | 颜色 | UI 文案 |
|------|------|------|---------|
| BULLISH | 偏多 | 红色 | 看多 |
| BEARISH | 偏空 | 绿色 | 风险 |
| WATCH | 观察 | 黄色 | 关注 |
| NEUTRAL | 中性 | 灰色 | 中性 |
| INVALID | 失效 | 暗红 | 失效 |

### 4.4 图表规范

K线与指标页面应统一：

- 红涨绿跌。
- 主图展示 K线、均线、BOLL、风险线、目标区间。
- 副图展示 VOL、MACD、RSI、KDJ、资金流。
- 信号点使用小型标记，不遮挡K线。
- 鼠标悬停展示 OHLCV、涨跌幅、成交额、换手率。
- 图表右侧保持价格轴，底部保持时间轴。

## 五、关键页面优化方案

## 5.1 市场总览

### 页面目标

让用户在 30 秒内判断当前市场是否适合运行策略。

### 推荐布局

```text
顶部：指数卡片
上证指数 / 深证成指 / 创业板指 / 沪深300 / 中证500

中部左：市场广度
上涨家数、下跌家数、涨停、跌停、成交额、量能变化

中部右：行业热力图
行业涨跌幅、资金流、强弱排名

底部左：策略环境评分
趋势环境、波动环境、流动性、风险偏好

底部右：今日风险雷达
放量下跌、破位、极端波动、指数背离
```

### 关键组件

- 指数行情卡
- 行业热力图
- 涨跌分布柱状图
- 市场环境评分卡
- 风险雷达列表

### 数据建议

- 指数行情
- 行业板块行情
- 全市场涨跌家数
- 涨跌停统计
- 成交额与换手率

## 5.2 自选监控

### 页面目标

帮助用户发现自选股中的“新变化”和“高优先级策略机会”。

### 推荐表格字段

| 字段 | 说明 |
|------|------|
| 股票 | 代码、名称、行业 |
| 最新价 | 当前价格 |
| 涨跌幅 | A股红涨绿跌 |
| 策略评分 | 0-100 |
| 信号状态 | 看多、风险、观察、中性 |
| 信号变化 | 新出现、增强、减弱、失效 |
| 风险线 | 跌破则策略失效 |
| 目标区间 | 策略观察目标 |
| 触发原因 | 最近主要信号 |
| 操作 | 查看图表、生成报告、加入验证 |

### 交互建议

- 支持按策略评分排序。
- 支持标签：短线观察、中线跟踪、风险警戒、策略验证中。
- 支持筛选：新信号、风险信号、高评分、行业。
- 点击股票进入指标 IDE，并自动加载该股票。
- 一键生成单股策略报告。

## 5.3 指标 IDE

### 页面目标

指标 IDE 应成为本项目的核心策略研究工作台。

### 推荐布局

```text
左侧：研究侧栏
  Tab 1 策略模板
  Tab 2 指标参数
  Tab 3 代码编辑
  Tab 4 质量检查

中间：K线与指标图表
  顶部工具栏：股票、周期、指标、复权、日期范围
  主图：K线 + 主图指标 + 信号点 + 风险线
  副图：VOL / MACD / RSI / KDJ / 资金流

右侧：策略建议面板
  当前信号
  置信度
  触发原因
  风险线
  目标区间
  建议仓位
  失效条件

底部：回测摘要
  收益、回撤、胜率、夏普、样本外表现
```

### 重点优化

1. 指标实例管理
   - 同一指标可添加多个实例。
   - 每个实例可设置参数、颜色、显示/隐藏、删除。
   - 激活指标以芯片形式展示。

2. 信号解释
   - 点击信号点展示触发原因。
   - 展示关联指标值，例如 MACD、RSI、成交量变化。
   - 显示信号置信度和历史胜率。

3. 策略建议面板
   - 不只显示“买卖”，而是输出策略建议。
   - 必须包含风险线和失效条件。
   - 必须显示数据依据。

4. 代码质量检查
   - 检查未来函数。
   - 检查缺失输出字段。
   - 检查空值处理。
   - 检查是否误用交易执行函数。

5. 一键联动
   - 当前指标参数一键进入回测。
   - 当前策略一键生成投研报告。
   - 当前信号一键加入自选监控。

## 5.4 策略验证页面

### 页面目标

回答一个核心问题：这个策略是否值得作为决策参考？

### 推荐布局

```text
左侧：回测配置
  股票池
  时间区间
  策略模板
  参数
  交易假设
  基准指数

右侧上：核心指标
  总收益、年化、最大回撤、夏普、胜率、盈亏比

右侧中：图表
  权益曲线 vs 基准
  回撤曲线
  月度收益热力图
  参数敏感性热力图

右侧下：报告
  策略逻辑
  回测假设
  优势
  风险
  失效条件
```

### 必须新增的回测假设展示

- 手续费
- 印花税
- 滑点
- T+1
- 涨跌停处理
- 停牌处理
- 复权方式
- 样本内/样本外切分

### 结果展示建议

不要只展示收益率，应同时展示：

- 最大回撤区间
- 连续亏损次数
- 单笔最大亏损
- 策略换手率
- 与基准相关性
- 不同市场环境下表现
- 参数稳定性

## 5.5 因子研究页面

### 页面目标

支持用户从因子角度构建选股逻辑。

### 推荐功能

- 因子库列表：Alpha101、GTJA、Qlib158、A股特色因子。
- 因子分类：动量、反转、估值、质量、成长、成交量、波动率。
- 因子评估：IC、IR、Rank IC、换手率、分组收益。
- 因子组合：权重配置、相关性检查、组合评分。
- 因子覆盖度：哪些股票数据不足。

### 推荐视图

- 因子列表表格
- 因子详情抽屉
- IC 时间序列图
- 分组收益柱状图
- 因子相关性矩阵
- 因子组合配置器

## 5.6 AI 投研报告页面

### 页面目标

从“聊天页面”升级为“结构化投研报告生成器”。

### 推荐布局

```text
顶部：报告对象
  股票 / 股票池 / 策略 / 因子组合

左侧：报告目录
  摘要
  技术面
  基本面
  资金面
  策略验证
  风险提示
  最终建议

中间：报告正文
  结构化内容
  图表
  数据引用

右侧：结论卡
  策略评分
  信号状态
  置信度
  风险等级
  关注区间
  风险线
```

### AI 内容要求

每条 AI 结论应包含：

- 结论
- 数据依据
- 置信度
- 风险说明
- 是否需要人工复核

禁止让 AI 直接输出：

- 自动买入
- 自动卖出
- 立即下单
- 自动调仓

推荐输出：

- 建议关注
- 策略偏多
- 风险升高
- 需要等待确认
- 信号失效

## 六、组件级设计规范

### 6.1 策略评分卡

字段：

- 策略评分：0-100
- 当前状态：看多/风险/观察/中性
- 置信度
- 最近触发时间
- 关键原因

### 6.2 风险提示卡

字段：

- 风险等级：低/中/高
- 风险类型：趋势破位、量能异常、波动放大、回撤扩大
- 触发条件
- 应对建议

### 6.3 信号解释抽屉

字段：

- 信号名称
- 触发时间
- 触发价格
- 指标值
- 历史同类信号胜率
- 风险线
- 失效条件

### 6.4 报告导出按钮

建议导出格式：

- Markdown
- HTML
- PDF
- JSON

按钮文案：

- 生成报告
- 导出 Markdown
- 导出 PDF
- 保存到报告中心

## 七、交互优化建议

### 7.1 全局交互

- 页面切换保持当前股票上下文。
- 自选股、指标 IDE、回测、报告之间可互相跳转。
- 所有表格支持排序、筛选、列配置。
- 所有图表支持时间范围联动。
- 所有策略建议支持展开证据链。

### 7.2 状态反馈

统一状态：

- 加载中
- 数据不足
- 计算失败
- 无信号
- 信号已失效
- 回测完成
- 报告生成中

### 7.3 空状态设计

避免只显示空白。

示例：

- 自选股为空：提示添加关注股票。
- 回测无数据：提示检查数据同步和时间区间。
- 策略无信号：提示当前市场不满足策略条件。
- AI 报告失败：提示缺少必要数据或模型配置。

## 十、优先级建议

| 优先级 | 优化项 | 原因 |
|--------|--------|------|
| P0 | 文案与产品边界统一 | 避免系统被误解为自动交易工具 |

| P1 | AI 报告结构化 | 提升投研价值和可归档性 |

| P2 | 因子研究可视化 | 支撑进阶策略研究 |
| P2 | 报告中心 | 支撑长期沉淀 |

## 十一、验收标准

### 11.1 体验验收

- 用户能在 3 次点击内从自选股进入指标 IDE。
- 用户能在指标 IDE 中看到信号原因、风险线、目标区间。
- 用户能从任意策略建议一键进入回测验证。
- 用户能从回测结果一键生成报告。
- 页面中不出现自动交易、自动下单、实盘托管类误导文案。

### 11.2 视觉验收

- 红涨绿跌符合 A 股习惯。
- 深色主题层级清晰，不出现大面积纯黑。
- 图表、表格、标签颜色一致。
- 关键信号和风险提示清晰可见。
- 移动端或窄屏不出现文字严重重叠。

### 11.3 技术验收

- 前端构建通过。
- 主要页面接口异常时有清晰错误态。
- 响应结构统一适配 `success/data/message/meta`。
- 图表组件在 2000 根 K线内交互流畅。
- 报告导出内容包含免责声明。

## 十二、最终建议

前端优化应优先把系统从“功能堆叠”整理为“策略研究工作流”：

```text
市场环境判断
  → 自选股监控
  → 指标 IDE 研究
  → 策略验证
  → AI 解释
  → 报告输出
```

这条链路应贯穿所有页面和组件。QuantDinger 值得借鉴的是研究、策略、回测和 AI 协作能力；本项目需要进一步 A 股本地化，并明确保留在人类决策支持层，不进入自动交易执行层。
