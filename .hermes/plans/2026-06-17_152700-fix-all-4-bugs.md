# 综合修复方案：indicator-ide.html 全部4个Bug

> **For Hermes:** 按此方案任务逐项实现，经用户确认后方可修改代码。

**目标：** 解决 `_ui-prototype/indicator-ide.html` 中发现的全部 4 个 Bug：
1. 主图左侧Y轴数值不显示（canvas初始尺寸错误）
2. 键盘左右键到数据边界后不触发dataZoom滚动
3. 副图数值不显示（CSS选择器错误 + Y轴show:false）
4. 从副图右侧离开后十字光标残留

**技术栈：** ECharts 5.4.3, 纯 HTML/CSS/JS 原型

## Bug 1：主图左侧Y轴数值不显示

### 根因

侧栏折叠 IIFE（第 1053-1059 行）改变了 `main-content.marginLeft`（200px → 64px），但**未调用 `chart.resize()`**。ECharts canvas 初始绘制宽度为 **648px**（容器实际 784px），导致左侧 55px 网格内的轴标签被压缩错位。

### 任务 1.1：侧栏折叠后强制 resize 所有图表

**文件：** `_ui-prototype/indicator-ide.html`
**行号：** 1053-1059

**改动：** 在 IIFE 末尾加入 50ms 延迟的 `chart.resize()` + 子图 resize，与 `toggleSidebar()` 函数保持一致。

**代码：**
```javascript
// ── 首次加载默认收起 ──
(function(){
  var s = document.querySelector('.sidebar'), m = document.querySelector('.main-content');
  if(s) s.classList.add('collapsed');
  if(m){ m.classList.add('collapsed'); m.style.marginLeft = '64px'; }
  var btn = document.querySelector('.sidebar-toggle');
  if(btn) btn.textContent = '▶';
  // ★ 修复：布局变更后强制重绘所有 ECharts 实例
  setTimeout(function(){
    if(typeof chart !== 'undefined' && chart.resize) chart.resize();
    for(var k in subInstances){
      try{ subInstances[k].resize(); }catch(e){}
    }
  }, 50);
})();
```

### 任务 1.2：修复 closeLine 初始宽度

**行号：** 668-669

**当前代码：** `chart.setOption({graphic:[{... x2:chart.getWidth()-20 ...}]})` — 此时 chart.getWidth() 返回错误值 648px。

**方案：** 删除第 668-669 行的 closeLine 静态初始化。`updateCloseLine()` 已在 mousemove 回调中动态创建 closeLine（第 876-891 行），会自动使用 `chart.getWidth()` 的正确值。

## Bug 2：键盘到数据边界不触发 dataZoom 滚动

### 根因

键盘事件处理函数（第 962-977 行）仅做 `Math.max(0, Math.min(dates.length-1, idx))` 的边界 clamp，到达数据边缘后十字光标不再移动，但**没有触发 dataZoom 滚动**。

用户期望的是：当十字光标在 index=0 时按左键（或在 index=749 时按右键），整个 dataZoom 窗口向左（或向右）平移，使得新数据进入可视区域。

### 任务 2：键盘边界触发 dataZoom 滚动

**行号：** 962-977

**改动：** 当 `idx` 被 clamp 到边界（0 或 length-1）且按键方向超出边界时，平移 dataZoom 窗口。

**核心逻辑：**
- 当前 `dataZoom` 范围：`{start, end}`，显示 `end-start`% 的数据
- 当光标在 index=0 且按 `ArrowLeft`：向左平移，如 `start -= 10%, end -= 10%`（最低 clamp 到 0%）
- 当光标在 index=749 且按 `ArrowRight`：向右平移，如 `start += 10%, end += 10%`（最高 clamp 到 100%）
- 平移后插值刷新所有子图的 dataZoom

**代码：**
```javascript
document.addEventListener('keydown', function(e){
  if(e.key==='ArrowLeft'||e.key==='ArrowRight'){
    e.preventDefault();
    keyboardMoving = true;
    var idx = cursorIdx + (e.key==='ArrowLeft'?-1:1);
    
    // 检测边界→触发dataZoom滚动
    if(idx < 0 || idx >= dates.length){
      var dz = chart.getOption().dataZoom[0];
      var range = dz.end - dz.start;           // 当前可视范围宽度
      var shift = Math.max(5, Math.round(range * 0.3));  // 每次平移30%
      var ns = dz.start - shift;
      var ne = dz.end - shift;
      if(e.key==='ArrowRight'){ ns = dz.start + shift; ne = dz.end + shift; }
      // clamp 到合法范围
      if(ns < 0){ ns = 0; ne = range; }
      if(ne > 100){ ne = 100; ns = 100 - range; }
      
      chart.dispatchAction({type:'dataZoom', start:ns, end:ne});
      for(var k in subInstances){
        try{subInstances[k].dispatchAction({type:'dataZoom', start:ns, end:ne});}catch(e2){}
      }
      // 滚动后将光标定位到新边界附近
      idx = e.key==='ArrowLeft' ? dates.length-1 : 0;
    } else {
      idx = Math.max(0, Math.min(dates.length-1, idx));
    }
    
    cursorIdx = idx;
    updateCloseLine(idx);
    chart.dispatchAction({type:'showTip', seriesIndex:0, dataIndex:idx});
    updateSubHeaders(idx);
    for(var k in subInstances){
      try{subInstances[k].dispatchAction({type:'showTip', seriesIndex:0, dataIndex:idx});}catch(e){}
    }
    keyboardMoving = false;
  }
});
```

**平移量说明：** 每次平移 `range × 30%`，相当于移动可视窗口的 30%，确保用户每次按键能连续滚动浏览。`Math.max(5, ...)` 保证最小平移 5%，避免范围过窄时几乎不动。

**测试方法：** 键盘移到最左侧 index=0 → 再按左键 → dataZoom.start 应减少（从 86 变为约 82），显示更多左侧数据。

## Bug 3：副图数值不显示

### 根因 A：CSS 选择器错误

**行号：** 737

**当前代码：** `document.querySelector('#sub-chart-area .sub-chart-'+k+' .sub-val')`

- 使用父子组合器 `.sub-chart-{k} .sub-val`，查找 `.sub-chart-{k}` 元素下的 **子孙** `.sub-val`
- 但实际 DOM 中两个 class 在 **同一个元素** 上：`<span class="sub-val sub-chart-成交额">`
- 所以选择器永不匹配，`hdr` 始终为 `null`，updateSubHeaders 无效果

**修正：** 改用 AND 组合器：

```javascript
var hdr = document.querySelector('#sub-chart-area .sub-val.sub-chart-' + k);
```

### 根因 B：副图 Y 轴数值标签被隐藏

**行号：** 561

**当前代码：** `function subYAxis(){ return {scale:true, show:false, splitLine:{show:false}}; }`

- `show:false` 隐藏了 Y 轴及其标签
- 主图 Y 轴的 `axisPointer.label`（显示当前价位值的浮标）也未在副图启用

**方案：** 将 `show:false` 改为 `show:true`，并加入副图专属的轴指针标签配置，与主图样式一致：

```javascript
function subYAxis() {
  return {
    scale: true,
    show: true,
    splitLine: { show: false },
    axisLabel: { fontSize: 9, color: '#64748B' },
    axisPointer: {
      label: {
        show: true,
        backgroundColor: '#334155',
        color: '#E2E8F0',
        fontSize: 10,
        padding: [2, 6, 2, 6]
      }
    }
  };
}
```

**例外处理：** 需要检查所有 inline 定义 yAxis 的 subDef（RSI、VOL、W&R、资金仓位等），确保它们也同步更新。已知 inline yAxis 的 subDef：
- `vol`（第 579 行）：`yAxis:{scale:true, show:false, splitLine:{show:false}}`
- `rsi`（第 612 行）：`yAxis:{min:0, max:100, scale:true, show:false, splitLine:{show:false}}`
- `wr`（第 615 行）：`yAxis:{min:0, max:100, scale:true, show:false, splitLine:{show:false}}`
- `资金仓位`（第 625 行）：`yAxis:{min:0, max:100, scale:true, show:false, splitLine:{show:false}}`

以上所有 `show:false` 均应改为 `show:true` 并加入 `axisPointer.label`。

## Bug 4：从副图右侧离开后十字光标残留

### 根因

主图有 `mouseleave` 回调（第 913-920 行），正确隐藏 tips 和 closeLine。

副图（第 768-802 行 `renderSubs`）**没有注册任何 `mouseleave` 处理**。当鼠标从副图 canvas 直接移到右侧面板时：
- 主图 mouseleave **不会触发**（鼠标不在主图 DOM 上）
- 副图无 mouseleave → 其 showTip 保留 → 通过同步机制维持主图和其他副图的 showTip
- 结果：十字光标在所有图表上残留

### 任务 4：给副图 DOM 元素添加 mouseleave 事件

**行号：** 768-802（renderSubs 函数内）

**改动位置：** `activeSubs.forEach(function(name){...})` 循环末尾，创建子图 chartDiv 后，在其上绑定 mouseleave 事件。

**代码：**
```javascript
// 在创建 chartDiv 并 instance = def.render(...) 之后：
chartDiv.addEventListener('mouseleave', function(){
  chart.dispatchAction({type:'hideTip'});
  updateCloseLine(-1);  // 用无效索引隐藏closeLine
  for(var _k in subInstances){
    try{subInstances[_k].dispatchAction({type:'hideTip'});}catch(e){}
  }
});
```

**辅助修改：** `updateCloseLine` 函数（第 876-891 行）需要处理无效索引：

```javascript
function updateCloseLine(idx){
  if(idx == null || idx < 0 || idx >= closes.length){
    // 隐藏 closeLine
    chart.setOption({graphic:[{id:'closeLine',invisible:true}]},{replaceMerge:['graphic']});
    return;
  }
  // 原有逻辑不变...
}
```

## 风险与注意事项

1. **subYAxis 修改影响所有 subDef**：需要检查是否有些 subDef 在创建 chart 后又覆盖了 yAxis 配置（如 setOption 中传入的 yAxis 会完全替换，而非 merge）
2. **inline yAxis 的 subDef 要逐一确认**：VOL 的 yAxis 有 `{scale:true, show:false}` 不带 `splitLine`—添加 `axisPointer.label` 时要保留原有配置
3. **dataZoom 平移时的百分比对齐**：`dataZoom.start/end` 范围是 0-100，但数据点分布可能不是均匀百分比—平移后需要确认 visible 区域内的数据点与键盘光标索引一致
