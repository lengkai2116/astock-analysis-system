# 因子管理原型页面设计全量方案（修订v2）

> 基于 2026-06-23 至 2026-06-24 两轮沟通确认 + 修订反馈
>
> 修订要点：因子库改表格布局、LLM Wiki 因子分类标准补充、目标匹配改场景标签展示、GitHub 高星项目组合补充、新建组合交互流程设计

---

## 一、页面定位

因子管理页是**系统因子组合策略的后端管理平台**，职责清晰界定为：

- **不是**：股票的实质因子分析/选股/评分场所（那是选股系统、个股策略分析的职责）
- **不是**：策略模板生成场所（那是策略模板模块的职责）
- **是**：**单因子的管理平台**（浏览、搜索、了解功能、查看和调整参数）
- **是**：**因子组合的管理平台**（创建、编辑、删除用户自建组合；展示、调参系统预设组合）
- **是**：**选股系统 L3 多因子评分的数据源**（组合列表通过 API 真实同步到 screener.html #factor-combos 区域，禁止硬编码 Mock）

### 数据流关系

```
因子管理页（管理端）
  ├─ 因子库 → 管理 242 个单因子（表格浏览/搜索/分类/详情/参数/场景标签）
  └─ 组合方案 → 管理 N 个因子组合（系统预设 + 用户自建）
        │
        ▼ (API 实时同步)
选股系统 L3 多因子评分区域
  ⬜ 经典动量组合  ⬜ 量价突破组合  ⬜ 用户自建组合A ...
```

## 二、页面架构

### 2.1 布局方案

采用 **Tab 切换的双管理模式**：

```
┌─ 页面标题区 ──────────────────────────────────────┐
│  📊 因子管理                                        │
│  [📚 因子库]  [📋 组合方案]                          │
└─────────────────────────────────────────────────────┘
│                                                      │
│  当前 Tab 内容区域                                    │
│  （因子库 或 组合方案 二选一）                         │
│                                                      │
└─────────────────────────────────────────────────────┘
```

- 切换方式：顶部 Tab 按钮，高亮当前 Tab，点击切换
- 各 Tab 内部独立管理各自的布局和交互
- **不采用并排双页**，因为因子（242+）和组合方案（系统预设+用户自建，未来可能大量）各自独立管理，单 Tab 切换可避免拥挤

## 三、因子库 Tab

### 3.1 定位

管理所有 242 个已注册因子。面向**非专业量化用户**，以"因子能做什么"为核心组织信息，兼顾按分类浏览和模糊功能搜索两种发现方式。

### 3.2 布局

采用**表格展示**（替代卡片网格），理由：
- 242 个因子卡片化展示会形成很长页面，不利于快速浏览
- 表格可按列排序、横向对比，信息密度更高
- 点击表格任意行展开侧滑详情面板（3.6 模式一），不破坏浏览连续性

```
┌─ 分类筛选条 + 搜索 ───────────────────────────────────┐
│  [🔍 搜索因子功能描述、名称或场景...（模糊搜索）]         │
│                                                        │
│  全部(242) │ 动量(24) │ 反转(12) │ 成交量(18) │ ...     │
│  更多 ▼ ───── 波动率(15) │ 趋势(14) │ 位置(8)          │
│        ───── 基本面(12) │ 情绪(8) │ 统计(20)          │
│        ───── 规模(6) │ 价值(10) │ 成长(8) │ 质量(12)  │
└────────────────────────────────────────────────────────┘
│                                                         │
┌─ 因子表格 ─────────────────────────────────────────────┐
│  # │ 因子名称    │ 功能说明          │ 分类    │ 场景标签 │
│  1 │ RSI_14      │ 判断超买超卖的...  │ 动量    │ 短线,抄底  │
│  2 │ 14日RSI     │                    │ 超买超卖│  反弹     │
│  3 │ ROC(20)     │ 短期价格变动...    │ 动量    │ 中短线,   │
│    │             │                    │         │ 追强势   │
│  ... (每行高度紧凑，点击任意位置展开详情面板)              │
└────────────────────────────────────────────────────────┘
  ↑ 表头可点击排序（按名称、分类）
```

**列定义：**

| 列 | 内容 | 说明 |
|:---|:-----|:------|
| 因子名称 | 中文名（主） + 英文名（副，小字） | 例：14日相对强弱指数 `RSI_14` |
| 功能说明 | 一句话白话说明 | 例：判断价格是否处于超买(>70)或超卖(<30)状态 |
| 分类 | 中文分类标签 | 例：超买超卖 · 震荡指标 |
| 场景标签 | 该因子适用的场景标签（1-3个短标签） | 例：🏷️短线 🏷️抄底 🏷️全市场 |
| 数据源 | 数据来源标识 | 例：QLib |

### 3.3 分类筛选标准

#### 3.3.1 分类来源

基于 LLM Wiki 知识库调研（35+ 因子相关概念文件），结合业界通用分类标准和系统现有 category 体系，采用**两层分类**——上层为用户可理解的"大类标签"，下层为技术性"子分类"：

| 大类（用户面向） | 子分类（技术） | 后端映射 category | 数量 | 说明 |
|:---------------|:--------------|:-----------------|:----|:------|
| 📈 **价格动量** | 价格动量、动量振荡器、趋势动量 | momentum | ~24 | 测量价格变动的速度和力度 |
| 🔄 **超买超卖** | 短期反转、价格偏离、超买超卖 | reversal | ~12 | 判断价格是否偏离正常范围 |
| 💧 **成交量与资金** | 成交量趋势、量比、成交额、能量指标 | volume | ~18 | 分析成交量的变化和资金流向 |
| 📊 **趋势追踪** | 移动平均、布林带、趋向指标 | trend | ~14 | 识别和跟随价格运行方向 |
| 🌊 **波动率** | 价格波动、价格区间、标准差 | volatility | ~15 | 衡量价格波动幅度和活跃度 |
| 📍 **价格位置** | 价格水平、布林带位置 | position | ~8 | 判断当前价格在近期区间中的位置 |
| 🏢 **基本面** | 估值、盈利、成长、偿债 | —（后端暂无） | ~12+ | 基于财务数据的因子（需数据驱动） |
| 😊 **情绪感知** | 情绪指标、拥挤度 | —（后端暂无） | ~8 | 感知市场过热或过冷的状态 |
| 📐 **统计指标** | 偏度、峰度、回撤、比率 | academic | ~20 | 基于统计学和风险度量的分析 |
| 🏋️ **规模因子** | 市值、规模效应 | — | ~6 | 公司规模相关的因子 |
| 💎 **价值因子** | 估值、价值回归 | — | ~10 | 估值相关的因子 |
| 🌱 **成长因子** | 盈利增长、成长性 | — | ~8 | 成长性相关的因子 |
| ✅ **质量因子** | 盈利能力、财务质量 | — | ~12 | 公司质量相关的因子 |

> **注：** 标"—"的类别在后端现有 category 体系中暂无直接映射，表示这些因子在原型阶段用中文分类标签展示即可，无需改动后端分类体系。模糊搜索可覆盖这些因子的查找。

#### 3.3.2 模糊搜索

搜索框支持**模糊匹配**以下字段（实时过滤表格）：
- 因子**中文名**（如搜"强弱" → 命中 RSI）
- 因子**英文名**（如搜"ROC" → 命中 ROC(20)）
- 因子**功能说明**内容（如搜"超买" → 命中 RSI、KDJ、WILLR 等）
- 因子**场景标签**（如搜"短线" → 命中所有标记了"短线"标签的因子）
- 因子**分类名称**（如搜"动量" → 命中动量大类）

**默认展示**：无搜索和筛选时，默认按分类聚合展示全量因子（可按表头排序），不做分页。242 行在宽表格中约 3-4 屏高度，配合分类筛选可快速定位。

**空状态**：搜索无命中时展示 "未找到匹配因子，试试其他关键词，如：短线、超买、放量、趋势"

### 3.4 表格行交互

| 交互 | 触发 | 效果 |
|:----|:-----|:------|
| 行点击 | 点击任意行的任意位置 | 展开侧滑详情面板（模式一） |
| 表头排序 | 点击"因子名称""分类"列头 | 正序/倒序切换 |
| 分类标签点击 | 点击某行的分类标签 | 快速筛选到同分类因子（等同于点分类筛选条） |

### 3.5 因子详情面板（模式一·侧滑面板）

点击表格行后从右侧滑入详情面板，主表格区域保留且半透明遮罩。

```
┌────────── 主表格（底色变暗遮罩）──────────┬─ 详情面板 ──┐
│                                            │  RSI_14      │
│                                            │  14日相对强弱指数│
│                                            │              │
│                                            │  📝 功能说明   │
│                                            │  判断价格是否   │
│                                            │  超买(>70)或   │
│                                            │  超卖(<30)     │
│                                            │              │
│                                            │  🏷️ 场景标签   │
│                                            │  [短线] [抄底] │
│                                            │  [反弹] [全市场]│
│                                            │              │
│                                            │  ⚙️ 参数(可调) │
│                                            │  period: [14] │
│                                            │              │
│                                            │  📊 典型场景   │
│                                            │  "当RSI>70时  │
│                                            │   提示超买风险" │
│                                            │              │
│                                            │  📎 关联因子   │
```

**详情面板内容：**

| 区块 | 内容 | 数据来源 |
|:----|:-----|:---------|
| 基本标识 | 中文名 + 英文名 | name_cn / name |
| 📝 功能说明 | 白话描述该因子能识别什么状态、如何解读数值 | description + 原型阶段扩展文本 |
| 🏷️ 场景标签 | 该因子适用的选股场景标签（1-5 个短标签） | 原型阶段预设标签字典 |
| ⚙️ 参数(可调) | 参数名 + 当前值 + 可编辑输入框 | params + user editable |
| 📊 典型场景 | 1-2 个该因子在实际中的使用示例 | 原型阶段 Mock |
| 📎 关联因子 | 同一分类或可配合使用的相似因子 | 原型阶段预设 |
| ➕ 加入组合方案 | 按钮：跳转到组合方案 Tab 的新建组合流程 | 跨 Tab 导航（见六、新建组合流程） |

### 3.6 场景标签设计（替代 §3.7 目标匹配交互）

**核心设计变更说明：**
- ❌ 移除原 §3.7 中"因子库内完成目标匹配→勾选→保存为组合"的交互流程
- ✅ 因子库只负责**管理和展示因子**，场景标签作为因子的元数据展示在详情面板中
- ✅ 将"目标匹配"能力迁移到**组合方案 Tab 的新建流程**中（见六）

**场景标签系统：**

每个因子在原型阶段预设 1-5 个场景标签，标签分三个维度：

| 维度 | 标签值 | 示例 |
|:----|:-------|:------|
| ⏱ 时间周期 | 超短线 / 短线 / 中短线 / 中线 / 长线 | RSI → 短线 |
| 🎯 策略倾向 | 追强势 / 抄底反弹 / 趋势跟踪 / 低波防御 / 价值回归 | ROC → 追强势 |
| 🌐 市场环境 | 牛市 / 震荡市 / 熊市 / 全市场 | ATR → 全市场 |

| MACD_DIF | 中短线 | 趋势跟踪 | 全市场 |

### 4.3 系统预设组合

#### 4.3.1 来源说明

系统预设组合来源于三个渠道：
- **LLM Wiki 知识库**：Fama-French 三因子/五因子、A股五大有效因子实证、M8精选因子模型、状态依赖因子合成
- **GitHub 高星项目**：QuantsPlaybook（100+策略复现）、quantitative_analysis（12内置因子组合）、FactorHub（因子生命周期管理）等项目的因子组合方案
- **系统现有能力**：基于系统已实现的5个因子（ROC/VOL_RATIO/ATR/RSI/LINEARREG_SLOPE）的组合

#### 4.3.2 预设组合完整列表

| # | 组合名称 | 因子构成 | 权重 | 来源 | 功能说明 |
|:-:|:--------|:--------|:----|:-----|:--------|
| 1 | **Fama-French 三因子** | 市场因子(MKT) + 规模因子(SMB) + 价值因子(HML) | 40%/35%/25% | Fama & French 1993（学术经典） | 最经典的多因子模型，三大核心维度解释股票收益差异 |
| 2 | **Fama-French 五因子** | 市场 + 规模 + 价值 + 盈利(RMW) + 投资(CMA) | 30%/25%/20%/15%/10% | Fama & French 2015（学术扩展） | 在三因子基础上增加盈利质量和投资效率维度 |
| 3 | **A股核心五因子** | 市场因子 + 税收因子 + 市盈率因子 + 规模因子 + 速动因子 | 等权 (各20%) | A股实证（市值加权LASSO 2008-2020） | **A股最直接的实证结果**，经83个月数据验证 |
| 4 | **M8精选八因子** | 市场 + 预期增长(REG) + PEAD + 精细化价值(HMLM) + 短期反转(STR) + 行业领先滞后(ILR) + 规模(SMB) + 盈利可预测(EPRD) | 等权 (各12.5%) | Feng等 (2026) M8模型 | 从97个候选中经高维Alpha检验筛选，年化夏普2.73 |
| 5 | **短线动量** | ROC(20) + RSI(14) + VOL_RATIO(5) + 短期反转因子 | 30%/25%/25%/20% | QuantsPlaybook 因子构建类参考 | 适合短线追强势，捕捉价格动量+量能配合 |
| 6 | **价值防御** | 市盈率因子 + ATR(14) + 低波动(STD_20反向) + 规模因子 + 乖离率(BIAS 20) | 30%/20%/20%/15%/15% | 综合设计（类似Barra风险模型防御风格） | 适合低风险偏好的稳健配置，熊市/震荡市表现较好 |
| 7 | **聪明钱因子组合** | 聪明钱因子 + 筹码分布因子 + 换手率因子 + RSI(14) | 30%/30%/25%/15% | QuantsPlaybook 开源证券研报复现 | 捕捉主力资金行为，识别"聪明钱"动向 |
| 8 | **量价核心** | ROC(20) + VOL_RATIO(5) + RSI(14) + ATR(14) + LINEARREG_SLOPE(20) | 25%/20%/25%/15%/15% | 系统现有能力（5因子） | **系统已经实现的因子权重配置**，可直接运行 |
| 9 | **情绪感知** | RSI(14) + VR(26) + 换手率因子 + BOLL_PCT + 市场状态因子 | 25%/25%/20%/15%/15% | 状态依赖因子合成路径（LLM Wiki） | 感知市场过热/过冷/震荡状态，适合情绪择时 |

## 六、新建组合交互流程（核心新增）

### 6.1 流程总览

新建组合操作入口在**组合方案 Tab**（点击「+ 新建组合」按钮），不设在因子库内部。整个流程分 5 步：

```
Step 1: 描述目标（可选）      ─── 输入自然语言目标描述
↓                               系统推荐适用因子候选列表
Step 2: 筛选/选择因子         ─── 从候选列表或全量因子中勾选
↓                               查看各因子详情/功能说明/标签
Step 3: 配置权重              ─── 权重滑块 / 等权 / 手动输入
↓                               实时归一化至100%
Step 4: 调整参数              ─── 各因子的 period 等参数自定义
↓
Step 5: 命名保存              ─── 输入组合名称 + 功能描述
                                保存到"用户自建"列表
```

### 6.2 Step 1：描述目标（可选）

```
┌─ 新建组合 ──────────────────────────────────────────────┐
│  💡 你希望这个组合帮你选出什么样的股票？（选填）              │
│  ┌──────────────────────────────────────────────────┐   │
│  │ 例如：今天买入明天卖出 / 长期持有优质公司 / ...    │   │
│  └──────────────────────────────────────────────────┘   │
│  [🔍 匹配适用因子]                                       │
│                                                          │
│  或直接从下方因子列表中自行选择                             │
│                                                          │
│  ─────────────────────────────────────────────────────   │
│  ▼ 因子选择区（展开后见 Step 2）                           │
└──────────────────────────────────────────────────────────┘
```

**交互说明：**
- 目标描述为**可选**，用户可跳过直接进入因子选择
- 输入后点击「匹配适用因子」→系统按关键词匹配场景标签→筛选出候选因子（见 6.3）
- 不描述目标则直接展示全量因子候选列表（按分类分组）

**空状态引导：**
- 输入框 placeholder 展示示例："例如：短线快进快出 / 长期持有优质公司 / 捕捉强势股回调 / ..."
- 不输入直接选择也完全可用

- **Fama-French 三因子模型** — 市场/规模/价值
- **A股五大有效因子** — 市场/税收/市盈率/规模/速动
- **世界量化因子分类** — 动量/价值/质量/低波动/规模/成长
- **因子预处理** — 行业/市值中性化标准
- **风格时变性** — 牛熊市因子风格切换
- **五维统一分类框架** — 形态/位置/量价/结构/情绪五维
