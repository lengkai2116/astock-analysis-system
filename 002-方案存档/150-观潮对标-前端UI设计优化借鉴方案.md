# 观潮对标 — 前端UI设计优化借鉴方案

> 本文档基于 `149-观潮网站系统能力调研与对标分析报告.md` 的研究成果，聚焦 **前端UI设计及用户使用感知提升** 维度，对比观潮与当前项目的差距，提出可落地的借鉴改善方案。

## 2. 核心借鉴：设计系统升级

### 2.1 半透明叠加色体系

**观潮做法**：观潮没有使用多层级纯色背景，全部通过白色在深色底上的不同透明度构建层级，一套变量适配双主题。

**现状 (148 规范)**：我们的深色主题使用纯色绝对值 (`--bg-surface: #1A1A1A`)，浅色主题尚未落地。

**改进方案**：引入观潮的 rgba 半透明叠加模式替换深色主题的纯色绝对值。

```css
/* 深色主题 — 半透明叠加模式 (替换纯色绝对值) */
[data-theme="dark"] {
  /* 保留我们更亮的画布底色 */
  --bg-canvas: #0A0A0A;

  /* 层级通过叠加白色实现 (类似观潮) */
  --overlay-white: rgba(255, 255, 255, 0.04);   /* 最浅叠加 (hover/次级) */
  --overlay-light: rgba(255, 255, 255, 0.06);   /* 面板底色 */
  --overlay-medium: rgba(255, 255, 255, 0.09);   /* 强调面板 */
  --overlay-strong: rgba(255, 255, 255, 0.12);   /* 激活/选中 */

  /* 映射到语义 Token (向后兼容 148 规范) */
  --bg-surface: var(--overlay-light);             /* 替代 #121214 */
  --bg-subtle: var(--overlay-white);              /* 替代 #1A1A1A */
  --bg-muted: var(--overlay-medium);              /* 替代 #262626 */
  --bg-active: rgba(59, 130, 246, 0.12);          /* 品牌色选中态 */
  --bg-selected: rgba(59, 130, 246, 0.08);        /* 品牌色选中态 */
}
```

**浅色主题**：观潮模式在浅色主题下自动反转（黑色叠加），我们也同样实现：

```css
/* 浅色主题 — 黑色叠加模式 */
[data-theme="light"] {
  --bg-canvas: #FAFAFA;                           /* 页面背景 */
  --overlay-black: rgba(0, 0, 0, 0.03);
  --overlay-light: rgba(0, 0, 0, 0.05);
  --overlay-medium: rgba(0, 0, 0, 0.08);
  --overlay-strong: rgba(0, 0, 0, 0.12);
  /* ... 映射同深色 */
}
```

**实施步骤**：
1. 在 `global.less` 中添加 `[data-theme]` CSS 变量块
2. 将原 `--bg-*` 纯色值替换为 `var(--overlay-*)` 引用
3. 添加主题切换机制（localStorage + CSS 变量切换）
4. 回退方案：为不支持 CSS 变量的浏览器保留纯色后备

### 2.2 三层 Token 隔离

**观潮做法**：Primitive (p-) → Semantic (s-) → Component (c-)，每个组件模块有独立变量空间（`--ai-card-*` / `--tv-empty-*`）。

**现状 (148 规范)**：仅有 Layer 0 (基础 Token) 全局变量，无组件级命名空间。

**改进方案**：在 148 规范基础上追加组件级 Token 规范（观潮 Component Layer）。

**命名约定**：
```
--[component-namespace]-[property]-[variant]
--pro-table-header-bg
--pro-table-row-hover-bg
--kline-chart-crosshair-color
--ai-card-padding
```

**试点组件**（Phase 1，从 ProTable 开始）：
```css
/* --pro-table-* 命名空间 */
--pro-table-header-bg: var(--bg-subtle);
--pro-table-row-hover-bg: var(--bg-active);
--pro-table-row-height: 40px;
--pro-table-cell-padding: var(--space-8) var(--space-12);

/* --kline-chart-* 命名空间 */
--kline-chart-grid-color: rgba(255,255,255,0.06);
--kline-chart-crosshair-color: rgba(255,255,255,0.3);
--kline-chart-volume-height: 80px;
```

**实施步骤**：
1. 设计组件 Token 命名规范文档
2. 从 ProTable 试点，提取 5-8 个组件级 Token
3. 推广到 KLineChart、Sidebar、StatsCard 等高频组件
4. 建立组件级 Token 索引表

### 2.3 观潮设计参数修正（148 规范修订）

| 148 规范当前值 | 观潮值 | 建议修订 | 理由 |
|---------------|--------|---------|------|
| `--radius-md: 10px` | 6px | **8px** (折中) | 金融终端信息密度高，8px 兼顾美观与空间效率 |
| 无 `--space-5` | 20px | **新增 `--space-5: 20px`** | 观潮使用 20px 作为中间间距，弥合 16/24 跳跃 |
| 边框 `#333333` | rgba(255,255,255,0.06) | **rgba(255,255,255,0.08) 深色** | 半透明边框不占额外色值，双主题自适应 |
| Hover 底色 `#262626` | rgba(255,255,255,0.04) | **rgba(255,255,255,0.06)** | 同上，双主题自适应 |
| `--bg-surface` 深色 `#1A1A1A` | `#09090B` | **改为 `--overlay-light`** | 见 2.1 半透明体系 |
| 浅色主题未定义 | 自动反转 | **落地浅色主题** | 双主题是成熟产品的必要条件 |
| Inter 静态字体 | Inter Variable | **切为 Inter Variable** | 可变字体一个文件替代多个字重，加载更快 |
| 无 `--font-mono` 变量 | JetBrains Mono | **保留 `JetBrains Mono`** | 已在 148 定义 |

### 2.4 字体优化

**现状**：使用 Inter 静态字体，需加载多个字重文件。

**改进**：切换到 Inter Variable（可变字体）。

```css
/* Inter Variable — 单一文件加载 */
@font-face {
  font-family: 'Inter Variable';
  src: url('/fonts/Inter-Variable.woff2') format('woff2');
  font-weight: 100 900;  /* 单一文件覆盖所有字重 */
  font-display: swap;
}

/* 主字体更新 */
--font-sans: 'Inter Variable', 'Inter', -apple-system, BlinkMacSystemFont,
              'PingFang SC', 'Microsoft YaHei', 'Noto Sans SC', sans-serif;
```

**收益**：减少 3-5 个 HTTP 请求，文件总大小从 ~60KB（4 个字重）降到 ~30KB（单一可变），同时获得 100-900 所有字重支持。

## 3. 图表系统增强

### 3.1 多 K 线分屏

**观潮做法**：通过 `chart-cell--active` CSS 类实现多图联动，支持 2-4 图分屏。

**对标差距**：本项目使用 KLineCharts 单图组件，无多图支持。

**改进方案**：

```vue
<!-- Phase 2+ : 多 K 线分屏组件 -->
<template>
  <div class="chart-grid" :class="`chart-grid--${layout}`">
    <!-- 主图 (默认 1 图) -->
    <div class="chart-cell"
         :class="{ 'chart-cell--active': activeIndex === 0 }"
         @click="activeIndex = 0">
      <KLineChart :symbol="symbol"
                  :period="periods[0]"
                  :indicators="indicators[0]" />
    </div>
    <!-- 副图 (2-4 图模式) -->
    <div class="chart-cell"
         v-for="(period, i) in secondaryPeriods"
         :key="i"
         :class="{ 'chart-cell--active': activeIndex === i + 1 }">
      <KLineChart :symbol="symbol"
                  :period="period"
                  :indicators="indicators[i + 1]"
                  :sync-crosshair="true" />
    </div>
  </div>
</template>
```

**布局模式**：

| 布局 | 说明 | CSS Grid |
|------|------|---------|
| `single` | 单图 (默认) | `1fr` |
| `dual-h` | 双图水平 | `1fr 1fr` |
| `dual-v` | 双图垂直 | `1fr / 1fr` |
| `triple` | 三图 | `1fr 1fr / 1fr` |
| `quad` | 四图 | `1fr 1fr / 1fr 1fr` |

**必要前置条件**：
- 实现十字光标同步混合（`chartSyncMixin`）：鼠标移动时广播当前值到其他图
- 实现时间轴对齐：所有分屏图共享同一个 X 轴时间范围
- 实现指标独立配置：每个图可独立选指标（关注观潮的 `chart-indicator-manager`）

### 3.2 K 线图型扩展

**现状**：仅支持标准 K 线和分时图。

**观潮支持的图型**：标准K线、分时图、Heikin Ashi、Renko、Kagi、砖形图(P&F)、线形图、面积图、柱状图、收盘价线、等量图共 11 种。

**改进方案**：

| 图型 | 优先级 | 实现方式 | 前置条件 |
|------|--------|---------|---------|
| Heikin Ashi | P1 | 数据转换层，将 OHLC 转换为 HA 值 | KLineCharts 支持自定义数据转换 |
| 收盘价线 | P1 | KLineCharts 内置 type: 'line' | 无需开发 |
| 面积图 | P1 | KLineCharts 内置 type: 'area' | 无需开发 |
| Renko | P2 | 数据聚合层，将日线转为砖形 | 需实现聚合算法 |
| 砖形图 (P&F) | P3 | 独立计算模块 | 最复杂，远期 |

**实施建议**：通过 KLineCharts 的 `customType` 或 `extendData` API 实现，避免修改 KLineCharts 源码。

### 3.3 时间周期扩展

**现状**：仅支持 3 档（日/周/月）。

**观潮支持的周期**：1秒/5秒/15秒/30秒/1分/5分/15分/30分/60分/2小时/4小时/日/2日/3日/周/月/3月/6月/年共 20+ 档。

**改进方案**（分层实现）：

| 阶段 | 新增周期 | 数据源要求 |
|------|---------|-----------|
| Phase 3（对齐151: 分钟级数据通道§4.3） | 1分/5分/15分/30分/60分 | 需接入分钟级 Tushare API，数据层优先级依赖 151 号方案的分钟级数据通道 |
| Phase 2 (中期) | 2小时/4小时/2日/3日 | 由小时/日线聚合 |
| Phase 3 (远期) | 1秒/5秒/15秒/30秒 | 需 WebSocket 实时流 |

**实施前提**：数据层需增加分钟级数据获取能力（参见 `151-观潮对标-系统能力提升与稳定性优化方案.md` 数据层章节）。

### 3.4 画图工具

**观潮做法**：完整 82 类画图工具（趋势线/斐波那契/矩形/椭圆/文本标注/箭头/测量工具等）。

**现状**：本项目无画图工具（KLineCharts 基础支持但未启用）。

**改进方案**：

| 画图类型 | 示例 | 优先级 | 预计工作 |
|---------|------|--------|---------|
| 趋势线 | 射线/水平/垂直 | P1 | KLineCharts 内置支持 |
| 通道线 | 平行通道/等距通道 | P1 | KLineCharts 内置支持 |
| 斐波那契 | 回调/扩展/扇形 | P1 | KLineCharts 内置支持 |
| 几何图形 | 矩形/圆形/椭圆 | P2 | 自定义 overlay |
| 测量工具 | 价格测量/周期测量 | P2 | 自定义 overlay |
| 文本标注 | 文本/备注/箭头 | P2 | 自定义 overlay |
| 多段线 | 徒手画线/多边形 | P3 | 自定义 overlay |
| 风控线 | 止盈/止损参考线 | P1 | 业务自定义 |

**实施路径**：
1. 启用 KLineCharts 内置画图插件 API
2. 封装 `DrawToolbar` 组件（工具选择 + 颜色/线型配置）
3. 实现画图数据持久化（IndexedDB 存储, 参考观潮 drawings/batch API）
4. 按优先级分批上线画图类型

## 4. 交互体验升级

### 4.1 数据源状态指示

**观潮做法**：数据源三态（normal/degraded/fallback）通过状态栏 cloud-sync 图标指示。

**现状**：无任何数据源状态指示，数据断流时用户无感知。

**改进方案**：实现 `DataSourceStatus` 三态指示组件。

```vue
<!-- 数据源状态组件 -->
<template>
  <div class="data-source-status" :class="`data-source-status--${status}`">
    <span class="data-source-status__dot"></span>
    <span class="data-source-status__label">{{ statusLabel }}</span>
    <Tooltip v-if="message">{{ message }}</Tooltip>
  </div>
</template>

<!-- 三态定义 -->
status: 'normal'    → 绿灯 + "数据正常"
status: 'degraded'  → 黄灯 + "部分延迟 (12s)"
status: 'fallback'  → 红灯 + "已切换至备用数据源"
```

**放置位置**：Sidebar 底部状态栏、KLineChart 左上角、仪表盘顶部。

> **后端依赖**：本组件依赖 `151-系统能力提升与稳定性优化方案.md` §4.1 DataSourceManager + `GET /api/v3/data-source/status` 端点。数据源状态的后端自动切换/降级由 151 号方案负责，本组件仅负责前端展示。

### 4.2 快捷键系统

**观潮做法**：可编辑 30 类快捷键（导航/搜索/画图/交易/分析）。

**现状**：无快捷键系统。

**改进方案**：基于 VueUse `onKeyStroke` 实现轻量快捷键系统。

**Phase 1 快捷键**（10 个基础快捷键）：

| 快捷键 | 功能 | 实施方式 |
|--------|------|---------|
| `/` 或 `Ctrl+K` | 全局搜索 | 搜索弹窗激活 |
| `g + i` | 跳转仪表盘 | 路由导航 |
| `g + s` | 跳转选股 | 路由导航 |
| `g + b` | 跳转回测 | 路由导航 |
| `g + a` | 跳转AI分析 | 路由导航 |
| `g + w` | 跳转自选 | 路由导航 |
| `r` | 刷新当前数据 | 触发数据刷新 |
| `Ctrl+F` | 页面内搜索 | 列表搜索 |
| `Esc` | 关闭弹窗/取消 | 通用 |
| `?` | 快捷键帮助 | 弹出快捷键说明面板 |

**架构设计**：

```javascript
// stores/shortcuts.js — 快捷键 Store
export const useShortcutStore = defineStore('shortcuts', () => {
  const shortcuts = ref([
    { id: 'goto-dashboard', keys: ['g', 'i'], action: 'router.push("/dashboard")' },
    { id: 'global-search', keys: ['/'], action: 'openGlobalSearch()' },
    // ...
  ])

  // 支持用户自定义快捷键
  const customBindings = persistedRef('shortcut-bindings', {})

  function getBinding(id) { return customBindings.value[id] || shortcuts.value.find(s => s.id === id)?.keys }

  function setBinding(id, keys) { customBindings.value[id] = keys }

  return { shortcuts, customBindings, getBinding, setBinding }
})
```

**改进方案**：

```javascript
export default defineConfig({
  build: {
    rollupOptions: {
      output: {
        manualChunks: {
          'vendor-vue': ['vue', 'vue-router', 'pinia'],
          'vendor-antd': ['ant-design-vue', '@ant-design/icons-vue'],
          'vendor-echarts': ['echarts', 'echarts-gl'],
          'vendor-kline': ['klinecharts'],
          'vendor-vueuse': ['@vueuse/core'],
          'data-services': [
            './src/services/chartService.js',
            './src/services/screenerService.js',
            './src/services/aiAnalysisService.js',
          ],
          'analysis-views': [
            './src/views/screener/index.vue',
            './src/views/backtest/index.vue',
          ],
        }
      }
    }
  }
})
```

在 `index.html` 中添加 `<link rel="modulepreload">` 预加载关键分包。
