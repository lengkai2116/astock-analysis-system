## 二、核心发现：提供文档未覆盖的关键内容

你提供的文档已经覆盖了周期差异根源、缠论级别映射、UIDF 三层结构、凌波多周期联立等核心内容。知识库中还包含了以下**补充性内容**：

### 2.1 周期错配——信号冲突的首要来源

**来源：[周期错配.md](概念层)**

这是系统设计中极易忽视但后果严重的问题：

- **用5分钟K线判断日线趋势方向** → 小周期噪音覆盖大周期信号
- **用周线多因子评分指导日内交易** → 信号频率与目标周期不匹配
- **60分钟缠论买卖点直接用于月线仓位决策** → 仓位管理混乱

> 对系统的影响：当前系统中，VolumePriceStrategy 的日线信号和 60 分钟信号在同一条 pipeline 中直接合并评分，存在周期错配风险。需要引入分层机制（详见下文）。

### 2.2 分层决策框架——"主决策周期"概念

**来源：[分层决策框架.md](概念层)**

核心原则：必须选定一个 **"主决策周期"**，所有最终决策以此周期为准，其他周期辅助但不否决。

| 层级 | 关系 | 功能 | 规则 |
|------|------|------|------|
| 背景/趋势周期 | 比决策周期高2-3级 | 只判断大势（多头/空头） | 不做交易决定 |
| **决策周期** | **主周期** | **买卖指令的唯一来源** | **必须有与背景周期一致的方向** |
| 执行周期 | 比决策周期低1级 | 只用于精确入场 | 不改方向 |

**红绿灯规则**：三周期同向才可交易。

### 2.3 状态依赖的动态周期——核心补充

**来源：[状态依赖的动态周期.md](概念层)**

你提供的文档§4.3 有提及，但知识库中有更具体的权重数据：

| 市场状态 | 大周期权重 | 小周期权重 | 操作规则 |
|----------|-----------|-----------|---------|
| 趋势市 | 80%（日线/周线） | 微调 | 大周期主导，小周期仅用于微调入场时机 |
| 震荡市 | 被压制 | 权重提升（60分钟/30分钟） | 小周期信号更可靠 |
| 高波动市 | 全部降低权重 | 全部降低权重 | 等待波动收缩，减少交易 |

**工程意义**：我们目前系统的策略没有市场状态识别→周期权重动态调整的机制。所有策略在所有状态下使用固定权重。这是后续需要引入的增强。

### 2.4 参数平原——科学选定周期的验证方法

**来源：[参数平原.md](概念层)、[滚动窗口参数平原检验.md](概念层)**

这是你提供文档§5.2 的完整版本：

**参数平原判据（判断策略是否稳健的标准）：**
- 参数跨度 ≥ 30%（如窗口20→40，性能相近）
- 绩效波动 < 15%
- 单连通区域（无断裂）

**方法论**：将分析周期作为参数，在合理范围（30分钟到周线）内网格搜索策略表现。如果20+个连续周期参数下表现相近 → 策略对周期不敏感，选居中周期。如果只有个别参数突出 → 可能是过拟合。

**对系统的指导意义**：我们的 S(20)/M(60)/L(120) 框架选定的三个周期点是否落在参数平原内？需要回测验证。

### 2.5 从单周期到多周期的渐进式升级

**来源：[从单周期到多周期的渐进式升级.md](概念层)**

> **"不要一开始就搭建包含6个周期的复杂系统"**

推荐路径：单周期验证 → 双周期验证（加高层次过滤器）→ 多周期共振（加精细执行周期）→ 状态依赖的动态周期。每增加一个周期维度，研究员自由度指数级上升——3周期×5参数每个 = 15倍自由度。

### 2.6 情绪周期经验性时长

**来源：[情绪周期各阶段经验性时长.md](概念层)**

基于2018-2025年A股数据的量化统计：

| 阶段 | 常规时长 | 极端情况 |
|------|---------|---------|
| 冰点期 | 5-8天 | 2022年4月12天 |
| 启动期 | 3-5天 | 2024年2月2天 |
| 主升期 | 8-15天 | 2023年6月AI行情22天 |
| 高潮期 | 2-3天 | 2025年1月仅1.5天 |
| 退潮期 | 5-10天 | 2021年9月7天 |

**意义**：情绪周期各阶段的时长数据可以直接用于系统设计——例如"冰点期≤8天"，选股系统在冰点第5天起加大关注；"高潮期≤3天"，信号过滤层在高潮期降低买入信号权重。

### 2.7 斐波那契数列变盘周期

**来源：[斐波那契数列变盘周期.md](概念层)**

吴国平系统阐述的变盘时间点预测方法：

| 周期分类 | 关键数字 | 适用场景 |
|---------|---------|---------|
| 小周期 | 21天、34天 | 最容易发生变盘 |
| 中周期 | 55天、89天 | 中级趋势转向 |
| 大周期 | 144天、233天 | 重大转折点 |

**多周期共振**：大周期和小周期数字共振时，变盘可靠性极高。实例：上证5178点调整到2780点见底，正好是第233个交易日。

### 2.8 Anna Coulling 量价多时间框架

**来源：[量价分析多时间框架分析.md](概念层)**

安娜·库林的"三车道高速公路"比喻：
- **快车道（5分钟）** → 最先出现变化
- **中间车道（15分钟）** → 主交易图表
- **慢车道（30分钟）** → 揭示更长趋势

**涟漪效应**：变化首先出现在最快时间框架，如果真实则传导到中间，再传导到慢速。多时间框架确认 = 更高可靠性。

### 2.9 财信证券三维情绪模型

**来源：[财信证券三维情绪模型.md](概念层)**

从三个频率维度观察情绪：

| 维度 | 用途 | 关键数据 |
|------|------|---------|
| 高频 | 日内决策 | 期指升贴水、期权PCR |
| 中频 | 日线级别操作 | 机构资金流向、交易活跃度 |
| 低频 | 中长期仓位管理 | 行业间相关系数（0.83警戒线预测牛熊转折） |

## 四、对我们系统时间窗口框架的建议

我们之前初步提出的 **S(20)/M(60)/L(120)** 三层框架——与知识库内容对照后，有以下发现：

### 4.1 框架定位修正

知识库的共识是 **四层而非三层**。UIDF 三层结构中，执行层和过滤层都在日线级别但功能不同。凌波框架和分层决策框架也强调需要 3-4 级分层。建议调整为：

| 层 | 当前建议 | 知识库依据 | 映射到现有策略参数 |
|---|---------|-----------|-----------------|
| **确认层（趋势）** | **周线/120日** | UIDF确认层、决策框架背景层、凌波大周期 | MA120位置、周线MACD、周线趋势 |
| **决策层（主周期）** | **日线/60日** | 分层决策的"主决策周期"、UIDF执行层 | MA60位置、ChipScorer、日线缠论、量价四阶段 |
| **执行层（入场）** | **日线/20日** | UIDF过滤层、决策框架执行层上界 | MA20支撑、量比基线、RSI14、volatility |
| **观测层（精确）** | **60分钟**（预留） | MACD买入法、凌波小周期、Anna库林 | 精确入场信号、背驰确认 |

### 4.2 关键建议：引入"主决策周期"概念

当前系统的策略评分没有区分"哪个周期是主周期"——所有周期信号在 Pipeline 中合并评分。按知识库的分层决策框架，应该：

1. **选定日线（60日）为主决策周期**：所有买卖信号的裁决基准
2. **120日/周线为背景周期**：只输出趋势方向（多/空/震荡），不参与具体评分
3. **20日为执行周期**：精确入场时机微调，不推翻主周期方向
4. **60分钟为观测周期**：保留供后续精确入场使用

### 4.3 论证：为什么是日线为主决策周期

| 理由 | 依据 |
|------|------|
| 系统当前所有数据颗粒度以日线为主 | 日线OHLCV 600万条，分钟线仅30天 |
| 策略目标持仓周期 5-20 天（波段/趋势跟踪） | A股散户典型持股周期 |
| 缠论日线中枢跨度 1周-3个月，覆盖典型持仓 | 中枢跨度时长经验数据 |
| UIDF 和分层决策框架的"标准配置"都是日线为主周期 | 多个框架共识 |

### 4.4 现有参数归并与简化建议

系统当前有 **14+ 种不同窗口**：5/10/20/26/30/40/60/90/120/250/365+macd12/26/9

建议归并路径（不改变策略逻辑，只简化参数配置）：

| 当前分散窗口 | 归并方案 | 依据 |
|------------|---------|------|
| 5天、10天 | 保留（执行层精确信号） | 5天适合量比/动量快线，10天短线辅助 |
| **20天、26天** | **统一为20日（S级）** | 26=MACD默认已算入，对外统一暴露20日 |
| 30天、40天 | **归入M级(60日)的增强子窗口** | 30是StageDetector最小窗口，40是divergence最小门，均可作为M级的参数子集 |
| **60天、90天** | **统一为60日（M级）** | 90是策略健康监控，可纳入60日的延伸度量 |
| **120天、250天** | **统一为120日（L级）** | 250仅用于上市资格判断，归入L级的边界条件 |
| 365天 | 保留（指数数据请求，不影响策略） | 只用于指数历史数据，不参与策略评分 |

### 4.5 参数平原验证建议

**建议实施**：在确定 S/M/L 框架后，需要对每个窗口做参数平原验证：

1. 对 VolumePriceStrategy 和 ChipScorer，将窗口参数在 ±30% 范围内做网格搜索（20→26→34→40→52→68→90→120）
2. 观察各策略在不同窗口下的评分稳定性
3. 如果 40-90 范围内评分波动 < 15%，说明该策略存在参数平原，M=60 是合理选择
4. 如果评分只在 60 处突出，在其他窗口显著下降 → 需要警惕过拟合

## 五、对系统分阶段实施建议

根据知识库的渐进式升级原则，建议分阶段推进周期框架重构：

### Phase 0（当前可做，不涉及前端）
- **文档级：明确系统的三层/四层周期框架定义**（本文即此输出）
- 不修改策略代码，先确认框架设计

### Phase 1（不需要前端对接）
- **引入"主决策周期"概念**：在 SignalComputationService 中给日线信号更高权重（或设置方向约束，如"周线看空时日线买入信号不执行"）
- **参数统一**：将 14+ 窗口简化为 S(20)/M(60)/L(120) + 少数保留窗口
- **状态识别**：实现基础的市场状态识别（趋势/震荡/高波动），为动态权重做准备

### Phase 2（待前后端打通时）
- **UI 中展示分层信息**：个股策略分析页中显示"确认层→决策层→执行层"各层判定
- **60分钟观测层对接**：利用现有分钟线数据增加精确入场信号

### Phase 3（后续完善）
- **状态依赖的动态周期权重**：根据市场状态自动调整各周期信号权重
- **参数平原周期性检验**：定期验证策略窗口选择是否仍在参数平原内

## 七、总结

知识库中关于分析周期规则的内容非常丰富，在以下方面对你已有的文档形成了关键补充：

1. **"主决策周期"概念**——系统选择日线（60日）作为主决策周期，所有信号以此为准
2. **周期错配防范**——当前 Pipeline 直接合并所有周期评分存在风险，需要分层
3. **状态依赖的动态周期权重**——趋势/震荡/高波动下各周期权重不同
4. **参数平原验证方法**——选择 S/M/L 三个窗口后需验证其是否在参数平原内
5. **情绪周期精确时长数据**——可用于选股系统的时机过滤
6. **渐进式升级路径**——确认 S/M/L 三层框架可先从单层开始，逐步扩展

**建议从之前的 S(20)/M(60)/L(120) 三层调整为四层**：确认层(周/120日) → 决策层(日/60日) → 执行层(日/20日) → 观测层(60min)，其中日/60日为主决策周期。
