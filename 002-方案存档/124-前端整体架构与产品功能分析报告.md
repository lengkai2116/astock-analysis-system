**关键组件**：ECharts, Ant Design Card/Statistic, 自定义 DashboardWidget

### 4.2 个股策略分析（indicator-ide, 834行）

**功能概述**：系统最核心的页面，对单只股票进行全维度分析。

| 功能 | 技术实现 | API 依赖 |
|------|---------|---------|
| K线图表 | klinecharts 封装组件 | `/api/v3/chart/kline` |
| 指标叠加（主图 MA/BOLL） | klinecharts overlay API | 前端计算 |
| 副图指标（成交量/MACD/RSI/KDJ） | klinecharts subchart | 前端计算 |
| 周期切换（日/周/月） | Ant Radio Group | 参数传递给 Kline API |
| 股票搜索 | Ant Select 搜索模式 | `chartService.searchStocks()` |
| B/S 信号标记 | klinecharts createOverlay + simpleAnnotation | `chartService.fetchSignals()` |
| 自选股快捷切换 | Ant Dropdown | localStorage |
| 返回选股系统 | 工具栏按钮 | 路由跳转 |
| 代码编辑器（可折叠） | CodeMirror 5 | 无（本地编辑） |
| 策略运行 | POST 请求 | `/api/v3/strategy/run` |
| L2 主力分析信号 | 分类信号卡片 | `/api/v2/strategy/outputs` |
| L3 策略验证信号 | 分类信号卡片 | `/api/v2/strategy/outputs` |
| 自定义策略信号 | 橙色边框卡片 | CodeEditor 运行结果 |
| 综合操作建议 | 底部汇总卡片 | 前端加权计算 |
| 星级评分 | ⭐1-5 可视化 | 根据 confidence 换算 |

**L2/L3 信号分类逻辑**：
- **L2 主力分析**: 策略名含"筹码/主力/chip/ASR/集中度/main force"的信号
- **L3 策略验证**: 策略名含"缠论/chanlun/因子/factor/量价/volume/策略"且非 L2 的信号
- **自定义策略**: CodeEditor 运行产生的信号，橙色边框区分

**综合操作建议计算规则**：
```
1. 统计多头/空头信号数量 → 确定操作方向
2. 各信号 confidence 均值 → 综合赢率
3. 取各信号中最保守的止损线
4. 取各信号中最积极的止盈目标
```

### 4.3 选股系统（screener, 230行）

**功能概述**：全市场批量筛选，三步流水线过滤出精选股票。

| 功能 | 技术实现 | API 依赖 |
|------|---------|---------|
| 三层筛选流程可视化 | PipelineFlow 组件 | 无（前端状态展示） |
| Layer 1 风险剔除 | 全市场过滤 | `/api/v3/screener/layer1` |
| Layer 2 主力识别 | 主力资金分析 | `/api/v3/screener/layer2` |
| Layer 3 策略验证 | 多策略交叉验证 | `/api/v3/screener/layer3` |
| 选股结果表格 | ScreeningResults（Ant Table） | 上一步结果 |
| 信号融合配置 | SignalFusionConfig 组件 | `/api/v3/screener/fusion-config` |
| 评分/阶段/ASR/集中度 | 颜色编码（金/银/铜牌） | 后端返回数据 |
| 双击跳转个股分析 | `$router.push({ query: { symbol } })` | 路由跳转 |
| 导出 CSV | Blob + URL.createObjectURL | 无（前端生成） |

**三栏布局**：
```
┌─────────────────┬───────────────────┬──────────────────┐
│  筛选流程         │  选股结果          │  信号融合配置     │
│  PipelineFlow    │  ScreeningResults  │  SignalFusion    │
│  (260px)         │  (flex:1)          │  (300px)         │
└─────────────────┴───────────────────┴──────────────────┘
```

## 六、用户视角的合理性与功能性分析

### 6.1 用户画像

系统面向 A 股量化投资者/分析人员，典型用户工作流：

```
【盘后分析场景】
选股系统 → 发现标的 → 个股策略分析 → 确认操作 → 添加到自选
                                                 ↓
                                             报告中心存档

【盘中监控场景】
仪表盘 → 自选监控(实时行情) → 个股策略分析 → 确认操作
                                  ↑
                              AI 分析辅助

【策略开发场景】
策略模板 → 代码编辑器修改 → 运行验证 → 信号确认 → 回测验证
```

### 6.2 流程合理性分析 ✅

| 流程 | 合理性 | 依据 |
|------|--------|------|
| 选股 → 个股分析的闭环 | ✅ 合理 | 双击跳转携带 symbol 参数，用户不必重复搜索 |
| 个股分析 → 选股的返回 | ✅ 合理 | 工具栏"选股"按钮一键返回 |
| 自选 → 个股分析 | ✅ 合理 | 双击/分析按钮，携带 symbol |
| 个股分析 → 代码编辑 → 信号展示 | ✅ 合理 | 右侧面板三段式清晰分区 |
| 因子组合 → 回测 | ✅ 合理 | 组合保存后可快速启动回测 |
| 模板 → 编辑器 | ✅ 合理 | 复制到剪贴板后可直接粘贴运行 |
| 策略运行 → 报告 | ✅ 合理 | 报告中心可查看历史分析结果 |

### 6.4 潜在问题与改进建议 ⚠

### 6.5 (real 6.7) 总结

**优势**：
1. **功能闭环完整** — 从选股→分析→自定义→回测→报告的全链路打通
2. **信息分层清晰** — 123号方案解耦后，选股和分析两个场景不再互相干扰
3. **右侧面板三段式合理** — L2/L3/Custom 的分类方式对应不同的分析深度
4. **技术栈统一** — 全栈基于 Vue 2 + Ant Design，维护成本可控
5. **模块化程度高** — 6个服务层、16个组件、9个页面，层次分明

**风险**：
1. **Vue 2 → Vue 3 迁移** — 需规划迁移路径，涉及 Ant Design 从 1.x 到 4.x
2. **后端数据真实度** — 部分功能依赖 Mock 数据，影响用户体验验证
3. **状态管理碎片** — 需逐步收敛到 Vuex 或推进 Composition API
4. **AI 分析断点** — 页面已建设但后端未就绪，形成功能断点

**建议优先级**：
1. 🔴 高：后端真实数据接入（选股、信号、策略输出）
2. 🔴 高：Vue 3 迁移规划
3. 🟡 中：全局股票搜索条
4. 🟡 中：回测 → 报告链路打通
5. 🟢 低：快捷键系统
6. 🟢 低：移动端响应式优化
