## 一、理解门控

| 问题 | 回答 |
|------|------|
| 1. 这段代码解决什么问题？ | 5 个独立缺陷：(a) backtest 页面因缺少 `useAppStore` import 导致 `ReferenceError`，(b) reports-center 使用已废弃的 AntDV 3 `createForm` API 导致 `TypeError`，(c) `formatter` 类型不匹配字符串→函数，(d) `bootstrap()` 在 Pinia 初始化前执行存在潜在风险，(e) 7 个路由文件缺少模块注释 |
| 2. 为什么选择此方案？ | 最小化本地修改——补 import、替换 AntDV 3 回调 API 为 AntDV 4 `:model` + `ref` 模式、重排序 main.js 调用。无超出问题范围的重构。 |
| 3. 代码失败时会发生什么？ | 低风险——所有修复边界清晰。ErrorBoundary 组件已在（已提交代码中）包裹 router-view，渲染异常可优雅降级而非白屏。表单验证失败由 try/catch 捕获并通过 `$message.error` 显示。 |
