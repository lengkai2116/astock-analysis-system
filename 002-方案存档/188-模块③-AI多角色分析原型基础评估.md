# 模块③（AI多角色分析）原型基础评估

**依据**：186号方案 §4.4（模块③与UI类7项 / #47-#53）、183号方案（三模块策略架构评估）、实际审查 `_ui-prototype/ai-analysis.html`

**评估结论**：✅ **现有的 ai-analysis.html 可以直接作为模块③的实施基础，无需新建页面。**

---

## 一、评估背景

186号方案 §4.4 列出了模块③相关的 7 项 P2 待办（#47-#53），其中 **#47（新建前端 `/stock-analysis` 页面）** 原判断需要新建一个页面来承载模块③功能。经实际审查原型目录后发现，`_ui-prototype/ai-analysis.html` 已是一个**完整的 AI 多角色分析独立页面**，定位与模块③完全一致。

用户确认：在此页面基础上进行后续修改即可，现阶段还未到达该页面的沟通修订阶段，预留此文档供后续直接调用。

## 三、模块③需求与现有页面对照

| # | 模块③要求 | ai-analysis.html 现状 | 差距 | 改造难度 |
|---|----------|----------------------|------|---------|
| **47** | 新建 stock-analysis 页面 | **已有独立AI分析页，无需新建** | ✅ 无差距 | **无需** |
| **48** | 策略现状面板（缠论/量价/筹码/因子） | 该页面没有此项 | 此项属于个股策略分析页(`indicator-ide.html`)，不是模块③的功能 | **无需**（归属其他页面） |
| **49** | AI解读区（操作计划/入场/止损/目标/风险） | 已有4概览卡片（总评/趋势/信号/置信度）+综合研判文字 | 需从自由文字改为结构化 JSON Schema 渲染 | **低** |
| **50** | 辩论环节（观点冲突→第二轮辩论→共识区） | 无。当前是线性进度 p0→p1→p2→p3 依次完成 | 需在单个角色结果出现矛盾时触发第二轮 | **中高**（核心功能） |
| **51** | 结果持久化+历史查询 | 全 Mock，无真实 API 调用 | 需建表 + 后端路由 + 历史列表 UI | **中** |
| **52** | 共识矩阵+分歧可视化（雷达图 + ✓/⚡标记） | 已有5角色评分条（技术6.8/基本面7.5/消息面4.2/情绪面5.5/综合6.8） | 已有雏形。需升级为 ECharts 雷达图 + 共识/分歧状态标记 | **低中** |
| **53** | UI展示分层周期信息 | 没有 | 此项属于个股策略分析页(`indicator-ide.html`)，不是模块③的功能 | **无需**（归属其他页面） |

### 关键结论

- **#47、#48、#53** 三项实际上不适用于此页：
  - #47：AI分析页已存在，无需新建
  - #48（策略现状面板）和 #53（分层周期信息）属于 **个股策略分析页**（即原型中的 `indicator-ide.html`），不是模块③的功能
- **因此模块③实际需在 ai-analysis.html 上实施的改造为 4 项**（#49 解读区、#50 辩论、#51 持久化、#52 共识矩阵），而非原计划的 7 项
- 现有页面骨架已经覆盖了模块③ **约 70% 的 UI 需求**

## 四、现有页面骨架详细分析

### 4.1 页面布局

```
侧栏导航 ┃  头部: 🤖 AI 分析 · DeepSeek 投研
（分析组）┃  股票选择: [代码输入框] [下拉选股] [🤖 开始分析]
         ┃  模型标识: 模型: DeepSeek v4-flash
         ┃
         ┃  进度面板（4步骤: 技术→基本面→情绪→综合）
         ┃      ↓ 完成动画后隐藏
         ┃  结果面板
         ┃    ├── 4概览卡片（总评/趋势/信号/置信度）
         ┃    └── 4Tab（综合/技术/基本面/消息面）
```

### 4.2 可复用部分

| 现有元素 | 复用方式 |
|---------|---------|
| 股票选择区（代码输入+下拉选股+开始按钮） | **直接保留**，仅需将 Mock 改为传入真实 ts_code |
| 进度面板（4步骤 + 进度条动画） | **保留骨架**，步骤扩展为5-6步（对应5个AI角色），进度逻辑改为按真实 API 响应推进 |
| 4概览卡片（总评/趋势/信号/置信度） | **保留并结构化**，字段映射到 `interpret_status()` 的 JSON Schema |
| 4Tab 切换 | **扩展为6Tab**：综合 + 技术/基本面/情绪/缠论/筹码 五角色 |
| 5角色评分条 | **保留并扩展**，增加共识标记和分歧标记 |

### 4.3 需要新增/重构的部分

| 功能 | 描述 |
|------|------|
| **辩论环节面板** | 在综合 Tab 下方增加"辩论区"：显示哪些角色观点一致、哪些分歧、分歧点是什么、第二轮辩论输入输出 |
| **共识雷达图** | 使用 ECharts 雷达图展示 5 角色在 5 维度（趋势/结构/量价/情绪/风险）的评分分布 |
| **历史查询列表** | 页面左侧或独立区域，展示当前股票的历史 AI 分析记录，支持点击查看历史报告 |
| **后端联调** | 调用 `interpret_status()` + 各角色真实 LLM API，而非 Mock |

## 五、未来改造路线（P2阶段，仅供预先规划）

```
步骤 1 — 改造进度面板（最浅）
  将 Mock 步骤改为真实 API 调用
  步骤数从 4 扩展为 5-6（增加缠论角色、筹码角色）

步骤 2 — Tab 扩展
  4Tab → 6Tab（综合 + 技术/基本面/情绪/缠论/筹码）

步骤 3 — 概览卡片结构化渲染
  将 interpret_status() 的 JSON 输出直接渲染到卡片
  操作计划/入场区间/止损/目标/风险提示各字段独立展示

步骤 4 — 共识雷达图
  引入 ECharts，在综合 Tab 中新增雷达图区域

步骤 5 — 辩论环节（核心高难度）
  实现观点冲突检测 → 触发第二轮 LLM 辩论 → 共识合成

步骤 6 — 持久化+历史
  AiAnalysis 模型建表 → 写库 → 查询接口 → 历史列表 UI
```
