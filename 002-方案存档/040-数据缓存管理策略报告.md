## 🔧 缓存数据库建立机制

### 1. 数据库架构

```sql
-- 股票缓存表
CREATE TABLE stock_cache (
    ts_code TEXT,           -- 股票代码（如：000001.SZ）
    data_type TEXT,         -- 数据类型（balance/cashflow/fina/income）
    data TEXT,              -- JSON格式的财务数据
    update_time TIMESTAMP,  -- 更新时间
    PRIMARY KEY (ts_code, data_type)
)

-- 请求计数表
CREATE TABLE request_count (
    date TEXT PRIMARY KEY, -- 日期（如：2026-05-16）
    count INTEGER DEFAULT 0 -- 当日API请求次数
)
```

### 2. 数据获取流程

```
┌─────────────────────────────────────────────────────────┐
│  1. 检查本地缓存                                          │
│  ├─ 缓存存在且未过期 → 直接返回本地数据 ✅               │
│  └─ 缓存不存在或已过期 → 进入步骤2                       │
│                                                         │
│  2. 限流检查                                             │
│  ├─ 未超限 → 继续                                        │
│  └─ 已超限 → 等待（最多等待60秒）⏳                     │
│                                                         │
│  3. 调用Tushare API                                      │
│  ├─ 成功 → 保存到本地缓存 → 返回数据 ✅                  │
│  └─ 失败 → 重试（最多3次）                              │
└─────────────────────────────────────────────────────────┘
```

### 3. 核心代码逻辑

```python
def get_stock_data_with_cache(ts_code: str, data_type: str, fetch_func):
    # 步骤1：检查缓存
    cached = cache.get(ts_code, data_type)
    if cached is not None:
        return pd.DataFrame(cached)  # 直接返回，无需API调用
    
    # 步骤2：限流等待
    rate_limiter.wait_if_needed()
    
    # 步骤3：调用API并缓存
    df = fetch_func(ts_code=ts_code)
    cache.set(ts_code, data_type, df.to_dict('records'))
    return df
```

### 2. 长期存储优势

✅ **无需重复下载**：
- 缓存数据永久保留
- 下次运行时直接使用本地数据
- 节省API调用次数（每7天更新一次即可）

✅ **降低API依赖**：
- 网络异常时仍可使用本地数据
- 避免API限流问题
- 提高系统稳定性

✅ **性能提升**：
- 本地读取速度极快（毫秒级）
- 无需等待API响应
- 筛选效率大幅提升

### 3. 当前配置

```python
# 缓存配置
CACHE_EXPIRE_DAYS = 7  # 缓存有效期（天）
```

**含义**：数据获取后保留7天，7天后自动过期，下次使用时重新获取。

## 📈 数据更新策略

### 1. 自动更新机制

```
首次运行（2026-05-16）
  ├─ 000001.SZ → 获取 → 缓存7天 → 过期日：2026-05-23
  ├─ 000002.SZ → 获取 → 缓存7天 → 过期日：2026-05-23
  └─ ...
  
第二次运行（2026-05-17）
  ├─ 000001.SZ → 缓存未过期 → 直接使用 ✅
  ├─ 000002.SZ → 缓存未过期 → 直接使用 ✅
  └─ ...

第三次运行（2026-05-24，超过7天）
  ├─ 000001.SZ → 缓存已过期 → 重新获取
  ├─ 000002.SZ → 缓存已过期 → 重新获取
  └─ ...
```

### 2. 手动更新方式

```bash
# 方式1：删除过期数据后重新获取
# 直接运行脚本，已过期数据会自动更新
python run_optimized_filter.py

# 方式2：强制刷新特定股票
# 在Python中手动清除缓存
import sqlite3
conn = sqlite3.connect('stock_cache.db')
conn.execute("DELETE FROM stock_cache WHERE ts_code = '000001.SZ'")
conn.commit()
conn.close()

# 方式3：完全清除缓存重新获取
rm stock_cache.db
python run_optimized_filter.py
```

### 3. 推荐更新频率

| 数据类型 | 更新频率 | 说明 |
|---------|---------|------|
| **财务数据** | 每周一次 | 季报/年报更新频率 |
| **行情数据** | 每日一次 | 实时性要求高 |
| **股票列表** | 每月一次 | 新股上市频率 |
