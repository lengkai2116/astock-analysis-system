### PE=0 数据的真实含义

1540 只 PE_TTM=0 的股票中：
- **平均换手率 4.24%**（高于全市场均值 4.1%）
- **平均流通市值 71亿**
- 集中行业：软件服务(141)、电气设备(84)、元器件(80)、专用机械(72)
- **不是"无数据"的废股票**，而是 Tushare 在亏损时填充 PE_TTM=0
- 这些股票中很多是正常交易的成长型公司（如科创板、创业板亏损期的科技股）

**结论**：PE=0 不可单独作为剔除条件，否则会误杀 28% 的市场，尤其伤害成长股。

### ⚠️ 行业异常：银行股

银行板块剔除率高达 **81%**（34/42只），原因是 Tushare 对银行股返回 PE_TTM=0（银行业会计准则导致），需要**金融行业豁免**：

```python
# 在 PE=0 亏损检测中加入行业豁免
if industry in ['银行', '保险', '证券', '多元金融']:
    return True  # 跳过 PE=0 检测
```

## 架构建议

### 统一过滤链

```
UnifiedRiskFilter
├── Layer -1: 大盘环境     (MarketEnvironmentFilter → 仓位管理)
├── Layer 0:  全局熔断     (CircuitBreaker → 全局控制)
├── Layer 1:  硬性否决     (六大禁区 — 逐步实现)      ← 当前阶段
├── Layer 2:  基本面量化   (ROCE + 稳健性)            ← 下阶段
├── Layer 3:  技术面风险   (流动性/筹码/缠论/资金流向) ← 当前阶段(部分)
└── Layer 4:  操作纪律     (冰箱里没有糖果黑名单)
```

### P0 阶段推荐过滤器（基于方案G）

```python
def filter_p0(self, symbol, name, industry, pe_ttm, turnover_rate, circ_mv):
    # 规则1: ST/退市 → 一票否决
    if 'ST' in name or '退市' in name:
        return False
    
    # 规则2: 低流动性 → 一票否决
    if turnover_rate < 1.0:
        return False
    
    # 规则3: 极高估值 → 一票否决 (金融除外)
    if pe_ttm > 300 and industry not in FINANCIAL_SECTORS:
        return False
    
    # 规则4: 亏损微盘 → 一票否决 (三重条件防误伤)
    if pe_ttm == 0 and circ_mv < 30 and turnover_rate < 3.0:
        if industry not in FINANCIAL_SECTORS:
            return False
    
    return True
```
