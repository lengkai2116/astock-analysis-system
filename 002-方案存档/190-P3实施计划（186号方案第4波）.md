#### #54 — 缠论"当下判断"机制

**当前状态：** `FractalDetector`（chanlun_strategy.py:248-307）使用 `confirm_bars=1` 机制，分形只有确认后才发射。无任何"未确认分形"或"实时方向识别"方法。

**实施：** 在 `FractalDetector` 中新增 `get_last_unconfirmed_fractal(highs, lows, last_confirmed_idx)` 方法：
- 从最后一个已确认分形的下一个K线开始扫描
- 检测是否正在形成新的顶分形/底分形
- 如果未确认但有潜在顶分形迹象 → 方向判定为 `PENDING_DOWN`
- 如果未确认但有潜在底分形迹象 → 方向判定为 `PENDING_UP`
- 返回 `{'has_unconfirmed': bool, 'direction': str, 'potential_type': str, 'start_idx': int}`

在 `ChanlunAnalyzer.analyze()` 末尾增加 `pending_judgment` 字段到输出结构。

#### #55 — 多因子动态权重矩阵

**当前状态：** `_compute_factor_signal`（signal_computation_service.py:1041）使用硬编码权重 `{'ROC':0.25, 'VOL_RATIO':0.20, 'ATR':0.15, 'RSI':0.25, 'LINEARREG_SLOPE':0.15}`。`StateDependentFactorWeight`（P2-#40已实现）有动态权重矩阵但未接入。

**实施：** 在 `_compute_factor_signal` 中：
1. 拿到 `market_state`（已传入上下文）
2. 实例化 `StateDependentFactorWeight` 调用 `compute_weighted_score(factor_scores, market_state)`
3. 将结果与原始加权做比较，差异>0.2时标记为 `dynamic_weight_adjusted`
4. 保留原硬编码权重作为降级后备

#### #57 — 状态依赖动态周期权重

**当前状态：** `PRIMARY_CYCLE=60` / `SECONDARY_CYCLE=120` / `EXECUTION_CYCLE=20` 三个常量定义在 `volume_price_strategy.py:282-284` 但完全未被使用。P1-#25 市场状态识别已完成。`_get_adaptive_windows` 方法存在但被注释掉。

**实施：**
1. 在 `volume_price_strategy.py` 的 `VolumePriceRelationAnalyzer.analyze()` 中根据 `market_state` 调整三个周期的权重：
   - TRENDING_BULL → PRIMARY_CYCLE权重80% / SECONDARY_CYCLE 20%
   - TRENDING_BEAR → PRIMARY_CYCLE权重40% / SECONDARY_CYCLE 60%（放大背景周期权重）
   - RANGING → PRIMARY_CYCLE权重60% / SECONDARY_CYCLE 40%
   - HIGH_VOL → PRIMARY_CYCLE权重30% / SECONDARY_CYCLE 30% / EXECUTION_CYCLE 40%
2. 在 `signal_computation_service.py` 的 `compute_for_stock()` 中将 cycle_weights 传递到各策略调用中

#### #58 — 参数平原周期性检验

**当前状态：** `optimizer.py` 已有 `PRESET_GRIDS` 四策略预置网格 + `run_preset_grid_search()` 函数。`scheduler.py` 已有月/周定时任务框架，但无任何参数优化相关定时任务。

**实施：** 在 `scheduler.py` 中新增每月定期任务：
1. 每月第一个周日运行 `run_preset_grid_search` 
2. 记录各策略最优参数与当前参数的偏差
3. 偏差>20%时发出告警日志
4. 结果写入 `logs/parameter_plateau_check.json`

#### #61 — 板块轮动模型

**当前状态：** Tushare `industry()` / `concept()` / `index_member()` API已接入（P1完成），但仅用于金融股过滤和简单行业归集。无板块轮动评分模型。

**实施：** 新建 `backend/app/engine/framework/sector_rotation_model.py`：
- `SectorRotationModel` 类
- `get_sector_data(ts_code, df)` — 获取该股所属行业的板块指数数据
- `calc_sector_strength(industry_code, all_industries)` — 计算板块相对强度（动量差）
- `calc_sector_score(ts_code, df, market_state)` — 综合评分：
  - 板块相对强度排名分位（动量差）
  - 板块内个股涨跌比（宽度）
  - 板块资金流向（成交量变化比）
- `evaluate()` 返回轮动状态：LEADING/STRENGTHENING/WEAKENING/LAGGING

**集成：** `signal_computation_service.py` 中 `compute_for_stock()` 调用 `evaluate()`，结果加权到最终信号。

#### #62 — 情绪拥挤度因子

**当前状态：** Tushare `margin()` API（融资融券）已接入（P1-P2完成）但未用于策略。无任何拥挤度因子。

**实施：** 新建 `backend/app/engine/framework/crowding_factor.py`：
- `CrowdingFactor` 类
- `calc_margin_ratio(margin_data)` — 融资余额/流通市值比 → 拥挤度指标
- `calc_turnover_ratio(volumes, cap)` — 换手率拥挤度
- `calc_concentration(highs, lows, closes)` — 价格收敛度（波动率下降=拥挤）
- `calc_crowding_score(margin_df, df, market_state)` — 综合评分：
  - 融资净买入/余额变化率 > 20% → HIGH_CROWDING
  - 换手率 > 95%分位 → HIGH_CROWDING
  - 布林带宽度 < 20%分位 → HIGH_CROWDING
  - 综合：2/3项满足 → 拥挤警告
- `evaluate(ts_code, df)` → 返回挤压度等级 + 风险建议

**集成：** `signal_computation_service.py` 中加权到因子信号通道。

#### #63 — 鹰眼大宝剑双系统共振

**当前状态：** `ResonanceService`（resonance_service.py）已实现三策略共振评分，但无专门的"趋势系统+情绪系统"双共振模型。

**实施：** 新建 `backend/app/engine/framework/eagle_sword_resonance.py`：
- `EagleSwordResonance` 类
- **鹰眼系统（趋势系统）**：
  - 缠论方向判定（上涨/下跌/盘整）
  - 量价趋势强度（MA排列+ADX）
  - 格兰维尔信号（买入/卖出）
- **大宝剑系统（情绪系统）**：
  - BOCIASI快线信号强度
  - BOCIASI慢线风险等级
  - CrowdingFactor拥挤度信号
- **双系统共振判定**：
  - 趋势向上 + 情绪积极 → BUY (confidence 0.85+)
  - 趋势向上 + 情绪中性 → WATCH_BUY (confidence 0.60)
  - 趋势向下 + 情绪消极 → SELL (confidence 0.85+)
  - 趋势向下 + 情绪中性 → WATCH_SELL (confidence 0.60)
  - 趋势中性 + 情绪积极 → CAUTIOUS_BUY (confidence 0.50)
  - 趋势中性 + 情绪消极 → CAUTIOUS_SELL (confidence 0.50)
  - 趋势向上 + 情绪消极 → CONFLICT (降低持仓)
  - 趋势向下 + 情绪积极 → CONFLICT (观望)
- `evaluate(chanlun_result, volume_price_signal, bociasi_quick, bociasi_slow, crowding, market_state)` → 双系统共振信号
- 输出：'action' + 'confidence' + 'system_state'（鹰眼+大宝剑各自状态）

**集成：** `signal_computation_service.py` 中 `compute_for_stock()` 调用 `evaluate()`，结果作为顶层综合信号。

#### #56 — CSCV计算过拟合概率

**当前状态：** 完全不存在。需从零实现组合对称交叉验证（Combinatorial Symmetric Cross-Validation）。

**实施：** 新建 `backend/app/engine/framework/cscv_validator.py`：
- `CSCVValidator` 类
- 核心方法 `compute_pbo(signals, returns, n_splits=6, n_trials=1000)`：
  1. 将时间序列分为N个等分块
  2. 从N个块中选K个作为训练集（所有组合）
  3. 剩余N-K个作为测试集
  4. 对每个组合：训练集上寻找最优参数 → 测试集上计算夏普比
  5. 构建夏普比矩阵（行=组合, 列=参数配置）
  6. 计算PBO（Probabilistic Backtest Overfitting）：夏普比排名优于中位数的概率
- `calc_rank_probability(sharpe_matrix)` — 计算过拟合概率
- `evaluate_strategy(strategy_name, param_grid, historical_returns)` — 综合评估
- `is_robust(pbo, threshold=0.05)` → PBO < 5% 为稳健

**集成：** 作为独立验证工具，暂不自动接入实时策略流。后续可在scheduler中定期执行。
