## 2. 回测的目的和意义

### 2.1 核心目的

根据知识库中[策略回测与评估](concepts/celve-huice-yu-pinggu.md)的概念定义，回测的根本目的在于：

1. **验证策略有效性**：在历史数据上模拟策略运行，检验策略是否具备统计意义上的正期望收益
2. **评估稳健性**：通过多种验证方法（走步前推、CSCV、蒙特卡洛等）判断策略表现的稳定性，区分"真实信号"与"数据噪声"
3. **过拟合检测**：识别策略是否过度拟合历史数据——这是量化策略失败的最主要原因

### 2.2 核心误区警示

知识库特别强调了以下认知陷阱：

- **"高夏普比率 = 好策略"**：夏普比率假设收益服从正态分布，但A股具有典型厚尾特征，高夏普可能由极端交易日集中贡献（[[90%收益来自2%交易日]]）
- **"回测三年年化30% = 好策略"**：A股特定风格可能在三年内持续跑赢，但风格切换后策略可能大幅回撤（[[风格时变性]]）
- **"优化后参数更优"**：每次优化消耗研究员自由度，最终参数可能仅在测试集上有效（[[优化悖论]]）

### 2.3 多重验证铁律

知识库中的[多重验证铁律](concepts/多重验证铁律.md)明确要求：**至少采用三种独立的方法对同一策略进行稳健性验证**，以避免单一评估指标（如夏普比率、年化收益率）导致的[[IRR陷阱]]。

三道核心验证：
1. **蒙特卡洛检验** — 随机打乱交易顺序或生成随机价格路径
2. **组合对称交叉验证（CSCV）** — Bailey等（2014）提出的PBO评估框架
3. **走步前推与滚动验证** — 模拟实盘过程

### 2.4 偏差防控意义

回测的可靠性核心在于**数据的时间准确性**。知识库强调了：

- **前视偏差**（Look-ahead Bias）：回测中最严重的陷阱，因使用未来数据导致绩效严重失真
- **隐性前视偏差**（Universe Leakage）：先删除无标签样本后标准化，导致统计量被未来数据污染
- **PIT数据库**（Point-in-Time）：确保回测使用数据与该交易日实际可获取的数据完全一致

## 3. 回测方法与流程

### 3.1 基本回测方法

| 方法 | 描述 | 来源页面 |
|------|------|----------|
| **历史回测（标准回测）** | 使用完整历史数据对策略进行一次性回测 | 策略回测与评估 |
| **滚动回测** | 固定窗口长度，向前滚动重复测试 | 策略回测与评估 |
| **蒙特卡洛模拟** | 随机生成替代价格路径或打乱交易顺序，评估收益分布和破产概率 | 蒙特卡洛檢驗 / 多重验证铁律 |

### 3.2 高级验证方法

#### 3.2.1 走步前推（Walk-Forward Analysis）

知识库中[走步前推](concepts/走步前推.md)详细定义了四种变体：

| 变体 | 说明 |
|------|------|
| **固定窗口** | 训练窗口长度固定，向前滚动 |
| **扩张窗口** | 累积所有历史数据 |
| **滚动窗口** | 固定长度，向前滚动（最标准形式） |
| **锚定窗口** | 训练期起点固定，终点滚动 |

**核心输出指标——一致性（Consistency）**：

`一致性 = 1 − |训练胜率 − 验证胜率| / 训练胜率`

| 一致性范围 | 解读 |
|-----------|------|
| ≥ 0.95 | 几乎完美（没有过拟合） |
| 0.85 ~ 0.95 | 轻微过拟合 |
| 0.75 ~ 0.85 | 明显过拟合 |
| < 0.75 | 严重过拟合 |

**最低验证标准**：
- 夏普衰减 < 0.3
- 收益衰减比例 < 30%
- 回撤恶化 < 5%
- 模拟实盘夏普 > 0.5

#### 3.2.2 遮盖式走步回测（Masked Walk-Forward）

来自LLM-TRADE框架的进阶方法，通过遮盖未来数据块防止前视偏差。相比标准走步前推，更严格地控制了信息泄露风险。

#### 3.2.3 组合对称交叉验证（CSCV）

[组合对称交叉验证（CSCV）](concepts/组合对称交叉验证cscv.md)由Bailey等（2014）提出，是量化回测过拟合概率（PBO）的标准评估框架：

**核心机制**：
1. 将时间序列划分为S个等长子样本
2. 生成所有可能的训练集-测试集组合（C(S, S/2)种）
3. 在每个组合中优化参数并评估
4. 计算最优策略在CSCV结果中的排名分位数 → PBO

**PBO解读**：
- **PBO < 20%**：策略稳健性高，可考虑部署
- **PBO 20%-30%**：策略稳健性较好
- **PBO 30%-60%**：中等过拟合风险
- **PBO > 60%**：极高风险，回测结果不可信

**A股实证结果**（来自知识库）：
| 算法 | PBO | 评价 |
|------|-----|------|
| XGBoost | 8.6%–14.7% | 过拟合风险最低 |
| SVM | 35.8%–37.1% | 中等风险 |
| LSTM | 48.4%–57.2% | 高过拟合风险 |

#### 3.2.4 双重随机化验证

[双重随机化验证](concepts/双重随机化验证.md)通过双重随机机制测试策略的泛化稳健性：

1. **标的随机**：从未测试过的股票中随机抽取
2. **时间窗口随机**：随机截取365-730天连续区间
3. **重复成百上千次** → 得到策略表现的概率分布

有效对抗针对特定标的或特定时间窗口的过拟合。

#### 3.2.5 参数平原分析

[参数平原](concepts/参数平原.md)（Parameter Plateau）判断策略是否捕捉到真实市场机制：

| 判据 | 要求 |
|------|------|
| 宽度 | 参数跨度 ≥ 30%（如窗口20→40） |
| 深度 | 绩效波动 < 15% |
| 连通性 | 单连通区域，无断裂 |

参数平原 vs 参数山峰：
- **平原**：低敏感度、高稳健性、低过拟合风险
- **山峰**：高敏感度、低稳健性、高过拟合风险

#### 3.2.6 融合框架回测验证

[融合框架回测验证](concepts/ronghe-kuangjia-huice-yanzheng.md)覆盖了三个核心方法：

| 方法 | 说明 |
|------|------|
| **路径优选与多维打分合成** | 多重路径选择与综合评分 |
| **多层收缩与正则化** | 防止复杂模型过拟合 |
| **状态识别与动态调制HMM+** | 隐马尔可夫模型识别市场状态并动态调整 |

**研究机构结果**：华泰证券融合框架夏普1.72，中邮证券夏普1.29。

### 3.3 完整回测流程

知识库综合多个概念页面的内容，回测标准流程如下：

```
┌─────────────────────────────────────────────────────────────────┐
│ 第一层：数据准备                                                 │
│  ├─ PIT数据库（Point-in-Time）确保时间准确性                      │
│  ├─ 因子计算（避免 rolling center=True / shift(-1)）             │
│  ├─ 截面标准化（先标准化，再删除无标签样本）                      │
│  └─ 严格按日期分组操作                                          │
├─────────────────────────────────────────────────────────────────┤
│ 第二层：基础回测                                                 │
│  ├─ 历史回测（完整数据一次性运行）                                 │
│  └─ 滚动回测（固定窗口向前滚动）                                  │
├─────────────────────────────────────────────────────────────────┤
│ 第三层：偏差审计                                                 │
│  ├─ 前视偏差四层审计：因子层 → 信号层 → IC层 → 回测层             │
│  ├─ 隐性前视偏差（Universe Leakage）检测                          │
│  └─ T+1规则验证（信号日T → 执行日T+1）                           │
├─────────────────────────────────────────────────────────────────┤
│ 第四层：稳健性验证（多重验证铁律）                                 │
│  ├─ 走步前推（一致性指标 ≥ 0.85）                                │
│  ├─ 组合对称交叉验证CSCV（PBO < 20%）                             │
│  ├─ 蒙特卡洛模拟（收益/回撤分布）                                 │
│  ├─ 双重随机化验证（随机标的+随机时间窗口）                        │
│  ├─ 参数平原分析（宽度≥30%，波动<15%）                            │
│  └─ 遮盖式走步回测（可选）                                        │
├─────────────────────────────────────────────────────────────────┤
│ 第五层：综合评估                                                 │
│  ├─ 多元指标评估（夏普/索提诺/卡玛/信息率/RAR/R立方等）           │
│  ├─ 一致性分析（走步前推的验证流程间一致性）                       │
│  ├─ 自由度数审计（追踪研究员自由度消耗）                           │
│  └─ 基准对比（HS300/ZZ500等）                                     │
└─────────────────────────────────────────────────────────────────┘
```

### 3.4 关键指标体系

知识库中涉及的所有绩效指标分类：

| 类别 | 指标 | 来源 |
|------|------|------|
| **收益类** | 总收益（Total Return）、年化收益（Annual Return）、超额收益 | V2引擎 + 知识库 |
| **风险类** | 波动率（Volatility）、最大回撤（Max Drawdown）、回撤恢复时间 | V2引擎 + 知识库 |
| **风险调整收益** | 夏普比率（Sharpe Ratio）、索提诺比率（Sortino Ratio）、卡玛比率（Calmar Ratio）、卡尔玛比率、信息比率（Information Ratio）、RAR、R立方 | 多重验证铁律 |
| **交易类** | 胜率（Win Rate）、盈亏比（Profit/Loss Ratio）、交易次数 | V2引擎 |
| **稳健性** | PBO过拟合概率、走步前推一致性、参数平原宽度/深度/连通性、置信区间 | CSCV/走步前推/参数平原 |
| **基准对比** | 基准收益（Benchmark Return）、超额收益（Excess Return）、Beta、跟踪误差（Tracking Error）、信息比率（Information Ratio） | BenchmarkService |

### 3.5 过拟合检测体系

知识库覆盖了完整的过拟合检测体系：

| 检测方法 | 核心指标 | 判据 |
|----------|---------|------|
| 走步前推 | 一致性（Consistency） | ≥ 0.85合格，≥ 0.95优秀 |
| CSCV | PBO（过拟合概率） | < 20%低风险，> 60%不可信 |
| 参数平原 | 宽度≥30%+波动<15%+单连通 | 满足三条=捕捉真实机制 |
| 双重随机化 | 多轮统计显著性 | 置信区间不包含零 |
| 自由度数审计 | 自由度消耗总量 | 需设定预算上限 |
| 研究员自由度 | 每一步消耗 | 线性增加验证步骤，固定大部分决策 |
| 前视偏差审计 | 四层审计清单通过率 | 零Critical级问题 |

### 3.6 偏差防控体系

| 偏差类型 | 描述 | 防控方法 |
|----------|------|----------|
| 前视偏差（Look-ahead） | 使用未来数据 | 四层审计流程 + PIT数据库 |
| 隐性前视偏差（Universe Leakage） | 标准化顺序错误 | 先标准化，再删样本 |
| 幸存者偏差 | 只考虑存续标的 | PIT数据库 + 双重随机化 |
| 时间窗口选择偏差 | 选取有利时间段 | 走步前推 + 双重随机化 |
| 参数优化悖论 | 过度优化消耗自由度 | 参数平原 + 自由度数审计 |

### 4.4 差距分析总结

#### 4.4.1 按优先级分组

| 优先级 | 差距项数 | 说明 |
|--------|---------|------|
| 🔴🔴 严重（P0级） | 7项 | 走步前推/一致性、CSCV/PBO、蒙特卡洛、双重随机化、参数平原、基准对比、前视偏差审计 |
| 🔴 中（P1级） | 12项 | 更多指标展示、回撤曲线后端生成、遮盖式回测、融合框架、多标的回测等 |
| 🟡 轻量（P2级） | 7项 | Mock转真实对接、交易规则展示、佣金配置等 |
| 🟢 低（P3级） | 3项 | 偏度、热力图、研究员自由度等 |

#### 4.4.2 核心缺口

**最大缺口1：稳健性验证完全缺失**
知识库最强调的就是"单一回测指标不可信，必须多重验证"。当前系统没有任何稳健性验证功能（走步前推、CSCV、蒙特卡洛、双重随机化），这是从"玩具回测"到"专业回测"的关键跨越。

**最大缺口2：基准对比缺失**
后端BenchmarkService提供了7个指数和完整的基准对比指标，但前端不仅没有展示，甚至连基准选择入口都没有。没有基准对比，回测收益指标缺乏参照系。

**最大缺口3：偏差防控缺失**
没有前视偏差审计工具，没有参数平原可视化，回测结果可信度无从验证。CSCV的PBO计算已实现（cscv_validator.py, #56），参数平原网格搜索已实现（optimizer.py, #45），但完全没有前端入口。

**最大缺口4：指标展示不完整**
后端引擎产出17个指标，但前端仅展示5个Mock指标——12个已有能力被浪费。

## 6. 结论

**知识库维度**：LLM Wiki知识库在量化策略回测领域的内容深度和广度远超预期。它不仅覆盖了基础的回测方法和指标（历史回测、滚动回测、夏普比率等），更完整收录了学界和业界的前沿验证体系——Bailey等（2014）的CSCV/PBO框架、LLM-TRADE的遮盖式走步回测、GroAries的双重随机化验证、以及融合框架回测的券商级研究成果。**知识库中直接相关概念页面12+个，间接相关30+个，构成了一个从基础到前沿的完整回测方法论体系。**

**系统现状维度**：当前回测系统存在严重的"头重脚轻"——后端引擎已达到专业水平（A股规则完整、指标丰富、基准对比完善），但前端UI仅展示了Mock数据，且知识库中80%以上的方法论（稳健性验证、偏差防控、过拟合检测）在当前前端完全不可见。

**核心差距**：从"功能型回测"到"专业级回测"的跨越，关键在于三层能力建设：
1. **展示层**：后端17个指标 → 前端展示（已具备基础，需对接）
2. **验证层**：CSCV/PBO + 参数平原（后端已实现，#45/#56） → 前端可视化（差距最小，收益最大）
3. **评估层**：走步前推 + 蒙特卡洛 + 双重随机化（需新开发） → 前端集成（中长期）

**推荐优先顺序**：基础对接（P0） → CSCV+参数平原可视化（P0，后端已有） → 走步前推+蒙特卡洛（P1） → 偏差审计工具（P1）
