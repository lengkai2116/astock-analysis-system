## 三、DeepSeek AI API配置说明

### 3.1 配置步骤

#### 步骤1：获取DeepSeek API Key

1. 访问 [DeepSeek API](https://platform.deepseek.com/)
2. 注册并登录账号
3. 在控制台创建新的API Key

#### 步骤2：更新.env配置

```bash
# .env文件
DEEPSEEK_API_KEY=your-actual-api-key-here
DEEPSEEK_BASE_URL=https://api.deepseek.com/v1
DEEPSEEK_MODEL=deepseek-chat

# 选择使用DeepSeek
LLM_PROVIDER=deepseek
```

#### 步骤3：重启后端服务

```bash
cd backend
python run.py
```

### 3.2 配置验证

```bash
# 测试API调用
curl -X POST http://localhost:5000/api/v2/reports/strategy/generate \
  -H "Content-Type: application/json" \
  -d '{"description": "均线金叉策略"}'
```

### 3.3 支持的LLM类型

| 类型 | LLM_PROVIDER | 说明 |
|-----|-------------|------|
| DeepSeek | `deepseek` | DeepSeek官方API（推荐） |
| LM Studio | `lm_studio` | 本地大模型（需运行LM Studio） |
| Mock | `mock` | 模拟模式（默认，无需配置） |

## 八、风险与注意事项

### 8.1 DeepSeek API使用注意

1. **API Key安全**：不要将真实API Key提交到代码仓库
2. **费用控制**：DeepSeek API按token计费，设置合理的预算
3. **错误处理**：API调用失败时系统会自动降级到Mock模式

### 8.2 性能注意

1. **回测性能**：多标的回测可能需要较长时间，考虑异步处理
2. **AI生成**：DeepSeek API调用可能需要几秒钟，添加加载提示

### 8.3 数据库注意

1. **迁移顺序**：先运行迁移，再初始化预设模板
2. **数据备份**：重要数据操作前先备份
