## 一、书本策略体系的完整还原

### 1.1 全书8章的逻辑链条

```
第1章：筹码是什么 → 主力vs散户的持仓成本特征 → 主力操盘过程
         ↓
第2章：主力操盘在筹码上的具体表现 → ASR/CYQKL/SSRP三个核心信号
         ↓
第3章：价格异动 → 分时图信号(5种) + K线形态(4种) → 价格信号确认
         ↓
第4章：建仓阶段 → 4种抢筹方式 → 筹码向低位集中
         ↓
第5章：洗盘阶段 → 5种洗盘手法 → 筹码换手+量能萎缩
         ↓
第6章：拉升阶段 → 5种筹码转移模式 → 低位→高位转移
         ↓
第7章：出货阶段 → 5种卖点形态 → 筹码向高位集中完毕
         ↓
第8章：下跌阶段的4种支撑形态 → 在支撑位寻找反弹机会
```

### 3.2 各模块详细设计

#### 模块1：PreFilter（前置过滤层）

**类名**: `ChipPreFilter`

**策略逻辑**:

```
步骤1：大盘环境检查
──────────────────
输入: 沪深300日线数据（QMT实时）
计算: close > ma60(沪深300)
输出: {'condition': 'GOOD', 'position_multiplier': 1.0}
      或 {'condition': 'POOR', 'position_multiplier': 0.5}
影响: 所有后续信号的仓位建议乘以position_multiplier
书本依据: 第7章§7.4 — "大盘环境不佳时总仓位上限降至40%"

步骤2：新股排除
──────────────────
输入: ts_code + 上市日期（QMT财务数据）
判断: 上市日期 至今 >= 250个交易日
排除: 不满足条件的股票
书本依据: 第9章§9.6 — "上市不满1年排除"

步骤3：流动性过滤
──────────────────
输入: 换手率（QMT日线）
判断: 换手率 >= 2%
排除: 换手率<2%的股票
书本依据: 第3章§3.6 — "换手率<2%为低活跃度，不适合操作"

步骤4：市值适配
──────────────────
输入: 流通市值（QMT财务数据）
输出: 不同市值对应的参数集
       小盘<50亿 → 标准参数
       中盘50~200亿 → 放宽成交量倍数0.2
       大盘200~1000亿 → CYQKL阈值降低5
       超大盘>1000亿 → 不作为主策略
书本依据: 第9章§9.4 — "不同市值适用性差异"
```

**文件**: `backend/app/engine/framework/chip_pre_filter.py`（新建）

#### 模块2：ChipAnalyzer（筹码分布分析层）

**类名**: `ChipAnalyzer`（重构现有 `ChipIndicators` + `ChipDistributionService`）

**新增功能**:

```
2.1 主力/散户筹码区分
──────────────────
方法: 持仓稳定性法 + 价格位置法
逻辑:
  A. 扫描所有price_bin，追踪连续n日筹码比例变化
  B. 连续60日未显著变化(标准差<阈值)的price_bin → 标记为"稳定筹码"(≈主力)
  C. 底部30%价格区间的筹码 → 优先标记为主力
  D. 剩余筹码 → 标记为"浮筹"(≈散户)
输出: chip_bins 增加 {holder_type: 'major'/'retail'} 字段
书本依据: 第1章§1.1.2 — "主力筹码集中度较高、存在于价格低位、转移速度慢"

2.2 筹码峰类型识别（增强版）
──────────────────
新增识别:
  - 脉冲筹码峰: >=3个峰 + 相邻间距>5% + 各峰5%~15%
  - 历史筹码峰: 对比60日前的筹码分布，找出留存峰
书本依据: 第4章§4.1.4

2.3 支撑位/阻力位识别
──────────────────
逻辑:
  - 找出所有显著的筹码峰(ratio>3%)
  - 低于当前价的筹码峰 → 支撑位
  - 高于当前价的筹码峰 → 阻力位
  - 按筹码量排序，标记TOP3支撑和TOP3阻力
输出: {'support_levels': [price1, price2, ...],
       'resistance_levels': [price1, price2, ...]}
书本依据: 第8章全章
```

**文件**: 重构 `backend/app/engine/framework/chip_analyzer.py`

#### 模块3：PhaseDetector（操盘阶段判定层）

**类名**: `PhaseDetector`（重写现有 `TradingPhaseDetector`）

**设计要点**:

```
阶段判定状态机（每个阶段独立评分+置信度）：

┌──────────────────────────────────────────────────────────┐
│ PhaseDetector.detect()                                    │
│                                                          │
│ 输入: chip_bins + indicators + kline_data                │
│                                                          │
│ 计算各阶段得分:                                           │
│   building_score = Σ(条件J1~J6中满足的)                    │
│   washing_score = Σ(条件X1~X6中满足的)                    │
│   raising_score = Σ(条件L1~L6中满足的)                    │
│   shipping_score = Σ(条件C1~C6中满足的)                   │
│   support_score = Σ(条件D1~D4中满足的)                    │
│                                                          │
│ 最高分 → 当前阶段                                        │
│ 置信度 = 最高分 / 次高分                                 │
│                                                          │
│ 输出: {'phase': 'RAISING',                               │
│        'confidence': 0.85,                               │
│        'scores': {...},                                  │
│        'phase_history': [...]}                           │
└──────────────────────────────────────────────────────────┘
```

**阶段切换规则**（防止频繁振荡）：

```
阶段切换容忍度:
  - 仅当新阶段得分 >= 当前阶段得分 × 1.2 时才切换
  - 建仓→洗盘：低位筹码峰形成确认
  - 洗盘→拉升：量能持续放大>=5日
  - 拉升→出货：高位筹码>=50%
  - 出货→下跌：跌破筹码主峰
```

**文件**: 重写 `backend/app/engine/framework/phase_detector.py`

class KLinePatternVerifier:
    """K线形态识别器"""
    
    @dataclass
    class Pattern:
        name: str          # 'big_yang' / 'doji' / 'gap_up' / 'gap_down' / '冲高回落' / '高开低走大阴线'
        strength: float    # 0~1
        direction: str     # 'bullish' / 'bearish'
    
    def verify(self, df: pd.DataFrame, indicator_df: pd.DataFrame) -> List[Pattern]:
        """识别所有形态"""
        patterns = []
        
        # 大阳线: 实体 > 振幅×0.7 AND 涨幅 > 3%
        if self._is_big_yang(df):
            patterns.append(Pattern('big_yang', 0.7, 'bullish'))
        
        # 十字星: 实体 < 振幅×0.1
        if self._is_doji(df):
            patterns.append(Pattern('doji', 0.5, 'neutral'))
        
        # 跳空高开: open > prev_high × 1.005
        if self._is_gap_up(df):
            patterns.append(Pattern('gap_up', 0.6, 'bullish'))
        
        # 跳空低开: open < prev_low × 0.995
        if self._is_gap_down(df):
            patterns.append(Pattern('gap_down', 0.6, 'bearish'))
        
        # 冲高回落: high > close × 1.05 AND 实体 < 振幅×0.3
        if self._is_high_wave(df):
            patterns.append(Pattern('冲高回落', 0.65, 'bearish'))
        
        # 高开低走大阴线: open > prev_close AND close < open AND 实体 > 振幅×0.5
        if self._is_big_bear_high_open(df):
            patterns.append(Pattern('高开低走大阴线', 0.7, 'bearish'))
        
        return patterns
```

class ConfirmLayer:
    """多层辅助验证"""
    
    def verify(self, signal: Dict, context: Dict) -> Dict:
        """
        对候选信号进行多层验证，返回调整后的置信度和仓位
        """
        confidence = signal['confidence']
        position = signal['position']
        
        # 6.1 换手率验证
        turnover = context.get('turnover_rate', 0)
        if turnover < 0.02:
            confidence *= 0.5  # 低换手 → 折扣
        elif turnover > 0.10:
            if signal['signal_type'] == 'SELL':
                confidence *= 1.2  # 高换手 → 卖出加强
            else:
                confidence *= 0.8  # 高换手 → 买入警惕
        
        # 6.2 转移速度验证
        transfer_speed = context.get('transfer_speed', 0)
        if transfer_speed > 0.01:   # >1%/日 → 加速
            if signal['signal_type'] == 'BUY':
                confidence *= 0.8  # 加速期追高有风险
                position *= 0.7
        
        # 6.3 五档盘口不平衡验证（QMT）
        if context.get('has_qmt', False):
            imbalance = context.get('order_imbalance', 0)
            # 买入时检查买盘 > 卖盘
            if signal['signal_type'] == 'BUY' and imbalance > 0.2:
                confidence *= 1.15
            elif signal['signal_type'] == 'SELL' and imbalance < -0.2:
                confidence *= 1.15
            # 信号与盘口相反 → 折扣
            elif signal['signal_type'] == 'BUY' and imbalance < -0.1:
                confidence *= 0.8
        
        # 6.4 假突破过滤器
        if not self._false_breakout_check(context):
            return {'action': 'HOLD', 'reason': '假突破过滤', 'position': 0}
        
        return {'confidence': confidence, 'position': position}
    
    def _false_breakout_check(self, context) -> bool:
        """假突破过滤器 - 三维确认"""
        # 书本第9章§9.3
        breakout_day = context.get('breakout_day', {})
        
        # 维度1：量能确认
        if breakout_day.get('vol_ratio', 0) < 1.5:
            return False
        
        # 维度2：时间确认（突破后3日未回落）
        if not self._check_breakout_duration(context):
            return False
        
        # 维度3：深度确认
        if breakout_day.get('cyqkl', 0) < 0.2:
            return False
        
        return True
```

class ChipPositionManager:
    """基于操盘阶段+多层风控的仓位管理"""
    
    def calculate_target_position(
        self,
        phase: str,                  # 操盘阶段
        signal: Dict,                # 信号详情
        market_condition: str,       # 大盘环境 'GOOD'/'POOR'
        current_position: float,     # 当前仓位 %
        has_position: bool           # 是否已持仓
    ) -> Dict:
        """计算目标仓位"""
        
        # 步骤1：基础仓位（按阶段）
        base_position = self._phase_base_position(phase)
        
        # 步骤2：信号调整（按信号类型）
        signal_adjustment = self._signal_adjustment(signal)
        
        # 步骤3：大盘环境调整
        market_multiplier = 1.0 if market_condition == 'GOOD' else 0.5
        
        # 步骤4：熔断检查
        circuit_breaker = self._check_circuit_breaker()
        
        target = base_position * signal_adjustment * market_multiplier * circuit_breaker
        
        # 步骤5：限仓
        max_allowed = {
            'BUILDING': 0.3, 'WASHING': 0.5, 'RAISING': 0.7,
            'SHIPPING': 0.0, 'SUPPORT': 0.3
        }.get(phase, 0.3)
        if market_condition == 'POOR':
            max_allowed = min(max_allowed, 0.4)
        
        target = min(target, max_allowed)
        
        return {'target_position': target, 'action': 'BUY' if target > current_position else 'SELL'}
    
    def _phase_base_position(self, phase: str) -> float:
        return {
            'BUILDING': 0.3,     # 建仓期：最大30%
            'WASHING': 0.5,      # 洗盘期：最大50%
            'RAISING': 0.7,      # 拉升期：最大70%
            'SHIPPING': 0.0,     # 出货期：清仓
            'SUPPORT': 0.3       # 支撑期：最大30%
        }.get(phase, 0.3)
    
    def _check_circuit_breaker(self) -> float:
        """
        熔断检查（书本第9章§9.5）
        大盘单日跌幅>5% → 平仓50%
        大盘连续3日下跌且累计>8% → 清空
        """
        # 需要接入指数行情
        return 1.0  # 正常情况

```python
class ChipRiskExecutor:
    """止损止盈执行引擎"""
    
    def __init__(self):
        self.entry_prices: Dict[str, float] = {}     # 入场价格
        self.entry_dates: Dict[str, int] = {}         # 入场日期
        self.peak_prices: Dict[str, float] = {}       # 最高价跟踪
        self.entry_chip_bins: Dict[str, List] = {}    # 入场时筹码数据
    
    def execute(self, portfolio: Dict, market_data: Dict) -> List[Dict]:
        """对所有持仓执行风控检查"""
        actions = []
        
        for symbol, pos in portfolio.items():
            current_price = market_data.get_price(symbol)
            entry_price = self.entry_prices.get(symbol)
            peak_price = self.peak_prices.get(symbol, entry_price)
            
            # 更新最高价
            if current_price > peak_price:
                self.peak_prices[symbol] = current_price
            
            # 8.1 硬止损（书本第7章§7.2）
            if entry_price and current_price < entry_price * 0.92:
                actions.append({'symbol': symbol, 'action': 'SELL_ALL', 'reason': '硬止损(8%)'})
                continue
            
            # 8.2 跟踪止损（书本第7章§7.2）
            if peak_price > entry_price:
                drawdown = (peak_price - current_price) / peak_price
                if drawdown > 0.15:
                    actions.append({'symbol': symbol, 'action': 'SELL_50', 'reason': '跟踪止损(回撤15%)'})
            
            # 8.3 时间止损（书本第7章§7.2）
            holding_days = self.entry_dates.get(symbol, 0)
            if holding_days >= 20 and current_price <= entry_price * 1.01:
                actions.append({'symbol': symbol, 'action': 'SELL_50', 'reason': '时间止损(20日未盈利)'})
        
        return actions
    
    def check_take_profit(self, symbol: str, indicators: Dict, 
                          chip_bins: List[Dict], current_price: float) -> List[Dict]:
        """止盈检查（书本第7章§7.3）"""
        actions = []
        
        # 8.4 筹码止盈
        if self._chip_take_profit(chip_bins):
            actions.append({'action': 'SELL_70', 'reason': '筹码止盈(低位筹码峰消失)'})
        
        # 8.5 RSI止盈
        rsi = indicators.get('rsi', 50)
        if rsi >= 85 and self._rsi_high_consecutive(3):
            actions.append({'action': 'SELL_50', 'reason': 'RSI止盈(≥85持续3日)'})
        
        # 8.6 放量滞涨止盈
        if indicators.get('vol_ratio', 0) >= 2.0 and abs(indicators.get('return', 0)) < 0.005:
            actions.append({'action': 'SELL_30', 'reason': '放量滞涨止盈'})
        
        # 8.7 移动止盈
        peak = self.peak_prices.get(symbol, current_price)
        if (peak - current_price) / peak > 0.10:
            actions.append({'action': 'SELL_ALL', 'reason': '移动止盈(回撤10%)'})
        
        return actions
    
    def _chip_take_profit(self, chip_bins) -> bool:
        """检查低位筹码峰是否接近消失"""
        # 底部30%区间筹码量 < 10%
        low_chip_ratio = sum(c['chip_ratio'] for c in chip_bins 
                            if c['price_bin'] <= 最低价 + 价格范围*0.3)
        return low_chip_ratio < 0.10
```
