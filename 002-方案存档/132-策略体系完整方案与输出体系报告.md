## 二、核心策略一：筹码分布策略（Chip Distribution Strategy）

**文件位置**: `backend/app/engine/framework/chip_strategy.py` / `backend/app/engine/pipeline.py` / `backend/app/engine/chip_strategy_impl.py`

**实现状态**: 完整实现，为系统最成熟策略

### 2.1 设计依据

基于《筹码分布量化策略技术说明书》，通过筹码分布数据识别主力资金行为。

### 2.2 子模块详述

#### 2.2.1 ChipUniverseSelectionModel（股票池选择）

```
类: ChipUniverseSelectionModel
功能: 从全市场筛选具备主力条件的股票
参数:
  - lookback_period: 120 (回看周期)
  - top_n: 50 (输出前N只)
核心: ChipScorer.score() 对每只股票评分
输出: 股票代码列表，按评分降序排列
```

#### 2.2.2 ChipAlphaModel（信号生成）

```
类: ChipAlphaModel
功能: 生成筹码分布交易信号Insight
信号阈值:
  - score >= 7.0 → LONG, 置信度 min(0.9, score/10), 权重0.8
  - score >= 5.0 → LONG, 置信度 min(0.7, score/10), 权重0.5
  - score >= 3.0 → FLAT (观望)
  - score < 3.0  → SHORT (做空)
```

#### 2.2.3 ChipScorer（评分器）

```
类: ChipScorer
功能: 基于OHLCV数据计算综合评分(0-10)
评分维度:
  1. 价格动量 (30%权重)
     - recent_return > 0: +3.0
     - recent_return > -0.05: +1.5
  2. 成交量评分 (40%权重)
     - vol_ratio >= 1.5: +4.0
     - vol_ratio >= 1.2: +2.5
     - 基准: +1.0
  3. 价格位置评分 (30%权重)
     - 30%~60%分位: +3.0
     - 60%~80%分位: +2.0
     - 其他: +1.0
```

#### 2.2.4 ChipRiskManagementModel（风险管理）

```
类: ChipRiskManagementModel
参数:
  - stop_loss_pct: 0.08 (止损8%)
  - take_profit_pct: 0.15 (止盈15%)
  - max_single_position_pct: 0.2 (单票最大20%)
  - max_total_exposure: 0.8 (总仓位上限80%)
功能:
  1. 检查止损止盈
  2. 控制单票最大仓位
  3. 总仓位风险缩减
```

#### 2.2.5 TradingPhaseDetector（操盘阶段检测）

**文件**: `backend/app/engine/chip_strategy_impl.py`

```
类: TradingPhaseDetector
检测5个操盘阶段:

1. 建仓期 (BUILDING) - 评分维度
   - 低位筹码峰: 底部30%价格区间筹码>=40%
   - ASR>=70持续>=10交易日
   - 横盘时间: 价格±15%范围>=60交易日
   - 股价处于250日低30%范围

2. 洗盘期 (WASHING) - 评分维度
   - 股价从高点回落10%-33%
   - 量能萎缩
   - RSI在30-55区间
   - 价格未破主力成本(SSRP)

3. 拉升期 (RAISING) - 评分维度
   - 低位筹码减少
   - 高位筹码增加
   - 成交量放大
   - 价格突破
   - ASR快速回落

4. 出货期 (SHIPPING) - 评分维度
   - 高位筹码峰形成
   - 低位筹码基本消失
   - RSI顶背离
   - 量能萎缩
   - 价格跌破高位筹码峰

5. 下跌支撑期 (SUPPORT) - 评分维度
   - 低获利(profit_ratio<35%)
   - RSI超卖(<30)
   - 价格低于SSRP的90%
```

#### 2.2.6 ChipDistributionSignalGenerator（信号生成器）

```
类: ChipDistributionSignalGenerator
信号优先级: S_WASH_STOP > S_SELL > S_BUY > S_WASH_END > S_BOUNCE

六大信号:

S_BUY (主买入信号)
  触发: 拉升期 + 价格突破60日高 + 量>=100日均量×1.5 + CYQKL>=20 + 获利>=60%
  仓位: 70%

S_WASH_END (洗盘结束买入)
  触发: 洗盘期 + 地量(<=100日均量×0.4) + RSI 30-55企稳>=3日 + 价>=低位筹码峰下限
  仓位: 50%

S_BOUNCE (超跌反弹买入)
  触发: 下跌期/支撑期 + 价格在低位筹码峰±5% + RSI<30 + 获利<20%
  仓位: 30%

S_SELL (主卖出信号)
  触发: 任一满足 - 出货期确认 / 获利>=70%且5日跌超5% / RSI>=80
  仓位: 清仓(0%)

S_WASH_STOP (洗盘止损)
  触发: 价格跌破SSRP的90%
  仓位: 清仓(0%)

HOLD (无明确信号)
  仓位: 保持不变
```

## 三、核心策略二：缠论策略（ChanLun Strategy）

**文件位置**: `backend/app/engine/framework/chanlun_strategy.py`

**实现状态**: 核心逻辑完整实现，策略验证层集成就绪

### 3.1 数据结构定义

```
KLine      - K线数据 (idx, open, high, low, close, date, volume)
Fractal    - 分型结构 (top/bottom, idx, price)
Stroke     - 笔结构 (up/down, start/end idx+price, amplitude)
Segment    - 线段结构 (up/down, strokes list, range)
Zhongshu   - 中枢结构 (high, low, center, direction, range_width)
Divergence - 背驰结构 (trend/consolidation/zhongshu, confidence)
BuySellPoint - 买卖点 (1-3类, confidence, reason)
```

### 3.2 分析流水线

```
ChanlunAnalyzer.analyze(df: pd.DataFrame) -> Dict
               │
               ├─ 1. 数据预处理 (_preprocess)
               │   └─ OHLCV → KLine[]
               │
               ├─ 2. 包含关系处理 (KLineMerger.merge)
               │   └─ 向上处理: 新高=max(高), 新低=max(低)
               │   └─ 向下处理: 新高=min(高), 新低=min(低)
               │
               ├─ 3. 分型识别 (FractalDetector.detect)
               │   └─ 顶分型: 中间高>左右高
               │   └─ 底分型: 中间低<左右低
               │
               ├─ 4. 笔构建 (StrokeBuilder.build)
               │   └─ 条件: 分型交替, 间距>=3K线, 方向有效
               │
               ├─ 5. 线段识别 (SegmentAnalyzer.build)
               │   └─ 条件: >=3笔, 方向交替, 重叠检查
               │
               ├─ 6. 中枢识别 (ZhongshuAnalyzer.find)
               │   └─ 条件: >=3段, 重叠区域计算
               │
               ├─ 7. 背驰判断 (DivergenceDetector.detect)
               │   ├─ 趋势背驰: 创新高/低, 力度减弱(<80%)
               │   ├─ 盘整背驰: 回调力度>离开力度
               │   └─ 中枢破坏: 离开力度<返回力度
               │
               └─ 8. 买卖点识别 (BuySellPointDetector.find)
                    ├─ 一类买点: 底分型+底背驰
                    ├─ 二类买点: 回调不破中枢
                    ├─ 三类买点: 离开中枢后的回抽未回中枢
                    ├─ 一类卖点: 顶分型+顶背驰
                    ├─ 二类卖点: 反弹不破中枢
                    └─ 三类卖点: 离开中枢后的反弹未回中枢
```

### 3.3 ChanlunScorer（缠论评分系统）

```
评分体系 (0-100, 默认基线50):
  - 买点: 一类+20, 二类+15, 三类+10
  - 卖点: 一类-15, 二类-10, 三类-8
  - 底背驰: +15
  - 顶背驰: -10
  - 上升趋势: +10
  - 下降趋势: -5
  - 中枢形成: +5

推荐映射:
  70-100: STRONG_BUY
  55-70:  BUY
  45-55:  HOLD
  30-45:  SELL
  0-30:   STRONG_SELL
```

### 3.4 ChanlunAlphaModel

```
类: ChanlunAlphaModel (L3策略验证)
流程: 接收筛选后股票数据
      1. 对每只股票运行ChanlunAnalyzer.analyze()
      2. 使用ChanlunScorer.score() 评分
      3. score>=60 → LONG, <=40 → SHORT, 其他跳过
      4. 生成Insight (权重0.3)
```

## 四、核心策略三：因子评分系统（Factor Scoring Strategy）

**文件位置**: `backend/app/services/signal_computation_service.py`

**实现状态**: 实时计算服务集成

### 4.1 多因子评分模型

| 因子 | 计算方式 | 权重 | 说明 |
|------|----------|------|------|
| 动量因子 | (close[-1]-close[-20])/close[-20] | 30% | 20日价格动量 |
| 成交量因子 | vol[-1] / mean(vol[-20:-1]) | 25% | 量比指标 |
| 波动率因子 | std(closes[-20:]) / mean(closes[-20:]) | 20% | 波动性评估 |
| RSI因子 | 1 - abs(RSI-50)/50 | 25% | RSI适中度 |

```
信号判定:
  composite >= 0.6 → BULLISH (买入)
  composite >= 0.4 → WATCH (关注)
  composite < 0.4  → NEUTRAL (观望)
```

### 4.2 信号输出结构

每个因子信号包含完整的交易建议：
- strategy_name, signal, confidence
- entry_zone, risk_line, target_zone
- position_suggestion, holding_period
- evidence (因子明细)
- risk_notes (风险提示)

## 六、多层筛选系统（Multi-Layer Stock Screener）

**文件位置**: `backend/app/engine/framework/screener.py` / `backend/app/routes/screener.py`

**实现状态**: 完整实现，含API路由

### 6.1 三层架构

```
Layer 1: 风险剔除 (DarwinRiskFilter)
─────────────────────────────────────
输入: 全市场股票 (~5000只)
规则:
  1. _filter_st - 剔除ST/*ST
  2. _filter_continuous_loss - 剔除连续亏损(预留)
  3. _filter_low_liquidity - vol尾20日均量>10000
  4. _filter_high_valuation - 剔除高估值(预留)
输出: 通过风险过滤的股票池

Layer 2: 主力识别 (ChipScorer)
─────────────────────────────────────
输入: Layer1输出
方法: 对每只股票运行ChipScorer.score()
筛选: score>0的股票
排序: 按评分降序
截断: 前100只
输出: 候选股票列表(含评分和阶段)

Layer 3: 策略验证 (StrategyValidationLayer)
─────────────────────────────────────
输入: Layer2输出
验证:
  - 缠论Alpha验证 (ChanlunAlphaModel)
  - 因子评分验证 (预留)
融合: SignalFusion多策略信号融合
输出: 最终精选股票列表(含置信度和建议)
```

### 6.2 SignalFusion（信号融合器）

```
默认权重:
  - chip (筹码): 0.4
  - chanlun (缠论): 0.3
  - factor (因子): 0.3

融合公式:
  fused_value = Σ(signal_value × weight) / Σ(weight)

判定:
  fused_value > 0.3  → BUY
  fused_value < -0.3 → SELL
  其他              → HOLD
```

### 7.2 绩效指标体系

| 指标 | 计算方式 | 说明 |
|------|----------|------|
| total_return | (final - initial)/initial | 总收益率 |
| annual_return | (1+total)^(252/trading_days)-1 | 年化收益率 |
| volatility | daily_returns.std()×√252 | 年化波动率 |
| max_drawdown | max((cummax-value)/cummax) | 最大回撤 |
| sharpe_ratio | √252×excess_returns.mean()/returns.std() | 夏普比率 |
| sortino_ratio | √252×excess_returns.mean()/downside_std | 索提诺比率 |
| win_rate | (daily_returns>0).mean() | 胜率 |
| profit_loss_ratio | avg_win/avg_loss | 盈亏比 |
| max_consecutive_wins | 最大连续盈利天数 | 连续性指标 |
| max_consecutive_losses | 最大连续亏损天数 | 连续性指标 |
| beta | cov(returns, benchmark)/var(benchmark) | Beta系数 |
| tracking_error | (returns-benchmark).std()×√252 | 跟踪误差 |
| information_ratio | excess_return/tracking_error | 信息比率 |

## 八、Algorithm Framework（策略算法框架）

**文件位置**: `backend/app/engine/framework/__init__.py`

### 8.1 框架结构

借鉴 QuantConnect LEAN 的模块化架构：

```
Algorithm (核心算法类)
  ├─ set_universe_selection(UniverseSelectionModel) - 股票池选择
  ├─ set_alpha(AlphaModel) - 信号生成
  ├─ set_portfolio_construction(PortfolioConstructionModel) - 组合构建
  ├─ set_risk_management(RiskManagementModel) - 风险管理
  ├─ set_execution(ExecutionModel) - 执行模型
  └─ on_data(date_time, data) → (insights, orders)
      1. 选择股票池
      2. 生成信号(Insight)
      3. 构建组合目标
      4. 应用风险管理
      5. 生成执行订单
```

### 8.2 内部组件
