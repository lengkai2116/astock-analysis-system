  "data": {
    "aggregated_status": {

      "verification_chains": [
        {"chain_id": "chip_volume", "chain_name": "筹码×成交量",
         "passed": true/false, "confidence_multiplier": 1.X,
         "conflict_detail": "冲突描述", "evidence": ["证据1", "证据2"]}

      "chain_passed_ratio": 0.X,
      "conflicts_for_ai": [{"chain_id": "...", "chain_name": "...", "conflict_detail": "..."}],

**三级AI仲裁模式（依赖chain_passed_ratio）：**
| chain_passed_ratio | ai_mode | 决策逻辑 |
|:-------------------|:---------|:---------|
| ≥ 75% | summary_only | 直接使用验证链共识，AI仅做摘要，置信度×avg_mult |
| 25%~75% | arbitrate_conflicts | 待解决冲突发送给AI仲裁，仲裁加权作为最终决策 |
| ≤ 25% | full_analysis | 全部信息发送给AI做综合分析，固定置信度0.2 |

**四条验证链（#65 新增，均通过 status_intercept:verification_chains 输出）：**
| 链ID | 链名称 | 验证规则 |
|:-----|:-------|:---------|
| chip_volume | 筹码×成交量 | ACCUMULATING+放量→通过(×1.15)；ACCUMULATING+缩量→未通过(×0.85) |
| chanlun_factor | 缠论×因子 | 因子BULLISH+缠论有支撑→通过(×1.1)；因子BEARISH→未通过 |
| emotion_fundamental | 情绪×筹码 | BOCIASI看涨+筹码ACCUMULATING→通过(×1.15)；相反→未通过(×0.8) |
| technical_fundamental | 技术全维度 | 看涨占比≥66%→强通过(×1.2)；≥50%→通过(×1.0)；否则未通过(×0.8) |

> **2026-06-23 修订（v2.0+）：** 基于197号(知识库调研)、198号(底盘设计)、199号(修订闭环)和200号(信号源重构与多维验证走廊)方案，回测系统已完成全新设计，v2.1修订已实施。详见[166号§10](002-方案存档/166-前端UI原型逐页调整记录.md)。

**旧问题：** `views/backtest/index.vue` 的 `finishBacktest()`（第 418 行）返回完全硬编码的假数据，不调用任何后端 API。

**影响：**
- 回测结果不可信
- 因子组合选择无实际效果（UI 上切换组合改变的只是 TotalReturn 从 15.68 变为 25.68）
- 无法复现/无法验证
- 无回测结果中的诊断能力（策略健康度/冲突成本/可信度）
- 无回测信号源——不知道"用什么信号来回测"

**新设计解决方案（200号方案）：**
1. **组合策略引擎为信号源**：回测信号来自后端逐日期运行的UPF→ConflictArbiter→UIDF链路（不是实操参考规则）
2. **横截面多股票回测**：默认20~30只组合验证（统计显著）+ 单股票快速模式（试算）
3. **3-tier模式**：快速(单股,~5s) / 标准(20~30只+全4判据,~15-25min) / 深度(30只+全量,~40-50min)
4. **多维验证走廊**：4判据×K股票 2D矩阵 + 联合判定🟢/🟡/🔴
5. **非专业用户兜底**：4类故障定位修订建议 + 情境分析 + 异步执行不阻塞
