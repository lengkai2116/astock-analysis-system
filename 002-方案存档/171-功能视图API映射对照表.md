## 四、关键差距分析

### 4.1 回测系统 — 硬编码 Mock（P0）

**问题：** `views/backtest/index.vue` 的 `finishBacktest()`（第 418 行）返回完全硬编码的假数据，不调用任何后端 API。

**影响：**
- 回测结果不可信
- 因子组合选择无实际效果（UI 上切换组合改变的只是 TotalReturn 从 15.68 变为 25.68）
- 无法复现/无法验证

**后端已有能力：** 11 个 `/api/v3/backtest/*` 端点，包括：
- `POST /api/v3/backtest/run` — 执行完整回测——使用策略配置+行情数据
- `GET /api/v3/backtest/metrics` — 获取指标
- `POST /api/v3/backtest/benchmark/run` — 带基准对比的回测

### 4.2 模拟交易 — 前端有/后端有，但视图未连接（P1）

**问题：** 后端有完整的 `/api/v3/sim/*` 模拟交易模块（下单/撤单/持仓/账户管理），`dataService.js` 中已封装所有方法（7 个），但没有视图直接使用。

**影响：** 模拟交易功能存在但不可通过 UI 访问。

### 4.3 权益曲线 — `GET /api/v1/account/equity-curve` 未对接（P2）

**问题：** 账户管理页面的权益曲线 Tab（`EquityTab.vue`）使用 echarts 绘制图表，但数据源来自哪里不明确——没有调用 `accountService.getEquityCurve()`。

### 4.4 多个功能模块有"后端做了但前端没做"的能力（P2-P3）
