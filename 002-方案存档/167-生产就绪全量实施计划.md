## 领域 E — 错误韧性（Error Resilience）

**目标：** 即使外部依赖或后端故障，系统仍能给出清晰提示并优雅降级。

### E01 — 全局 HTTP 错误处理器

| 检查项 | 方法 | 判据 | 优先级 |
|--------|------|------|--------|
| E01.1 `@app.errorhandler(400)` | curl 触发 400 | 返回 JSON，非 HTML | P1 |
| E01.2 `@app.errorhandler(401/403)` | curl 触发 | 统一 JSON + 友好错误消息 | P1 |
| E01.3 `@app.errorhandler(404)` | curl 不存在的路由 | 返回 `{"success":false,"error":"接口不存在"}` | P1 |
| E01.4 `@app.errorhandler(405)` | curl 错误的 method | 统一 JSON | P2 |
| E01.5 `@app.errorhandler(429/500/502/503)` | 模拟上游故障 | 统一 JSON + 不泄漏堆栈 | P1 |

### E02 — 前端 API 调用策略

| 检查项 | 方法 | 判据 | 优先级 |
|--------|------|------|--------|
| E02.1 axios 超时 15s | 审查 request 配置 | `timeout: 15000` | P1 |
| E02.2 401 自动刷新 token | 审查拦截器 | 401 时尝试 refresh token → retry | P1 |
| E02.3 幂等请求 500 重试 | 审查拦截器 | 500 自动重试 1 次（GET/HEAD 等幂等方法） | P1 |
| E02.4 失败提示不泄漏内部细节 | 审查 | 显示「服务暂时不可用」而非堆栈 | P1 |

### E03 — Tushare 熔断器

| 检查项 | 方法 | 判据 | 优先级 |
|--------|------|------|--------|
| E03.1 连续失败自动切 AKShare | 审查 `data_source_manager.py` | N 次连续失败后切换 | P1 |
| E03.2 间隔恢复尝试 | 审查代码 | M 分钟后尝试恢复 Tushare | P1 |
| E03.3 切换事件写入日志 + 通知前端 | 审查代码 | 日志 + WebSocket 或 API 通知 | P1 |

### E04 — 组件级优雅降级

| 检查项 | 方法 | 判据 | 优先级 |
|--------|------|------|--------|
| E04.1 首次加载失败 → 错误提示 + 重试按钮 | Playwright 拦截 API 返回 500 | 显示错误状态 + 可点击重试 | P1 |
| E04.2 已有缓存后失败 → 显示缓存数据 + 黄条 | Playwright 先加载一次（有数据），再断网刷新 | 显示旧数据 +「数据可能不是最新」 | P1 |
| E04.3 子组件崩溃不影响主布局 | Playwright 注入错误 | ErrorBoundary 捕获错误，不白屏 | P1 |

### E05 — ErrorBoundary 覆盖率

| 检查项 | 方法 | 判据 | 优先级 |
|--------|------|------|--------|
| E05.1 每个路由页面被 ErrorBoundary 包裹 | 审查路由配置或 App.vue | `<ErrorBoundary>` 包裹 `<router-view>` | P1 |
| E05.2 关键子组件独立 ErrorBoundary | 审查代码 | K线图/选股结果/信号列表各自有边界 | P2 |

## 假设与默认选择

- **浏览器目标**：Chrome 最新版为第一目标，Firefox/Safari 为第二目标。
- **测试工具**：Playwright（复用现有框架）+ Lighthouse CI + k6。
- **Mock 迁移优先级**：仪表盘 4 个区块为最高优先级（直接面向用户首屏）。
- **错误降级策略**：有缓存数据时显示缓存+黄条提示（优先于白屏+红色错误）。
- **备份策略**：PostgreSQL 每日 pg_dump + 周备份，Redis RDB 保留 7 天，DuckDB 同步后备份。
- **安全配置**：所有安全头 + CSP 在 Nginx 层实施（不改后端代码）。
- **无障碍基线**：axe-core 集成到 Playwright 自动化测试中。
