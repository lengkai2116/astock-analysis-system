# 报告中心全量设计实施报告

## 1. 定位与范围

### 1.1 页面定位

报告中心是全系统**报告的统一存档检索中心**，不直接生成报告，而是接收来自5个上游模块的存档数据，提供搜索/筛选/查看/下载/删除功能。

| 🔄 方向 | 说明 |
|:---------|:------|
| **数据流入** | 各模块主动推送（自动或手动保存）到报告中心 |
| **用户操作** | 搜索 → 筛选 → 查看详情（弹窗）→ 下载 / 删除 |
| **数据流出** | JSON文件下载 → 本地留存 / 平台审阅 |

### 1.2 范围边界

| 包含 | 不包含 |
|:-----|:-------|
| 5类报告的存档展示与检索 | 报告生成（各模块自行负责） |
| 报告详情弹窗（内容预览+结构化数据） | ECharts图表渲染（交给ReportViewer生产组件） |
| 时间段批量下载（复盘中心诊断报告） | 报告编辑/修改 |
| 下载状态管理（已下载/未下载标注） | 报告标签/分类用户自定义 |

## 2. 报告类型体系

### 2.1 类型定义

| 前端类型标识 | 显示名称 | 来源模块 | 触发方式 | 当前对应 |
|:------------|:---------|:---------|:---------|:---------|
| `strategy` | 策略存档 | 个股策略分析（indicator-ide） | 用户手动保存 | ← 新增（ReportViewer原 `single_stock`） |
| `analysis` | 分析报告 | AI分析（ai-analysis） | 用户手动保存（导出→改为保存） | ← 新增（原导出改为保存） |
| `backtest` | 回测报告 | 回测系统（backtest） | 每次运行后自动归档 | 已有（改名ReportViewer无影响） |
| `diagnosis` | 诊断报告 | 复盘中心（playback） | 已完结单元自动生成 | 已有（通过 `review_reports` localStorage） |
| `sandbox` | 测试报告 | 策略沙箱（strategy-sandbox） | 每次测试后自动归档 | ← 新增 |

> **移除类型：** `research`（研究报告）— 系统中无此类型产出模块

### 2.2 报告类型变更对照

| 当前原型 | 变更为 |
|:---------|:-------|
| 单股票策略 | → **策略存档**（strategy） |
| 回测报告 | 保留（backtest） |
| ~~研究报告~~ | **移除** |
| AI分析 | → **分析报告**（analysis） |
| （无） | **诊断报告**（diagnosis） |
| （无） | **测试报告**（sandbox） |

### 2.3 各报告数据的当前生产函数映射

| 模块 | 当前报告产出函数 | 报告中心消费方式 | 变更方案 |
|:-----|:---------------|:----------------|:---------|
| indicator-ide | 无独立报告函数（五维叙事嵌入页面） | 新增「保存策略」按钮 → 写入 `review_reports` + `review_report_data_{id}` | 新增 `saveStrategyToReports(unitData)` |
| ai-analysis | `exportReport()` 仅为alert占位 | 新增「保存报告」按钮（替换导出）→ 写入 `review_reports` + `review_report_data_{id}` | 修改 `exportReport()` → `saveAnalysisReport()` |
| backtest | 9区域结果显示在页面（无独立报告持久化） | 运行结束自动归档 → 写入 `review_reports` + `review_report_data_{id}` | 新增 `archiveBacktestReport(config, results)` |
| playback | `saveReportToLS()` 已实现（review_reports/review_report_data_{id}） | 自动读取 `review_reports` 显示 | **无需修改**，仅在reports-center侧处理数据 |
| strategy-sandbox | 测试结论显示在页面（无独立报告持久化） | 测试结束自动归档 → 写入 `review_reports` | 新增 `archiveSandboxReport(result)` |

## 3. 数据流架构

### 3.1 跨页通信协议（localStorage）

维持 `review_reports` 作为唯一跨页元数据通道，扩展其结构以支持多类型：

```json
// review_reports — 报告列表元数据
[
  {
    "id": "RR-20260610-001",
    "type": "strategy",           // 新字段：strategy/analysis/backtest/diagnosis/sandbox
    "source": "indicator-ide",    // 来源模块标识
    "ts_code": "000762.SZ",
    "name": "西藏矿业 — 多策略分析",
    "date": "2026-06-10",
    "generated_at": "2026-06-10T14:30:00.000Z",
    "strategy_label": "量价+缠论+筹码",
    "total_score": 72,
    "pnl_pct": 5.2,
    "summary": "多策略全方位分析……前期摘要80字",
    "downloaded": false,           // 新字段：下载状态
    "file_size": null              // 新字段：下载后记录文件大小
  }
]

// review_report_data_{id} — 完整报告数据（与模块无关的通用结构）
{
  "id": "RR-20260610-001",
  "type": "strategy",
  "title": "西藏矿业 — 多策略分析",
  "ts_code": "000762.SZ",
  "generated_at": "2026-06-10T14:30:00.000Z",
  "content": "# 报告标题\n\n## 概述\n...Markdown正文...",
  "strategy": { /* 策略相关结构化数据 */ },
  "interpretation": {
    "overall_score": 72,
    "risk_level": "中等",
    "summary": "...",
    "strengths": ["..."],
    "weaknesses": ["..."],
    "suggestions": ["..."]
  },
  // 以下为类型特有字段（可选）
  "analysis_roles": {},           // analysis类型：6角色研判
  "backtest": {},                 // backtest类型：回测数据
  "diagnosis": {},               // diagnosis类型：分层诊断数据
```

### 3.2 各模块写入口

| 模块 | 写入时机 | 写入函数 | 写入内容 |
|:-----|:---------|:---------|:---------|
| indicator-ide | 用户点击「保存策略」 | `saveStrategyToReports()` | type=`strategy`, content=Markdown叙事 |
| ai-analysis | 用户点击「保存报告」（替换导出） | `saveAnalysisReport()` | type=`analysis`, content=Markdown研报+6角色数据 |
| backtest | 每次运行结束自动 | `archiveBacktestReport()` | type=`backtest`, content=Markdown概要+回测结构化数据 |
| playback | 已完结单元生成报告时（已实现） | `saveReportToLS()` 已有 | type=`diagnosis`, content=Markdown概要+分层诊断数据 |
| strategy-sandbox | 测试结束自动 | `archiveSandboxReport()` | type=`sandbox`, content=Markdown概要+测试指标数据 |

### 3.3 数据流图

```
indicator-ide  ── 保存策略  ──→  review_reports[type=strategy]
ai-analysis    ── 保存报告  ──→  review_reports[type=analysis]
backtest       ── 自动归档  ──→  review_reports[type=backtest]
playback       ── 自动生成  ──→  review_reports[type=diagnosis]  （已实现）
strategy-sandbox ── 自动归档──→  review_reports[type=sandbox]
                                 │
                                 ▼
                          报告中心 reports-center
                          (读取 review_reports → 展示)
```

## 5. 报告详情查看器（原型期自建弹窗）

### 5.1 决策：原型自建弹窗 vs 复用 Vue 组件

| 方案 | 优点 | 缺点 | 结论 |
|:-----|:-----|:-----|:-----|
| 复用 `ReportViewer.vue` | 生产一致，减少双份代码 | 原型页为纯HTML，嵌入Vue组件需Vue运行时+构建+依赖（marked/highlight/echarts），增加复杂度 | **不采用** |
| **原型自建弹窗** | 无外部依赖，与playback各弹窗风格一致，快速迭代 | 原型期与生产期存在差异，上线前需替换 | ✅ **推荐采用** |

### 5.2 弹窗规格

复用 playback.html 弹窗设计模式（modal-overlay + modal-content + close 三要素），但做以下定制：

| 特性 | 规格 |
|:-----|:------|
| **背景** | `background: var(--bg-canvas)` 不透明，非半透明遮罩 |
| **边框** | `border: 2px solid var(--border-strong)` 明显边框 |
| **内容滚动** | `overflow-y: auto` 内容超出弹窗时右侧自动出现滑块 |
| **弹窗尺寸** | 宽750px，高度自适应但 `max-height: 85vh` |

```
┌─ 弹窗容器 (modal-overlay, bg不透明, 有边框) ──────────┐
│  ┌─ 头部 ─────────────────────────────────────────┐  │
│  │  <标题>  [类型标签]  [下载状态]     [× 关闭]  │  │
│  ├─ 元信息行 ─────────────────────────────────────┤  │
│  │  🔖 来源 · 📅 日期 · ⭐ 评分 · 📎 关联股票     │  │
│  ├─ 内容区 (overflow-y:auto) ────────────────────┤  │
│  │  Markdown 渲染 / 结构化数据                    │  │
│  │  ...                                          │  │
│  │  ... (可滚动)                                  │  │
│  ├─ 底部操作栏 ──────────────────────────────────┤  │
│  │  [📥 下载 JSON]  [✕ 关闭] (仅诊断/测试有下载) │  │
│  └─────────────────────────────────────────────────┘  │
└────────────────────────────────────────────────────────┘
```

### 5.3 弹窗数据源

弹窗从 `review_report_data_{reportId}` 读取完整报告数据，渲染引擎做以下事情：

1. 提取 `data.title` → 标题
2. 提取 `data.type` → 类型标签
3. 提取 `data.generated_at` + `data.ts_code` → 元信息
4. 提取 `data.content` → Markdown 渲染为 HTML（使用 `marked` 或手动转换）
5. 提取 `data.interpretation` / `data.strategy` / `data.backtest` 等 → 预览侧栏

**弹窗底部下载按钮规则：** 与报告卡片一致——仅 `diagnosis` 和 `sandbox` 类型显示「📥 下载 JSON」按钮；策略存档/分析报告/回测报告在弹窗内也无下载按钮。关闭按钮始终存在。

### 5.4 与 ReportViewer.vue 的数据结构对齐

确保原型弹窗消费的数据结构与 ReportViewer.vue 的 `parseReportData()` 兼容，以便上线后直接替换为 Vue 组件时无需数据迁移：

| 字段 | ReportViewer 解析 | 原型弹窗 | 兼容性 |
|:-----|:-----------------|:---------|:-------|
| `data.title` | `this.reportTitle = data.title` | 弹窗标题 | ✅ |
| `data.report_type` | `this.reportType = data.report_type` | 类型标签 | ✅ |
| `data.ts_code` | `this.tsCode = data.ts_code` | 元信息 | ✅ |
| `data.generated_at` | `this.generatedAt = formatDate(data.generated_at)` | 元信息 | ✅ |
| `data.content` | `this.content = data.content` (Markdown) | 渲染正文 | ✅ |
| `data.strategy` | `this.strategy = data.strategy` | 策略Tab | ✅ |
| `data.backtest` | `this.backtest = data.backtest` | 回测Tab | ✅ |
| `data.interpretation` | `this.interpretation = data.interpretation` | 解释侧栏 | ✅ |
| `data.analysis_roles` | 无（分析报告新增） | 角色Tab | ✅ 不会冲突 |
