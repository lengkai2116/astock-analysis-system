## 1. 决策背景

### 1.1 问题发现

2026-06-25 沟通确认：原「虚拟验证」页面（`_ui-prototype/strategy-sandbox.html`）的**双轨验证设计**在系统演进后已无实质意义：

| 验证轨 | 原目标 | 当前覆盖情况 | 结论 |
|:-------|:-------|:------------|:-----|
| **轨B：虚拟持仓** | 模拟持仓跟踪策略表现 | **复盘中心（playback）** 已全面覆盖虚拟实盘复盘能力 | ⛔ 废弃 |
| **轨A：信号赢率** | 统计信号预测正确率 | **回测系统（backtest）** 的回测结果面板应承担该职能 | ⛔ 废弃 |

### 1.2 决策

用户确认将虚拟验证页面改造为**策略沙箱**——一个针对**单个策略/因子组合**进行快速历史数据测试的轻量工具，定位为回测系统的前置快速验证环节。

### 1.3 用户画像

非专业量化用户。核心特征：

1. 懂得基本策略概念（涨跌/信号/收益），但不懂量化专业术语（夏普/回撤/胜率等）
2. 需要秒级反馈来验证"这策略行不行"，不需要全量回测的耗时
3. 能接受参数调整但不了解调整后对结果的影响程度
4. 需要一个「试错」空间，而非「决策」工具

## 2. 定位与设计原则

### 2.1 产品定位矩阵

| 维度 | 策略沙箱 | 回测系统 |
|:-----|:---------|:---------|
| 测试范围 | **单策略/单因子** | 多策略组合+多标的 |
| 耗时 | 秒级/分钟级 | 分钟级~小时级 |
| 用户目的 | "这策略好使吗？" | "该用哪个配置？" |
| 输出层次 | 信号级明细 | 组合级稳健性评估 |
| 专业门槛 | 低（纯自然语言） | 中（含量化指标+判据） |
| 数据量 | 几周~几年的单股数据 | 多股票×多年×多参数 |

### 2.2 核心设计原则

1. **自然语言优先** → 全界面无量化术语，所有指标配说明弹窗
2. **非专业用户引导** → 参数折叠+"不熟悉可跳过"+"空态引导"
3. **知识库锚定三个维度** → 来源自197号回测知识库调研报告
4. **轻量定位** → 不引入PBO/CSCV/走步前推等专业验证方法
5. **结果可行动** → 三态结论标签（✅适用/⚠️谨慎/📌转回测），"纳入回测"一键闭环

### 2.3 三个知识库锚定维度

| # | 维度 | 来源（197号报告） | 原型实现 |
|:--|:-----|:-----------------|:---------|
| ① | **基准对比** | 无基准的收益数字无意义 | ECharts双线权益曲线（策略净值vs大盘基准净值） |
| ② | **行情分段看** | 单一窗口回测不反映策略全部表现 | 自动划分上涨/下跌/震荡三段，各段展示收益+正确率+信号次数 |
| ③ | **参数稳定度** | 参数平原概念简化 | 可选开关，周期5→60天步进柱状图+自然语言解读 |

## 6. 行为准则：测试与回测分离

策略沙箱严格遵守三项行为准则，确保与回测系统的职责边界清晰：

1. **不替代回测**：沙箱测试结果仅作为"快速摸底"，不输出任何"决策建议"或"最优方案"
2. **不跨策略组合**：始终针对单策略/单因子组合测试，不涉及多策略权重
3. **不做稳健性评估**：参数稳定度仅展示波动趋势，不给出"参数平原"或"PBO通过"等专业判定。如需稳健性验证→引导用户到回测系统

## 7. 生产环境转化评估

### 7.1 三个核心问题的答案

在原型阶段，代码均为前端硬编码Mock。以下逐项分析生产环境需要的改造：

#### 问题1：策略选择是否连接真实策略系统？

| 评估项 | 当前状态 | 生产要求 | 差距 |
|:-------|:---------|:---------|:-----|
| 策略来源 | 8个radio硬编码(S1-S5 + F1-F3) | 从strategy-template.html读取localStorage策略配置 | ❌ 未连接 |
| 因子来源 | 3个radio硬编码(F1-F3) | 从factor-manager.html读取localStorage因子组合 | ❌ 未连接 |
| 参数参与计算 | 滑块值仅前端展示 | 参数需作为信号计算入参传给后端 | ❌ 无效参数 |
| 信号计算引擎 | 无 | 需后端信号计算服务或本地策略引擎 | ❌ 不存在 |

**生产方案**：

```
策略模板localStorage键: strategy_template_config_{id}  
  → 存储策略类型、参数默认值范围、信号计算逻辑标识  
因子组合localStorage键: factor_combination_{id}
  → 存储因子列表、权重配置

沙箱读取后:
  → 将策略类型+参数+因子权重拼接为计算请求
  → 传给后端 POST /api/v3/sandbox/run
  → 后端调用实际信号计算引擎
```

#### 问题2：测试数据是否真实A股历史数据？

| 评估项 | 当前状态 | 生产要求 | 差距 |
|:-------|:---------|:---------|:-----|
| 股票行情 | 无数据参与，纯随机 | 按日期范围读取后端行情数据库(日线/周线/月线) | ❌ 无数据 |
| 权益曲线 | `Math.random()` 生成 | 基于实际信号+实际行情的逐日净值计算 | ❌ 全Mock |
| 指标计算 | 固定值展示 | 从净值序列计算：总收益/年化/最大回撤/胜率等 | ❌ 全Mock |
| 基准对比 | 随机生成第二条线 | 读取沪深300等基准指数的实际历史净值 | ❌ 全Mock |
| 行情分段 | 不基于数据，固定三段文案 | 基于实际K线走势的涨/跌/震荡分类算法 | ❌ 全Mock |

**生产方案**（数据流全景）：

```
后端数据层:
  行情数据: stock_daily / stock_weekly 表（Tushare/AKShare 归档）
  基准指数: index_daily 表（沪深300等）
  信号计算: 策略类型 × 参数 × 股票代码 → 信号时间序列

前端请求:
  POST /api/v3/sandbox/run {
    strategyId: "S1",
    params: { window: 20, threshold: 70, stoploss: 5 },
    stockCodes: ["000762.SZ", "000001.SZ"],
    startDate: "2024-01-01",
    endDate: "2026-06-01",
    benchmark: "沪深300",
    capital: 1000000
  }

后端响应:
  {
    equityCurve: [ {date, strategyValue, benchmarkValue} ],
    metrics: { totalReturn, annualReturn, maxDrawdown, winRate, ... },
    phases: [ { type: "up", start, end, return, winRate, signalCount }, ... ],
    signals: [ { date, direction, entry, exit, pnl, correct, phase }, ... ],
    stability: { windowVariation: [ {window, annualReturn}, ... ] }
  }
```

#### 问题3：结果解读是否真实分析而非硬编码？

| 评估项 | 当前状态 | 生产要求 | 差距 |
|:-------|:---------|:---------|:-----|
| 结论文案 | 固定模板拼接（"5只表现不错"等）| 基于实际指标+行情段表现的规则推理 | ❌ 模板硬编码 |

**生产方案**（结论生成规则引擎示例）：

```javascript
function generateConclusion(metrics, phases, stability) {
  var conclusions = [];
  
  // 规则1: 总体正收益判断
  if(metrics.totalReturn > 0) conclusions.push('整体表现不错');
  else if(metrics.totalReturn > -0.1) conclusions.push('整体表现一般');
  else conclusions.push('整体表现欠佳');
  
  // 规则2: 行情适应性判定
  var upPhase = phases.find(p => p.type === 'up');
  var downPhase = phases.find(p => p.type === 'down');
  if(upPhase && upPhase.winRate > 0.6) {
    conclusions.push('上涨行情中有效性较高');
    tags.push('上涨行情适用');
  }
  if(downPhase && downPhase.winRate > 0.6) {
    conclusions.push('下跌行情中有较好防御');
  }
  
  // 规则3: 参数稳定度判定
  if(stability && stability.variation < 0.15) {
    conclusions.push('参数稳定度高，策略鲁棒性好');
  } else if(stability && stability.variation > 0.3) {
    conclusions.push('参数敏感度高，建议回测系统进一步验证');
    tags.push('推荐回测系统进一步验证');
  }
  
  return { text: conclusions.join('；'), tags };
}
```
