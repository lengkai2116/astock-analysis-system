# 回测结果驱动策略修订闭环设计方案

## 1. 问题与现状

### 1.1 当前断层

```
回测完成 → 用户看到报告 → "好或不好" → 用户自行判断 → （结束）
                                                           ↑ 缺少结构化路径将结论
                                                             反馈到策略参数层
```

| 层面 | 已有 | 缺失 |
|------|------|------|
| **回测输出** | `BacktestResultV2`(17指标), trades, daily_equity, benchmark | 无"推荐参数"字段输出 |
| **参数空间** | `PRESET_GRIDS` 4策略网格, `run_preset_grid_search()` | 从未被回测实际驱动 |
| **参数写入** | 各策略 `_default_config()` / 构造函数 `lookback_period` | 无可编程写入API、无版本管理、无回退 |
| **前端UI** | 5大指标卡片 + 策略健康度 + 解读 | 无"应用优化结果"交互区 |

### 1.2 核心矛盾

**回测引擎的产出（绩效指标）和策略引擎的入参（参数配置）之间缺少一个标准的翻译层。**

```
回测引擎产出: Sharpe=0.85, TotalReturn=+35.2%, MaxDrawdown=-12.3%
                 ↓  缺参数与绩效的映射关系
策略引擎入参: lookback_period=120, window_5=5, threshold=0.15
```

**新增的数据类：**

```python
@dataclass
class ParamChange:
    strategy: str          # 策略名称（chanlun/volume_price/chip/factor）
    param_name: str        # 参数名（lookback_period/window_5 等）
    old_value: Any         # 原值
    new_value: Any         # 优化后的最优值
    impact: str            # 影响描述（HIGH/MEDIUM/LOW）

@dataclass
class OptimizeResult:
    preset_name: str
    target_metric: str
    best_params: Dict[str, Any]
    best_score: float
    all_scores: List[Dict]  # 所有参数组合的评分排名
    plateau_width: float     # 参数平原宽度（稳健性指标）
    plateau_score: float     # 参数平原得分（0~1）
    changes: List[ParamChange]
    confidence: str          # HIGH/MEDIUM/LOW
```

**优化目标（按优先级）：**

| 优先级 | 目标函数 | 适用场景 |
|--------|---------|---------|
| P0 | 夏普比率 | 默认通用目标 |
| P1 | 卡玛比率（年化收益/最大回撤） | 风险敏感型用户 |
| P2 | 年化收益率 | 进取模式优化 |
| P3 | 信息比率 | 需要与基准对比时 |

### 3.3 参数变更影响评估

用户需要知道"改了参数会怎样"。影响评估基于两个维度：

**① 敏感度（Sensitivity）**：该参数变化 1% 对目标指标的影响幅度
  - High：±5% 参数变化 → ≥2% 绩效变化
  - Medium：±5% 参数变化 → 0.5-2% 绩效变化
  - Low：±5% 参数变化 → <0.5% 绩效变化

**② 平原宽度（Plateau Width）**：最佳值附近平坦区域宽度
  - Wide（稳健）：最佳值 ±20% 范围内绩效衰减 <5%
  - Medium（可接受）：最佳值 ±10% 范围内绩效衰减 <5%
  - Narrow（尖锐/易过拟合）：最佳值 ±5% 范围内绩效衰减 ≥5%

**推荐置信度**基于①×②的组合判定：

```
敏感度 High + 平原 Wide  →  HIGH（强烈推荐调整）
敏感度 Medium + 平原 Wide →  HIGH
敏感度 High + 平原 Medium →  MEDIUM（建议调整但需谨慎）
敏感度 Low + 平原 Any     →  LOW（调整无意义，不推荐改变）
任何组合 + 平原 Narrow    →  LOW（过拟合风险高，不推荐改变）
```

| 交互 | 设计 |
|------|------|

| **恢复默认** | 回到 PRESET_GRIDS 中定义的 default 值 |

### 4.3 对非专业用户的多档交互

| 用户类型 | 看到的界面 |
|---------|-----------|
| 👤 初级 | 只有「一键优化」按钮+结果摘要，自动应用最佳推荐 |
| 👤 中级 | 参数差异表可见但不是可编辑，只能选择"应用"或"忽略" |
| 👤 高级 | 完整参数面板可见，可手动修改单个参数值再应用 |

**推荐**：默认显示中级模式（差异表可见，不可编辑），用户可展开高级模式。

## 8. 风险与注意事项

### 8.1 优化耗时

`run_preset_grid_search` 在 PRESET_GRIDS 全量运行时：
- 缠论7档 × 量价7档 × 筹码5档 × 因子4档 = 980种组合
- 每组合回测1次 ≈ 0.5~2s（取决于数据量）
- 总耗时：~500~2000s → **必须异步执行**

**解决方案**：
1. Phase 0 先实现**逐策略独立优化**（4次调用，每次7档×1策略=7组合，~10s/策略）
2. Phase 1 再实现**全策略联合优化**（异步任务+WebSocket推送进度）

### 8.2 参数空间膨胀

当前 PRESET_GRIDS 仅 `lookback_period` 一个维度。未来增加多维度（如缠论增加 `min_klines`、`min_confidence`）会导致组合数指数级增长。

**解决方案**：
1. 默认仅优化 lookback_period（对绩效影响最大）
2. 高级模式开放多维度优化，但限制最多2个维度同时优化
3. 需要时为多维度场景切换到 RandomSearchOptimizer（效率更高）

### 8.3 过拟合风险

在回测数据上跑优化 → 找到的最佳参数可能过拟合历史。

**防护**：
1. 内置参数平原宽度判定（平原窄=过拟合，不推荐）
2. 优化结果必须附带 plateau_width 字段
3. 前端根据置信度决定是否自动推荐（HIGH才显示"强烈推荐"标签）
4. 建议用户在回测数据后半段做验证（out-of-sample validation）
