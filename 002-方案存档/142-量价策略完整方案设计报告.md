# 量价策略完整方案设计报告

## 一、策略定位与背景

### 1.1 策略定位

量价策略是 A 股股票分析系统中**第四个核心策略**，与筹码策略、缠论策略、因子评分策略同级并列：

```
SignalComputationService.compute_for_stock()
    ├─ _compute_chip_signal()            ── 筹码分布策略（L2）
    ├─ _compute_chanlun_signal()         ── 缠论策略（L3）
    ├─ _compute_volume_price_signal()    ── 量价策略（本次新增）
    └─ _compute_factor_signal()          ── 因子评分策略（L3）
```

### 1.2 设计依据

| 来源 | 核心贡献 |
|------|---------|
| 132号文档 | 策略体系完整方案（三层金字塔+多策略融合+统一输出格式） |
| 134号文档 | 策略系统实施路线图（PreFilter→Analyze→Detect→Signal→Verify） |
| LLM Wiki | 格兰威尔八准则、量价时空、量托量压、波段四阶段、戴维·朗德里三重过滤 |
| WyckoffTradingAgent | 量价分析工程化参考 |
| 现存 chanlun_strategy.py | 分层分析架构模板（合并→分型→笔→段→中枢→买卖点） |

### 1.3 与缠论策略的关键区别

| 维度 | 缠论策略 | 量价策略 |
|------|---------|---------|
| 理论基础 | 缠中说禅理论（分型/笔/段/中枢/背驰） | 格兰威尔+威科夫+量价时空 |
| 分析对象 | K线形态结构（顶底分型） | 成交量与价格的关系 |
| 核心判断 | 走势终完美（背驰→转折） | 量在价先（量确认价） |
| 确认方式 | 笔/段/中枢的结构确认 | 成交量方向与价格方向的匹配 |
| 阶段判断 | 笔状态（延续/回调/反弹） | 波段四阶段（上涨/触顶/下跌/探底） |

## 二、核心设计原则

### 2.1 右侧交易铁律

```
根本原则：仅在趋势已经明确确立后，才输出具备操作性的信号。

┌──────────────────────────────────────────────────────┐
│                  右侧交易决策矩阵                      │
│                                                      │
│  阶段状态          │ 信号类型   │ 输出内容             │
│  ─────────────────────────────────────────────       │
│  UPTREND_ACTIVE   │ BUY/WATCH │ 买入建议+入场区间     │
│  UPTREND_TOPPING  │ WATCH     │ 减仓提醒+风险提示     │
│  DOWNTREND_ACTIVE │ SELL      │ 卖出建议+止损         │
│  DOWNTREND_BOTTOM │ WATCH     │ 观察信号+预警        │
│  CONSOLIDATION    │ HOLD      │ 等待方向+关键位       │
│  TRANSITION       │ WATCH     │ 趋势转折预警          │
└──────────────────────────────────────────────────────┘

在任何情况下，量价策略不输出"抄底买入"（逆趋势）或"逃顶卖出"（逆趋势）的操作信号。
底背离/顶背离信号仅作为 WATCH 预警，不作为直接操作信号。
```

### 2.2 分析逻辑链

```
分析必须严格按以下顺序执行，前序结论约束后序判断：

Phase 1: 阶段状态判定（宏观层面）         ← 先答"股票处于什么阶段？"
    │
Phase 2: 成交量状态评估（中观层面）       ← 再答"成交量在做什么？"
    │
Phase 3: 量价关系分析（微观层面）         ← 再答"量价如何互动？"
    │
Phase 4: 信号生成（约束输出）            ← 最后在右侧交易约束下输出信号
```

**为什么这个顺序不可颠倒？**

1. **阶段必须先于量价关系**：同样的"价涨量增"，在上升初期是健康信号，在连续大涨后的高位可能是最后的疯狂。阶段不同的同一形态含义相反。
2. **量价关系必须先于信号**：量价背离是警告而非操作信号。警告需要结合阶段来判断：上升趋势中的顶背离是减仓提醒，但不是做空信号。
3. **右侧交易必须在最后**：信号在约束层进行"右侧过滤"，确保不输出逆趋势的操作建议。

## 三、Phase 1：阶段状态判定

### 3.1 波段四阶段定义

基于《量价时空：波段操作精解》波段四阶段模型，结合趋势结构分析：

```
                  ┌─────────────────────────┐
                  │  2. 多头触顶阶段         │
         价格     │  (UPTREND_TOPPING)       │
          ↑       │  放量冲顶/缩量新高/     │
          │  ┌────┤  K线反转/顶部形态       │
          │  │ 2   │                       │
          │  │    ┌┴──────────────────────┐
          ├──┤  1 │  3. 空头下跌阶段       │
          │  │    │  (DOWNTREND_ACTIVE)    │
          │  │    │  放量下跌/趋势下行/    │
          │  └────┤  均线空头排列          │
          │  1. 多│                       │
          │  头上 │                       │
          │  涨阶 ├───────────────────────┤
          │  段   │  4. 空头探底阶段      │
          │ UPTRE │  (DOWNTREND_BOTTOMING) │
          │ ND_AC │  缩量止跌/底部形态/   │
          │ TIVE  │  放量试探性反弹       │
          └───────┴───────────────────────┘
                     时间 →
```

### 3.2 阶段判定规则

#### 3.2.1 UPTREND_ACTIVE（多头上涨阶段）

**判定条件（全部满足）**：
- 价格高于MA60，MA60方向朝上
- 最近2个波段低点逐步抬高（higher lows）
- 最近2个波段高点逐步抬高（higher highs）
- 成交量在上涨波段中放大，回调波段中缩小

**特征**：
- K线形态：旗形、楔形、三角形、通道线等上涨持续形态
- 均线：多头排列（MA5>MA10>MA20>MA60）
- 量价：价涨量增 > 价涨量平 > 价涨量缩（健康度降序）

#### 3.2.2 UPTREND_TOPPING（多头触顶阶段）

**判定条件（满足2项以上）**：
- 价格创出新高但成交量未跟随放大（顶背离）
- 出现明显的顶部反转K线形态（长上影线、十字星、吞没形态）
- 放量但价格不涨（放量滞涨）
- 价格跌穿关键趋势线或MA20
- MA5下穿MA10或MA20

**特征**：
- K线形态：头肩顶、双重顶、V形反转、假突破
- 均线：多头排列开始走平或交叉
- 量价：价涨量缩、放量滞涨

#### 3.2.3 DOWNTREND_ACTIVE（空头下跌阶段）

**判定条件（全部满足）**：
- 价格低于MA60，MA60方向朝下
- 最近2个波段高点逐步降低（lower highs）
- 最近2个波段低点逐步降低（lower lows）
- 成交量在下跌初期放大（恐慌），后期可能萎缩

**特征**：
- K线形态：下降旗形、楔形、三角形、通道线等下跌持续形态
- 均线：空头排列（MA5<MA10<MA20<MA60）
- 量价：价跌量增（初期）→ 价跌量平 → 价跌量缩（衰竭，接近探底）

#### 3.2.4 DOWNTREND_BOTTOMING（空头探底阶段）

**判定条件（满足2项以上）**：
- 价格创出新低但成交量未跟随放大（底背离）
- 价格在低位缩量横盘，出现底部K线反转形态（锤子线、早晨之星）
- 放量突破下降趋势线或关键压力位
- MA5上穿MA10，MA10开始走平
- 出现量托形态（VolMA5上穿VolMA10再上穿VolMA20）

**特征**：
- K线形态：头肩底、双重底、V形底、圆形底、岛形反转
- 均线：空头排列开始收敛
- 量价：价跌量缩后出现放量反弹试探

### 3.3 阶段判定所需数据维度

| 维度 | 计算指标 | 回看窗口 |
|------|---------|---------|
| 趋势方向 | MA60方向、波峰波谷序列 | 120日 |
| 价格位置 | 距60日高点/低点的百分比 | 60日 |
| 波段结构 | higher high/low lower high/low | 60日 |
| 均线排列 | MA5/10/20/60关系 | 60日 |

## 四、Phase 2：成交量状态评估

### 4.1 成交量趋势

在阶段判定之后，聚焦近期的成交量变化方向：

```
判定逻辑（基于近20日 vs 前60日均量）：
  - vol_20avg / vol_60avg > 1.3  → EXPANDING（放量趋势）
  - vol_20avg / vol_60avg < 0.7  → CONTRACTING（缩量趋势）
  - else                          → STABLE（平量趋势）
```

**阶段关联约束**：
- UPTREND_ACTIVE + EXPANDING → 趋势健康，上涨有量能支撑
- UPTREND_ACTIVE + CONTRACTING → 上涨动力不足，可能在接近触顶
- DOWNTREND_ACTIVE + EXPANDING → 恐慌抛售中，卖压释放
- DOWNTREND_ACTIVE + CONTRACTING → 下跌动力衰竭，可能在接近探底

### 4.2 均量线结构

```
均量线 (VolMA5 / VolMA10 / VolMA20)：

  VolMA5 > VolMA10 > VolMA20 → BULLISH（量能多头排列）
  VolMA5 < VolMA10 < VolMA20 → BEARISH（量能空头排列）
  交叉/无序                   → MIXED（量能混乱）

量托形态：VolMA5 ↑ VolMA10（金叉）→ 共同 ↑ VolMA20（形成封闭三角）
量压形态：VolMA5 ↓ VolMA10（死叉）→ 共同 ↓ VolMA20（形成封闭三角）
```

### 4.3 量异常检测

| 类型 | 条件 | 含义 |
|------|------|------|
| 倍量 | vol > 均量 × 3 | 极端活跃，关键博弈区域 |
| 放量 | vol > 均量 × 1.5 | 显著活跃，关注方向 |
| 地量 | vol < 均量 × 0.3 | 极度低迷，变盘前兆 |

**关键点位的量异常**特别重要：
- 前高位置放量：突破还是受阻？
- 前低位置放量：破位还是支撑？
- 倍量位置将成为未来参照位（GBP：Greatest Volume Point）

## 五、Phase 3：量价关系分析

### 5.1 八种基本量价形态（当前状态）

```
形态编号  价格方向   量方向     形态名称        信号含义
──────────────────────────────────────────────────────
 VP-1      上涨       放量     价涨量增        健康上涨（+）
 VP-2      上涨       平量     价涨量平        温和上涨（+）
 VP-3      上涨       缩量     价涨量缩        动力不足（-）
 VP-4      横盘       放量     价平量增        方向选择（±）
 VP-5      横盘       平量     价平量平        持续观望（0）
 VP-6      横盘       缩量     价平量减        变盘前兆（±）
 VP-7      下跌       放量     价跌量增        恐慌抛售（-）
 VP-8      下跌       平量     价跌量平        阴跌继续（-）
 VP-9      下跌       缩量     价跌量缩        下跌衰竭（+）
```

### 5.2 量价背离检测

**顶背离检测**：
```
条件：
  1. 最近60日价格出现两个波峰：P1 < P2（P2创新高）
  2. P2对应时间段成交量：V2 < V1（量未创新高）
  3. 置信度 = min(1.0, (P2-P1)/P1 × 10) × min(1.0, (V1-V2)/V1 × 5)

阶段约束：仅在 UPTREND_ACTIVE 和 UPTREND_TOPPING 中检测顶背离有意义
```

**底背离检测**：
```
条件：
  1. 最近60日价格出现两个波谷：T1 > T2（T2创新低）
  2. T2对应时间段成交量：V2 > V1（成交量反而放大）
  3. 置信度 = min(1.0, (T1-T2)/T1 × 10) × min(1.0, (V2-V1)/V1 × 5)

阶段约束：仅在 DOWNTREND_ACTIVE 和 DOWNTREND_BOTTOMING 中检测底背离有意义
```

### 5.3 量托/量压判定

| 信号 | 触发条件 | 阶段约束 |
|------|---------|---------|
| 量托 | VolMA5↑VolMA10(金叉) + 两者均上穿VolMA20 | 仅 UPTREND_ACTIVE / DOWNTREND_BOTTOMING |
| 量压 | VolMA5↓VolMA10(死叉) + 两者均下穿VolMA20 | 仅 UPTREND_TOPPING / DOWNTREND_ACTIVE |

### 5.4 量价关系与阶段的交叉矩阵

```
                    价涨量增  价涨量缩  价平量增  价跌量增  价跌量缩
UPTREND_ACTIVE      强烈买入   警告      关注       异常      健康回调
UPTREND_TOPPING     最后疯狂  顶背离★   放量滞涨★  趋势反转  缩量下跌
DOWNTREND_ACTIVE    反弹诱多   超跌反弹  横盘整理   恐慌抛售★  下跌中继
DOWNTREND_BOTTOMING 试探建仓  底部吸筹  关注转势★  加速赶底  底部确认
CONSOLIDATION       关注向上   弱势平衡  方向选择   关注向下  变盘前兆★

★ = 该组合在该阶段中最具操作参考价值
```

## 六、Phase 4：信号生成（右侧交易约束）

### 6.1 信号体系总表

```

UPTREND_ACTIVE   价跌量缩 + 均线支撑   VP_PULLBACK    BUY    0.60~0.75
UPTREND_ACTIVE   放量突破前高          VP_BREAKOUT    BUY    0.70~0.85

──────────────────────────────────────────────────────────────

──────────────────────────────────────────────────────────────

──────────────────────────────────────────────────────────────
CONSOLIDATION    缩量整理               VP_SQUEEZE     HOLD   0.45~0.55
```

### 6.2 关键约束说明

1. **右侧交易铁律**：量价策略**绝不**在 DOWNTREND_ACTIVE 阶段输出 BUY 信号，也**绝不**在 UPTREND_ACTIVE 阶段输出 SELL 信号。

2. **背离仅作预警**：
   - 上升趋势中的顶背离 → WATCH（提醒减仓），不是 SELL
   - 下降趋势中的底背离 → WATCH（提醒观察），不是 BUY

3. **阶段过渡时的信号降级**：
   - UPTREND_TOPPING 中的买入信号 → 降级为 WATCH
   - DOWNTREND_BOTTOMING 中的买入信号 → 保持 WATCH（等待确认后再升级）

```
        ├── apply_right_side_filter() 右侧交易约束过滤
        ├── calc_entry_zone()         入场区间计算
        ├── calc_risk_line()          止损线计算
        └── build_evidence()          证据链构建
```

### 7.3 数据类定义

```python
@dataclass
class Stage:
    """阶段状态"""
    name: str            # UPTREND_ACTIVE / UPTREND_TOPPING / DOWNTREND_ACTIVE / DOWNTREND_BOTTOMING / CONSOLIDATION
    confidence: float    # 判定置信度 0~1
    trend_structure: str # HH/HL 或 LH/LL
    ma_alignment: str    # 多头/空头/混乱
    price_position: str  # 高位/中位/低位

@dataclass
class VolumeState:
    """成交量状态"""
    trend: str           # EXPANDING / CONTRACTING / STABLE
    volma_structure: str # BULLISH / BEARISH / MIXED
    has_volume_tuo: bool # 是否有量托
    has_volume_ya: bool  # 是否有量压
    volume_keypoints: List[float]  # 倍量关键点位

@dataclass
class VolumePriceSignal:
    """量价信号"""
    signal_id: str       # VP_TREND / VP_DIVERGENCE_TOP 等
    direction: str       # BUY / SELL / WATCH / HOLD
    confidence: float
    entry_zone: tuple
    risk_line: float
    evidence: List[str]
```

## 附A：与用户确认的设计决策

| # | 决策项 | 本方案采取的立场 | 说明 |
|---|--------|----------------|------|
| 1 | 阶段判定模型 | 波段四阶段（上涨/触顶/下跌/探底） | 基于知识库"量价时空"和"波段操作"，比主力四阶段更客观 |
| 2 | 右侧交易严格执行 | 仅趋势确认后输出操作信号 | 满足用户强制要求 |
| 3 | 背离信号的动作 | 仅WATCH，不跨阶段生成BUY/SELL | 防止抄底/逃顶策略失效 |
| 4 | 分析顺序不可颠倒 | 阶段→量→价→信号 | 保证前序约束后序 |
| 5 | 量托/量压周期 | 默认 VolMA5/10/20 | 基于知识库推荐 |
| 6 | 回看周期 | 120日（阶段）+ 60日（细节） | 平衡趋势识别和时效性 |
