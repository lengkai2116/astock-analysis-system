## 三、问题修复记录

### 3.1 已修复的问题

1. **Redis配置错误**
   - 问题: realtime.py尝试访问`redis_manager.redis_client`
   - 解决: 改为`redis_manager.client`
   - 文件: `backend/app/routes/realtime.py`

2. **DuckDB并发问题**
   - 问题: DuckDB文件被其他进程锁定
   - 解决: 添加容错处理（只读模式、备份恢复、内存模式）
   - 文件: `backend/app/data/enhanced_cache_manager.py`

3. **数据目录路径**
   - 问题: 使用只读目录`/data`
   - 解决: 改为用户可写目录
   - 文件: `.env`

## 四、如何访问系统

### 4.1 方式一：使用Docker服务（推荐）

打开浏览器访问：

```
http://localhost:3000
```

这将访问已经运行的DeerFlow前端服务。

### 4.2 方式二：本地开发模式

#### 步骤1: 启动后端
```bash
cd backend
python3 run.py --port 5000
```

#### 步骤2: 启动前端（另一个终端）
```bash
cd frontend/vue-project
npm install  # 如果首次运行
npm run dev
```

#### 步骤3: 访问
```
http://localhost:5173
```
