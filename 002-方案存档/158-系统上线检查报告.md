### 🔴 P0 — 阻塞（上线前必须修复）

| ID | 问题 | 影响范围 | 修复方案 |

### 🟠 P1 — 高风险（上线前建议修复）

| ID | 问题 | 影响范围 | 修复方案 |
|----|------|---------|---------|
| **C2/E8** | 缺少全局 `@app.errorhandler` | 404/500 等未处理错误返回 Flask HTML 页面，泄露服务端信息，前端无法以 JSON 格式处理 | 在 `create_app()` 中添加 `@app.errorhandler(404)` 和 `@app.errorhandler(500)` 返回 JSON |
| **E4** | `SECRET_KEY` 后备值为硬编码弱密钥 | Session/CSRF token 可被预测 | docker-compose 中移除后备值：`SECRET_KEY=${SECRET_KEY}`（强制在 `.env` 中设置） |
| **C5** | docker-compose 中 `LLM_PROVIDER` 默认 `deepseek` 而非 `mock` | 未配置 DeepSeek 时，AI 功能直接调用 API 失败而非优雅降级 | 改为 `LLM_PROVIDER=${LLM_PROVIDER:-mock}` 与 config.py 默认一致 |
| **A7** | `__init__.py` 第 188 行游离 `return app` + 重复 `from app import models` | 死代码不影响运行但造成维护混淆，且在 `_route_provider` 函数后执行 | 移除第 188-195 行的游离代码（`return app`, `from app import models`, `import logging`） |
| **B15** | nginx 安全头在首页响应中缺失 | `X-Frame-Options`/`X-Content-Type-Options`/`X-XSS-Protection` 对主页面不生效，仅对 `/socket.io/` 和 `/health` 生效 | 在 nginx `location = /index.html` 中显式添加安全头，或在 `location /` 中使用 proxy_set_header |
| **E2** | 三份 `.env` 文件共存 | 配置管理混乱，`python-dotenv` 无明确优先级文档 | 删除 `.env.local` 和 `backend/.env`，将变量统一到根 `.env`（或 `.env.production`） |
