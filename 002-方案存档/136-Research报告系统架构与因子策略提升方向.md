## 二、层面一：系统架构与 AI 能力提升方向

### 2.1 当前系统架构快照

```
市场数据层 (Tushare/DataManager)
    ↓
PreFilter 层 (大盘环境/流动性/新股/市值)
    ↓
三层策略引擎:
  ├─ 筹码分布策略 (PhaseDetector → SignalGenerator → PositionManager → RiskExecutor)
  ├─ 缠论策略 (ChanlunStrategy)
  └─ 达尔文选择策略 (DarwinSelection)
    ↓
SignalFusion → 策略权重加权融合
    ↓
DeepSeek AI 分析服务 (独立调用，非内嵌)
    ↓
前端展示 / StrategyOutput 持久化
```

### 2.2 突破方向 A：SMC 概念融入策略信号体系

**研究依据**：
> "SMC（聪明钱概念）从博弈论角度追踪机构资金的行为痕迹，使用订单块、流动性猎杀、结构破位等工具。SMC 在多个维度上相对传统 TA 具有优势：信号拥挤度（好）、AI 辅助友好度（优）、多时间框架一致性（优）、胜率天花板（夏普 1.2-2.0）。"

**SMC 与系统的映射关系**：

| SMC 概念 | 系统已有对应 | 差距 | 突破方向 |
|----------|------------|------|---------|
| 订单块 (Order Block) | 筹码峰识别（支撑/阻力位） | 筹码峰基于分布统计，订单块基于价格行为 | 在 KLinePatternVerifier 中增加订单块识别逻辑 |
| 流动性猎杀 (Liquidity Sweep) | 无 | 完全缺失 | 在 ConfirmLayer 中增加流动性猎杀检测（高低点假突破后的反转） |
| 结构破位 (Structure Break) | PhaseDetector 的阶段切换 | 缺少"趋势线破位"和"区间破位"的独立检测 | 新增结构分析模块或增强 KLinePatternVerifier |
| 公平价值缺口 (FVG) | 跳空缺口检测已有 | 未区分"突破缺口"vs"衰竭缺口" | 增强跳空缺口分类逻辑 |
| SMC 多时间框架一致性 | 仅在日线框架 | 缺少分钟线与日线的信号验证 | ConfirmLayer 增加时间框架一致性验证 |

**实施建议**：
- **近期**：在 `KLinePatternVerifier` 中增强跳空缺口类型识别（突破/中继/衰竭），增加结构破位检测
- **中期**：在 `ConfirmLayer` 中增加流动性猎杀检测模块，与假突破过滤器配合使用
- **长期**：构建独立的 SMC SignalVerifier 模块，作为第 7 层接入策略流水线

### 2.3 突破方向 B：AI 增强的信号置信度与趋势质量评估

**研究依据**：
> "AI 增强后的趋势跟踪策略可实现显著改善：胜率从 38% 提升至 64%，年化回报率从 18.3% 提升至 31.7%，最大回撤从 -23.4% 降低至 -14.2%。"
> "AI 能识别传统方法无法发现的多维模式，如价格行为、成交量分布、期权流、跨市场相关性和微观结构订单失衡的'模式叠加'。"

**当前系统的 AI 能力快照**：
- `deepseek_analysis_service.py` — DeepSeek API 调用，分析个股和策略信号
- 当前用法：独立 API 调用，将 K 线数据 + 筹码指标发给 DeepSeek，返回文本分析
- 问题点：AI 分析结果**未被回纳入决策循环**，仅作为前端展示

**突破方向**：

```
当前:  策略信号 → 前端展示 + 独立 AI 分析（旁路）
目标:  策略信号 → AI 置信度评估 → 信号权重调整 → 最终决策
```

**具体路径**：

**A-1. AI 信号验证层**（近期，低风险）
- 在 `SignalFusion` 或 `ConfirmLayer` 中增加 AI 验证入口
- 将关键信号（S_BUY/S_SELL/S_DIVERG_SELL）的上下文（K线形态、筹码指标、阶段判定结果）传给 DeepSeek
- DeepSeek 返回信号置信度调整（0.5x~1.5x）
- 接入方式：异步调用，超时降级为 1.0x

**A-2. AI 趋势质量分级**（中期）
- PhaseDetector 判定为拉升期后，调用 AI 评估趋势质量（强趋势/弱趋势/过渡期）
- 对应调整 PositionManager 的仓位上限
- 与研究中的 "AI 将市场分为五种状态" 对应

**A-3. AI 风格诊断**（中期）
- 在 backtest_evidence.py 的 evaluator 中增加 AI 分析接口
- 对策略回测结果做风格诊断："该策略适配什么市场环境？最怕哪种行情？"
- 输出直接接入市场状态判断模块

### 2.4 突破方向 C：市场状态判断与策略调度

**研究依据**：
> "针对市场风格切换，实践者提出三种核心应对方案：多策略组合主动对冲、市场状态判断与策略调度、风格过滤器。"
> "设计规则（如中证2000相对沪深300超额收益为负时暂停小盘策略）进行策略启停。"

**当前状态**：
- `MarketEnvironmentFilter` 仅检查沪深300与60日均线的关系（GOOD/POOR）
- `SignalFusion` 使用固定权重加权（无动态调度）
- 没有"策略启停"机制

**突破方向**：

**C-1. 市场状态检测升级**（近期）
- 当前：沪深300 vs MA60（单一维度）
- 目标：四维市场状态：
  - 趋势强度维度：大盘趋势（沪深300/MA60）
  - 风格维度：大小盘分化（中证2000/沪深300 相对收益）
  - 情绪维度：换手率和涨跌比
  - 波动率维度：ATR 和 VIX 类指标

**C-2. 策略调度器**（中期）
- 新建 `strategy_scheduler.py`，管理策略启停和权重调整
- 输入：市场状态四维数据
- 输出：各策略的激活状态和权重乘数
- 示例规则：
  - 大盘强势（沪深300 > MA60）+ 小盘活跃（中证2000超额>0）→ 筹码策略权重 1.0
  - 大盘弱势 + 小盘活跃 → 筹码策略权重 0.7（谨慎做多）
  - 大盘弱势 + 小盘弱势 → 筹码策略权重 0.3，仅 S_BOUNCE 信号有效
  - 大盘单日跌>5% → 所有策略暂停新开仓

**C-3. SignalFusion 动态加权**（近期）
- 当前 `SignalFusion.fuse()` 使用固定权重
- 改为从 `strategy_scheduler` 读取动态权重

### 2.5 突破方向 D：多智能体架构雏形（Quant 4.0 方向）

**研究依据**：
> "Quant 4.0 并非对传统 ML 模型的替代，而是为其加装'智能外壳'——通过多智能体辩论实现从盲目算力穷举到逻辑驱动因子的转变。"
> "Man Group 的 AlphaGPT 即采用此架构，AI 自动生成研究思路、编写代码并进行回测。"

**当前架构局限性**：
- 三大策略（达尔文/筹码/缠论）独立运行
- AI 分析是旁路输出，不参与决策
- 没有"策略提案 → 策略评估 → 策略选择"的辩论机制

**突破方向**（中长期）：

```
提案智能体（策略生成）:
  ├─ 筹码策略引擎 (现有)
  ├─ 缠论策略引擎 (现有)
  ├─ 达尔文筛选引擎 (现有)
  └─ [未来] AI 提案策略 (DeepSeek 生成临时规则)

评估智能体:
  ├─ SignalWinRateEvaluator (现有 Phase4)
  ├─ AI 置信度评估 (方向 B)
  └─ 市场状态判断 (方向 C)

仲裁智能体:
  ├─ SignalFusion (升级版)
  ├─ StrategyScheduler (方向 C)
  └─ 风险总监 (当前 RiskExecutor/PositionManager)
```

**近期可实施的最小化版本**：
- 为每个信号增加"AI 置信度评分"字段（方向 B-1）
- SignalFusion 将 AI 评分作为加权因子
- 形成"策略提案 → AI 评估 → 融合决策"的三段式流程

---

### 2.6 突破方向 E：策略失效诊断体系

**研究依据**：
> "策略失效最常见的原因并非逻辑错误，而是验证时间窗口恰好选在了有利的市场环境中——即'时间窗口过拟合'。"
> "分段回测：将回测时间段切分为不同市场阶段，分别考察策略的胜率、回撤和年化。"

**当前状态**：
- `SignalWinRateEvaluator` 已做全局胜率评估和条件概率分支
- 但缺乏"分段回测"能力——按市场风格切分时间窗口分别评估

**突破方向**（中期）：

**E-1. 分段回测引擎**
- 在 `backtest_evidence.py` 中增加市场阶段划分逻辑
- 划分标准：大盘趋势（牛/熊/震荡）+ 大小盘风格（大盘占优/小盘占优/均衡）
- 每个阶段输出独立胜率、夏普比率、最大回撤
- 自动检测策略的"适配市场"和"天敌市场"

**E-2. 策略健康度仪表盘**
- 对每个活跃策略持续跟踪近期表现（滚动30/60日窗口）
- 当滚动胜率跌破阈值时发出预警
- 结合方向 C 的策略调度器自动降权

### 3.2 Research 中的对标素材：中金79个高频因子

**研究依据**：
> "中金公司发布了规模达 79 个的高频价量因子库，涵盖动量反转、波动性、高阶特征、流动性、量价相关性、筹码分布、拥挤度和资金成交等八大类别。"
> "多个因子周度年化多空收益夏普比率超过 5.0"
> "近三成的因子月度 IC 均值绝对值超过 5%，月度 ICIR 超过 0.65"
> "傅里叶变换机构拥挤度因子（crowd_fftv20_3s_w0_std）月度和周度 ICIR 分别为 1.37 和 1.38"

**与现有因子的对比**：

| 类别 | 中金因子 | 我们 | 差距 |
|------|---------|------|------|
| 动量反转 | ✅ 包含 | ✅ momentum/reversal | 中金使用高频分钟数据，我们是日线 |
| 波动性 | ✅ 包含 | ✅ volatility | 频率差异 |
| 高阶特征 | ✅ 包含 | ⚠️ 部分 | 中金使用傅里叶变换等复杂数学变换 |
| 流动性 | ✅ 包含 | ⚠️ 基础 | 中金有精细的微观流动性指标 |
| 量价相关性 | ✅ 包含 | ⚠️ 部分 | 中金使用滚动相关系数 |
| **筹码分布** | **✅ 包含** | **✅ 独立策略** | **中金的筹码因子是 79 个之一，我们是独立策略——这是优势** |
| **拥挤度** | **✅ 核心亮点** | **❌ 缺失** | **最大差距——crowd_fftv20 ICIR 1.37** |
| 资金成交 | ✅ 包含 | ⚠️ 基础 | 中金使用分钟级资金流数据 |

### 3.3 突破方向 F：拥挤度因子构建

**最大差距——缺失拥挤度因子**。

中金的拥挤度因子（尤其傅里叶变换机构拥挤度 crowd_fftv20）ICIR 高达 1.37，是目前已知最强的截面因子之一。该因子的核心思想：通过高频量价数据计算机构资金的交易拥挤程度。

**实施路径**：

**F-1. 基于现有数据的简化拥挤度指标**（近期）
- 使用日线数据近似计算拥挤度：
  - 换手率异常放大（当前换手率 / 20日均值 > 阈值）
  - 成交量集中度（前N日成交量方差）
  - 价格波动率压缩后的突然放大
- 无需新数据源，基于 `chip_indicators` 中已有的换手率和成交量数据

**F-2. 筹码拥挤度指标**（中期）
- 利用筹码分布数据计算：
  - 筹码峰聚集度（单一峰占比 > 30% 且 换手率高 = 拥挤）
  - ASR 持续高位（ASR>70% 持续 10 日以上 = 高拥挤）
  - 这些数据 `chip_indicators` 中已有计算

**F-3. Tushare 资金流拥挤度**（按需）
- 使用 `moneyflow` 接口计算机构资金流入集中度
- 需要确认 Tushare 权限

### 3.4 突破方向 G：因子模型管理优化

**当前问题**：
- 因子注册表是平面的（FactorRegistry），无分层管理
- 缺乏因子 IC 衰减监控
- 缺乏因子生命周期管理
- 因子权重静态，不随环境变化

**实施路径**：

**G-1. 因子分层管理**（近期）
```
FactorTier:
  ├─ CORE（核心因子：ICIR > 0.8，始终参与筛选）
  │   ├─ 动量类因子
  │   ├─ 质量类因子
  │   └─ 筹码类因子
  ├─ AUX（辅助因子：ICIR 0.3-0.8，按市场环境启用）
  │   ├─ 换手率类
  │   ├─ 波动率类
  │   └─ 拥挤度类
  └─ ENV（环境因子：描述市场状态，不直接参与选股）
      ├─ 大盘趋势
      ├─ 大小盘风格
      └─ 市场情绪
```

**G-2. IC 衰减监控**（中期）
- 在 `registry.py` 或新建 `factor_monitor.py`
- 对每个因子的月度 IC、ICIR、多空收益做滚动跟踪
- 当连续 3 个月 IC 下降 → 自动从 CORE 降级到 AUX
- 当 IC 转负 → 自动标记为 DEPRECATED（不参与筛选但保留历史数据）

**G-3. 因子生命周期**（中期）
```
DRAFT（初版） → TESTING（回测验证） → ACTIVE（上线） → MONITORING（监控） → DEPRECATED（废弃）
```
- 每个阶段有明确的准入/准出条件
- 与 `backtest_evidence.py` 的回测引擎对接

### 3.5 突破方向 H：因子验证方法优化

**研究依据**：
> "量化选股系列分析报告：规避隔日竞争，基于 T-0.5 的量价因子交易窗口优化"
> "当日收盘价交易的组合年化超额收益为 20.21%，延后至次日开盘价交易，超额收益损失 1.11 个百分点"

**当前验证体系**：
- 回测引擎支持 IC 分析、分组回测
- `SignalWinRateEvaluator` 支持信号赢率评估
- 但未做"交易窗口优化"分析

**实施路径**：

**H-1. 滚动 IC 窗口**（近期）
- 当前：全时间段一次计算 IC
- 目标：按季度滚动计算 IC，观察稳定性

**H-2. 分段 IC 分析**（近期）
- 在中金八大类因子框架下，按不同市场环境计算分段 IC
- 找出每个因子"最有效"和"最无效"的市场阶段
- 与方向 G-1 的因子分层管理对接

**H-3. T-0.5 交易窗口优化**（中期）
- 研究在收盘前交易（提前收盘 1-2 小时计算因子）vs 次日开盘交易的收益差异
- 需要分钟级数据支持
- 核心洞见：牺牲尾盘少量数据（与原始因子相关性 0.99+），显著提升超额收益

### 3.6 突破方向 I：风格过滤器

**研究依据**：
> "风格过滤器：在原策略基础上增加外部环境判断层。"
> "关键词含'小市值、微盘'的策略可能适配小盘活跃、情绪高涨期，失效于流动性危机、监管收紧期。"

**当前状态**：
- `MarketCapAdapter` 已做市值分档（小/中/大/超大）
- 但未根据市场风格调整策略参数

**实施路径**：
- 在 `ChipPreFilter` 中增加风格判断层
- 规则示例：
  - 小盘占优期：筹码策略正常执行（中小盘筹码更集中，KB 依据）
  - 大盘占优期：提升 CYQKL 阈值、降低仓位上限
  - 流动性危机期：策略暂停，仅保留 S_BOUNCE
- 风格判断数据源：`BenchmarkService` 已有的指数数据（沪深300 / 中证500 / 中证1000 / 中证2000）

## 四、优先级汇总

## 五、下一步建议

建议从**近期优先级最高、数据依赖最小、风险最低**的方向开始：

1. **B-1 AI 信号验证层** — 现有 DeepSeek 服务已就绪，只需将 AI 分析结果回纳入 `ConfirmLayer` 或 `SignalFusion` 的决策链
2. **C-1 市场状态四维升级** — 仅需扩展 `MarketEnvironmentFilter` 的判断逻辑，数据来源 `BenchmarkService` 已有
3. **F-1 简化拥挤度因子** — 基于现有换手率和成交量数据，在 `chip_indicators` 或 `factors/builtin` 中新增
4. **G-1 因子分层管理** — 在 `FactorRegistry` 中增加 Tier 字段，最小化代码改动

这四个方向各选一个先做方案确认，然后逐一定计划实施。
