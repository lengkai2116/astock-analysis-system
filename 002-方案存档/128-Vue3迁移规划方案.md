# Vue 3 迁移规划方案

## 一、迁移背景

### 1.1 为什么必须迁移

| 因素 | 状态 | 紧迫性 |
|------|------|--------|
| Vue 2 官方支持 | 已于 2023-12-31 **EOL** 🚫 | 🔴 |
| Vue 2 最后一个版本 | 2.7.16 (2024 年 1 月) | — |
| Ant Design Vue 1.x | 已停止维护 | 🔴 |
| Vue 生态系统重心 | 全部转向 Vue 3 | 🔴 |
| 新特性/安全更新 | 不再有 | 🔴 |
| 现有项目可运行 | ✅ 当前可正常开发 | 🟡 但窗口期有限 |

### 1.2 当前技术栈

| 组件 | 当前版本 | 目标版本 | 变化幅度 |
|------|---------|---------|---------|
| Vue | 2.7.16 | 3.5.x | 🔴 大规模 |
| Vue Router | 3.5.4 | 4.x | 🟡 中 |
| Vuex | 3.6.2 | Pinia 2.x | 🟡 中 |
| Ant Design Vue | 1.7.8 | 4.x | 🔴 大规模 |
| vue-i18n | 8.27.2 | 9.x | 🟡 中 |
| Vite | 5.4.21 | 6.x | 🟢 低(已有Vite) |
| ESLint | 7.32.0 | 9.x | 🟡 中 |
| less | 3.13.1 | 4.x | 🟢 低 |

## 二、代码库规模

### 2.1 文件统计

| 类型 | 数量 | 需修改的行数估计 |
|------|------|----------------|
| `.vue` 文件（组件+页面） | **19** 个 | 每条都要过 |
| `.js` 文件（服务/工具/路由/Store） | **13** 个 | 路由/Store/i18n 必须改 |
| 入口文件 | 1 (`main.js`) | 重写 |
| 样式文件 | 1 (`global.less`) | 部分变量可能变 |
| 配置文件 | 2 (vite.config, eslint) | 要改 |
| **总计** | **~36 个** | **全部需审查或修改** |

### 2.2 Vue 2 特有模式使用情况

| 模式 | 出现次数 | 迁移方式 |
|------|---------|---------|
| `Vue.use(...)` | 4 | → `app.use()` |
| `new Vue({...}).$mount()` | 1 | → `createApp(App).mount()` |
| `Vue.prototype.xxx` | 1 (axios) | → `app.config.globalProperties` |
| Options API (`data/computed/watch`) | **19 文件全部在用** | 可保留（Vue 3 兼容） |
| `this.$store.state.xxx` | 多文件 | → useStore() / 或保留 Options |
| `this.$router.push()` | 多文件 | → useRouter() / 或保留 |
| `Vue.mixin` / `Vue.extend` | 0 | ✅ 无影响 |
| `v-model` | 多文件 | Antd 4.x 语法有变 |
| `slot-scope` / `scopedSlots` | 少量 | → `v-slot` |

## 四、Ant Design Vue 1.x → 4.x 关键变化

这是迁移中工作量最大的部分。以下列出高频使用的组件 API 变化：

### 4.1 组件变化清单

| 组件 | 1.x 用法 | 4.x 用法 | 风险 |
|------|---------|---------|------|
| **a-table** | `@rowClick` / `@change` | `@rowClick` 基本兼容，columns 定义兼容 | 🟡 |
| **a-table** | `slot="xxx" scope="props"` | 统一 `#xxx="{ props }"` | 🟡 |
| **a-modal** | `@ok` / `@cancel` | 基本兼容 | 🟢 |
| **a-button** | `@click` / `icon="xxx"` | 4.x 图标分离（需要额外安装 `@ant-design/icons-vue`） | 🔴 |
| **a-icon** | `<a-icon type="star" />` | ❌ **移除** → `<StarOutlined />` 或 `svg` | 🔴 |
| **a-tag** | `color="..."` / `closable` | 基本兼容 | 🟢 |
| **a-select** | `v-model` / `@search` | 基本兼容 | 🟢 |
| **a-dropdown** | `v-model` | 略有变化 | 🟡 |
| **a-menu** | `theme="dark"` | 基本兼容 | 🟢 |
| **a-radio-group** | `button-style="solid"` | 基本兼容 | 🟢 |
| **a-spin** | `:spinning` | 基本兼容 | 🟢 |
| **a-empty** | `description` | 基本兼容 | 🟢 |
| **a-divider** | `type="vertical"` | 基本兼容 | 🟢 |
| **a-tooltip** | `title` prop | 基本兼容 | 🟢 |
| **a-tabs** | `@change` | 基本兼容 | 🟢 |
| **$message** | `this.$message.info()` | 改为 `message.info()` | 🟡 |

| **Form** | `a-form` / `a-form-item` | `a-form` 变化较大 | 🔴 |

## 五、迁移策略

### 5.1 推荐策略：渐进式迁移

```
总工期估计：5-7 人/天（单人约 1-2 周）
策略：不可一次性全部迁移，采用渐进式方案
```

#### 策略选型

| 策略 | 方案 | 工期 | 风险 | 推荐度 |
|------|------|------|------|--------|
| **A: 渐进式** | 保持 Options API + 逐组件迁移 | 1-2周 | 🟡 低 | ⭐⭐⭐⭐⭐ **推荐** |
| B: 全部重写 | 从零开始新项目 | 3-4周 | 🔴 高 | ❌ |
| C: 混用 | 新功能用 Vue 3，旧功能保留 Vue 2 | 无限期 | 🔴 极高 | ❌ |

### 6.4 Store 迁移 (Vuex → Pinia)

```javascript

  state: { sidebarCollapsed: false, ... },

  actions: { async loadFactors({ commit }) { ... } },
  getters: { currentSymbolKey: state => ... }
})

// ───────────────────────────────────────

// Pinia (目标) - 更简洁
import { defineStore } from 'pinia'
export const useAppStore = defineStore('app', {
  state: () => ({
    sidebarCollapsed: false,
    symbol: '000001.SZ',

  getters: {
    currentSymbolKey: (state) => `${state.market}:${state.symbol}`
  },
  actions: {
    toggleSidebar() {
      this.sidebarCollapsed = !this.sidebarCollapsed
    },
    async loadFactors() { ... }
  }
})

// 使用 Pinia（组件内）
// Options API: computed: { collapsed() { return useAppStore().sidebarCollapsed } }
// Composition API: const store = useAppStore(); store.toggleSidebar()
```

### 7.2 Options API → Composition API（可选，非强制）

Vue 3 完全兼容 Options API，**迁移时可以保留现有 `data/computed/watch/methods`** 写法。

```vue
<script>
// Vue 2 / Vue 3 均兼容的写法（无需立即改造）
export default {
  data() { return { count: 0 } },
  computed: { double() { return this.count * 2 } },
  methods: { increment() { this.count++ } }
}
</script>

// 如果想迁移到 Composition API（可选优化）
<script setup>
import { ref, computed } from 'vue'
const count = ref(0)
const double = computed(() => count.value * 2)
function increment() { count.value++ }
</script>
```

> **建议**：阶段二保持 Options API，仅确保 Vue 3 兼容。Composition API 迁移可作为单独阶段。

## 十、风险与注意事项

### 10.1 高风险项

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|---------|
| Antd 4.x 图标全部重写 | 100% | 🔴 高 | `@ant-design/icons-vue` + 全局替换 |
| Antd 4.x API 不兼容 | 80% | 🔴 高 | 逐个组件验证，无法兼容的用 wrapper 适配 |
| `@ant-design-vue/pro-layout` 无 Vue 3 版 | 50% | 🟡 中 | 替换为手动布局或替代库 |
| Less 主题覆盖失效 | 70% | 🟡 中 | 改用 CSS Variables |
| `VueAxios` 插件 Vue 3 不兼容 | 60% | 🟡 中 | 改为直接 import |
| 第三方库（klinecharts）调用方式变化 | 30% | 🟡 中 | 检查文档 |
