# 选股系统与个股分析解耦重构方案

## 一、问题回顾

### 1.1 当前架构的问题

当前将「选股」和「个股分析」两个用户需求绑定在同一个 IDE 页面中，用标签页切换来区分模式：

```
指标 IDE (indicator-ide)
├── [单股分析] 标签 ← 个股策略分析
└── [框架选股] 标签 ← 选股系统
```

这种设计导致了三个核心问题：

| 问题 | 表现 | 后果 |
|------|------|------|
| **用户心智冲突** | 一个页面试图满足两个不同场景 | 用户需要频繁切换标签页，使用流程断裂 |
| **状态管理复杂** | 两模式共享同一个 Vue 实例的状态 | 模式切换需要保存/恢复状态，容易出错 |
| **L2/L3 信号混排** | 右侧策略面板不分维度 | 用户无法区分"主力分析"和"策略验证"各自给出的建议 |

### 1.2 用户真实需求

```
需求 A：选股                                需求 B：个股策略分析
"最近哪些股票有主力介入？"                   "这只股票现在应该怎么操作？"

┌─────────────────────────┐                ┌─────────────────────────┐
│ 输入：全市场股票池       │                │ 输入：具体股票代码       │
│ 处理：L1风险→L2主力→L3  │                │ 处理：多维度策略分析     │
│ 输出：精选股票排行榜     │                │ 输出：操作建议+赢的概率  │
└─────────────────────────┘                └─────────────────────────┘

两者互不依赖，各有独立的入口、流程和产出。
```

## 二、新架构设计

### 2.1 架构总览

```
┌──────────────────────────────────────────────────────────────┐
│                    系统导航（Sidebar）                         │
├──────────────────────────────────────────────────────────────┤
│  📊 仪表盘 / 选股系统 / 个股分析 / 回测系统 / ...            │
│         ↑                  ↑                                  │
│    ┌────┴─────┐     ┌─────┴──────┐                          │
│    │ 选股系统   │     │ 个股策略分析  │ ← 这里跳转 →          │
│    │ /screener │     │ /indicator  │ 双击股票/从选股进入      │
│    └────┬─────┘     └─────┬──────┘                          │
│         │                  │                                   │
│         │  批量扫描        │  单只深度分析                      │
│         │  ⬇               │  ⬇                                │
│         │  L1→L2→L3       │  L2筹码+L3缠论+L3因子+自定义       │
│         │  排行榜输出      │  操作建议+赢的概率                  │
```

**核心思想**：两个独立页面，通过 URL 参数传递 `symbol`，各自拥有完整的布局自由。

### 4.3 右侧信号面板详细设计

#### 4.3.1 数据结构（对应后端 `StrategyOutput` 模型）

```json
{
  "strategy_name": "筹码主力分析",
  "signal": "BULLISH",          // BULLISH / BEARISH / NEUTRAL / WATCH
  "confidence": 0.72,           // 赢的概率 0-1
  "entry_zone": [185.0, 192.0], // 建仓区间
  "risk_line": 178.0,           // 止损线
  "target_zone": [210.0, 225.0], // 目标区间
  "position_suggestion": "20%",  // 仓位建议
  "holding_period": "1-3个月",   // 持仓周期
  "evidence": [                  // 依据

    "ASR持续上升至0.72"
  ],
  "risk_notes": [                // 风险提示

  ]
```

#### 4.3.3 三个维度

| 维度 | 信号来源 | 展示内容 |
|------|---------|---------|
| **L2 主力分析** | chip_strategy.ChanlunScorer / ChipAlphaModel | 主力阶段、ASR、集中度、主力评分 |
| **L3 策略验证** | chanlun_strategy.ChanlunScorer + factor + volume | 缠论买卖点、因子评分、量价（待添加） |
| **自定义策略** | CodeEditor 运行结果 | 用户自定义指标/策略的输出信号 |

#### 4.3.4 赢的概率评分标准

**待确认标准 —— 建议方案：**

| 分数区间 | 星级 | 含义 | 对应行动 |
|---------|------|------|---------|
| 85-100% | ⭐⭐⭐⭐⭐ | 强烈信号 | 重点配置 |
| 70-84%  | ⭐⭐⭐⭐ | 明显信号 | 正常配置 |
| 55-69%  | ⭐⭐⭐ | 参考信号 | 轻仓参与 |
| 40-54%  | ⭐⭐ | 弱信号 | 持续观察 |
| <40%    | ⭐ | 无信号 | 不操作 |

各策略维度建议的评分方法：

| 维度 | 评分方法 |
|------|---------|
| 筹码主力 | ASR(30%) + 集中度(20%) + 获利率(20%) + 成交量(15%) + RSI(15%)，阶段加分（建仓+2）→ 映射到100分 |
| 缠论 | 买点存在(35%) + 背驰信号(20%) + 趋势方向(15%) + 中枢状态(5%) + 卖点存在(-25%) → 映射到100分 |
| 因子 | 多因子加权综合得分 → 映射到100分 |
| 自定义 | 用户策略自行定义和输出 confidence 字段 |

```python
        'custom': 0.15    # 自定义权重
    }
    # 加权计算综合赢率
    # 取多数策略一致的信号方向作为综合建议
```
