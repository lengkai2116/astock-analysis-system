# 量价策略深化方案设计报告（v3）

## 一、深化原则

v3 是在 v2 框架上的**细化与辅助增强**，而非全面替换。核心不变的是：

```
分析逻辑链（不可动摇）：

Phase 1: 阶段状态判定 → 股票处于什么阶段？
Phase 2: 成交量状态评估 → 成交量在做什么？
Phase 3: 量价关系分析 → 量价如何互动？
Phase 4: 信号生成 → 在右侧交易约束下输出信号
```

新增内容作为各 Phase 内部的**辅助校准维度**，为决策提供更多依据，但不改变主逻辑链的方向和顺序。

| Phase | 主逻辑（v2，不变） | 辅助增强（v3 新增） | 增强目的 |
|-------|-------------------|-------------------|---------|
| Phase 1 | 波段四阶段判定 | + 多周期价格分位评估 | 判断当前价格在历史中的高低位置，辅助阶段置信度校准 |
| Phase 2 | 成交量趋势+结构 | + 主力入市阶段推断 | 从量价模式反推主力行为，辅助判断成交量变化的背后原因 |
| Phase 3 | 量价关系分析 | + 多空动量评估 | 评估当前趋势的动能强弱，辅助判断后续趋势可持续性 |
| Phase 4 | 信号生成 | 上述辅助维度参与信号校准 | 多方面综合提升信号质量 |

## 二、Phase 1 增强：阶段判定 + 价格分位

### 2.1 主逻辑（保持 v2 不变）

```
波段四阶段判定：
  UPTREND_ACTIVE      — MA60向上 + HH/HL序列确认
  UPTREND_TOPPING     — 冲顶特征（缩量新高/放量滞涨/顶部K线形态）
  DOWNTREND_ACTIVE    — MA60向下 + LH/LL序列确认
  DOWNTREND_BOTTOMING — 探底特征（缩量止跌/底部形态/放量试探反弹）
  CONSOLIDATION       — 均线纠缠，无明确方向
```

### 2.2 辅助维度：多周期价格分位

量价分析中，同样的量价形态在不同价格位置含义不同。价格分位帮助回答"当前价格在历史中处于什么位置"。

**分位计算**：

```
价格分位 = (当前价 - N日最低价) / (N日最高价 - N日最低价)

三周期：
  短期分位 = 30日价格分位
  中期分位 = 60日价格分位
  长期分位 = 120日价格分位
```

**分位区间**：

| 区间 | 范围 | 含义 |
|------|------|------|
| 低位区 | 0~25% | 价格处于该周期底部 |
| 中低位区 | 25~40% | 偏低但脱离底部 |
| 中位区 | 40~60% | 价格处于该周期中部 |
| 中高位区 | 60~75% | 偏高但未到顶部 |
| 高位区 | 75~100% | 价格处于该周期顶部 |

知识来源：知识库"量价时空"——量价形态需结合价格所处位置判断；"简单估值法"——价格低于/高于合理估值时的操作策略不同。

### 2.3 辅助校准规则

分位不改变阶段判定结果，但影响置信度和附加风险提示：

```
分位对阶段判定的辅助校准：

  UPTREND_ACTIVE + 短期低位区  → 置信度不变，附加"刚启动，上行空间充足"
  UPTREND_ACTIVE + 短期高位区  → 置信度 × 0.85，附加"高位风险，关注触顶信号"
  DOWNTREND_ACTIVE + 短期低位区 → 置信度不变，附加"超跌区域，关注探底信号"
  DOWNTREND_ACTIVE + 短期高位区 → 置信度 × 1.05，附加"下跌初期，风险较大"
  DOWNTREND_BOTTOMING + 长期低位区 → 置信度 × 1.10，附加"长周期底部区域"
```

**新增输出字段**（在阶段判定结果中）：

```
stage.valuation_position = {
    "short_30d": 0.55,    # 中位区
    "mid_60d": 0.42,     # 中低位区
    "long_120d": 0.35,    # 中低位区
    "note": "短中长均处于中位区"
}
```

## 三、Phase 2 增强：成交量状态 + 主力行为推断

### 3.1 主逻辑（保持 v2 不变）

```
成交量趋势：
  EXPANDING     — 近20日均量 / 前60日均量 > 1.3
  CONTRACTING   — < 0.7
  STABLE        — 0.7~1.3

均量线结构：
  BULLISH  — VolMA5 > VolMA10 > VolMA20
  BEARISH  — VolMA5 < VolMA10 < VolMA20
  MIXED    — 交叉无序

量异常检测：倍量(>3×)、放量(>1.5×)、地量(<0.3×)
量托/量压：VolMA5/10/20 金叉/死叉封闭三角
```

### 3.2 外盘/内盘/大单数据可行性评估

| 数据需求 | 可行性 | 原因 |
|---------|--------|------|
| 外盘/内盘（主动买卖盘） | ❌ 不可用 | 需 Level-2 Tick 数据，日线 OHLCV 不含 |
| 大单/散户资金统计 | ❌ 不可用 | 需 Level-2 逐笔成交数据 |
| Tushare moneyflow（主力净流入） | ⚠️ 可选 | 需 Tushare Pro 积分 ≥2000，数据有延迟 |

**结论**：量价策略的核心数据仅需日线 OHLCV，不依赖 Level-2 数据。Tushare moneyflow 作为可选增强（不阻塞）。

### 3.3 辅助维度：从量价形态反推主力行为

在无法获取外盘/内盘/大单数据的情况下，通过量价形态模式反推主力所处阶段。借鉴现有 `chip_strategy_impl.py` 的 `TradingPhaseDetector` 评分模式。

**主力四阶段推断**（作为量状态的辅助解读）：

```
推断逻辑：基于量价形态评分，判断当前最可能的主力阶段

  BUILDING (建仓): 低位横盘 + 温和放量 + 价平量增
  WASHING  (洗盘): 缩量回调至均线 + 量萎缩 + 价不创新低
  RAISING  (拉升): 价涨量增 + 均线多头 + 放量突破
  SHIPPING (出货): 高位放量滞涨 + 量价背离 + 缩量上涨
```

评分细则参考知识库"主力行为分析"四阶段特征和"放量缩量模式"的成交量变化模式。

### 3.4 辅助校准规则

主力阶段推断不改动量状态评估的主结论，但为量状态提供辅助解读：

```
量状态 + 主力阶段 → 辅助解读：

  EXPANDING + BUILDING  → "放量建仓中，底部量能积累"
  EXPANDING + RAISING   → "放量拉升中，上涨有量能支撑"
  EXPANDING + SHIPPING  → "放量出货中，注意高位风险"
  CONTRACTING + WASHING → "缩量洗盘中，浮筹清洗"
  CONTRACTING + SHIPPING → "缩量出货中，主力撤退迹象"
```

**新增输出字段**（在量状态评估结果中）：

```
vol_state.institutional_inference = {
    "likely_phase": "RAISING",
    "confidence": 0.68,
    "interpretation": "近期价涨量增、均量线多头排列，符合拉升期特征"
}
```

## 四、Phase 3 增强：量价关系 + 多空动量

### 4.1 主逻辑（保持 v2 不变）

```
九种量价形态：VP-1~VP-9（价涨量增 → 价跌量缩）

量价背离检测：
  顶背离 — 价格创波段新高，对应成交量未创新高
  底背离 — 价格创波段新低，对应成交量反而放大

阶段×量价交叉矩阵（v2 内容保留）
量托/量压（v2 内容保留）
```

### 4.2 辅助维度：多空动量评估

在完成量价关系分析后，进一步评估当前趋势的动能强弱，判断趋势是否健康、可持续。与缠论"走势能量"概念一致——多空能量在 K 线和成交量中有客观体现。

**动量评估公式**：

```
多空动量分 = K线力度分(0.3) + 量能方向分(0.35) + 价格加速度分(0.35)

各子维度：

① K线力度（近10日）:
   阳线实体均值 / 阴线实体均值
   > 1.5 → 多方主导  |  < 0.67 → 空方主导  |  中间 → 均衡

② 量能方向（近10日）:
   上涨日均量 / 下跌日均量
   > 1.2 → 上涨有量  |  < 0.8 → 下跌放量  |  中间 → 均衡

③ 价格加速度:
   5日均涨幅 vs 20日均涨幅
   > 1.5 → 加速上涨  |  < 0.5 → 减速上涨（动力衰减）
   或对应下跌方向的加速/减速

综合得分范围: -1.0 ~ +1.0
  +0.5 ~ +1.0  → 多方动能充足
  +0.2 ~ +0.5  → 多方偏强
  -0.2 ~ +0.2  → 动能均衡
  -0.5 ~ -0.2  → 空方偏强
  -1.0 ~ -0.5  → 空方动能充足
```

知识来源：知识库"走势能量"——走势是能量的释放、纠缠、衰竭过程；"多方炮空方炮"——K 线力度判断多空强弱；"量价背离"——顶/底背离是能量衰竭的标志。

### 4.3 辅助校准规则

动量评估不改动量价关系分析的主结论，但为量价信号提供"趋势能否持续"的参考：

```
量价形态 + 动量 → 辅助判断：

  VP-1（价涨量增）+ 多方动能充足 → "上涨健康，趋势有望延续"
  VP-1（价涨量增）+ 多方动能偏弱 → "上涨中量能不足，关注后续能否补量"
  VP-3（价涨量缩）+ 多方动能衰减 → "上涨动力衰竭，警惕回调"
  VP-7（价跌量增）+ 空方动能充足 → "恐慌抛售中，不宜抄底"
  VP-9（价跌量缩）+ 空方动能衰减 → "下跌动力衰竭，关注底部信号"
```

**新增输出字段**（在量价关系分析结果中）：

```
relation.momentum = {
    "score": 0.52,
    "level": "多方动能充足",
    "candle_force": "多方主导 (阳/阴=1.8)",
    "volume_direction": "上涨有量 (买量/卖量=1.3)",
    "price_accel": "温和加速",
    "interpretation": "当前上涨有量能支撑，趋势健康"
}
```

## 五、Phase 4 更新：信号生成（融合辅助维度）

v3 中 Phase 4 的主逻辑不变——仍然是右侧交易约束下的信号生成。辅助维度（价格分位、主力推断、动量评估）融入信号校准环节。

### 5.1 信号生成流程（更新版）

```
Step 4-1: 基础信号匹配（阶段 × 量价形态 → 12种信号ID）
  → 同 v2，保持不变

Step 4-2: 信号置信度校准
  → 在基础置信度上，辅助维度进行微调：

  价格分位校准：
    低位区 + 买入信号 → +0.05
    高位区 + 买入信号 → -0.10（高位风险）
    低位区 + 卖出信号 → -0.05（低位不宜杀跌）

  主力推断校准：
    RAISING + 买入信号 → +0.05
    SHIPPING + 买入信号 → 降级为 WATCH（主力出货时不宜买入）

  动量评估校准：
    多方动能充足 + 买入信号 → +0.05
    空方动能充足 + 买入信号 → 降级为 WATCH（逆势不宜买入）
    价格涨但动量衰减（背离）→ -0.10（顶背离预警）

Step 4-3: 右侧交易铁律过滤（同 v2，不变）
  → DOWNTREND_ACTIVE 中不输出 BUY
  → UPTREND_ACTIVE 中不输出 SELL
  → 背离信号仅输出 WATCH
```

### 5.2 信号体系（与 v2 一致的 12 信号 + 辅助维度说明）

```
信号ID          方向    基础条件              辅助维度参考
──────────────────────────────────────────────────────────
VP_TREND        BUY     UPTREND_ACTIVE+价涨量增  +低位区+RAISING+多方动能→最强
VP_PULLBACK     BUY     UPTREND_ACTIVE+价跌量缩  均线支撑位+缩量洗盘→入场时机
VP_BREAKOUT     BUY     UPTREND_ACTIVE+放量突破   前高放量+多方动能→确认有效
VP_WEAKENING    WATCH   UPTREND_ACTIVE+价涨量缩   +高位区+动量衰减→警惕触顶
──────────────────────────────────────────────────────────
VP_DIV_TOP      WATCH   UPTREND_TOPPING+顶背离    +SHIPPING+动量衰减→强烈预警
VP_STAGNATION   WATCH   UPTREND_TOPPING+放量滞涨   +高位区+SHIPPING→触顶确认
VP_TOPPING      WATCH   UPTREND_TOPPING+量压+破MA20 +高位区→减仓信号
──────────────────────────────────────────────────────────
VP_PANIC        SELL    DOWNTREND_ACTIVE+价跌量增  +空方动能充足→恐慌中
VP_BREAKDOWN    SELL    DOWNTREND_ACTIVE+放量破位   +空方动能→破位确认
──────────────────────────────────────────────────────────
VP_DIV_BOT      WATCH   DOWNTREND_BOTTOM+底背离    +低位区+缩量企稳→观察反转
VP_BOTTOMING    WATCH   DOWNTREND_BOTTOM+缩量横盘   +长期低位区+BUILDING→跟踪
VP_REVERSAL     WATCH   DOWNTREND_BOTTOM+放量突破   +低位区+多方动能初现→待确认
──────────────────────────────────────────────────────────
VP_SQUEEZE      HOLD    CONSOLIDATION+缩量整理      +中位区+动能均衡→等待方向
```

## 六、软件架构（与 v2 一致，辅助维度内嵌）

主文件：`backend/app/engine/framework/volume_price_strategy.py`（~1000行）

```
VolumePriceStrategy（主入口）
    │
    ├── StageDetector（Phase 1：阶段判定）
    │   ├── detect_stage()          ← 主逻辑：波段四阶段（v2）
    │   ├── calc_price_position()   ← 辅助：三周期价格分位（v3新增）
    │   └── calibrate_by_position() ← 辅助：分位校准阶段置信度（v3新增）
    │
    ├── VolumeStateAnalyzer（Phase 2：成交量状态）
    │   ├── analyze_volume_state()     ← 主逻辑：量趋势+结构+量托量压（v2）
    │   ├── detect_inst_phase()       ← 辅助：从量价模式推断主力阶段（v3新增）
    │   └── interpret_vol_inst()      ← 辅助：量状态+主力联合解读（v3新增）
    │
    ├── VolumePriceRelationAnalyzer（Phase 3：量价关系）
    │   ├── analyze_relation()         ← 主逻辑：九形态+背离+交叉矩阵（v2）
    │   ├── calc_momentum()           ← 辅助：多空动量评估（v3新增）
    │   └── interpret_vp_momentum()   ← 辅助：形态+动量联合解读（v3新增）
    │
    └── VolumePriceSignalGenerator（Phase 4：信号生成）
        ├── match_base_signal()        ← 主逻辑：阶段×量价→信号ID（v2）
        ├── calibrate_confidence()     ← 增强：融合分位+主力+动量校准（v3更新）
        ├── apply_right_side_filter()  ← 主逻辑：右侧交易铁律（v2）
        ├── calc_entry_zone()
        ├── calc_risk_line()
        └── build_evidence()           ← 增强：含辅助维度证据（v3更新）
```

## 附A：设计决策确认

| # | 决策项 | 立场 | 依据 |
|---|--------|------|------|
| 1 | 辅助维度定位 | 校准微调，不改变主逻辑 | 量价分析是策略核心，辅助维度只做 ±0.05~0.10 的置信度微调 |
| 2 | 外盘/内盘/大单数据 | 不纳入，以量价形态反推主力 | 需 Level-2 数据，日线 OHLCV 不含；量价模式反推已足够 |
| 3 | 价格分位周期 | 30/60/120 日三周期 | 缩短周期，更贴近A股波段操作节奏；30日=近期波动，60日=中期趋势，120日=中期趋势定位 |
| 4 | 主力推断方法 | 评分制，借鉴 `TradingPhaseDetector` | 与筹码策略保持一致 |
| 5 | 动量权重 | 量能方向 0.35 > K线力度 0.30 = 价格加速度 0.35 | 量能方向是量价策略核心，权重微高 |
| 6 | 辅助维度校准幅度 | 单维度 ±0.05~0.10 | 辅助维度不主导信号，只提供边角校准 |
