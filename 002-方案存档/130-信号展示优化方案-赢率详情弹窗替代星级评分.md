# 信号展示优化方案

## 背景

原设计方案（见 123-选股系统与个股分析解耦重构方案）在信号卡片中使用星级评分：

```
⬆ 买入         赢率 72%  ⭐⭐⭐⭐
```

用户认为星级评分过于主观，建议：
1. 只展示赢率数值
2. 点击赢率后弹窗显示各策略维度的详细分析说明
3. 借助 AI 能力生成可读性强的解释文本

## 可行性评估

### 技术前提

当前系统已有的能力：

| 能力 | 状态 | 说明 |
|------|------|------|
| 策略信号计算 | ✅ | SignalComputationService 可计算 Chip / Chanlun / Factor 三维信号 |
| DeepSeek API 集成 | ✅ | deepseek_analysis_service.py 已就绪，5 角色分析可用 |
| 弹窗组件 | ✅ | Antd `<a-modal>` 已在多处使用 |
| 信号数据 | ✅ | 各维度评分数据已包含在 signal dict 中 |

### 需要改动的工作量评估

| 改动项 | 涉及文件 | 预估工时 |
|--------|---------|---------|
| 信号卡片 UI：去掉星级，赢率改为可点击 | `StrategySignalPanel.vue` | 0.5h |
| 新增详情弹窗组件 | `SignalDetailModal.vue`（新建） | 1.5h |
| AI 分析接口：根据信号数据生成解释文本 | `deepseek_analysis_service.py` 新增方法 | 1h |
| 前端 API 对接 | 新增 `signalExplain()` 调用 | 0.5h |
| 数据联动：筛选结果双击跳转 IDE 携带信号数据 | `ScreeningResults.vue` → `indicator-ide` | 0.5h |
| **合计** | | **约 4h** |

## 设计方案

### 1. 信号卡片新格式

```
┌─────────────────────────────────────┐
│  ⬆ 买入     赢率 72%  [查看详情 →] │
│  依据: MACD金叉确认, 成交量放大...   │
│  止损: 16.80    目标: 19.50         │
│  仓位: 20%     持有: 1-3个月       │
└─────────────────────────────────────┘
```

- 去掉 `⭐⭐⭐⭐`
- `72%` 改为带下划线的可点击文本
- 右侧增加 `[查看详情 →]` 链接

### 3. AI 分析接口设计

```
POST /api/v3/ai/signal-explain

Request:
{
  "ts_code": "000001.SZ",
  "signals": [
    {
      "strategy_name": "筹码主力分析",
      "dimensions": { "asr": 65, "concentration": 72, "profit_rate": 58, "volume": 70, "rsi": 55 },
      "phase_bonus": { "building": 2 },
      "score": 6.8
    },
    ...
  ]
}

Response:
{
  "success": true,
  "data": {
    "explanations": [
      {
        "strategy": "筹码主力分析",
        "ai_summary": "该股筹码集中度处于较高水平...",
        "ai_advice": "建议关注后续拉升信号...",
        "risk_tip": "注意大盘系统性风险..."
      },
      ...
    ],
    "composite_advice": "三维信号共振：筹码面主力控盘明显..."
  }
}
```

### 4. 降级策略

如果 DeepSeek API 不可用（未配置 key 或网络不通），弹窗仍然打开，但 AI 解读区域显示固定模板文本，结构不变：

```
AI 解读:
（当前为离线模式，基于规则生成）
筹码集中度较高，ASR指标中等偏上，成交量温和放大...
```

## 所需改动清单

### 后端

| 文件 | 改动 |
|------|------|
| `services/deepseek_analysis_service.py` | 新增 `explain_signal()` 方法，接收信号维度数据，返回 AI 解读 |
| `routes/ai_analysis.py` | 新增 `POST /api/v3/ai/signal-explain` 路由 |

### 前端

| 文件 | 改动 |
|------|------|
| `components/StrategySignalPanel.vue` | 赢率改为可点击，去掉星级 |
| `components/StockScreener/SignalDetailModal.vue` | **新建**：弹窗组件 |
| `services/aiAnalysisService.js` | 新增 `explainSignal()` API 调用 |

## 不涉及改动的

- 信号计算逻辑不变（`SignalComputationService`）
- 后端评分算法不变
- 筛选结果表格不变（仍显示综合评分排序）
- SignalFusionConfig 权重配置不变
