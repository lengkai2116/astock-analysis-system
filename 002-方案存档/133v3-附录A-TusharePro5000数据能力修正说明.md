**真正需要QMT Level1才能补的只有：**

2. **盘中实时信号推送** → 非T+1信号
3. **全推行情高速筛选** → 全市场实时监控

### 当前代码状态总结

所谓"需要QMT"的项目，实际情况：

1. **换手率 → 代码已经完整（Provider + Cache + DataManager），芯片指标函数 `calculate_all_indicators()` 的参数列表里已经预留 `turnover_rate` 形参**，只是调用方从未传入。工作量：改一行代码。

2. **大单流向 → Provider的 `get_moneyflow()` 已实现**，但缺 DuckDB 缓存层和 DataManager 封装。需要补充约50行代码。

3. **分钟线 → Provider的 `get_minute_data()`（内部调 pro_bar）已实现**，策略中K线形态识别用日线OHLCV就够了，分钟线只用于盘中实时增强。

4. **指数环境过滤 → BenchmarkService 已实现**，内部已调 index_daily，只需在策略入口加一行调用。

这样就非常清晰了——之前我在报告中大量标注"需QMT"是不准确的。正确的画面是：**代码基础设施几乎都写好了，缺的是策略层把它们串联起来的一根线。**, QMT真正不可替代的只有实时五档盘口这一项。
