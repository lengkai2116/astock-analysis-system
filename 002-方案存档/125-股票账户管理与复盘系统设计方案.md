# 股票账户管理与复盘系统设计方案

## 一、项目概述

### 1.1 背景与目标

当前系统已完成选股、个股分析、回测、报告等核心功能，但缺少**用户股票账户**的管理维度。用户无法在系统内记录自己的实际交易、跟踪资金盈亏、也不支持对交易过程进行系统性复盘。

本方案设计一个独立的**账户管理与复盘页面**（`/account`），让用户：

1. **记录实盘交易** — 自行填买入/卖出记录
2. **跟踪资金曲线** — 实时展示盈亏汇总
3. **智能复盘** — 结合系统策略数据与用户买卖记录，输出复盘报告
4. **持续改进** — 通过复盘发现交易问题，优化策略

### 1.2 用户画像

| 角色 | 特征 | 需求 |
|------|------|------|
| 个人投资者 | 有实盘记录，手动记账 | 记账+汇总盈亏+简单复盘 |
| 量化交易者 | 有策略回测+实盘数据 | 对比回测vs实盘差异+深度归因 |
| 学习型用户 | 复盘交易过程 | 多维复盘+建议+报告输出 |

## 二、复盘维度的研究基础

### 2.1 中国A股专业复盘体系

参考A股顶级游资和机构投资者的复盘框架，成熟的交易复盘应覆盖以下维度：

| 复盘维度 | 核心问题 | 参考体系 |
|---------|---------|---------|
| **大盘环境** | 指数当前处于什么阶段？量能是否配合？ | 缠论/波浪理论 |
| **板块轮动** | 当前主线板块是什么？是否有板块效应？ | 题材-板块-个股三层分析 |
| **个股操作** | 买卖点是否合理？是否符合计划？ | 交易计划-执行-检查循环 |
| **资金管理** | 仓位控制是否合理？风险敞口多大？ | 凯利公式/固定比例 |
| **心态纪律** | 是否严格执行计划？有无情绪化交易？ | 交易心理学 |
| **策略归因** | 盈利是靠策略还是运气？亏损的原因？ | 归因分析框架 |

### 2.2 西方交易日志体系

| 复盘维度 | 核心指标 | 参考来源 |
|---------|---------|---------|
| **P&L归因** | Win Rate, RR Ratio, Expectancy | Market Wizards |
| **Setup分析** | 哪种形态的表现最好/差 | Trade Journal 实践 |
| **执行质量** | Slippage, 提前/延迟入场 | Quantitative Self |
| **心理状态** | 情绪评分, 决策质量评分 | Trading Psychology |
| **市场环境** | VIX, 趋势vs震荡, 波动率 | Regime Detection |

### 2.3 本系统采用的综合复盘模型

结合以上体系，提炼出适合本系统的**七维复盘模型**：

```

                  │  ① 大盘环境评估    │
                  └────────┬──────────┘
                           ↓
                  ┌───────────────────┐
                  │  ② 板块与题材分析  │
                  └────────┬──────────┘
                           ↓
        ┌───────────────────┬───────────────────┐
        ↓                   ↓                   ↓
  ┌───────────┐    ┌───────────────┐    ┌───────────────┐
  │ ③ 个股操作  │    │ ④ 策略执行     │    │ ⑤ 资金管理     │
  │   买入/卖出  │    │   信号匹配度    │    │   仓位控制     │
  │   时机评估  │    │   策略合规性    │    │   风险敞口     │

        └───────────────────┬───────────────────┘

                  └────────┬──────────┘

                  │  ⑦ 综合归因与改进   │
                  │  (报告输出)         │
                  └───────────────────┘
```

#### 维度详解

**① 大盘环境评估**
- 目标：评估当前交易周期内的市场整体环境
- 输入：指数K线（上证/深证/创业板）、成交量、涨跌家数、北向资金
- 输出：市场阶段判定（牛市/熊市/震荡/结构性行情）、风险等级
- 数据来源：系统已有 `/api/v1/market/overview`

**② 板块与题材分析**
- 目标：识别复盘周期内的主线和轮动节奏
- 输入：行业板块涨跌幅排名、概念板块热度、涨停/跌停分布
- 输出：主线板块判定、个股操作与板块的匹配度
- 数据来源：需新增板块行情 API

**③ 个股操作评估**
- 目标：逐笔分析买卖操作的合理性
- 输入：用户录入的买卖记录、系统当时的策略信号、K线走势
- 分析维度：
  - 买入时机：是否在策略建议的信号窗口内？
  - 卖出时机：止盈/止损是否合理？有无过早/过晚？
  - 操作一致性：是否按计划操作？
- 输出：每笔操作评分（1-5分）+ 文字建议

**④ 策略执行评估**
- 目标：检查交易是否遵循了系统策略和交易计划
- 输入：操作记录 vs 系统信号 vs 用户自定义策略
- 分析维度：
  - 策略合规率：符合策略信号的交易占比
  - 信号覆盖度：系统发出的信号被执行的占比
  - 策略偏差：偏离策略的原因分析
- 输出：执行评分 + 偏差列表

**⑤ 资金管理评估**
- 目标：评估仓位控制、资金分配、风险管理的合理性
- 计算指标：
  - 单笔风险敞口（占本金%）
  - 总仓位集中度
  - 最大回撤
  - 连续亏损后的仓位变化
- 输出：资金管理评分 + 风险提示

**⑥ 心态与纪律复盘**
- 目标：识别情绪化交易、纪律性问题
- 信号：
  - 追涨杀跌（买入在涨幅>5%后/卖出在跌幅>5%后）
  - 频繁交易（单日操作次数 > 阈值）
  - 报复性交易（大亏后立即重仓）
  - 犹豫不决（信号出现但未执行）
- 输出：心态评分 + 具体问题点

**⑦ 综合归因与改进**
- 目标：将上述六个维度的分析聚合，输出行动项
- 输出结构：
  - 本次复盘周期总评分
  - 各维度雷达图
  - TOP 3 可改进事项
  - 下一周期重点关注

```python
class Trade(Base):
```

**API 设计**:
| 方法 | 路由 | 说明 |
|------|------|------|
| GET | `/api/v1/account/positions` | 当前持仓列表（实时价格待集成） |
| GET | `/api/v1/account/summary` | 账户总览指标（资产/盈亏总额） |
| GET | `/api/v1/account/history` | 历史持仓快照（持仓变化曲线） |

**自动计算规则**:
```
持仓均价 = 买入总金额 / 买入总数量（含多次买入加权）
已实现盈亏 = Σ(卖出金额 - 对应买入成本)
浮动盈亏 = Σ(现价 - 持仓均价) × 持仓数量
总盈亏 = 已实现盈亏 + 浮动盈亏
总资产 = 现金余额 + 持仓市值
收益率 = 总盈亏 / 总投入本金
```

**绩效指标体系**:

| 指标 | 计算方式 | 说明 |
|------|---------|------|
| 累计收益率 | (最终净值 - 初始净值) / 初始净值 | 总收益 |
| 年化收益率 | (1 + 累计收益率)^(365/天数) - 1 | 年化对标 |
| 最大回撤 | Max(Peak-to-Trough) | 最大亏损幅度 |
| 夏普比率 | (年化收益 - 无风险利率) / 年化波动率 | 风险调整收益 |
| 胜率 | 盈利交易次数 / 总交易次数 | 交易准确性 |
| 盈亏比 | 平均盈利 / 平均亏损 | 赔率 |
| 期望值 | 胜率 × 平均盈利 - (1-胜率) × 平均亏损 | 单笔期望收益 |

##### 复盘引擎核心逻辑

```python
class ReviewEngine:
    """复盘引擎 - 七维评估"""

    def calculate_composite_score(self, user_trades, market_data, strategy_signals, period):
        """综合评分 = 各维度加权"""
        dimensions = {
            'market': self.eval_market(market_data, period),
            'sector': self.eval_sector(user_trades, market_data, period),
            'trade': self.eval_trade_ops(user_trades, strategy_signals),
            'strategy': self.eval_strategy_exec(user_trades, strategy_signals),
            'capital': self.eval_capital_mgmt(user_trades),
            'psychology': self.eval_psychology(user_trades)
        }
        weights = {'market': 0.15, 'sector': 0.15, 'trade': 0.20,
                   'strategy': 0.20, 'capital': 0.15, 'psychology': 0.15}
        total = sum(dimensions[k]['score'] * weights[k] for k in weights)
        return {'dimensions': dimensions, 'total': round(total, 1)}

    def eval_trade_ops(self, trades, signals):
        """个股操作评分：买卖点匹配度"""
        for trade in trades:
            nearby_signals = self._find_nearby_signals(trade, signals)
            position_quality = self._eval_position_in_kline(trade)
            score = self._calc_trade_score(nearby_signals, position_quality)
            trade.review = {'score': score, 'signals': nearby_signals, 'position': position_quality}

    def eval_strategy_exec(self, trades, signals):
        """策略执行评估：信号覆盖度 + 合规率"""
        compliant = sum(1 for t in trades if self._matches_signal(t, signals))
        signal_executed = sum(1 for s in signals if self._was_executed(s, trades))
        return {
            'compliance_rate': compliant / len(trades) * 100,
            'signal_execution_rate': signal_executed / len(signals) * 100,
            'deviations': self._find_deviations(trades, signals)
        }

    def eval_psychology(self, trades):
        """心态识别：追逐/恐慌/报复/犹豫"""
        red_flags = []
        for i, trade in enumerate(trades):
```

```sql
CREATE TABLE account_trades (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,

    price           DECIMAL(10,2) NOT NULL,       -- 成交价格
    quantity        INTEGER NOT NULL,             -- 成交数量
    amount          DECIMAL(14,2) NOT NULL,       -- 成交金额
    commission      DECIMAL(10,2) DEFAULT 0,      -- 手续费
    tax             DECIMAL(10,2) DEFAULT 0,      -- 印花税
    notes           TEXT,                         -- 备注
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### 6.2 持仓快照表

```sql
CREATE TABLE account_positions (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    ts_code         VARCHAR(16) NOT NULL,

    avg_cost        DECIMAL(10,2) NOT NULL,       -- 持仓均价
    quantity        INTEGER NOT NULL,             -- 持仓数量
    realized_pnl    DECIMAL(14,2) DEFAULT 0,      -- 已实现盈亏

    UNIQUE(ts_code, snapshot_date)
);
```

```sql
CREATE TABLE account_equity (

    cash_balance    DECIMAL(14,2) NOT NULL,       -- 现金余额
    market_value    DECIMAL(14,2) NOT NULL,       -- 持仓市值
    daily_pnl       DECIMAL(12,2) DEFAULT 0,      -- 当日盈亏
    cumulative_pnl  DECIMAL(14,2) DEFAULT 0,      -- 累计盈亏
    cumulative_return DECIMAL(6,4) DEFAULT 0,     -- 累计收益率
```
