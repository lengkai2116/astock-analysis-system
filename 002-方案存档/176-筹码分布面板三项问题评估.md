### ① 价格标签位置：纵坐标移至右侧 — 图表左移

#### 当前实现

`renderChipChart()`（`indicator-ide.html:1058-1066`）：

```javascript
var priceAxisLeft = 42;  // 价格标签从左边界占宽
var barAreaW = containerW - priceAxisLeft - 4; // 筹码线绘制区域剩余宽度

// 价格标签（左侧）
var priceLabels = [hi, hi-(hi-lo)*0.75, (hi+lo)/2, lo+(hi-lo)*0.25, lo];
for(var pi = 0; pi < priceLabels.length; pi++){
  var py = (1 - (priceLabels[pi] - lo) / range) * containerH;
  var labelTop = Math.max(0, Math.min(containerH - 10, py - 5));
  html += '<div style="position:absolute;left:1px;top:...;width:38px;...">'+fmt+'</div>';
}

// 筹码线从 priceAxisLeft=42px 开始
html += '<div style="position:absolute;left:42px;top:...;height:2px;width:...;">';
```

**当前布局：**
```
|  left:1-38px →  price labels (5档价位)  |  left:42px → 筹码线开始  |  右侧 ~88px 空白  |
```

#### 用户需求（修正理解）

> 不是"标签对齐方式"的问题，而是**整个筹码纵坐标（价格标签）移到图表右侧**，筹码线条从左侧开始绘制，使图表视觉上左移。

#### 修正方案

| 修改项 | 当前值 | 目标值 |
|:-------|:-------|:-------|
| `priceAxisLeft`（筹码线起始X） | 42px | **2px**（左侧仅留2px边距）|
| 价格标签定位 | `left:1px; width:38px; text-align:right` | `right:2px; width:38px; text-align:right` |
| 白色线宽度 | `containerW - priceAxisLeft` | `containerW - 4` |
| 筹码线可用宽度（barAreaW） | containerW - 42 - 4 | containerW - 4 - 42 |

**新布局：**
```
|  left:2px → 筹码线开始（左移40px）  |  右侧 price labels (38px)  |
```

| 评估项 | 说明 |
|:-------|:------|
| 代码改动 | `renderChipChart()` 中约4行定位值修改 |
| 风险 | 低。纯定位改动，不涉及数据计算逻辑 |
| 副作用 | 标签在右侧可能溢出 → CSS已有 `overflow:hidden`，安全 |
| 注意事项 | 价格标签需 `text-align:right` 并从右边界向左排布；应使用 `right:2px` 而非 `left:xxx` 定位 |

### ② 筹码分布线间距过大 — 改为以 0.01 价位为档位间隔

#### 当前实现

`renderChipChart()`（`indicator-ide.html:1033`）：

```javascript
var bins = 30;  // 分成30个价位区间
```

每条线高 2px。30 条线分布在 325px 容器中，每档占据 10.83px，线高 2px → **间隙 8.83px**，稀疏条纹效果。

#### 用户需求（修正理解）

> 不是固定设一个 bins 常量，而是**以小数点后 2 位（0.01）为单位计算每个价位档**。即每个 0.01 的价格区间对应一根筹码线。

#### 修正方案

**bins 的动态计算：**

```javascript
// 价格区间 ÷ 0.01 = 档位数，下限20防止极端窄区间时只有几根线
var bins = Math.max(20, Math.round((hi - lo) / 0.01));
```

**不同价格区间下的 bins 及线高：**

| 价格区间 (hi-lo) | bins | 每档高度 (325/bins) | 推荐线高 | 视觉效果 |
|:-----------------|:-----|:-------------------|:---------|:---------|
| ~1.0 元 | 100 | 3.25px | 3px | 紧密，轻微间隙 |
| ~2.0 元 | 200 | 1.625px | 2px | 几乎致密 |
| ~5.0 元 | 500 | 0.65px | 1px | 重叠连续 |
| ~0.2 元 (下限) | 20 | 16.25px | 2px | 稀疏（但罕见）|

**线高动态调整：** 为避免稀疏间隙，线高随 bins 动态填充垂直空间：

```javascript
var barHeight = Math.max(1, Math.round(containerH / bins) - 1);
// 但控制上限，避免线过粗
barHeight = Math.min(barHeight, 4);
```

同时，筹码分布 `counts` 生成逻辑中的加权乘积也需相应缩放：

```javascript
// 原: Math.max(1, Math.round(weight * 30));
// 改为用 bins 比例缩放:
var chipMultiplier = Math.max(10, Math.round(bins * 0.3));
counts[i] = Math.max(1, Math.round(weight * chipMultiplier));
```

| 评估项 | 说明 |
|:-------|:------|
| 代码改动 | `renderChipChart()` 中 1 处常量 + 1 处动态线高 + 1 处数据缩放 |
| 性能风险 | 低。极端情况（范围 10 元 = 1000 条 DOM），浏览器轻松处理 |
| 注意事项 | `maxBarPct` 保持不变；白色线 z-index 需在所有筹码线上方 |

### ④ 筹码白线与十字光标横线不在同一水平线

#### 问题描述

鼠标悬停在 K 线上时，筹码分布面板的白色当前价线（`#FFFFFF`）与十字光标的水平横线（`ch` graphic element）**不在同一垂直高度**。两条线代表同一个价格（当日收盘价），但视觉上处于不同水平位置。

#### 实际测量数据（浏览器验证，cursorIdx=1, closePrice=12.66）

| 测量项 | 值 |
|:-------|:---|
| ECharts canvas 距页面上边距 | 142.17px |
| grid[0] top（canvas 内） | 10px → 页面上 **152.17px** |
| grid[0] height | 325px → grid 底部页面上 **477.17px** |
| ECharts Y 轴显示范围 | **12.00 — 20.00**（范围 8 元）|
| 十字光标横线 Y（canvas 内） | 308.19px → 页面上 **450.36px** |
| chip-chart div 距页面上边距 | **134.38px** |
| chip-chart height | 325px → 底部页面上 **459.38px** |
| **当前白线 Y（chip-chart 内）** | **162.50px** → 页面上 **296.88px** |
| **正确对齐 Y（chip-chart 内）** | **315.98px** → 页面上 **450.36px** ✅ |
| **偏差** | **162.50 - 315.98 = -153.48px** ❌ |

#### 根因分析

白线与十字光标使用**完全独立的坐标系统**：

| 坐标系统 | 来源 | Y 轴范围 | closePrice 映射位置 |
|:---------|:-----|:---------|:-------------------|
| **十字光标横线** | `chart.convertToPixel()` 基于 ECharts grid Y 轴 | 12.00 — 20.00（8元跨度）| 94.8% 高度（近底部）→ Y=450.36px |
| **筹码白线（现）** | 独立 60 日价格范围 `[11.55, 13.77]`（2.23 元跨度）| 2.23 元跨度 | 50% 高度（正中间）→ Y=296.88px |

**问题本质：** `renderChipChart()` 使用 60 日 lookback 计算自己的价格区间 `[lo, hi]`，再在此区间内映射 closePrice。而 ECharts grid 的 Y 轴基于**当前可见数据**的极值，两个范围完全不同 → 相同价格的 Y 像素位置不同。

#### 修正方案

**核心思路：** `renderChipChart()` 在显示白色线时，放弃独立坐标计算，改用十字光标的**实际 Y 像素**通过 DOM `getBoundingClientRect()` 精确换算。

**修改①：`updateCrosshair()` — 传递 Y 像素**

第 832 行：

```javascript
// 改前:
if(chipOpen && typeof renderChipChart === 'function') renderChipChart(idx, closePrice);
// 改后:
if(chipOpen && typeof renderChipChart === 'function') renderChipChart(idx, closePrice, y);
```

**修改②：`renderChipChart()` — DOM 坐标对齐**

```javascript
function renderChipChart(idx, closePrice, crosshairY){
  ...
  // ③ 白色当前价线
  var whiteY;
  if(crosshairY != null){
    // 有十字光标 → 计算绝对对齐 Y
    // 公式: chipWhiteY = crosshairY - (chipChartRect.top - ehCanvasRect.top)
    var chipRect = container.getBoundingClientRect();
    var ecRect = chart.getDom().getBoundingClientRect();
    whiteY = crosshairY - (chipRect.top - ecRect.top);
    // 边界钳位
    whiteY = Math.max(0, Math.min(containerH, whiteY));
  } else {
    // 无十字光标（如 toggleChipPanel 展开）→ 使用自身坐标
    whiteY = (1 - (closePrice - lo) / range) * containerH;
  }
  ...
}
```

**修改③：`toggleChipPanel()` — 不传 crosshairY（走旧路径）**

第 1122 行无需改动，因为未传入第三个参数 → `crosshairY` 为 `undefined` → 走 `else` 分支（自身坐标）。

#### 坐标系对齐示意图

```
页面坐标系：
┌──────────────────────────────────────────────┐
│ ECharts Canvas (top: 142.17px)                │ chip-chart (top: 134.38px)           │
│  ┌─ grid[0] (top: 152.17px) ─────────┐       │  ┌────────────────────────────────┐   │
│  │                                     │       │  │                                │   │

│  │                                     │       │  │                                │   │
│  │  十字光标: Y=308.19 (canvas内)      │       │  │  白线(正确): Y=315.98 ✅        │   │
│  │  → 页面上 450.36px                 │       │  │  → 页面上 450.36px             │   │
│  │                                     │       │  │                                │   │
│  └─────────────────────────────────────┘       │  └────────────────────────────────┘   │
│                                               │                                      │
└──────────────────────────────────────────────┘                                      │

└──────────────────────────────────────────────────────────────────────────────────────┘
```

#### 风险评估

| 项目 | 说明 |
|:-----|:------|
| 修改点 | `updateCrosshair()` 传参 + `renderChipChart()` 新增 DOM 坐标计算 |
| DOM 依赖性 | `getBoundingClientRect()` 依赖渲染完毕后的布局信息 — 通常在 updateCrosshair 被调用时布局已稳定 |
| 性能 | `getBoundingClientRect()` 每帧数十次调用无压力，此处仅 2 次/十字光标移动 |
| 边界情况 | `whiteY` 钳位到 [0, containerH] 防止极端数据时溢出 |
| **总偏差消除** | **从 153.48px → 0px** |

## 三、实施建议

### 3.1 修改优先级

| 优先级 | 问题 | 理由 |
|:-------|:-----|:------|
| **P0** | **④ 白线未对齐（新发现）** | **数据精准度缺陷：白色标记线位置错误，误导用户判断当前价在筹码中的位置** |
| **P1** | ② 线间距 + ③ 鼠标离开恢复 | 功能缺陷级：间距不准影响数据可读性，鼠标离开不恢复影响交互完整性 |
| P2 | ① 价格标签移右侧 | 布局优化级：当前功能可用，右侧布局更符合视觉习惯 |

### 3.2 预计总工作量

| 文件 | 修改处 | 新增/改行数 |
|:-----|:-------|:------------|
| `_ui-prototype/indicator-ide.html` | 4处 | ~25行 |
| 沟通记录 + 归档 | 追加 | ~30行 |
| **合计** | | **< 60 行** |

### 3.3 风险与注意事项

- 问题② 极端情况下（hi-lo > 10 元）bins 可达 1000+，DOM 元素约 1000 个，浏览器性能无压力
- 问题① 价格标签移到右侧后，需要确认 white close-price line 也左移到 `left:2px` 起始（同步左移）
- ④ 白线对齐使用 `getBoundingClientRect()` 依赖布局稳定。`updateCrosshair` 在键盘/鼠标事件后触发，布局已稳定 → 无风险
- 四个问题修改互不依赖，可一次性实施
