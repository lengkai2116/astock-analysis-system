# 策略模板原型页面设计全量方案

## 1. 页面定位与范围界定

### 1.1 页面角色

**策略模板** 是策略类后台管理的核心枢纽页面，承担以下职责：

| 职责 | 说明 |
|:-----|:------|
| **浏览中心** | 展示所有可用策略的清单、分类、详情 |
| **管理中心** | 管理策略的创建、编辑、删除、参数调节 |
| **分发节点** | 为选股系统 L3、个股策略分析页提供可用的策略源 |

### 1.2 管理范围（边界明确）

**属于本页面管理：**
- 系统策略：量价策略、缠论策略、筹码策略、（S1-S7）7个新增独立策略、达尔文风险策略
- Vibe Coding 自建策略：自然语言描述→AI生成→命名保存→可编辑/删除

**不属于本页面管理：**
- 因子组合策略（含系统预设和用户自建）：由因子管理页（factor-manager.html）管理，直接对接到选股系统 L3，不经过策略模板页
- AI策略：由AI分析页（ai-analysis.html）单独管理
- 因子库管理：由因子管理页管理

### 1.3 相关页面职责隔离

| 页面 | 职责 |
|:-----|:------|
| 策略模板（本页） | 策略后台管理：浏览/查看/创建/编辑/删除**系统策略**和**Vibe自建策略** |
| 选股系统 L3 | 策略使用：选择已有策略用于选股筛选条件（系统策略来自策略模板页，因子组合来自因子管理页） |
| 个股策略分析页 | 策略执行：使用已选策略对个股进行分析 |
| 因子管理页 | 因子库浏览 + 组合方案管理（CRUD），直接对接到选股系统 L3 |
| AI分析页 | AI策略运行和管理 |

## 2. 策略分类体系（已校正）

### 2.1 全量策略分类

经过多轮沟通校正，系统策略分类如下：

```
系统策略（11个独立策略）
├── 量价策略 (VolumePriceStrategy)
│   └── 单一独立策略，内含50+形态检测规则（非子策略）
├── 缠论策略 (ChanlunStrategy)
│   └── 单一独立策略，内含三类买卖点/8种形态/中枢突破检测（非子策略）
├── 筹码策略 (ChipStrategy)
│   └── 单一独立策略，内含筹码分布分析/主力追踪/密集支撑（非子策略）
├── 达尔文风险策略 (DarwinRiskStrategy)
│   └── L1层过滤器，系统级风险筛选
├── S1 市场情绪周期策略 (MarketSentimentCycleStrategy)
├── S2 主力行为追踪策略 (MainForceTrackingStrategy)
├── S3 趋势通道识别策略 (TrendChannelStrategy)
├── S4 涨停板短线策略 (LimitUpShortTermStrategy)
├── S5 多层次风险控制策略 (MultiLevelRiskControlStrategy)
├── S6 波浪理论阶段识别策略 (WaveTheoryStrategy)
└── S7 斐波那契时间周期策略 (FibonacciTimeCycleStrategy)

Vibe Coding 自建策略（本页创建）
└── 自然语言→AI生成→用户命名保存→可编辑/删除
```

### 2.2 核心纠正记录

| 纠正项 | 初始认识 | 最终结论 |
|:-------|:---------|:---------|
| 量价/缠论/筹码分类 | 认为包含多种子策略 | 均为单一独立策略，内部规则非外部可选项 |
| AI策略归属 | 归入策略模板 | 在AI分析页单独管理 |
| S1-S7归类 | 未明确归类 | 全部归入系统策略，与量价/缠论/筹码同级 |
| 因子组合管理归属 | 策略模板管理 | 因子管理页管理，直接对接到选股系统L3，不经过策略模板页 |

### 3.2 区域详述

#### 3.2.1 左侧窄导航（30px宽）

| 控制 | 说明 |
|:-----|:------|
| 定位 | 左侧固定30px竖排，hover展开为标签 |
| 功能 | 策略来源分类快捷切换 |
| 项 | 全部 / 系统策略 / Vibe策略 |
| 系统策略展开子项 | 量价 / 缠论 / 筹码 / S1-S7 / 达尔文风险 |
| Vibe策略子项 | 用户自建策略列表（动态） |

#### 3.2.2 主区域（卡片流）

**工具栏：**
- 搜索框：按名称/描述模糊搜索
- 分类标签行：全部 / 量价策略 / 缠论策略 / 筹码策略 / 情绪周期 / 主力行为 / 趋势通道 / 涨停板 / 风控 / 波浪理论 / 斐波那契 / Vibe自建
- 视图切换按钮：卡片视图 / 表格视图
- 排序下拉：按名称 / 按更新时间 / 按类型
- **[➕ 创建策略]** 按钮：位于页头右侧

**卡片网格（默认视图）：**
- 每张卡片内容：策略图标 / 策略名称 / 一句话描述 / 分类标签（小圆点色标）/ 状态标识（系统/Vibe）
- 卡片hover：阴影 + 轻微上移
- 点击卡片：打开右侧详情面板

**空状态：**
- 系统策略分类下无结果：友好提示"当前分类暂无策略，请调整筛选条件"
- 自建策略为空："暂无自建策略，点击上方「创建策略」使用 Vibe Coding 生成"

**表格视图（备选）：**
- 列：策略名称 / 类型 / 分类 / 参数数量 / 更新时间 / 操作（查看/编辑/删除）
- 支持表头排序

#### 3.2.3 右侧详情面板（320px宽，可折叠）

**功能定位：** 仅用于查看详情和编辑操作，不负责创建（创建走全屏模态弹窗）。

**面板内容（按数据分类折叠）：**

| 区段 | 内容 |
|:-----|:------|
| ① 概览 | 策略图标 + 名称 + 分类 + 简短描述 + 状态标签（系统/Vibe/因子） |
| ② 参数面板 | 各参数滑块/输入框，可调节（系统策略默认值不可改但可截屏记录 → 编辑时才生效） |
| ③ 输入说明 | 数据源说明、格式要求（文本展示） |
| ④ 输出预览 | JSON 结构展示（语法高亮），Mock示例数据（如果是Vibe策略可编辑） |
| ⑤ 关联概念 | LLM Wiki关联文件列表（仅展示，链接暂不可点） |
| ⑥ 代码预览 | 策略代码（仅Vibe策略可编辑，系统策略为只读代码骨架） |

**底部操作栏：**
- 系统策略：`[📋 复制到自建]`（基于系统模板复制为新自建策略）`[使用此策略]`
- Vibe自建策略：`[✏️ 编辑]` `[🗑️ 删除]` `[使用此策略]`

**折叠交互：** 面板右上角折叠/展开按钮。折叠后主区域占满宽。

**入口②：从系统模板复制**

- 新名称自动生成：`[原策略名] - 副本n`
- 保存为自建策略，与原系统策略独立

#### 3.3.2 Vibe Coding 流程状态

| 状态 | 界面表现 | 说明 |
|:-----|:---------|:------|
| 初始 | 空文本域 + [AI生成]按钮 | 无描述输入时按钮disabled |

| 调整完成 | 代码区域可编辑 | 用户可手动修改AI生成的代码 |

## 5. 跨页面同步映射

### 5.1 策略来源与使用链路

```
┌──────────────┐               ┌──────────────────┐     ┌───────────────────┐
│ 因子管理页    │ ────────────→ │ 选股系统 L3       │     │ 策略模板页         │
│ (factor-man) │               │ (screener L3)     │     │ (strategy-tpl)    │
│              │               │                   │     │                   │
│ 因子库        │               │ checkbox选择策略   │     │ 系统策略           │
│ 组合方案      │               │ 因子组合（动态同步）│     │ Vibe自建策略       │
│              │               │                   │     │                   │
└──────────────┘               └────────┬──────────┘     └────────┬──────────┘
                                        │                        │
                                        ▼                        ▼
                               ┌──────────────────┐     ┌───────────────────┐
                               │ 个股策略分析页     │     │ 回测系统           │
                               │ (indicator-ide)  │     │ (backtest)         │
                               │ → 执行策略分析     │     │ → 组合策略回测      │
                               └──────────────────┘     └───────────────────┘
```

### 5.2 同步约束

| 数据来源 | 目标页面 | 同步方式 | 约束说明 |
|:---------|:---------|:---------|:---------|
| 策略模板页所有策略（含系统策略+Vibe自建） | 选股系统 L3 | 动态同步（策略列表API） | L3的checkbox列表必须从策略模板动态同步 |
| 策略模板页所有策略 | 个股策略分析页 | 动态同步 | indicator-ide的策略配置来源 |
| Vibe自建策略 | 选股系统 L3 | 同上 | 保持一致 |

### 5.3 原型约束

当前阶段（原型沟通），各页面使用 localStorage 模拟数据同步。规则：
- 在策略模板页创建的Vibe自建策略 → 存入 localStorage key `vibe_strategies`
- 选股系统 L3 页面从同一个 key 读取列表 → 实现跨页模拟同步
- 因子管理页组合直接对接选股系统 L3，不经过策略模板页
- 统一在 [171号文档](002-方案存档/171-功能视图API映射对照表.md) 中标注 Mock 数据范围


### 6.3 错误与边界状态

| 场景 | 处理方式 |
