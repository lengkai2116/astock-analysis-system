## 2. 观潮策略核心机制深度解析

### 2.1 模式检测引擎（analysis.js 165KB）

反编译 reveals 观潮的模式检测引擎按以下架构组织：

```
PatternRouter
├── ReversalPatterns    (头肩/双底双顶/三重顶底/圆弧顶底/123反转)
│   └── each: detection() + confidence() + evolution() + level()
├── ContinuationPatterns (旗形/三角/楔形/矩形)
│   └── each: same interface
├── BreakoutPatterns    (突破/缺口/放量/均线/新高)
│   └── each: same interface
├── ComboPatterns       (组合信号: 背离+金叉+放量等)
│   └── each: preconditions() + triggers() + invalidation()
└── AIEnhancedPatterns  (AI视觉匹配)
    └── match() + score() + context()
```

**关键发现**：每个模式都有统一的接口契约，包含以下属性：
- **name**: 模式唯一标识
- **direction**: bullish / bearish / neutral
- **strength**: 置信度 0~100
- **stage**: forming / confirming / completed / invalidated
- **evolving**: True/False（是否在演变中）
- **completion**: 完成度 0~100%
- **eta**: 预计完成所需 K 线数量
- **conditions**: 触发条件列表
- **invalidation**: 失效条件
- **levels**: { support, resistance, target, invalidation } 价格层级
- **interpretation**: 自然语言解读

### 2.5 共振评分系统

### 2.6 视觉模式识别（intelligence.js 94KB）

反编译发现 `VisualPatternRecognition` 类结合 **Qwen VL（通义千问视觉语言模型）** 进行模式识别：

```javascript
// 推断的 VisualPatternRecognition 接口
VisualPatternRecognition
  .scan(symbol, period)       → PatternResult[]
  .match(chartCanvas, type)   → ConfidenceScore

```

**与 AI 的协作流程**：
1. 用户触发模式扫描 → K 线图截图 + 上下文发送到 Qwen VL API
2. Qwen VL 返回视觉识别的候选模式列表 + 每个模式的置信度评分
3. 引擎将 AI 视觉输出与规则引擎结果交叉验证
4. 输出最终模式卡片（AI Combo Card / Evolving Card）

**关键前提**：此项能力依赖**多模态视觉语言模型**（如 Qwen-VL / GPT-4V），仅支持文本的 LLM 无法完成。

### 4.1 策略一：统一模式分类体系（P0）

from enum import Enum

class PatternCategory(Enum):
    REVERSAL = 'reversal'          # 反转形态
    CONTINUATION = 'continuation'  # 持续形态
    BREAKOUT = 'breakout'          # 突破形态
    GAP = 'gap'                    # 缺口形态
    DIVERGENCE = 'divergence'      # 背离形态
    COMBO = 'combo'               # 组合信号
    TREND = 'trend'               # 趋势形态
    VOLUME = 'volume'             # 量价形态

class PatternStage(Enum):
    FORMING = 'forming'           # 形成中
    CONFIRMING = 'confirming'     # 确认中（等待第2根K线）
    COMPLETED = 'completed'       # 已完成
    INVALIDATED = 'invalidated'   # 已失效

@dataclass
class PatternLevel:
    support: Optional[float] = None
    resistance: Optional[float] = None
    target: Optional[float] = None
    invalidation: Optional[float] = None

@dataclass
class PatternResult:
    """统一模式结果 — 每个模式返回此结构"""
    name: str                    # 模式唯一标识: 'double_bottom', 'bull_flag'
    category: PatternCategory    # 所属分类
    direction: str               # 'bullish' | 'bearish' | 'neutral'
    strength: float              # 置信度 0~1
    stage: PatternStage          # 当前生命周期阶段
    completion: float            # 完成度 0~100%
    eta: Optional[int]           # 预计还需 K 线数量
    conditions: List[str]        # 触发条件列表

### 4.2 策略二：模式生命周期追踪（P1）

**目标**：将观潮的 evolving 机制引入现有策略。

**现状**：所有模式检测是一次性快照 — "此刻是否满足条件"。

**改进**：为关键模式增加生命周期追踪。

```python
# backend/app/engine/patterns/tracker.py — 模式追踪器
class PatternTracker:
    """跨 K 线追踪模式演变"""
    
    def __init__(self):
        self.active_patterns: Dict[str, PatternResult] = {}
        self.history: List[PatternResult] = []
    
    def update(self, df: pd.DataFrame) -> List[PatternResult]:
        """每根新 K 线更新所有活跃模式的状态"""
        updates = []
        
        for name, pattern in self.active_patterns.items():
            if pattern.stage == PatternStage.FORMING:
                # 检查是否满足确认条件
                if self._check_confirmed(df, pattern):
                    pattern.stage = PatternStage.CONFIRMING
                    pattern.completion = 60
                # 检查是否失效
                elif self._check_invalidated(df, pattern):
                    pattern.stage = PatternStage.INVALIDATED
                    pattern.completion = 100
                else:
                    # 更新完成度
                    pattern.completion = self._calc_completion(df, pattern)
                    pattern.eta = self._calc_eta(df, pattern)
                updates.append(pattern)
            elif pattern.stage == PatternStage.CONFIRMING:
                # 等待第2根K线确认
                if self._check_completed(df, pattern):
                    pattern.stage = PatternStage.COMPLETED
                    pattern.completion = 100
                elif self._check_invalidated(df, pattern):
                    pattern.stage = PatternStage.INVALIDATED
                    pattern.completion = 100
                else:
                    pattern.completion = 85
                updates.append(pattern)
        
        return updates
```

**适用场景**：
- 三角整理（ascending_triangle_evolving）：追踪整理区间收敛，预测突破方向
- 头肩顶底（head_shoulders_top_evolving）：追踪左右肩形成进度
- 双底/双顶（double_bottom_evolving）：追踪第二个底/顶形成
- 缺口回补（gap_fill）：追踪缺口是否被回补

### 4.4 策略四：AI Combo Card 系统（P2）

**目标**：将多个独立模式信号智能组合为可读的操作建议卡片。

**现状**：信号在 StrategySignalPanel 列表中独立展示，无组合逻辑。

**改进**：增加 Combo Engine。

```python
# backend/app/services/combo_engine.py — 组合推荐引擎
class ComboEngine:
    """
    多模式组合推荐引擎
    
    将单个股票上检测到的多个独立模式，
    按方向/置信度/时效性智能组合为操作建议。
    参考观潮 ai-combo-card 体系。
    """
    
    def build_combos(self, patterns: List[PatternResult]) -> List[ComboCard]:
        """
        构建组合卡片
        
        组合逻辑:
        1. 按方向分组 (bullish / bearish)
        2. 同一方向多个模式 → 加权综合评分
        3. 模式间有逻辑关联 → 组合叠加评分 (如: 底背离 + 放量突破)
        4. 冲突方向 → 分离为不同卡片
        """
        cards = []
        
        # 看多组合
        bull_patterns = [p for p in patterns if p.direction == 'bullish']
        if bull_patterns:
            cards.append(self._build_combo(bull_patterns, 'bullish'))
        
        # 看空组合
        bear_patterns = [p for p in patterns if p.direction == 'bearish']
        if bear_patterns:
            cards.append(self._build_combo(bear_patterns, 'bearish'))
        
        return cards
    
    def _build_combo(self, patterns: List[PatternResult], direction: str) -> ComboCard:
        """
        构建单张组合卡片
        
        ComboCard 数据模型 (对应前端 ai-combo-card):
          name: str              # 组合名称
          strength: float        # 综合强度 0~100
          resolution: str        # 周期 (日线/周线)
          expires: str           # 有效期限
          patterns: List[Tag]    # 触发的模式标签列表
          dim_tags: List[Tag]    # 维度标签 (趋势/动量/量价)
          sr_tag: Tag            # 支撑/阻力评估
          vol_tag: Tag           # 成交量确认
          interpretation: str    # 综合分析解读
          hint: str             # 操作提示
          invalidation: str      # 失效条件
          levels: PriceLevels    # 关键价格层级
        """
```

### 4.7 策略七：规则引擎深度优化（P3）— 替代 AI 视觉方案

**背景核实**：本项目当前使用的 LLM（DeepSeek API + LM Studio）均为**纯文本模型**，不具备图像输入/视觉识别能力。与观潮使用的 Qwen VL（视觉语言模型）有本质差异，无法完成 K 线图的视觉形态匹配。

**替代方案**：放弃 AI 视觉识别路径，改为对规则引擎进行深度优化，弥补无法视觉识别的差距。

| 观潮 AI 视觉能力 | 规则引擎替代方案 | 工作量 |
|-----------------|----------------|--------|
| 视觉匹配头肩顶形态 | 增加多周期确认逻辑 + 量价交叉验证，提升形态识别准确率 | 2天 |
| AI 置信度评分 | 基于历史回测统计每个模式的实际胜率，作为置信度权重 | 3天 |
| AI 组合推荐 | ComboEngine 基于模式间逻辑关系（非视觉）组合推荐 | 已在 P2 覆盖 |
| 实时扫描多股票 | 扩展 PatternTracker 支持多股的并行扫描 | 2天 |

**结论**：在不引入多模态模型的前提下，通过规则引擎深度优化（增加确认逻辑、引入回测胜率权重、优化多周期验证），可在不依赖 AI 视觉能力的情况下达到接近的识别效果。
