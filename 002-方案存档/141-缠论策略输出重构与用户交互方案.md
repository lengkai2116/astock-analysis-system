# 缠论策略输出重构与用户交互方案

## 一、背景与问题定义

### 1.1 当前问题

当前 `_compute_chanlun_signal()` 的返回结果存在两大核心问题：

**问题 A — 输出字段是英文系统字段，用户无法直接理解**

返回的 Dict 中关键字段名为英文：
```python
{
    'signal': 'BULLISH',            # 用户看不懂
    'signal_label': '买入',          # label 对上了但字段名不对
    'entry_zone': [36.31, 38.55],   # 入场区间的概念不直观
    'risk_line': 36.33,             # 风险线的含义不明确
    'position_suggestion': '15%',   # 仓位建议太粗放
    'holding_period': '1~3个月',     # 基于什么逻辑？
    'evidence': [...],              # 证据列表有英文混杂
    'risk_notes': ['缠论信号滞后性'], # 风险提示太泛
}
```

**问题 B — 策略建议缺乏缠论内核，没有实际操作指导意义**

当前建议逻辑是简单的三段式：
- `trend=up` + `has_recent_buy` → BULLISH（买入），固定仓位 15%
- `trend=down` + `has_recent_sell` → BEARISH（卖出）
- 其他 → WATCH（关注）

这种判断完全忽略了缠论核心要素：
- ❌ 没有判断当前处于第几类买卖点
- ❌ 没有区分"当前笔阶段"对应的操作类型（延续中/转折中/中阴阶段）
- ❌ 没有基于买卖点类型的建仓策略（第一类买点试探建仓 vs 第三类买点追击建仓）
- ❌ 没有基于最新背驰段和中枢位置的风险评估
- ❌ 没有告诉用户"如果等待，具体要等什么信号"

### 1.2 用户实际需要什么

用户需要的不是 `signal=BULLISH` 这样的系统判断，而是在**当前时点**，基于缠论结构的**完整分析结论**：

```
锚定日：2026-05-08 | 西藏矿业 (000762.SZ)

【当前走势结构】
- 线段级别趋势：上升趋势（8段 / 1中枢）
- 当前笔阶段：上涨笔结束，已回调 10 个交易日，新下跌笔未确认
- 价格位置：37.43，高于中枢上沿 23.53（59.1%）

【中枢分析】
- 最新中枢区间：[21.05, 23.53]，中心 22.29
- 价格在中枢上方运行，处于强势区间
- 中枢形成后产生 3 个三类买点

【买卖点判断】
- 最近一个三类买点：2026-03-24 @25.28
- 当前状态：从最近买点已上涨 48%，进入高位区
- 抄底性价比明显下降

【操作建议】
当前建议：静待观察
原因：最近一个买点已在 2026-03-24 出现（@25.28），
当前价格 37.43 已远离买点区域（+48%）。
从笔结构看，上一笔上涨（笔129）终点 39.06 后已回调 10 个交易日，
当前处于新的下跌段走势中，但新下跌笔尚未确认。
等待信号：等待新下跌笔确认后，观察是否会：
  (a) 形成盘整背驰 → 可考虑第一类买点试探建仓
  (b) 在中枢上沿 23.53 附近企稳 → 可考虑第二类买点加仓
  (c) 直接跌破中枢 → 趋势转弱，继续等待

风险提示：当前距中枢上沿 +59%，一旦回调加深，
可能回到中枢内部甚至中枢下沿，波动空间大。
```

## 二、知识库与开源项目调研

### 2.1 LLM Wiki 知识库核心发现

从 A 股研究知识库中提取了以下缠论关键知识点：

| 概念 | 对策略重构的启示 |
|------|-----------------|
| **三类买卖点** | 第一类(背驰点)→试探建仓；第二类(回抽确认)→标准仓位；第三类(中枢破坏)→追击仓位。三类买卖点的操作策略完全不同，不能统一处理 |
| **缠中说禅MACD定律** | 第一类买点在0轴下形成，第二类买点在第一次上0轴后回抽确认。可用来辅助验证买卖点等级 |
| **买卖点级别定理** | 大级别买卖点必然在次级别体现。意味着日线级别的操作需要看60分钟/30分钟线确认 |
| **背驰-转折定理** | 趋势背驰后必然反弹回中枢DD。这是止损设置的理论依据 |
| **小背驰-大转折定理** | 小级别背驰+第三类买卖点=大级别转折。这是趋势反转预警的依据 |
| **区间套** | 从大级别→小级别逐级精确定位买卖点。可用于入场点的精细定位 |
| **走势终完美** | 任何走势终将完成，给出趋势延续还是转折的预判依据 |
| **三个独立程序系统** | 技术面+基本面+比价关系互相验证，可增加策略置信度 |

### 2.2 开源缠论项目参考

| 项目 | 可借鉴点 | 不适合我们的点 |
|------|---------|--------------|
| **Vespa314/chan.py** | 完整的缠论元素计算，多级别联立，500+特征工程，机器学习打分 | 完整版闭源，需联系作者 |
| **yijixiuxin/chanlun-pro** | 逐Bar增量计算，实时信号输出，监控与通知 | 核心算法闭源（PyArmor加密），需付费 |
| **haigechanlun/chanlun_auto_trading** | AI+规则引擎识别买卖点，动态仓位控制，缠论结构止损 | 核心模型闭源，针对加密货币 |

**关键启示**：市面成熟项目多为核心算法闭源。我们已有完整的 `chanlun_strategy.py`（1552行，6个数据类 + 7个分析类），缠论元素计算是完整的。问题出在**信号输出层**和**策略逻辑层**，并非底层计算。

### 3.2 策略逻辑重构

新的策略建议逻辑基于缠论核心定理，而非简单的 if-else：

```
输入: ChanlunAnalyzer 分析结果（笔、段、中枢、背驰、买卖点）

第一步：判断当前走势阶段
  1.1 最后一笔结束后的状态（延续/回调/反弹）
  1.2 当前价格相对最新中枢的位置（上方/内部/下方）
  1.3 是否有新背驰/新买卖点刚形成

第二步：按缠论分类决策
  ┌─ 趋势向上 + 价格在中枢上方：
  │   ├─ 刚出现第一类买点 → 建议：买入（试探建仓5%）
  │   ├─ 刚出现第三类买点且距中枢不远(<15%) → 建议：买入（标准建仓15%）
  │   ├─ 最近买点已大幅上涨(>30%) → 建议：静待（已无性价）
  │   ├─ 最后一笔向上延续中 → 建议：持仓观察
  │   └─ 最后一笔向上结束,回调中,未破中枢 → 建议：关注（等待回调企稳）
  │
  ├─ 趋势向上 + 价格在中枢内部：
  │   ├─ 下跌背驰刚出第一类买点 → 建议：买入（试探建仓）
  │   └─ 中枢内震荡 → 建议：静待（等方向突破）
  │
  ├─ 趋势向下 + 价格在中枢下方：
  │   ├─ 下跌背驰+第一类买点 → 建议：买入（试探建仓，严格止损）
  │   ├─ 出现第三类卖点 → 建议：清仓
  │   └─ 下跌笔延续中 → 建议：静待
  │
  ├─ 趋势不明 → 建议：静待（需更多数据确认方向）
  └─ 中阴阶段（背驰转折定理三种演化中）：
      └─ 建议：关注（设好预警，等待方向选择）
```

### 3.3 仓位管理增强

基于缠论的仓位管理，参考海龟交易法则的 N 值概念和朗德里 2%/6% 规则：

| 买卖点类型 | 建仓策略 | 单笔最大仓位 | 总仓位上限 |
|-----------|---------|-------------|-----------|
| **第一类买点**（背驰点） | 试探建仓，分 2 档：试探 5% → 确认+5% | 10% | 30% |
| **第二类买点**（回调确认） | 标准建仓，分 2 档：标准 10% → 加仓+5% | 15% | 40% |
| **第三类买点**（中枢突破） | 追击建仓，分 2 档：追击 10% → 加仓+5% | 15% | 50% |
| **第一类卖点**（背驰点） | 减半仓 50% | — | — |
| **第二类卖点**（反弹确认） | 再减 50%（总剩 25%） | — | — |
| **第三类卖点**（破中枢） | 清仓 | — | — |

## 四、方案评估

### 4.1 优势

- **符合缠论实际操作思维**：用户看到的不是系统判断，而是完整的走势结构分析 + 基于缠论的决策推理链
- **知行合一**：不仅告诉用户"怎么做"，还告诉"为什么这么做"和"下一步要等什么"
- **可落地性高**：底层 `ChanlunAnalyzer` 计算是完整的（1552行），只需要改造信号组合和输出层
- **支持扩展**：加入 `区间套` 多级别联立和 `MACD定律` 验证后，可自然扩展

### 4.2 风险与注意

- **中阴阶段的判断**：当下走势可能处于趋势→盘整或盘整→趋势的过渡期，算法层面需要做"走势分类"判断而非"预测"
- **背驰判断的准确性**：当前 DivergenceDetector 基于MACD面积法，可能存在滞后
- **多级别联立暂未实现**：当前只有日线级别，60分钟/30分钟的区间套分析需后续补充
