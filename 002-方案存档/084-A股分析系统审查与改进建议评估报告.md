## 第五部分：架构设计问题评估与建议

### 5.1 前后端通信问题

#### 问题现状
- WebSocket连接硬编码 `localhost:5001`
- Vite代理配置错误指向8080
- 缺乏重连机制

#### 改进方案

**建议实现**：

1. **统一配置管理**
   ```javascript
   // config.js
   export const API_CONFIG = {
     baseUrl: import.meta.env.VITE_API_BASE_URL || 'http://localhost:5001',
     wsUrl: import.meta.env.VITE_WS_URL || 'ws://localhost:5001'
   }
   ```

2. **WebSocket重连机制**
   ```javascript
   class ReconnectingWebSocket {
     constructor(url) {
       this.url = url
       this.reconnectDelay = 1000
       this.maxDelay = 30000
       this.connect()
     }
     
     connect() {
       this.ws = new WebSocket(this.url)
       this.ws.onclose = () => {
         setTimeout(() => this.connect(), this.reconnectDelay)
         this.reconnectDelay = Math.min(this.reconnectDelay * 2, this.maxDelay)
       }
     }
   }
   ```

### 5.2 数据层问题评估

#### 问题分析
- PostgreSQL + SQLite混合使用
- 缺乏统一数据访问层
- 缓存策略不明确

#### 改进建议

**短期方案**（1-2周）：
- 明确各数据库职责：PostgreSQL存元数据，SQLite存缓存
- 封装统一的数据访问接口
- 完善Redis缓存使用文档

**长期方案**（1-2月）：
- 考虑统一到PostgreSQL
- 引入ORM层
- 完善缓存失效策略
