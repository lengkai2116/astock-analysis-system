## 一、背景

在 Phase 1-5 完成筹码分布策略的完整实现（覆盖率约 75-80%）后，基于 LLM Wiki 知识库进行了第二次知识对照评估。评估确认核心策略实现正确，但存在 4 个精细化改进方向。

本次报告记录这 4 个方向的实施详情，包括：改动内容、知识库依据、实现方案、风险评估。

## 二、改进方向详述

### 方向1：S_DIVERG_SELL 改用 detect_rsi_divergence()

**改动文件**：`chip_strategy_impl.py` / `pipeline.py`

**原实现问题**：
原 `_check_s_diverg_sell()` 使用简化的"前后半段15日价格峰值对比法"，关键缺陷在于 `early_rsi` 和 `late_rsi` 都取了同一个当前 RSI 值（`indicators.get('rsi', 50)`），背离判断实质上失效——两个 RSI 变量始终相等，`rsi_change < 5` 条件永远为真，导致仅靠价格差异做判定。

**知识库依据**（背离形态）：
> "通常在二次背离之后会产生好的买点或卖点"
> "行情级别超过观察周期时，还会产生二次背离，甚至三次背离"

**新实现**：
- 调用 `ChipIndicators.detect_rsi_divergence(kline_data, period=14, lookback=20)`
- 该方法逐日计算完整 RSI 序列，找到局部价格极值点，对比对应位置的 RSI 值
- 顶背离计数映射：
  - count=1（初次背离）→ `position_adjustment=0.7`（减仓至70%）
  - count=2（二次背离）→ `position_adjustment=0.3`（减仓至30%）
  - count≥3（三重背离）→ `position_adjustment=0.1`（减仓至10%）

**风险**：无。使用已存在的精确方法替换失效的简化方法。

### 方向2：假突破过滤器前置到 S_BUY/S_SELL

**改动文件**：`chip_strategy_impl.py` / `pipeline.py`

**原实现问题**：
`ConfirmLayer` 中已有假突破三维检查，但**在信号生成后才调用**（后验验证），信号优先级的"先买入再验证"逻辑让假突破过滤仅在持仓后才生效。

**知识库依据**（假突破交易策略）：
> "利用三日原则判断，如果三个交易日内回补缺口，则是假突破"
> "突破时成交量未能持续放大"
> "MACD、KDJ等指标出现背离"

**新实现**：
新增 `_check_false_breakout()` 方法，在 `_check_s_buy()` 和 `_check_s_sell()` 内部作为前置条件：

```
非突破状态 → 直接通过（price在20日区间中部）

突破状态 → 三维确认：
  维度1-量能：vol_ratio >= 1.5
  维度2-时间：突破持续 >= 3日          ← 对应"三日原则"
  维度3-深度：CYQKL >= 0.2

  任一不通过 → return {triggered: False, reason: "假突破过滤: ..."}
```

**风险**：低。只在突破状态时激活，非突破状态直接通过。正确过滤伪信号。

### 方向3：筹码流动方向辅助阶段判定

**改动文件**：`chip_strategy_impl.py` / `pipeline.py`

**原实现问题**：
`TradingPhaseDetector` 的评分方法使用价格位置、成交量、RSI 等静态指标判定操盘阶段，没有利用"筹码在价格空间中的动态流动"信息。

**知识库依据**（筹码分布）：
> "筹码的流动方向一旦确定，往往有着极强的惯性"
> "筹码的流动速度则对应着趋势的推进力度"

**新实现**：

**接口升级**：
- `detect_phase()` / `_detect_phase()` 新增可选参数 `chip_bins_history: Optional[List[List[Dict]]]`
- 传入时调用 `chip_indicators.detect_chip_transfer()` 计算转移信息
- 未传入时跳过，不影响现有逻辑

**_score_raising 新增条件**：
| 条件 | 加分 | 含义 |
|------|------|------|
| 筹码向上转移 | +2.0 | 低位→高位转移确认拉升趋势 |
| 稳定分布+盈利盘≥50% | +1.0 | 拉升中的良性调整 |

**_score_washing 新增条件**：
| 条件 | 加分 | 含义 |
|------|------|------|
| 稳定分布+低位筹码变化≥-2% | +2.0 | 低位筹码锁定，洗盘接近结束 |
| 向下转移+低位筹码正增长 | +2.0 | 主力在低位吸筹 |

**风险**：低。依赖历史数据，数据不可用时自动跳过。

---

### 方向4：金字塔建仓替代一次性满仓

**改动文件**：`framework/chip_position_manager.py`

**原实现问题**：
当前 S_BUY 固定 70%、S_WASH_END 固定 50%、S_BOUNCE 固定 30%——一次性到位。没有分步建仓机制，在价格回调时抗波动能力弱。

**知识库依据**（短线风险控制）：
> "第一笔建仓：控制在3到5成"
> "接下来三天内股价走出预期走势：加仓2到3成"
> "没有走出预期走势：不加仓"

**新实现**：

**状态跟踪**：
```
_pyramid_state[symbol] = {
    'entry_price': float,   # 入场价
    'tier': int,            # 当前层级 1/2/3
    'entry_date': int,      # 入场交易日索引
}
```

**层级规则**：

| 层级 | 累计仓位 | 触发条件 | 意义 |
|------|---------|---------|------|
| Tier1 | 目标仓位×50% | 首次买入信号 | 首仓测试 |
| Tier2 | 目标仓位×80% | 3日后价格≥入场价 | 确认后加仓 |
| Tier3 | 目标仓位×100% | 6日后涨幅≥1% | 趋势确认满仓 |

**失败路径**：
- 价格跌破入场价5% → 停在当前层级，不再加仓
- 卖出/止损信号 → `reset_pyramid()` 清空状态
- 大盘环境差 → `market_multiplier=0.5` 在金字塔后仍生效
- 不传 symbol 参数 → 等同旧逻辑（一次性满仓）

**接口兼容性**：
`calculate_target_position()` 新增 4 个可选参数，均默认为 None/0。调用方不传时行为与旧版本一致。

**风险**：低。首次仓位更小，风险暴露更少。

## 五、策略链路总览（完整后）

```
PreFilter(市场/流动性/新股/市值) → ChipAnalyzer(指标+峰+转移)
  → PhaseDetector(评分+筹码转移) → FalseBreakoutFilter(前置三维确认)
    → SignalGenerator(6信号+S_DIVERG_SELL) → ConfirmLayer(换手/盘口)
      → PositionManager(金字塔建仓+大盘+熔断) → RiskExecutor(止损4+止盈4)
```

## 六、核心结论

1. **核心策略框架已完成** — 6层架构（PreFilter→Analyzer→PhaseDetector→SignalGenerator→PositionManager→RiskExecutor）全部实现
2. **7种信号** — S_BUY / S_WASH_END / S_BOUNCE / S_SELL / S_WASH_STOP / S_DIVERG_SELL + 假突破前置过滤
3. **风控体系** — 4止损（硬止损/跟踪/时间/跌破成本）+ 4止盈（筹码/RSI/放量滞涨/移动）+ 仓位管理（金字塔建仓/大盘环境/熔断）
4. **知识库对齐** — 核心指标 ASR/CYQKL/SSRP、4阶段判定、假突破三日原则、金字塔建仓规则均已与知识库内容核对一致
5. **后续方向** — 如需进一步提升，可考虑：大单流向集成（QMT Level2）、13种量价+筹码组合形态细分类、历史数据回测验证优化
