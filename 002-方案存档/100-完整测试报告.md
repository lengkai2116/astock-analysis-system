## 五、启动指南

### 5.1 后端启动

**注意**: 由于PostgreSQL和Redis依赖，推荐使用已配置的Docker环境，或先进行单元测试验证功能。

**快速功能测试**（无需数据库）:
```bash
cd backend
python3 test_api_simple.py
```

**完整启动**（需要PostgreSQL和Redis）:
```bash
cd backend
# 1. 确保Docker服务启动
# 2. 运行迁移
python3 -m migrations.upgrade_v2
# 3. 初始化模板
python3 -m app.services.init_system_templates
# 4. 启动服务
python3 run.py --port 5000
```

后端将在 **http://localhost:5000** 启动

### 5.2 前端启动

```bash
cd frontend/vue-project
npm install
npm run dev
```

前端将在 **http://localhost:5173** 启动

## 六、配置说明

### 6.1 环境变量配置 (.env)

```bash
# 数据库配置
DATABASE_URL=postgresql://stock_user:stock_password@localhost:5432/stock_analysis

# 缓存配置
REDIS_URL=redis://localhost:6379/0

# 数据目录
DATA_DIR=/Users/kalence/Desktop/01-A股股票分析系统/data

# LLM配置 - DeepSeek API
DEEPSEEK_API_KEY=sk-your-api-key-here
DEEPSEEK_BASE_URL=https://api.deepseek.com/v1
DEEPSEEK_MODEL=deepseek-chat

# LLM模式选择: mock | deepseek | lm_studio
LLM_PROVIDER=mock
```

### 6.2 LLM模式切换

| 模式 | LLM_PROVIDER | 说明 |
|---------|---------|---------|
| Mock | mock | 模拟输出，无需API Key（默认） |
| DeepSeek | deepseek | DeepSeek官方API |
| LM Studio | lm_studio | 本地大模型 |

## 八、API接口说明

### 8.1 策略相关 API

| 端点 | 方法 | 说明 |
|---------|------|------|
| /api/v2/strategy/outputs | GET | 获取策略输出列表 |
| /api/v2/strategy/outputs | POST | 创建策略输出 |
| /api/v2/strategy/outputs/<id> | DELETE | 删除策略输出 |
| /api/v2/strategy/templates | GET | 获取策略模板列表 |
| /api/v2/strategy/templates | POST | 创建策略模板 |

### 8.2 回测相关 API

| 端点 | 方法 | 说明 |
|---------|------|------|
| /api/v2/backtest/indices | GET | 获取基准指数列表 |
| /api/v2/backtest/index/<code> | GET | 获取指数数据 |
| /api/v2/backtest/run | POST | 运行回测 |

### 8.3 指标IDE相关 API

| 端点 | 方法 | 说明 |
|---------|------|------|
| /api/v2/indicator/parse | POST | 解析指标协议 |
| /api/v2/indicator/check | POST | 质量检查 |
| /api/v2/indicator/execute | POST | 执行指标代码 |

### 8.4 报告相关 API
