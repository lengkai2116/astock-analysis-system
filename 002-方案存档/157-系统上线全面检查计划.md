## 四、假设与默认选择

1. **Docker 环境可用**：假设宿主机已安装 Docker Desktop 或 Docker Engine，`docker compose` 命令可用。如不可用，Phase B 降级为纯静态检查 + 手动 `pip install` / `npm run build` 验证。
2. **后端 Mock 模式**：UI 回归测试时假设后端使用 `LLM_PROVIDER=mock`，DuckDB 使用本地缓存文件（如 `strategy_templates.db`），避免依赖外部 API。
3. **Playwright 依赖**：假设项目目录或全局已安装 Playwright Chromium。如不可用，A3-A6 降级为手动 curl + grep 验证 HTTP 响应状态。
4. **网络隔离**：`docker-compose.yml` 中 PostgreSQL/Redis 不暴露端口到宿主机，外部访问仅通过 backend 5001 端口和 frontend 9000 端口。
5. **生产 HTTPS**：Git 仓库中不包含 TLS 证书。HTTPS 证书应在生产环境独立配置（Nginx 或负载均衡器侧），不在本检查计划范围内。
6. **DeepSeek EOL**：`deepseek-chat-v4` 模型将于 2026/07/24 退役，建议在本检查完成后另行安排迁移到 `v4-flash`/`v4-pro`。本计划不包含该迁移的测试。
7. **并发限制**：Phase A 各检查项无共享资源冲突，支持并行执行；Phase B/C 有 Docker 资源竞争，建议串行。
