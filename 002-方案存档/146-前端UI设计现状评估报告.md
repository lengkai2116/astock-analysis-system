# 前端UI设计现状评估报告

## 1. 评估概要

### 1.1 评估目的

对 A 股股票分析系统当前前端 UI 的设计现状进行全面梳理与评估，明确已完成的内容、存在的差距、以及下一阶段的优化方向。本报告基于 2026-06-04 代码库快照进行分析。

### 1.2 评估范围

- 前端代码库：`frontend/vue-project/`
- 设计文档：`前端UI设计优化详细方案.md`
- 关联页面：全 10 个路由页面 + 共 15+ 个子组件
- 评估维度：技术架构、页面完成度、设计规范一致性、交互深度、组件复用性

### 1.3 评估方法

遍历项目全部前端源代码，逐页评估功能完备性；对照设计文档 `前端UI设计优化详细方案.md` 进行差距分析；从技术实现、交互设计、视觉规范三个层面给出综合评价。

### 2.3 状态管理架构

Pinia store 管理以下全局状态：

- **UI 状态**：`sidebarCollapsed`
- **当前上下文**：`symbol`、`market`、`timeframe`
- **自选股列表**：`watchlist`（5 只默认股票）
- **活跃指标实例**：`activeIndicators`（支持增删改）
- **因子列表**：`factors.all` + 按分类聚合 getter
- **因子组合**：`factorCombinations.all` + CRUD actions
- **策略列表**：`strategies.available` / `strategies.active`
- **回测结果**：`backtestResult`

## 4. 设计系统评估

### 4.1 已建立的规范

- **暗色主题**：硬编码在 `App.vue`（`theme-dark` class），页面级背景 `#0a1628`
- **CSS 变量体系**（`global.less`）：
  - `--color-up: #EF4444`（A股红色上涨）
  - `--color-down: #22C55E`（A股绿色下跌）
  - `--color-primary: #3B82F6`（蓝色主操作色）
  - `--color-accent: #F59E0B`（黄色关注色）
  - 背景色阶：`#0A1628` → `#141414` → `#1E293B`
  - 文字色阶：`#F1F5F9` → `#94A3B8` → `#64748B`
  - K线专用色：`--kline-up` / `--kline-down`
- **A 股涨跌规范已落地**：红涨绿跌
- **Ant Design 深色覆盖**：下拉框、输入框、选项卡、弹窗、tooltip、message 等全局组件样式覆盖
- **滚动条统一样式**：6px 宽，暗色轨道
- **全局字体栈**：`-apple-system, BlinkMacSystemFont, Segoe UI, PingFang SC, Microsoft YaHei, sans-serif`

### 4.2 缺失的规范

| 缺失项 | 设计文档建议 | 当前状态 |
|--------|-------------|---------|
| 设计 Token 文件 | `styles/tokens.less` | 不存在，变量散落在 global.less 和页面 scoped style 中 |
| 信号标签颜色规范 | 看多(红) / 风险(绿) / 关注(黄) / 中性(灰) / 失效(暗红) | 各页面各自定义，无统一组件 |
| 策略评分卡 | 0-100 分 + 状态 + 置信度 + 关键原因 | 无封装组件 |
| 风险提示卡 | 等级 + 类型 + 触发条件 + 应对建议 | 无封装组件 |
| 信号解释抽屉 | 信号名 + 触发时间/价格 + 指标值 + 历史胜率 + 风险线 | 无独立组件，部分逻辑在 SignalDetailModal |
| 系统边界声明 | 页面固定展示"仅供研究参考"免责声明 | 所有页面均缺失 |
| 文案边界 | 避免"买入/卖出/下单/自动交易/实盘" | 部分页面仍使用交易语义 |
| API 响应结构 | 统一 `success/data/message/meta` | 部分服务 expected 该格式，但未全局强制 |

## 8. 综合评估结论

### 8.1 整体判断

系统前端 UI 处于 **功能原型阶段** — 10 个页面均有模板框架、路由已串联、基础服务层（axios/socket）已就绪、暗色主题一致、全局状态管理通畅。项目具备完整的前后端交互链路雏形。

### 8.2 核心优势

- **覆盖全面**：从选股到策略研究到回测到报告的完整链路均有页面覆盖
- **基础扎实**：Vue 3 + Vite + Ant Design Vue + Pinia 的技术选型合理，扩展性好
- **实时功能**：Socket.IO 实时数据通道已打通
- **暗色主题**：统一的深色视觉风格，符合投研终端定位
- **组件初步拆解**：选股系统的流水线组件、策略信号面板等已有独立封装

### 8.3 核心短板

核心问题不在于"页面太少"，而在于 **组件深度和交互精细度不足**：

1. **证据链缺失**：每个策略建议没有展开能力 — 看不到触发原因、指标值、历史胜率、风险线
2. **验证闭环不完整**：策略从 IDE 到回测到报告没有一键联动
3. **AI 输出过浅**：AI 分析停留在对话日志层面，缺少结构化报告支撑
4. **结果展示不充分**：回测结果缺少多维可视化（权益曲线、回撤、敏感性等）
5. **设计系统不统一**：颜色变量有两套、信号标签无统一组件、免责声明全线缺失
6. **缺少市场总览入口**：用户无法快速判断当前市场环境是否适合运行策略

### 8.4 下一阶段方向

当前设计文档 `前端UI设计优化详细方案.md` 的建议方向仍然有效，建议优先聚焦 P0 层级：

1. **统一文案与信号规范** — 最小改动成本、最大影响面
2. **指标 IDE 升级为研究工作台** — 增加策略建议面板 + 信号解释 + 一键回测/报告
3. **回测结果页多维展示** — 解决"策略是否可信"核心问题
4. **AI 报告结构化** — 从对话升级为可归档的投研报告
5. **逐步补齐市场总览与因子可视化** — 支撑完整的策略研究工作流
