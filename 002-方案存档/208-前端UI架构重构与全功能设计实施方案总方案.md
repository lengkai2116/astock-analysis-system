## 一、架构总纲

### 1.1 视图层架构（17 页面）

整个前端按功能域划分为 5 个分组，每个分组有明确的用户目标：

```
┌─────────────────────────────────────────────────────────────────────────┐
│  GROUP 1: 概览 — "今天市场怎么样？"                                     │
├─────────────────────────────────────────────────────────────────────────┤
│  dashboard    仪表盘    ← 市场总览 + 信号汇总 + 快捷入口                │
│  login        登录页    ← Token 管理 + 系统入口                         │
├─────────────────────────────────────────────────────────────────────────┤
│  GROUP 2: 分析 — "这只股票能不能买/卖？"                                │
├─────────────────────────────────────────────────────────────────────────┤
│  indicator-ide    个股策略分析  ← K线 + 策略指标 + 信号                │
│  screener         选股系统      ← Pipeline 三层递进筛选                │
│  watchlist        自选监控      ← 实时行情 + 可配置表格                │
│  ai-analysis      AI 分析       ← DeepSeek 多角色投研                 │
│  chip-analysis    筹码分析 ★    ← 筹码峰 + 资金流向 + 主力追踪 ★新    │
├─────────────────────────────────────────────────────────────────────────┤
│  GROUP 3: 验证 — "我的策略到底赚不赚钱？"                              │
├─────────────────────────────────────────────────────────────────────────┤
│  backtest         回测系统      ← 配置 + 绩效 + 权益/回撤曲线          │
│  factor-manager   因子管理      ← 因子组合 + 评估                      │
│  strategy-templates 策略模板    ← 预设策略浏览与选用                   │
│  playback         复盘中心 ★    ← 时间轴回放 + 事件分析 ★新            │

├─────────────────────────────────────────────────────────────────────────┤
│  GROUP 4: 管理 — "系统运行得怎么样？"                                  │
├─────────────────────────────────────────────────────────────────────────┤

└─────────────────────────────────────────────────────────────────────────┘
```

**设计原则：**
- 每个 Group 覆盖一个**独立的用户心智模型**：看 → 想 → 验 → 管
- 用户从左到右依次使用，形成完整闭环
- 新增页面以 ★ 标记，分布在已有 Group 中，不破坏原有导航认知

### 1.3 数据流架构

```
外部 → API → Service → View/Component
                  ↘ Store ↗
```

**数据流规则：**

| 数据性质 | 存储位置 | 生命周期 | 访问方式 |
|---------|---------|---------|---------|
| 页面专有数据 | `data()` / `ref()` | 页面挂载期间 | 组件自身通过 Service 获取 |
| 跨组件共享数据 | Pinia Store | 应用运行期间 | `useXxxStore()` |
| 跨页面持久数据 | localStorage | 永久 | `store.js` 自动写入 |
| 后端预缓存数据 | IndexedDB (Worker) | 持久 | Worker 池异步访问 |
| 实时推送数据 | Pinia Store + Socket | 实时 | `socketService.js` |

**关键决策：**
- **Service 层采用增量扩展**：新建 Service 不替换已有的，旧 API 继续走旧 Service，新 API 走新 Service
- **Store 层最少新增**：仅 `market.js` 一个新增 Store，其余状态用 `data()` 管理
- **每个异步请求必须经过 Service**：禁止 View 直接调 `axios.get()`

```

  │     Universe → Alpha → Portfolio → Risk    │
  │     股票池    因子筛选   组合优化   风控     │
  │                                            │
  │  ② 执行筛选 → 结果列表                     │
  │     | 代码 | 名称 | 评分 | 信号 | ...       │
  │                                            │
  │  ③ 点击股票 → 打开 SignalDetailModal        │
  └──────────────────┬─────────────────────────┘
                      ↓ 发送到自选或分析

  ┌────────────────────────────────────────────┐
  │  ① 股票已自动加载（从选股器跳转）           │
  │  ② K线图 + 缠论叠加（分型/笔/中枢/买卖点）   │

  ┌────────────────────────────────────────────┐
  │  ① 股票已自动加载                          │
  │  ② DeepSeek 多角色并行分析                 │
  │  ③ 结构化评分 + 策略解读                    │
  └──────────────────┬─────────────────────────┘

  ┌────────────────────────────────────────────┐

  └────────────────────────────────────────────┘
```

**连贯性关键点：**
- Screener → IndicatorIDE：传递 `symbol` 参数（URL query 或 store）
- IndicatorIDE → AI Analysis：传递 `symbol` + `analysis_context`
- IndicatorIDE → Backtest：传递 `symbol` + `strategy_params`
- 所有跳转使用 `$router.push({ path, query: { symbol } })` 保持无刷新
