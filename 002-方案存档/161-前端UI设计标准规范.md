# 161 — 前端UI设计标准规范

## 规范说明

本文档作为后续全量检查（160）的 **校验基准**，定义了 A股分析系统前端 UI 所有视觉、交互、组件、文案、状态的合格判定规则。

### 规范来源

| 维度 | 提炼自 | 核心原则 |
|------|--------|---------|
| 空间与布局 | 148 §2 + 147 §1.3 | 8pt 网格、卡片密度、页面三段式 |
| 视觉Token | 148 §2 + global.less | CSS 变量体系、双主题、半透明叠加 |
| 排版规范 | 148 §2 | 6 级字号、行高、字重 |
| 交互行为 | 147 §6 + 148 §3 | hover/focus 过渡、三态覆盖 |
| 组件行为 | 146 §5 + 147 §2.1-2.2 | AntDV 4 标准用法、错误边界 |
| 文案边界 | 146 §7 + 158 §E | 无交易语义、免责声明、全中文 |
| 状态覆盖 | 146 §6.3 + 147 §5.2 | loading/empty/error/boundary 全覆盖 |

### 规范层级

每条规则标有唯一 ID（如 `S01.01`），对应检查清单中的条目引用。  
规则分三个等级：

| 等级 | 标记 | 含义 | 违规处理 |
|------|------|------|---------|
| **强制** | `[M]` | 必须遵守，违反即 P0/P1 | 必须修复 |
| **推荐** | `[R]` | 建议遵守，违反为 P2 | 建议修复 |
| **参考** | `[O]` | 理想态，违反为 P3 | 可选优化 |

## 维度一：空间与布局 (Space & Layout)

### S01 — 8pt 网格系统

| ID | 等级 | 规则 | 判定方法 |
|----|------|------|---------|
| S01.01 | `[M]` | 所有间距值必须从集合 `{2,4,6,8,10,12,16,20,24,32,40,48,64}`px 中选择 | 检查 CSS margin/padding/gap 值 |
| S01.02 | `[M]` | 卡片内边距统一为 `--space-24` (24px) | 检查 Card `padding` 或 `bodyStyle` |
| S01.03 | `[M]` | 卡片间距统一为 `--space-16` (16px) | 检查 grid-gap / Card 之间 margin |
| S01.04 | `[M]` | 页面左右边距统一为 `--space-24` (24px) | 检查页面级 padding |
| S01.05 | `[M]` | 按钮高度统一 `40px`，输入框高度统一 `40px` | 检查 AntDV 默认或自定义高度 |

### S02 — 卡片密度控制

| ID | 等级 | 规则 | 判定方法 |
|----|------|------|---------|
| S02.01 | `[M]` | 单张卡片最多承载 1 个核心主信息 + 不超过 2 个辅助信息 | 视觉检查卡片内容数 |
| S02.02 | `[R]` | 仪表盘概览统计卡（总资产/涨跌幅等）例外，数值可为主信息 | 确认数值卡片是否符合 |
| S02.03 | `[M]` | 卡片内禁用二次嵌套卡片（card-in-card） | 检查 DOM 结构 |
| S02.04 | `[M]` | 页面区块必须为全幅背景带，不能为浮动卡片 | 检查页面结构 |

### S03 — 页面结构规范

| ID | 等级 | 规则 | 判定方法 |
|----|------|------|---------|
| S03.01 | `[R]` | 每个功能区块遵循「大标题 + 辅助说明 + 核心操作按钮」三段式 | 视觉检查区块结构 |
| S03.02 | `[M]` | 页面顶部有清晰的页面标题和操作栏 | 检查 page-header 结构 |
| S03.03 | `[O]` | 页面内容最大宽度建议 1200px | 检查容器 max-width |

## 维度二：视觉Token (Visual Tokens)

### T01 — 色彩系统

| ID | 等级 | 规则 | 判定方法 |
|----|------|------|---------|
| T01.01 | `[M]` | 页面背景使用 Token 变量 `--bg-canvas`，禁止硬编码 hex | 检查 CSS 引用 |
| T01.02 | `[M]` | 卡片/面板背景使用 `--bg-surface` 或 `--bg-subtle`，禁止硬编码 | 检查 CSS 引用 |
| T01.03 | `[M]` | A股涨跌色：上涨 `--color-up: #EF4444`，下跌 `--color-down: #22C55E` | 检查所有涨跌展示元素 |
| T01.04 | `[M]` | 品牌主色使用 `--color-brand-500: #3B82F6`，禁止其他蓝色 | 检查蓝色元素 |
| T01.05 | `[M]` | 信号标签五色使用对应 Token：bullish/bearish/watch/neutral/invalid | 检查 SignalTag 实现 |
| T01.06 | `[M]` | 文字色使用 `--text-primary/secondary/muted/disabled` 层级 | 检查文字 CSS |
| T01.07 | `[M]` | 边框色使用 `--border-default/strong/focus`，禁止硬编码 | 检查边框样式 |
| T01.08 | `[R]` | 深色/浅色双主题均正确显示，用 `[data-theme]` 切换 | 执行 toggle 后检查 |
| T01.09 | `[M]` | 禁止使用装饰性渐变圆点/光晕/光斑（gradient orbs/bokeh） | 视觉检查 |
| T01.10 | `[M]` | 禁止使用紫色/紫蓝渐变主导的配色方案 | 视觉检查 |

### T02 — 圆角体系

| ID | 等级 | 规则 | 判定方法 |
|----|------|------|---------|
| T02.01 | `[M]` | 所有卡片/面板/弹窗圆角 `--radius-md: 8px` | 检查 border-radius |
| T02.02 | `[M]` | 所有输入框/Select/Dropdown 圆角 `--radius-sm: 6px` | 检查表单元素圆角 |
| T02.03 | `[M]` | 按钮圆角 `--radius-sm: 6px`（AntDV 默认） | 检查按钮样式 |
| T02.04 | `[M]` | Tag/Badge 圆角 `--radius-xs: 4px` | 检查标签元素 |

### T03 — 阴影体系

| ID | 等级 | 规则 | 判定方法 |
|----|------|------|---------|
| T03.01 | `[M]` | 仅保留两层阴影：正常态 `--shadow-default`、悬停态 `--shadow-hover` | 检查 box-shadow 值 |
| T03.02 | `[M]` | 禁止厚重的大投影/多层次阴影 | 视觉检查 |

## 维度三：排版规范 (Typography)

### G01 — 字号与行高

| ID | 等级 | 规则 | 判定方法 |
|----|------|------|---------|
| G01.01 | `[M]` | 字号仅从 `{12, 14, 16, 20, 24, 32}px` 中选择，禁止非标字号 | 检查 font-size 值 |
| G01.02 | `[M]` | 大标题（h1）行高 1.4，正文行高 1.6 | 检查 line-height |
| G01.03 | `[M]` | 不能使用 viewport-relative 字号（vw/vh） | 检查 font-size 单位 |
| G01.04 | `[M]` | `letter-spacing` 必须为 0 或 normal，不能为负值 | 检查 letter-spacing |
| G01.05 | `[R]` | 字体权重：正文 400、按钮/小标题 500、大标题/数值 600 | 检查 font-weight |

### G02 — 字体栈

| ID | 等级 | 规则 | 判定方法 |
|----|------|------|---------|
| G02.01 | `[M]` | 全局字体栈使用 `-apple-system, BlinkMacSystemFont, Segoe UI, PingFang SC, Microsoft YaHei, sans-serif` | 检查 global.less / CSS |

## 维度四：交互行为 (Interaction)

### I01 — 点击反馈

| ID | 等级 | 规则 | 判定方法 |
|----|------|------|---------|
| I01.01 | `[M]` | 所有可点击元素（按钮/链接/卡片 hoverable/表格行 clickable）必有 hover 状态视觉反馈 | 鼠标悬停验证 |
| I01.02 | `[M]` | 所有按钮点击时有 active 反馈（`scale(0.98)` 或颜色加深） | 点击验证 |
| I01.03 | `[M]` | hover/focus 过渡动画使用 `0.15s ease` | 检查 transition 值 |
| I01.04 | `[M]` | 不存在"点击后无响应"的死按钮 | 逐按钮点击验证 |
| I01.05 | `[M]` | 禁用态按钮有视觉区分（灰色/不可点击） | 检查 disabled 样式 |

### I02 — 过渡与动画

| ID | 等级 | 规则 | 判定方法 |
|----|------|------|---------|
| I02.01 | `[R]` | 交互动效应克制，仅 hover/focus 轻微过渡，无浮夸动画 | 视觉检查 |
| I02.02 | `[R]` | 主题切换动画使用 0.3s ease（已通过 theme.js 实现） | 切换主题验证 |
| I02.03 | `[M]` | Sidebar 折叠/展开动画流畅（transition 0.3s） | 折叠/展开验证 |

### I03 — 表单交互

| ID | 等级 | 规则 | 判定方法 |
|----|------|------|---------|
| I03.01 | `[M]` | 所有表单输入框有必填/选填标识 | 视觉检查 |
| I03.02 | `[M]` | 提交空表单时显示必填验证提示（AntDV 原生） | 空表单提交验证 |
| I03.03 | `[M]` | 表单提交期间按钮显示 loading 态 | 提交验证 |
| I03.04 | `[M]` | 表单提交成功/失败均有反馈（message.success / message.error） | 提交验证 |
| I03.05 | `[M]` | 所有弹窗可通过点击 mask 或按 Escape 关闭 | 弹窗交互验证 |

### I04 — 导航交互

| ID | 等级 | 规则 | 判定方法 |
|----|------|------|---------|
| I04.01 | `[M]` | Sidebar 当前页面路由高亮 | 逐路由检查 |
| I04.02 | `[M]` | Sidebar 折叠时只显示图标，展开时显示图标+文字 | 折叠/展开检查 |
| I04.03 | `[M]` | 下拉菜单（Dropdown）点击打开/关闭 | 验证 Dropdown |
| I04.04 | `[M]` | Tab 切换时内容同步切换无闪烁 | 验证 Tab 页 |

### C01 — 全局组件

| ID | 等级 | 规则 | 判定方法 |
|----|------|------|---------|

| C01.04 | `[R]` | 快捷键带组合键导航（g+i/s/f/w/a/b）正常工作 | 验证 |

### C02 — 数据展示组件

| ID | 等级 | 规则 | 判定方法 |
|----|------|------|---------|
| C02.01 | `[M]` | 表格（a-table）支持列排序（有 sort 图标） | 验证列头点击 |
| C02.02 | `[M]` | 表格中涨跌幅数值用红涨绿跌着色 | 视觉检查 |

| C02.05 | `[M]` | 代码编辑器（CodeMirror）语法高亮正常 | 验证代码编辑区 |

| C03.03 | `[M]` | Select 组件可搜索时显示搜索输入框 | 验证 |
| C03.04 | `[R]` | DatePicker/RangePicker 日期选择后回填正确 | 验证 |

| C04.02 | `[M]` | 弹窗关闭时内容清空/重置 | 打开关闭验证 |
| C04.03 | `[M]` | 删除操作有确认弹窗（`a-popconfirm` 或 modal） | 验证删除操作 |

## 维度六：文案边界 (Copy/Text Standards)

### X01 — 语义规范

| ID | 等级 | 规则 | 判定方法 |
|----|------|------|---------|
| X01.01 | `[M]` | **严禁** 出现"买入"、"卖出"、"下单"、"自动交易"、"实盘" | 全文搜索 |
| X01.02 | `[M]` | 交易相关文案使用信号语义替代：买入→关注信号，卖出→风险退出信号，交易记录→回测事件，持仓→模拟组合 | 全文搜索+核对 |
| X01.03 | `[M]` | 所有面向用户的提示/错误/确认信息使用中文 | 检查所有 message/tooltip |
| X01.04 | `[R]` | 按钮/链接文字不加 emoji 前缀（emoji 应作为图标独立使用） | 视觉检查按钮文案 |

### X02 — 免责声明

| ID | 等级 | 规则 | 判定方法 |
|----|------|------|---------|
| X02.01 | `[M]` | 所有展示策略信号/评分/建议的页面底部必须有免责声明 | 逐页检查 |
| X02.02 | `[M]` | 免责声明文字："本系统输出为策略研究与决策支持信息，不连接券商交易接口，不自动下单，不执行真实资金调仓。所有结论仅供研究参考。" | 文字核对 |
| X02.03 | `[R]` | 免责声明使用 `DisclaimerFooter` 统一组件 | 检查组件引用 |

### X03 — 标点与格式

| ID | 等级 | 规则 | 判定方法 |
|----|------|------|---------|
| X03.01 | `[R]` | 中文文案使用中文标点（逗号句号引号等） | 视觉检查 |
| X03.02 | `[R]` | 数字与单位之间有空格（如"3 只股票"） | 视觉检查 |

## 维度七：状态覆盖 (State Coverage)

### ST01 — Loading 状态

| ID | 等级 | 规则 | 判定方法 |
|----|------|------|---------|
| ST01.01 | `[M]` | 每个异步数据请求（API 调用）触发时显示 loading 指示器 | 逐一验证 API 调用页面 |
| ST01.02 | `[M]` | loading 状态下按钮/输入框不可重复操作 | 多次点击验证 |
| ST01.03 | `[R]` | 表格/列表数据加载时有 skeleton 骨架屏替代 Spin | 视觉检查 |

### ST02 — 空状态

| ID | 等级 | 规则 | 判定方法 |
|----|------|------|---------|
| ST02.01 | `[M]` | 每个列表/卡片网格在没有数据时显示空状态提示 | 逐一验证 |
| ST02.02 | `[R]` | 空状态使用统一组件（`a-empty` 或自定义 EmptyState）且有引导文案 | 视觉检查 |
| ST02.03 | `[O]` | 空状态包含操作引导（如"暂无自选股，点击添加"） | 视觉检查 |

### ST03 — 错误状态

| ID | 等级 | 规则 | 判定方法 |
|----|------|------|---------|
| ST03.01 | `[M]` | API 调用失败时向用户显示错误提示（非静默失败） | mock 错误验证 |
| ST03.02 | `[M]` | 错误提示使用中文（"请求失败"而非 "Request failed"） | 验证 |
| ST03.03 | `[M]` | 错误提示不泄漏内部实现细节（无堆栈信息、无 IP/路径） | 验证 |
| ST03.04 | `[M]` | 关键页面区域（K线图/表格）的局部错误不导致整个页面白屏 | mock 错误验证 |

### ST04 — 边界状态

| ID | 等级 | 规则 | 判定方法 |
|----|------|------|---------|
| ST04.01 | `[M]` | 页面可以处理 null/undefined 的 props/data 而不报错 | 代码审查 + mock |
| ST04.02 | `[M]` | ErrorBoundary 捕获渲染异常后显示降级 UI 而非白屏 | 触发错误验证 |
| ST04.03 | `[R]` | 信号已失效/过期的状态有视觉区分（灰色/暗红色文本） | 视觉检查 |
| ST04.04 | `[M]` | 首次加载（localStorage 空）时页面正常渲染 | 清除缓存后加载 |

## 附录：已注册 Token 变量一览

所有变量定义在 `frontend/vue-project/src/global.less` 中，以下为基础变量索引（不含衍生值）：

```less
/* 空间 */
--space-{2,4,6,8,10,12,16,20,24,32,40,48,64}

/* 叠加层 */
--overlay-{white,light,medium,strong,intense}

/* 背景 */
--bg-canvas, --bg-surface, --bg-subtle, --bg-muted, --bg-active, --bg-selected

/* 品牌色 */
--color-brand-{50,100,200,500,600,700}

/* 涨跌色 */
--color-up, --color-up-bg, --color-down, --color-down-bg, --color-flat

/* 信号色 */
--signal-{bullish,bearish,watch,neutral,invalid}

/* 文字 */
--text-{primary,secondary,muted,disabled,inverse}

/* 边框 */
--border-{default,strong,focus}

/* 圆角 */
--radius-{xs,sm,md,lg,full}

/* 阴影 */
--shadow-{default,hover}

/* 排版 */
--font-{sans,mono}
--text-{xs,sm,base,lg,xl,2xl,3xl}  /* 12,14,16,20,24,32px */
--leading-{tight,normal}           /* 1.4, 1.6 */
--weight-{normal,medium,semibold}  /* 400,500,600 */
```
