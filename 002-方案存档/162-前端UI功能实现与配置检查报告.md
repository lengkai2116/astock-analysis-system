## 四、综合问题分析

### 4.1 配置层问题

1. **`.env` 文件策略不一致**：根目录 `.env` 包含真实 `DEEPSEEK_API_KEY`、`SECRET_KEY`、`AUTH_TOKEN`；而 `.env.local` 包含开发配置（`sqlite`、`mock` LLM）。存在将敏感 Key 提交到版本库的风险，且开发/生产配置分离不够清晰。

2. **`LLM_PROVIDER` 默认值冲突**：`.env.local` 默认使用 `mock`，但 `docker-compose.yml` 硬编码 `${LLM_PROVIDER:-deepseek}`，当宿主环境未设置此变量时 Docker 内默认使用 `deepseek` 而非 `mock`。

### 4.2 渲染层问题

3. **硬编码色值**：`backtest/index.vue` 的 ECharts 轴线和标签颜色使用了 `#2a2a2a`、`#94a3b8`、`#1e293b` 等值；`dashboard/index.vue` 的评分逻辑、奖牌背景也用了硬编码十六进制。这些应使用 `global.less` 中定义的 CSS 变量 Token（ECharts 配置项虽然不支持 CSS 变量，但应使用定义在 `global.less` 或配置文件中统一的常量）。

4. **API 401/500 错误污染控制台**：后端 `AUTH_TOKEN` 认证导致所有 API 调用返回 401（页面通过优雅降级渲染，但 Nginx 下 token 未正确转发）；`/api/factor-combinations` 和账户概览接口返回 500。

### 4.3 体验层问题

5. **文案边界违规**：`views/account/components/TradeTab.vue` 仍出现"买入"、"卖出"等交易语义，违背 X01.01 强制规则。应替换为"关注信号"、"风险退出信号"等研究语义。

6. **免责声明全线缺失**：所有展示策略信号/评分的页面底部均未放置免责声明，违背 X02.01 强制规则。应创建统一 `DisclaimerFooter.vue` 组件并在所有策略输出页面引用。

7. **按钮 emoji 混用**：多数页面按钮文本以 emoji 开头（`📊 仪表盘`、`📈 策略中心`、`🔄 刷新`），与 X01.04 推荐规范建议不符。建议将 emoji 作为独立图标元素而非按钮文本前缀。
