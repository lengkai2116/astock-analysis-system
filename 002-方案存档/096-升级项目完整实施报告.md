### 2.3 第三阶段：指标IDE研究工作台 ✅

**完成时间**: 2026-05-24  
**工时**: 约13人天

#### 新增文件

| 文件路径 | 说明 |
|---------|------|
| `backend/app/services/indicator_contract.py` | 指标协议解析器 |

| `backend/app/services/indicator_sandbox.py` | 指标沙盒执行器 |
| `backend/app/routes/indicator_ide.py` | 指标IDE API路由 |

#### 核心功能

- ✅ **指标协议解析器**：
  - 解析 `my_strategy_name` 和 `my_strategy_description`
  - 提取 `@param` 注解生成参数配置
  - 解析 `output` 字典定义
  - 使用 AST 和正则表达式

- ✅ **指标质量检查器**：
  - 检查 `df.copy()` 使用情况
  - 检测未来函数（`shift(-N)`、`iloc[-N:]`）
  - 识别危险操作（文件、网络、子进程等）
  - 语法错误检查

- ✅ **指标沙盒执行器**：
  - 受限的Python执行环境
  - 只允许 pandas、numpy 基础操作
  - 禁止文件、网络、子进程等危险操作
  - 执行超时保护（默认5秒）

#### API端点

| 端点 | 方法 | 说明 |
|-----|------|-----|
| `/api/v2/indicator/parse` | POST | 解析指标代码 |
| `/api/v2/indicator/check` | POST | 质量检查 |
| `/api/v2/indicator/execute` | POST | 执行指标代码 |
| `/api/v2/indicator/validate` | POST | 综合验证 |

## 六、新增API文档

### 6.1 策略输出API

```bash
# 获取策略输出列表
GET /api/v2/strategy/outputs?ts_code=000001.SZ&limit=10

# 创建策略输出
POST /api/v2/strategy/outputs
{
  "ts_code": "000001.SZ",
  "strategy_name": "均线交叉策略",
  "signal": "bullish",
  "confidence": 0.75,
  "entry_zone": [12.30, 12.80],
  "risk_line": 11.60,
  "target_zone": [14.20, 15.00],
  "position_suggestion": "10%-20%",
  "holding_period": "5-20个交易日",
  "evidence": ["MA5上穿MA20", "成交量放大"],
  "risk_notes": ["接近前高压力位"],
  "signal_date": "2026-05-24"
}
```

### 6.2 回测API

```bash
# 运行回测
POST /api/v2/backtest/run
{
  "ts_code": "000001.SZ",
  "start_date": "2025-01-01",
  "end_date": "2026-01-01",
  "initial_capital": 100000,
  "strategy_code": "your_indicator_code_here"
}
```

### 6.3 指标IDE API

```bash
# 综合验证
POST /api/v2/indicator/validate
{
  "code": "your_indicator_code_here",
  "params": {"short_period": 5, "long_period": 20}
}
```

### 6.4 报告API

```bash
# 生成报告
```
