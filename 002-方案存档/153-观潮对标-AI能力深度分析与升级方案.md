# 观潮对标 — AI能力深度分析与升级方案

> 本文档通过反编译观潮 `intelligence.js`（94KB），结合 `analysis.js` 及 CSS 反向分析，对其 AI 能力架构进行深度还原分析。对比本项目当前 AI 系统（基于 DeepSeek 纯文本模型），评估差距并提出可落地的升级方案。
>
> **前提核实确认**：本项目使用的 DeepSeek API（`deepseek-chat`，2026/07/24 后迁移至 `deepseek-v4-flash`/`deepseek-v4-pro`）及 LM Studio 本地模型均为**纯文本模型**，不支持图像输入/视觉识别。观潮的 `VisualPatternRecognition` 依赖阿里 Qwen VL（视觉语言模型），本项目当前技术栈无法实现。因此本方案的 AI 升级聚焦**文本推理、上下文构建、信号融合、决策结构化**方向，不涉及视觉模式识别。

## 2. 观潮 AI 核心机制深度解析

### 2.1 MultiStepAnalysis — 多步诊断引擎

反编译揭示观潮 AI 诊断引擎的完整工作流程：

```
Step 1: Context Assembly（上下文装配）
  ├── 技术上下文: K线 + 指标 + 预检测模式信号 + 关键价格层级
  ├── 基本面上下文: 财务摘要 + 估值定位
  ├── 消息面上下文: NewsStream（公告/新闻/宏观/背景新闻）
  ├── 题材上下文: concept标签 + sector_strength
  └── 另类数据: northbound + money_flow + holder_trade + 股东户数

Step 2: Asset-Specific Diagnosis（资产类型特异化诊断）
  根据资产类型选择特异化 system prompt:
  ├── "你是专业A股量化诊断引擎。综合技术面、基本面、消息面、题材概念做多维推理。"
  ├── "你是专业ETF诊断引擎。综合净值锚定、市场统计、主题背景做多维推理。"
  ├── "你是专业期货诊断引擎。综合趋势结构、持仓结算、宏观驱动做多维推理。"
  ├── "你是专业加密资产诊断引擎。综合24/7价格结构、风险偏好、美元流动性做多维推理。"
  ├── "你是专业外汇诊断引擎。综合汇率趋势、宏观驱动、风险因素做多维推理。"
  ├── "你是专业国际商品诊断引擎。综合价格趋势、宏观驱动、供需线索做多维推理。"
  └── "你是专业指数诊断引擎。综合指数结构、市场广度、权重分布做多维推理。"

Step 3: Structured Output（结构化输出解析）
  要求 response_format = json_object，返回：
  ├── verdict: bullish|bearish|warning|catalyst|risk
  ├── confidence: 0~1
  ├── dimensions: [{name, score, evidence, direction}]
  ├── keyLevels: {support, resistance, invalidation, target}
  ├── interpretation: string
  ├── invalidation: string
  └── risk_control: {stopLoss, riskNotes}
```

### 2.4 AI 融合评分 V2

`ai_fusion_v2` 采用的评分算法信号：

```javascript
// intelligence.js 推断的融合算法
function fusionScore(signals) {
  // 多维度加权融合
  const dimensions = signals.map(s => ({
    name: s.dimension,
    score: s.score,       // 每个维度的独立评分
    weight: s.weight,     // 维度权重
    direction: s.direction // bullish/bearish
  }));

  // 共振检测: 多个同向维度同时高强度 → 共振加分
  const resonance = detectResonance(dimensions);
  // 矛盾检测: 方向冲突的维度 → 降权
  const conflict = detectConflict(dimensions);

  return {
    overallScore: weightedAverage(dimensions) + resonance - conflict,
    verdict: classifyVerdict(overallScore),
    keyLevels: extractKeyLevels(dimensions),
  };
}
```

### 2.5 预测校准系统

观潮的 `predictionStore` 和 `predictionTracker` 构成闭环：

```
AI 诊断输出预测（方向+关键价位）
  → 存入 predictionStore
  → 实际走势（5/10/20日后）回填 actualReturn5d/10d/20d
  → predictionTracker 计算准确率
  → calibration 校准下次诊断的置信度
  → 下次诊断注入 "历史校准" 上下文
```

**calibration 数据模型**（intelligence.js 反编译）：

```javascript
{
  symbol: string,
  predictionDate: string,
  direction: 'bullish' | 'bearish',
  confidence: number,
  actualReturn5d: number,
  actualReturn10d: number,
  actualReturn20d: number,
  hitTarget: boolean,
  hitStop: boolean,
  verdict: 'correct' | 'wrong' | 'pending'
}
```

## 4. 借鉴升级方案

### 4.1 策略一：多重上下文注入升级（P0）

**目标**：将 AI 分析 prompt 从"基础技术数据"升级为"多维市场上下文"。

**现状**：prompt 仅包含基础 OHLCV 数据 + 几个技术指标。

**改进**：构建多层次上下文注入器。

```python
# backend/app/services/ai_context_builder.py — 上下文构建器
class AIContextBuilder:
    """
    多维 AI 分析上下文构建器
    
    参考观潮 AnalysisContext 体系，为 AI 诊断提供丰富上下文。
    共 5 层上下文，每层独立获取、容错、可选。
    """
    
    def build(self, ts_code: str, df: pd.DataFrame) -> Dict:
        return {
            'technical': self._build_technical_context(df),
            'patterns': self._build_pattern_context(ts_code, df),
            'levels': self._build_key_levels(df),
            'fundamental': self._build_fundamental_context(ts_code),
            'news': self._build_news_context(ts_code),
            'market': self._build_market_context(),
            'sector': self._build_sector_context(ts_code),
            'calibration': self._build_calibration_context(ts_code),
        }

    def _build_technical_context(self, df) -> Dict:
        """技术面上下文：趋势/动量/量价/支撑阻力"""
        return {
            'trend': {'ma_alignment': 'bullish', 'stage': 'accumulation'},
            'momentum': {'macd': 'golden_cross', 'rsi': 55, 'kdj': '金叉'},
            'volume': {'vol_ratio': 1.5, 'vol_ma_ratio': '放量'},
            'patterns': ['底分型', '放量突破20日均线'],
            'levels': {'support': 15.20, 'resistance': 16.80},
        }

    def _build_news_context(self, ts_code) -> Dict:
        """消息面上下文（新增）"""
        # 来源：Tushare news API / 公告 API
        return {
            'announcements': ['2026-06-03: 业绩预告大幅增长'],
            'sector_news': ['新能源板块受政策利好'],
            'macro_news': ['央行释放流动性信号'],
        }

    def _build_sector_context(self, ts_code) -> Dict:
        """题材/板块上下文（新增）"""
        return {
            'concepts': ['锂电池', '新能源车'],
            'sector_strength': 'strong',  # 强于大盘
            'peer_comparison': {'rank': '前20%', 'peer_avg_return': 3.5},
        }

    def _build_calibration_context(self, ts_code) -> Dict:
        """历史校准上下文（新增）"""
        # 从 calibration store 获取历史预测准确率
        return {
            'last_prediction': {'direction': 'bullish', 'accuracy': 'correct'},
            'historical_accuracy': 0.65,
            'avg_return_5d': 2.3,
        }

    def to_prompt(self, context: Dict) -> str:
        """将结构化上下文渲染为 AI prompt 文本"""
        parts = []
        # 技术面
        parts.append("【技术面】")
        parts.append(f"趋势: {context['technical']['trend']}")
        parts.append(f"动量: {context['technical']['momentum']}")
        # 消息面
        parts.append("【消息面】")
        for news in context['news'].get('announcements', []):
            parts.append(f"  - {news}")
        # 题材板块
        parts.append("【题材概念】")
        parts.append(f"所属概念: {', '.join(context['sector'].get('concepts', []))}")
        parts.append(f"板块强度: {context['sector'].get('sector_strength', 'unknown')}")
        # ... 其他维度
        return "\n".join(parts)
```

### 4.3 策略三：AiSignalBus 轻量版（P1）

**目标**：构建事件驱动的 AI 信号广播机制，实现 AI 分析→模块联动的自动闭环。

**现状**：AI 分析结果独立展示在前端 AI 分析页面，不与其他模块联动。

**改进**：引入轻量级信号总线（EventEmitter 模式）。

```javascript
// frontend/src/services/aiSignalBus.js — 轻量版信号总线
class AiSignalBus {
  constructor() {
    this.listeners = new Map()
    this.lastSignals = new Map()
  }

  // 注册监听
  on(event, callback) {
    if (!this.listeners.has(event)) {
      this.listeners.set(event, [])
    }
    this.listeners.get(event).push(callback)
    // 如果有缓存的最新信号，立即触发
    if (this.lastSignals.has(event)) {
      callback(this.lastSignals.get(event))
    }
  }

  // 发射信号
  emit(event, data) {
    this.lastSignals.set(event, data)
    const cbs = this.listeners.get(event) || []
    cbs.forEach(cb => {
      try { cb(data) } catch (e) { console.warn(`[AiSignalBus] ${event} handler failed:`, e) }
    })
  }

  // 清除
  off(event, callback) {
    // ...
  }
}

// 全局单例
export const aiSignalBus = new AiSignalBus()
```

**事件定义**：

| 事件名 | 数据 | 触发时机 | 影响模块 |
|--------|------|---------|---------|
| `ai:diagnosis-complete` | {symbol, verdict, confidence, levels} | AI 诊断完成 | Watchlist → 更新共振灯；Chart → 标记价位 |
| `ai:pattern-detected` | {symbol, pattern, direction, strength} | AI 发现新形态 | Chart → 叠加形态标记；Panel → 追加 ComboCard |
| `ai:signal-update` | {symbol, signals[], resonanceScore} | 信号发生变化 | Watchlist → 排序更新；Dashboard → 指标更新 |
| `ai:alert-triggered` | {symbol, condition, price} | 价格触及预警 | Notification → 推送；Sound → 播放提示 |
| `ai:calibration` | {symbol, prediction, actualReturn} | 预测回填校准 | Calibration → 更新准确率 |

**后端对接**：后端 AI 分析完成后，通过 WebSocket 推送事件到前端。

```python
# backend/app/services/ai_signal_bus.py — 后端信号推送
class AiSignalBackend:
    """后端 AI 信号推送"""
    
    def __init__(self, socketio):
        self.socketio = socketio
    
    def broadcast_diagnosis(self, ts_code: str, result: Dict):
        """广播 AI 诊断结果到所有连接的客户端"""
        self.socketio.emit('ai:diagnosis-complete', {
            'symbol': ts_code,
            'verdict': result.get('verdict'),
            'confidence': result.get('confidence'),
            'levels': result.get('key_levels'),
        }, namespace='/ai')
```

```python
# 在 ComboEngine 中新增 AI 信号融合
class ComboEngine:
    def build_combos(self, patterns: List[PatternResult], ai_diagnosis: Optional[Dict] = None) -> List[ComboCard]:
        cards = []
        
        # 规则引擎模式 → ComboCard
        bull_patterns = [p for p in patterns if p.direction == 'bullish']
        if bull_patterns:
            cards.append(self._build_combo(bull_patterns, 'bullish'))
        
        # AI 诊断 → AI ComboCard（如果可用）
        if ai_diagnosis and ai_diagnosis.get('verdict') in ('bullish', 'bearish'):
            cards.append(self._build_ai_combo(ai_diagnosis))
        
        return cards
    
    def _build_ai_combo(self, diagnosis: Dict) -> ComboCard:
        """将 AI 诊断输出包装为 ComboCard"""
        return ComboCard(
            name="AI 综合诊断",
            strength=int(diagnosis['confidence'] * 100),
            patterns=diagnosis.get('dimensions', []),
            interpretation=diagnosis.get('interpretation', ''),
            levels=diagnosis.get('key_levels', {}),
            source='ai',
        )
```

```python
        'name': '技术分析师',

            'key_levels': {'support': 'float', 'resistance': 'float'},

            'indicators': {'macd': 'str', 'rsi': 'float', 'vol_ratio': 'float'},

class FusionEngine:
    """多角色评分自动融合"""
    
    WEIGHTS = {
        'technical': 0.25,
        'fundamental': 0.20,
        'macro': 0.20,
        'risk': 0.15,
        'fund_manager': 0.20,
    }
    
    def fuse(self, role_results: List[Dict]) -> Dict:
        weighted_score = sum(
            r.get('score', 50) * self.WEIGHTS.get(r['role_id'], 0.2)
            for r in role_results
        )
        directions = [r['direction'] for r in role_results if r.get('direction')]
        consensus = self._calc_consensus(directions)
        return {
            'overall_score': weighted_score,
            'consensus': consensus,
            'role_scores': {r['role_id']: r['score'] for r in role_results},
            'key_levels': self._merge_levels(role_results),
        }
```

### 5.3 响应时间实测与交互设计

> 本方案设计为**用户点击触发**（非自动），以节省 token 消耗，同时确保解读内容简洁聚焦。

#### 实测响应时间

| 环节 | 实测耗时 | 说明 |
|------|---------|------|
| LLM Wiki 搜索 | ~60ms | 本地 API，搜索 10 条结果，评分 30-96 |
| 读取知识页内容（含 includeContent） | ~60ms | 每页 900-1600 字符，非常轻量 |
| 策略输出准备 | <10ms | 已计算完成，直接取用 |
| Prompt 构建 | <10ms | 模板拼接，无网络开销 |
| DeepSeek API 调用（估计） | 2-5s | 按产出 300-500 tokens 估算 |
| **合计（用户感知）** | **2.5-5.5s** | 建议使用 streaming |

**实测数据说明**：
- 知识库页面内容典型大小：**906-1577 字符**（如《量价关系八准则》1314 字符、《筹码分布揭密》906 字符），非常适合作为 AI 上下文注入
- LLM Wiki 搜索模式当前为 keyword，评分范围 30-96，足够定位相关概念
- DeepSeek API 响应时间极简回复约 0.9s，真实解读（300-500 tokens）预计 2-5s

#### 交互设计

用户看到策略输出 -> 右侧有 AI 解读按钮（非自动触发）-> 用户点击 -> 展示 loading spinner -> 使用 SSE 流式输出 -> AI 解读逐行展示（首字出现时间 <2s）

#### 内容长度控制三重约束

1. 输出不超过 200 字（3-5 个要点），使用要点列表格式
2. 仅引用搜索结果中明确出现的知识点，不得编造
3. 不得修改策略输出的方向、置信度、价位等数值

**后续测试方向**：
| 测试项 | 目标 | 方法 |
|--------|------|------|
| 响应时间上限 | 确保 95% 请求在 6s 内完成 | 抽取 20 只不同股票，记录全链路耗时 |
| 内容长度最优值 | 找到信息量足够与阅读负担轻的平衡点 | A/B 测试 100/200/300/500 字输出长度 |
| 知识库搜索策略 | 找最相关的 1-2 页 vs 泛搜 5 页 | 对比不同搜索策略下的解读质量 |
| 流式输出体验 | 首字出现时间 < 2s | 实测 SSE 从点击到首字显示的耗时 |

### 5.4 与现有AI投研分析的功能区隔

#### 功能对照表

| 维度 | 现有 AI 投研分析 | 新增 AI 策略解读 |
|------|-----------------|-----------------|
| 触发方式 | 用户在 AI 分析页选择股票，点击开始分析 | 在策略信号面板中，点击 AI 解读按钮 |
| 输入数据 | K 线数据 + 基础指标 + 基本面数据 | 策略输出（方向/置信度/价位/证据）|
| 知识来源 | 通用股票分析知识（LLM 训练数据）| LLM Wiki 知识库（授权书内容）|
| 分析方式 | 5 角色平行分析，各自独立输出后合成 | 单引擎聚焦解读，围绕策略证据链展开 |
| 输出形式 | 多角色独立报告 + 经理最终合成，自由文本 | 结构化要点列表（3-5 条），紧扣策略信号 |
| 输出时长 | 2-5 分钟（5 个角色并行）| 2-5 秒（单次 API 调用）|
| 目标 | 这只股票怎么样？——独立全面研究 | 为什么策略给了这个信号？——策略原理阐释 |
| 依赖关系 | 独立运行，不依赖策略系统 | 依赖策略输出，无策略信号则无法解读 |
| 用户场景 | 用户想做独立的深度研究时 | 用户看到策略信号想理解其原理时 |

**简单比喻**：
- AI 投研分析 = 请一个 5 人专家团队对某只股票做独立调研
- AI 策略解读 = 请一个导师解释为什么系统给出了这个判断

两者是**互补关系**，不是替代关系。用户在策略面板看到信号，点击解读后理解了原理，如果需要更深入研究，可以切换到 AI 分析页做全面分析。

#### 界面整合方案

策略信号面板（现有）增加 AI 解读按钮 -> 点击后展开解读区域
AI 分析页（独立页面，不变）-> 用户切换到 /ai-analysis 页面做独立深度研究
