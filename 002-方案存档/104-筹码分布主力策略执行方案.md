### 3.1 扩展DuckDB表结构

**文件**: `backend/app/data/enhanced_cache_manager.py`

**新增表**:
```sql
CREATE TABLE IF NOT EXISTS chip_distribution_cache (
    ts_code VARCHAR,
    trade_date DATE,
    price_bin DECIMAL,      -- 价格区间（如：10.50）
    chip_ratio DECIMAL,     -- 该价格区间筹码比例（0-1）
    accumulated_ratio DECIMAL,  -- 累计筹码比例（用于筹码峰检测）
    peak_flag BOOLEAN,      -- 是否筹码峰
    update_time TIMESTAMP,
    PRIMARY KEY (ts_code, trade_date, price_bin)
)
```

**方法新增**:
- `cache_chip_distribution(ts_code, trade_date, chip_data)`: 缓存筹码分布
- `get_chip_distribution(ts_code, start_date, end_date)`: 获取筹码分布
- `batch_cache_chips(records)`: 批量缓存

### 3.2 实现筹码估算服务

**文件**: `backend/app/data/chip_distribution_service.py`

**核心逻辑**:
1. **价格网格**: 将历史价格区间切分为100-200个price_bin
2. **成交分配**: 当日成交量按OHLC区间分布到各价格区间
3. **衰减模型**: 换手率/固定衰减率模拟旧筹码替换
4. **筹码峰检测**: 局部最大值检测

**核心方法**:
- `calculate_chip_distribution(ts_code, lookback_days=120)`: 计算筹码分布
- `detect_chip_peaks(chip_data)`: 检测筹码峰
- `calculate_accumulated_ratio(chip_data)`: 计算累计比例

### 3.3 实现筹码因子计算

**文件**: `backend/app/data/chip_indicators.py`

**因子列表**:
| 因子 | 计算逻辑 | 用途 |
|------|---------|------|
| **ASR** | 当前价格±10%区间筹码占比 | 浮筹压力识别 |
| **CYQKL** | K线穿越筹码密集区强度 | 突破力度识别 |
| **SSRP** | 市场平均成本 | 主力成本区识别 |
| **筹码集中度** | 前N%价格区间筹码占比 | 筹码集中/分散 |
| **筹码获利率** | 当前价格以下筹码比例 | 获利盘分析 |
| **筹码转移率** | 近M日筹码向当前价格区域移动速度 | 筹码转移快慢 |

**核心方法**:
- `calculate_asr(chip_data, current_price, band_pct=0.1)`
- `calculate_cyqkl(chip_data, kline_data)`
- `calculate_ssrp(chip_data)`
- `calculate_concentration(chip_data, top_pct=0.2)`
- `calculate_profit_ratio(chip_data, current_price)`
- `calculate_transfer_rate(chip_history, window=20)`

## 四、技术设计细节

### 4.1 筹码估算算法（阶段一最小可用版）

```python
class ChipDistributionEstimator:
    """
    基于OHLCV的筹码估算器
    """
    
    def __init__(self, num_bins=150, decay_rate=0.005):
        self.num_bins = num_bins
        self.decay_rate = decay_rate  # 每日旧筹码衰减率
    
    def estimate(self, df_ohlcv):
        """
        估算筹码分布
        df_ohlcv: 包含ts_code, trade_date, open, high, low, close, vol, amount
        """
        # 1. 确定价格区间
        min_price = df_ohlcv['low'].min()
        max_price = df_ohlcv['high'].max()
        price_step = (max_price - min_price) / self.num_bins
        
        # 2. 初始化筹码分布
        chip_dist = np.zeros(self.num_bins)
        
        # 3. 逐天估算
        for idx, row in df_ohlcv.iterrows():
            # 旧筹码衰减
            chip_dist *= (1 - self.decay_rate)
            
            # 分配当日成交量到价格区间
            price_range = row['high'] - row['low']
            if price_range > 0:
                # 计算每个价格区间的成交量比例
                for bin_idx in range(self.num_bins):
                    bin_price = min_price + bin_idx * price_step + price_step/2
                    if row['low'] <= bin_price <= row['high']:
                        # 简单线性分配
                        chip_dist[bin_idx] += row['vol'] / self.num_bins
        
        # 4. 归一化
        total = chip_dist.sum()
        if total > 0:
            chip_dist = chip_dist / total
        
        return chip_dist, min_price, max_price, price_step
```

## 六、风险与注意事项

### 风险点
1. **筹码估算精度**: 基于OHLCV的估算精度有限，无法达到Level2数据的精度
2. **计算性能**: 全市场5000只股票×120天=60万次计算，需优化性能
3. **数据依赖**: 需要流通股本、换手率等daily_basic数据

### 注意事项
1. **阶段一先做单只股票测试**: 验证算法正确性
2. **复用现有缓存机制**: 尽可能使用DuckDB+Redis已有框架
3. **预留扩展接口**: 为将来Level2数据集成预留接口
