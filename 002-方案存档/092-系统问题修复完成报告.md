### P1 - 前端响应拦截规则与后端返回结构不兼容

**问题描述**:
- 前端仅接受 `res.code === 1/200` 的格式
- 后端大量接口返回 `{ success: true/false, data, message }` 格式
- 位置: `frontend/vue-project/src/utils/request.js:26-29`

**修复内容**:
- 实现双格式兼容逻辑
- 支持 `code` 和 `success` 两种判断方式
- 保持向后兼容

**修复代码**:
```javascript
// 兼容两种响应格式
if (res.code !== undefined) {
    // 格式 1: { code: 200, msg, data }
    if (res.code !== 1 && res.code !== 200) {
        message.error(res.msg || '请求失败')
        return Promise.reject(new Error(res.msg || 'Error'))
    }
} else if (res.success !== undefined) {
    // 格式 2: { success: true, message, data }
    if (!res.success) {
        message.error(res.message || res.msg || '请求失败')
        return Promise.reject(new Error(res.message || res.msg || 'Error'))
    }
}
```

**验证状态**: ✅ 已修复

### P2 - 实时行情涨跌幅计算逻辑不正确

**问题描述**:
- 昨收价和现价都读取 `latest['close']`
- 导致涨跌幅永远为0
- 位置: `backend/app/routes/realtime.py:90-91`

**修复内容**:
- 使用 `pre_close` 作为昨收价
- 添加了回退逻辑（如果没有pre_close则使用close）

**修复代码**:
```python
# 修复前
prev_close = float(latest.get('close', 0))
current_price = float(latest.get('close', 0))

# 修复后
prev_close = float(latest.get('pre_close', latest.get('close', 0)))
current_price = float(latest.get('close', 0))
```

**验证状态**: ✅ 已修复

### P2 - 缓存批量同步接口脚本路径问题

**问题描述**:
- 子进程使用相对路径 `bulk_sync.py`
- 工作目录不是 `backend/` 时失败
- 位置: `backend/app/routes/cache.py:189-191`

**修复内容**:
- 添加 os 模块导入
- 使用绝对路径构建脚本位置
- 通过 `__file__` 动态计算路径

**修复代码**:
```python
# 添加导入
import os

# 使用绝对路径
current_dir = os.path.dirname(os.path.abspath(__file__))
script_path = os.path.join(current_dir, '..', '..', 'bulk_sync.py')
script_path = os.path.abspath(script_path)

cmd = [sys.executable, script_path, '--limit', str(limit)]
```

**验证状态**: ✅ 已修复
