## 四、详细设计

### #31 — 量价形态扩展至50种

**当前状态：** EnhancedPatternDetector 已有 72 条 checks。但知识库要求的"50种"包含当前缺失的形态：
- 涨停形态(一字板/ T字板/ 地天板)
- 旗形整理(上升旗形 / 下降旗形)
- 楔形(上升楔形 / 下降楔形)
- 头肩顶/底
- W底/M顶
- 收敛三角形 / 扩散三角形
- 缺口形态(突破缺口 / 中继缺口 / 衰竭缺口)

**实施：** 补充10种以上新方法，确保"50种"规则库的完整性（含涨停/旗形/楔形等需额外数据的形态做较宽松的近似判定）。

### #32 — 缠论11个定理体系实现

**当前状态：** 完全不存在。代码中无任何定理类或定理判定函数。

**实施：** 新建 `ChanlunTheoremValidator` 类到 `chanlun_strategy.py`，实现以下11个定理：

| # | 定理名 | 判定逻辑 |
|---|--------|---------|
| T1 | 走势必完美 | 检查当前线段是否已包含至少3笔（一个走势类型的最小结构） |
| T2 | 级别递归定理 | 检查当前分析的级别是否一致（默认日线，不能混入周线级笔） |
| T3 | 中枢破坏定理 | 当第3段突破中枢且不回中枢→中枢被破坏 |
| T4 | 背驰与转折定理 | 背驰后最少回到最近一个中枢（与现有回溯检测联动） |
| T5 | 买卖点级别定理 | 买卖点应在对应级别的背驰中出现（日线背驰→日线级别买卖点） |
| T6 | 同级别分解定理 | 分解后各线段应在同一级别（笔数/幅度相近） |
| T7 | 趋势延伸与结束定理 | 趋势结束=最后一个中枢被破坏+反向线段确认 |
| T8 | 多级别联立定理 | 周线方向→日线决策，方向一致时信号可信（占位，等数据就绪） |
| T9 | 特征序列定理 | 特征序列的缺口判定标准（与P1-#8联动） |
| T10 | 动力结构定理 | 比较相邻同向段的力度（长度/幅度/MACD面积） |
| T11 | 走势类型转换定理 | 盘整→趋势/趋势→盘整的转换判定 |

**集成：** 在 `ChanlunAnalyzer.analyze()` 末尾调用 `theorem_validator.validate(segments, zhongshu_list, strokes, closes)`。

### #35 — 中枢内因子动态切换

**当前状态：** `Zhongshu` 已有 `contains_price()/is_above()/is_below()`, `signal_computation_service.py` 已有 `position_vs_zs` 判定。

**实施：** 新增 `ZhongshuFactorSwitch` 类，提供：
- `get_position(price, zhongshu) → 'inside'/'above'/'below'/'none'`
- `select_factors(position, base_factors) → List[str]` 返回适用的因子列表：
  - 中枢内 → 反转因子优先（均值回归）
  - 中枢外上方 → 动量因子优先（趋势延续）
  - 中枢外下方 → 超卖因子优先（超跌反弹）

### #36 — 冲突调和机制编码化

**当前状态：** 冲突检测存在（ResonanceService.MIXED等级 / ChanlunScorer方向矛盾减分），但无仲裁。

**实施：** 新建 `conflict_arbiter.py`，实现四级裁定规则：
1. **结构优先**：缠论判定优先级 > 量价判定（缠论反映结构本质）
2. **位置优先**：关键位置（中枢边界/MA120/250）的信号 > 非关键位置信号
3. **量价验证**：量价同步时信号可信 > 量价背离
4. **嵌套处理**：多空信号并列时按下列规则：同一级别冲突→取低置信度判定；不同级别→高级别优先

### #37 — 形态量化中枢v0.1

**实施：** 新建 `morpho_zhongshu_fusion.py`，α(缠论) + β(量价) 二维融合：
- 当 α=看涨 + β=看涨 → 强烈看涨 (confidence +0.2)
- 当 α=看涨 + β=中性 → 偏看涨 (confidence 不变)
- 当 α=看涨 + β=看跌 → 冲突状态 (降低confidence)
- 当 α=中性 + β=看涨 → 偏看涨
- 当 α=中性 + β=中性 → 中性
- 经验权重：α=0.55, β=0.45（非回测权重，v0.1暂定）

输入：`ChanlunAnalysisResult` + `VolumePriceSignal` → 输出：融合状态/置信度

### #38 — UPF统一实战框架

**实施：** 新建 `unified_practical_framework.py`，定义四层框架：
1. **走势结构层(StructureLayer)**：缠论分析入口
2. **形态验证层(PatternLayer)**：量价形态验证入口
3. **情绪环境层(SentimentLayer)**：BOCIASI快线+慢线入口
4. **因子决策层(FactorLayer)**：多因子评分入口

每层定义标准接口 `evaluate(context) → LayerResult`，最终汇总到 `UPFEngine.evaluate_all(ts_code, df) → FusedResult`。

### #39 — UIDF统一投资决策框架

**实施：** 新建 `unified_investment_decision_framework.py`，与UPF互补：
- 定义AI解读层的标准输入接口 `AIInterpretInput`
- 定义标准输出格式 `AIInterpretOutput（含操作计划/入场区间/止损/目标/风险提示）`
- 提供 `UIDFEngine.consolidate(upf_result, ai_output) → DecisionResult`

### #40 — 状态依赖因子合成

**实施：** 新建 `state_dependent_factor_weight.py`：
- 输入：`market_state` (P1-#25已实现) + 各策略信号
- 权重矩阵：

| 市场状态 | 动量因子 | 反转因子 | 情绪因子 | 筹码因子 |
|---------|---------|---------|---------|---------|
| TRENDING_BULL | 0.35 | 0.15 | 0.25 | 0.25 |
| TRENDING_BEAR | 0.10 | 0.40 | 0.20 | 0.30 |
| RANGING | 0.20 | 0.30 | 0.20 | 0.30 |
| HIGH_VOL | 0.20 | 0.20 | 0.40 | 0.20 |

- 输出：按权重重新计算的合成评分 + 置信度

### #43 — 量价窗口自适应

**实施：** `volume_price_strategy.py` VolumePriceRelationAnalyzer 新增 `_get_adaptive_windows(analysis_context) → Dict`：
- 根据 `market_state = TRENDING_BULL → lookback=15`（牛市短窗口，形态快）
- `TRENDING_BEAR → lookback=30`（熊市长窗口，需更多数据确认）
- `RANGING → lookback=20`（震荡市标准窗口）
- `HIGH_VOL → lookback=15`（高波动缩短窗口降低噪音）

### #45 — 参数平原网格搜索验证

**实施：** `optimizer.py` 新增预置网格定义：
- 缠论: `{'lookback_period': [60, 80, 100, 120, 140, 160, 180]}`
- 量价: `{'lookback_period': [60, 80, 100, 120, 140, 160, 180]}`
- 筹码: `{'lookback_period': [40, 60, 80, 100, 120]}`
- 因子: `{'lookback_period': [14, 20, 30, 40]}`

新增 `get_preset_grid(name) → dict` 方法 + `run_grid_search(name, objective_function)` 快捷入口。

### #46 — 筹码120根扩展模式

**实施：** `chip_strategy.py` 修复：
- `ChipScorer.score()`: `len(data) < 60` → `len(data) < self.lookback_period or 120`
- `ChipAlphaModel.generate_insights()`: `len(df) < 60` → `len(df) < self.lookback_period or 120`
- 新增 `get_available_windows() → [60, 120]` 方法
