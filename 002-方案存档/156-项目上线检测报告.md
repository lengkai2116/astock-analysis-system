### 3.3 异常页面诊断

#### 回测系统 (`/backtest`)
**错误：** `ReferenceError: useAppStore is not defined`
**原因：** 第 285、286、316 行调用 `useAppStore()`，但 `script` 顶部**缺少 `import { useAppStore } from '@/stores'`**
**补充：** 第 44 行 `formatter="¥ 10000"` 在 AntDV 4 中必须是函数而非字符串，会触发 `e.formatter is not a function`

#### 因子管理 (`/factor-manager`)
**错误：** `TypeError: Cannot convert undefined or null to object` → `Object.keys(null)`
**原因：** 第 294 行 `...mapState(useAppStore, ['factors'])`，当 `useAppStore().factors` 初始值为 `null` 时，Pinia/Vue 内部对状态的 `Object.keys()` 调用失败
**根因：** `stores/app.js` 中 `factors` 初始状态设为 `{ all: [], loading: false, error: null }`，但 `mapState` 解构时可能尝试枚举 `factors` 对象本身而非其子属性

#### 报告中心 (`/reports-center`)
**错误：** `TypeError: Cannot read properties of undefined (reading 'createForm')`
**原因：** 第 181 行 `this.$form.createForm(this)` — Ant Design Vue 4 **移除了** `this.$form` API，表单应使用 `<a-form>` 组件或 `Form.useForm()` Composition API 创建

#### 账户管理 (`/account`)
**错误：** 显示 ErrorBoundary "页面渲染异常"，但独立导航时无控制台错误
**原因：** ErrorBoundary 包裹 `<router-view>` 但**未在路由切换时重置 `hasError` 状态**。当上一个页面（报告中心）触发错误后，ErrorBoundary 保持错误状态，即使当前页面渲染正常也继续显示降级 UI
