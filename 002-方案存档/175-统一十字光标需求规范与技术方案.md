## 第一部分：需求规范

以下为本次十字光标改造的全部具体需求，来源于多次沟通确认。

### 需求一：单一十字光标

- 主图和所有副图**作为一个整体**看待
- 只建立一个完整的十字光标，而非在主图和副图分别建立再对齐
- 通过「两步走」逻辑实现：先定位日期画竖线，再查收盘价画横线

### 需求二：十字光标构成

| 元素 | 行为 | 范围 |
|------|------|------|
| **竖线** | 贯穿主图及所有副图 | 从第一个grid顶部到最后一个grid底部 |
| **横线** | 仅在主图区域显示 | 限制在主图grid宽度内 |
| **横线位置** | 与当日K线收盘价精确平齐 | 通过 `convertToPixel([dataIndex, closePrice])` 计算 |
| **层级** | 处于屏幕显示的最上层 | z-index = 100 / 101，其他图层不可遮盖 |

### 需求三：鼠标定位逻辑

鼠标在主图及副图的**任意位置**，都按以下规则定位：

1. 根据鼠标所在位置的X坐标 → 通过 `chart.convertFromPixel()` 转换为对应日期（dataIndex）
2. 竖线定位到该日期位置
3. 根据该日期 → 查找对应的K线收盘价（`closes[dataIndex]`）
4. 横线定位到该收盘价位置

> 特别说明：横线不跟随鼠标Y轴位置，始终锁定到收盘价。这是通过第二步「查收盘价→画横线」实现的。

### 需求四：数值显示

| 数值 | 显示位置 | 格式 |
|------|---------|------|
| **收盘价** | 主图横线最左端（Y轴附近） | 保留小数点后2位（如 `16.92`） |
| **日期** | 竖线与时间轴相交处（X轴下方） | 如 `10/17` |
| **副图指标值** | 副图标题往右拉开12px处 | 通过DOM文字显示（如 `DIF:0.45 DEA:0.53 MACD:-0.16`） |

### 需求五：交互方式

#### 5.1 右键拖拽平移
- 鼠标在主图按住**右键**拖动 → 主图和副图同步左右移动
- 对应时间轴及K线图表整体平移
- 需要阻止浏览器默认右键菜单

#### 5.2 键盘方向键
- 按键盘 → / ← → 调出十字光标
- 控制该唯一光标左右按**天**进行移动
- 当光标移动到图表边缘且继续同向按键时 → 图表进行对应方向的dataZoom平移
- 主图和副图同步平移

#### 5.3 鼠标移出清除
- 鼠标移出主图和副图范围时 → 十字光标线消失
- 不得遗留在屏幕上
- 鼠标控制十字光标的范围限制在主图和副图展示范围内

### 需求六：初始状态

- 页面首次加载 → 无十字光标
- 十字光标仅在用户通过鼠标或键盘操作时才出现

## 第二部分：技术方案

### 一、架构变更

**旧架构**：N个独立ECharts实例（主图1 + 副图N），各自axisPointer + 事件同步

**新架构**：1个ECharts实例，多grid布局，graphic全渲染十字光标

```
旧方案问题:
├─ axisPointer横线跟随鼠标Y → 无法Snap到收盘价
├─ N个实例状态不一致 → 十字光标残留
├─ 同步逻辑复杂 → _isSyncing防环、互派showTip
├─ 多个mouseleave → 难以彻底清除

新方案优势:
├─ graphic line精确Snap到收盘价
├─ 唯一数据源+唯一渲染 → 零残留风险
├─ 一个函数updateCrosshair(idx)统一入口
├─ chart-wrap单一mouseleave → 干净彻底
```

### 二、渲染流程

```
输入（鼠标位置 / 键盘 → 方向）
    │
    ▼
确定 dataIndex (0 ~ 749)
    │
    ├──► 第1步：竖线
    │      chart.convertToPixel({xAxisIndex:0}, dataIndex) → X坐标
    │      getGridRects() → 获取所有grid的Y范围
    │      绘制竖线 from grid[0].top → grid[N].bottom
    │      绘制日期标签在竖线底部
    │
    ├──► 第2步：横线
    │      closes[dataIndex] → 获取收盘价
    │      chart.convertToPixel([dataIndex, closePrice]) → Y坐标
    │      绘制横线（仅主图grid范围）
    │      绘制收盘价标签在横线左端
    │
    ▼
一次性 chart.setOption({graphic: [竖线, 横线, 价格标签, 日期标签]})
                              ↑ 同一帧内完成，整体出现
```

### 三、关键函数 `updateCrosshair(idx)`

```javascript
function updateCrosshair(idx) {
  // ── 隐藏分支 ──
  if (idx < 0 || idx >= dates.length) {
    chart.setOption({graphic: [全部invisible:true]});
    return;
  }

  _cursorIdx = idx;
  var closePrice = closes[idx];

  // dataZoom范围外 → 自动滚动使其可见
  var x = chart.convertToPixel({xAxisIndex:0}, idx);
  if (x == null) { 自动计算并滚动dataZoom到包含idx }

  // 收盘价Y坐标
  var y = chart.convertToPixel({xAxisIndex:0, yAxisIndex:0}, [idx, closePrice])[1];

  // 获取所有grid像素范围
  for (var g = 0; ; g++) {
    var rect = chart.getModel().getComponent('grid', g)?.getRect();
    if (!rect) break;
    gridRects.push(rect);
  }

  var graphics = [
    // ① 竖线：贯穿第一个→最后一个grid
    {id:'cv', type:'line', shape: {x1:x, y1:第一grid.y, x2:x, y2:最后grid.y+最后grid.height}},
    // ② 横线：仅主图grid
    {id:'ch', type:'line', shape: {x1:主图grid.x, y1:y, x2:主图grid.x+主图grid.width, y2:y}},
    // ③ 收盘价标签：横线左端
    {id:'cpl', type:'text', text: closePrice.toFixed(2), 红色加粗, textAlign:'right'},
    // ④ 日期标签：竖线底部
    {id:'cdl', type:'text', text: dates[idx], 红色, 加粗},
  ];

  chart.setOption({graphic: graphics}, {replaceMerge: ['graphic']});
  updateSubHeaders(idx);  // 更新副图DOM文字
}
```

### 四、事件映射

| 需求 | 实现 |
|------|------|
| **鼠标移动→十字光标** | `chart.getZr().on('mousemove')` → convertFromPixel → updateCrosshair |
| **键盘→/←→逐日移动** | `document.addEventListener('keydown')` → _cursorIdx ± 1 → updateCrosshair |
| **键盘到边缘→滚动** | 边界检测 → dataZoom平移30% → 光标跳转到对侧 |
| **右键拖拽** | zr mousedown(e.which===3) → mousemove → dataZoom dispatch |
| **鼠标离开→消失** | chart-wrap mouseleave → updateCrosshair(-1) |
| **初始无光标** | _cursorIdx = -1，rebuildChart不调用updateCrosshair |

### 五、副图配置

- 默认副图：**成交额** + **VOL成交量**（`activeSubs = ['成交额','vol']`）
- 副图渲染：只创建DOM header（标题 + 数值文字 + 切换按钮），**不创建独立ECharts实例**
- 所有副图数据作为series并入主chart，通过gridIndex分组
- 每个副图grid高度：116px（128px容器 − 12px上下边距）
- 副图数值显示：标题右侧 `margin-left: 12px` 的 DOM `.sub-val` 元素

### 六、多grid布局

| grid | 内容 | top | height |
|------|------|-----|--------|
| grid[0] | 主图K线 + MA/BOLL等叠加 | 10px | 325px |
| grid[1] | 副图1（成交额） | 360px | 116px |
| grid[2] | 副图2（VOL） | 504px | 116px |

- 所有grid共享X轴数据（`xAxisIndex`关联）
- 单一dataZoom作用所有grid
- 竖线X坐标统一使用 `xAxisIndex:0` 转换
