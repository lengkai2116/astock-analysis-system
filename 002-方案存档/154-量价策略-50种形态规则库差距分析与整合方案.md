## 2. 当前实现 vs 规则库：逐形态差距

### 2.1 当前实现了什么

当前 `volume_price_strategy.py` 的形态检测体系分两层：

```
第1层: 9种基础量价分类 (VP-1~VP-9)
  └─ _classify_vp_pattern(): 近4日价格方向 × 近7日量能方向

第2层: 5种增强形态 (EnhancedPatternDetector)
  ├─ 冲高回落急速缩量 (预涨) 
  ├─ 递增式放量下跌 (预跌)
  ├─ 堆量中跌回启动点 (预涨/黑马)
  ├─ 三连阳放量 (预涨)
  └─ 放量长阴破位 (预跌)
```

## 3. 现有检测逻辑 vs 规则库条目：逐条差异对比

### 3.1 "冲高回落"类形态冲突

这是最严重的**方向性冲突**：

| 对比项 | 当前 `_is_chonggao_huiluo_jisuo` | 规则库 "放量冲高回落" (预跌 2-4) |
|--------|----------------------------------|----------------------------------|
| 方向 | 预涨型（看多） | 预跌型（看空） |
| 价格条件 | `highs[-2] / closes[-3] > 1.03` | 股价创近20日新高 |
| 回落判定 | `pullback > 2%`（相对高点） | 收盘价 < 日内最高价 × 0.8 |
| 量能条件 | `volumes[-1] < volumes[-2] × 0.5` | 成交量 > 前20日均量 × 2.5 |
| 影线条件 | 无 | 上影线 > 实体长度 × 2 |
| 确认条件 | 无 | 次日收盘 < 前日收盘价 |

**结论**：当前检测的是"前一天大涨后今天缩量"，规则库检测的是"高位放量上影线"——完全不同的事件。

### 3.2 "三连阳放量" vs "红三兵"

| 对比项 | 当前 `_is_sanlian_yang_fangliang` | 规则库 "红三兵" (预涨 1-16) |
|--------|----------------------------------|-----------------------------|
| K线数量 | 3 根 | ≥ 3 根（一般为 5 根） |
| 开盘条件 | `closes[i] > opens[i]` | 每日开盘价 ≥ 前日收盘价（逐日高开） |
| 实体条件 | 无 | 每日实体长度递增 |
| 量能条件 | `volumes[i] > volumes[i-1]` | 逐日温和放大，但不超过前均量×2 |

**结论**：当前只检查连续3日阳线+量递增；规则库要求更严格（逐日高开+实体递增）。

## 4. 架构方案评估

### 方案 A：规则替换（激进）

用 50 种 IF-THEN 规则引擎替换当前的 9+5 检测体系。

**优点**：
- 直接对齐知识库
- 检测精度大幅提升
- 信号权重分级更合理

**风险**：
- 45 个新规则实现工作量 3-5 天
- 需要设计统一的规则注册和调度框架
- 7 种依赖筹码分布的规则需要等待筹码模块完成
- 需完整的回归测试覆盖

### 方案 B：增量扩展（推荐）

在现有架构基础上，按类别分批新增规则。

**实施路线**：

| 批 | 形态 | 数量 | 工作量 | 依赖 |
|----|------|------|--------|------|
| 1 | 独立量价规则（不依赖MA/形态/筹码） | ~12 | 1天 | 无 |
| 2 | 均线辅助规则 | ~8 | 0.5天 | 均线计算 |
| 3 | 形态识别规则（头肩/旗形/三重底等） | ~8 | 2天 | 形态识别器（152号Phase 0成果） |
| 4 | 多日模式规则（堆量/凹量/周线等） | ~8 | 1天 | 无 |
| 5 | 筹码分布联动规则 | 7 | 1天 | 筹码模块 |
| 6 | 黑马型特殊规则 | 7 | 1天 | 前序全部 |

### 方案 C：保持现状（不推荐）

当前 14 种形态与 50 种规则库差距过大，保留现状意味着知识库成果未被利用。
