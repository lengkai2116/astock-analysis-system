# 筹码分布主力策略模型可行性评估

评估对象：`/Users/kalence/Downloads/筹码分布揭密主力操盘手法 (淘金客) (z-library.sk, 1lib.sk, z-lib.sk).epub`

评估日期：2026-05-25

## 结论

可行，但应按“估算筹码分布 + 阶段识别 + 风控输出”的辅助决策模型落地，不应按“真实主力持仓识别器”宣传或使用。

本书方法可以整理为系统策略模型，核心是把筹码峰、筹码集中度、ASR、CYQKL、SSRP、筹码转移率、量价异动组合成主力阶段识别器，用于识别建仓、洗盘、拉升、出货和下跌支撑五类状态。当前系统已有日线数据缓存、技术指标、策略模板、策略输出和回测框架，能承载第一版模型。主要缺口是没有真实筹码分布数据、没有流通股本和换手率缓存、没有筹码分布快照表，必须先实现可回测的估算模型。

## 书中方法可模型化的内容

原书结构适合抽象为五层策略模型：

1. 成本分布层：用筹码分布近似投资者持仓成本，识别低位主峰、高位主峰、双峰、历史峰和筹码发散。
2. 指标层：计算 ASR、CYQKL、SSRP、筹码获利率、筹码集中度、筹码转移率。
3. 阶段识别层：按建仓、洗盘、拉升、出货、下跌支撑分别打分。
4. 信号层：输出买入观察、买入、持有、减仓、卖出、风险预警。
5. 风控层：结合大盘、流动性、涨跌停、回撤和跌破筹码峰等条件控制仓位。

## 建议策略模型

模型名称：筹码分布主力阶段识别模型

模型类型：选股 + 择时 + 风险预警，优先作为 `selection` 策略模板和策略输出服务，不建议第一版直接做自动交易。

输入数据：

- 必需：日线 OHLCV、成交额、涨跌幅。
- 强烈建议补充：流通股本、换手率、复权因子、行业/板块、指数走势。
- 增强数据：资金流向、分钟线、龙虎榜、Level-2 大单数据。

核心指标：

| 指标 | 作用 | 第一版可实现性 |
| --- | --- | --- |
| 筹码分布 | 估算不同价格区间的持仓成本 | 可用 OHLCV 估算 |
| 筹码峰 | 支撑、压力、主力成本区 | 可实现 |
| 筹码集中度 | 判断筹码是否从分散转为集中 | 可实现 |
| ASR | 衡量当前价附近浮筹压力 | 可实现 |
| CYQKL | 衡量 K 线穿越筹码区的强度 | 可实现 |
| SSRP | 市场平均成本线 | 可实现 |
| 筹码获利率 | 判断获利盘/套牢盘比例 | 可实现 |
| 筹码转移率 | 判断筹码是否快速上移或下移 | 可实现 |

阶段判定建议：

| 阶段 | 主要特征 | 输出建议 |
| --- | --- | --- |
| 建仓期 | 低位筹码单峰、集中度上升、温和放量、CYQKL 突破 | 观察或小仓试探 |
| 洗盘期 | 回调至筹码峰附近、缩量、获利率下降但支撑未破 | 观察或等待企稳 |
| 拉升期 | 价格站上 SSRP、低位主峰仍保留、筹码滚动上移、量价配合 | 持有或趋势跟随 |
| 出货期 | 高位筹码峰扩大、低位筹码明显减少、跌破高位主峰、ASR 高位失效 | 减仓或卖出 |
| 下跌支撑 | 价格回落至历史/低位筹码峰，配合超跌和企稳信号 | 仅短线反弹观察 |

## 当前系统适配度

当前系统具备较好的落地基础：

- 数据层已有 DuckDB 日线缓存，当前 `daily_cache` 约 611 万条记录。
- 后端已有 `DataManager`、`TushareProvider`、`EnhancedCacheManager`，可以扩展筹码指标缓存。
- 因子框架已有 `BaseFactor`，适合新增筹码类因子。
- 策略层已有 `StrategyOutput`、`StrategyTemplateV2`、策略模板 API，可承载信号输出。
- 回测层已有 A 股增强回测引擎，支持 T+1、涨跌停、手续费、滑点。
- `backend/app/engine/pipeline.py` 中已经预留 `ChipDistributionStrategy`，但目前没有实际逻辑。

主要缺口：

- 没有 `chip_distribution` 表或筹码快照缓存。
- 没有已落库的流通股本、换手率、复权因子。
- `TushareProvider` 没有实现 `daily_basic` 方法，虽然部分代码已有调用痕迹。
- 指标缓存为空，说明筹码类指标需要新增预计算任务。
- 分时和大单数据只能作为增强项，不能作为第一版硬依赖。

## 可行性分级

第一版可行性：高。

用历史日线 OHLCV 估算筹码分布，先实现全市场选股和单票阶段识别。该版本可回测、可展示、可迭代，但信号精度依赖估算算法。

增强版可行性：中。

补充换手率、流通股本、复权因子、资金流向、分钟线后，可以提高筹码衰减和筹码迁移估算质量，阶段识别也会更稳定。

高精度版可行性：中低。

如果要求接近商业软件筹码分布或真实持仓成本，需要采购筹码分布、Level-2 或更完整的逐笔成交数据。A 股真实投资者持仓成本并不公开，任何自建模型都只能是估算。

## 推荐落地方案

阶段一：筹码估算与指标因子

- 新增 `chip_distribution` 缓存表，按 `ts_code + trade_date + price_bin` 存储筹码比例。
- 新增筹码估算服务：基于 OHLCV、价格区间、成交量、换手率和衰减参数生成每日筹码分布。
- 新增筹码因子：ASR、CYQKL、SSRP、筹码集中度、筹码获利率、筹码转移率、筹码峰位置。

阶段二：主力阶段识别

- 新增 `chip_stage_service.py`，输出阶段、分数、证据、风险项。
- 建仓、洗盘、拉升、出货、下跌支撑分别建独立评分器。
- 将输出写入 `strategy_outputs`，复用现有信号展示结构。

阶段三：回测和参数校准

- 在 A 股增强回测引擎中接入阶段信号。
- 以沪深主板、创业板、科创板分别评估。
- 重点验证收益回撤比、胜率、持有期、换手率、极端行情失效场景。

阶段四：前端展示

- K 线图叠加筹码峰、SSRP、阶段标签。
- 策略模板页新增“筹码分布主力阶段识别模型”。
- 报告中心输出信号证据和风险原因。

## 风险与限制

1. 筹码分布不是公开真实持仓数据，自建结果本质是估算。
2. 原书案例偏技术分析和图形经验，不能直接等同于量化有效性。
3. 参数阈值必须经回测校准，不建议直接固定使用书中经验阈值。
4. 高位出货、低位建仓等标签容易产生事后解释偏差，必须避免未来函数。
5. 小盘股、停牌、连续涨跌停、除权除息会显著影响筹码估算。

## 最小可用实现范围

建议第一版只做日线级策略：

- 价格网格：按历史价格区间切分 100 到 200 个 price bin。
- 成交分配：将当日成交量按 OHLC 区间分布到 price bin。
- 衰减模型：用换手率或固定衰减率模拟旧筹码被新成交替换。
- 指标输出：ASR、CYQKL、SSRP、profit_rate、concentration、peak_price、transfer_rate。
- 策略信号：只输出 WATCH、BULLISH、BEARISH、NEUTRAL，不自动下单。

## 总体建议

建议推进，但采用分阶段落地：先做“可解释、可回测、可展示”的筹码估算版，再逐步接入更高质量的数据源。当前系统不需要重构即可承载该模型，主要工作集中在新增数据表、筹码指标计算、阶段识别服务和前端图表展示。

该模型最适合定位为“主力行为辅助判断 + 策略研究工具”，而不是独立交易圣杯。只要在产品中明确估算属性、回测边界和风险提示，工程上具备落地价值。
