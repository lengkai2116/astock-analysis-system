# Level2 数据权限必要性评估

评估日期：2026-05-25

## 结论

当前阶段 **不必须具备 Level2 数据权限**。

本系统现阶段定位是 A 股分析决策支持系统，核心功能包括日线/分钟线行情展示、技术指标、因子分析、策略模板、回测、报告中心、自选监控、筹码分布估算和主力阶段辅助识别。这些功能用日线 OHLCV、复权因子、换手率、资金流向、分钟线和普通实时快照即可完成。

Level2 数据应定位为 **增强项**，不是 MVP 或第一阶段上线的硬依赖。只有当系统明确进入盘口微观结构分析、高频短线策略、逐笔成交/逐笔委托分析、撤单率识别、盘口不平衡因子、大单主动买卖跟踪时，Level2 才变成强需求。

## 当前系统功能对 Level2 的依赖

| 功能模块 | 是否必须 Level2 | 说明 |
| --- | --- | --- |
| 股票列表、基础资料、行业信息 | 否 | 基础数据源即可 |
| 日线 K 线、技术指标 MA/MACD/RSI/KDJ/BOLL | 否 | OHLCV 足够 |
| 因子库与多因子筛选 | 否 | 当前因子主要基于日线/技术/量价 |
| 策略模板与策略输出 | 否 | 当前是研究和信号输出，不是盘口交易 |
| A 股增强回测 | 否 | 当前回测粒度是日线/信号级 |
| 报告中心、AI 分析 | 否 | 以结构化行情、指标和策略结果为主 |
| 自选股实时监控 | 否 | Level1 快照或 QMT 普通行情足够 |
| 实时价格推送 | 否 | QMT/miniQMT 快照、五档或网页快照可满足展示 |
| 筹码分布估算 | 否 | 第一版可用 OHLCV + 换手率估算 |
| 主力阶段识别 | 否 | 第一版以筹码估算、量价、资金流向为主 |
| 大单净流入/主力资金 | 部分需要 | 可先用 Tushare moneyflow 等日级资金流替代 |
| 盘口不平衡、撤单率、钓鱼单识别 | 是 | 需要逐笔委托/成交或深度盘口 |
| 高频分钟内策略 | 强烈需要 | 对盘口、tick、延迟和滑点敏感 |
| 自动交易执行优化 | 视目标而定 | 普通信号交易不必需，盘口拆单才需要 |

## 为什么当前不建议把 Level2 作为硬依赖

1. 成本和授权复杂度高。

Level2 是交易所或授权渠道提供的深度行情，商业合规、授权范围、终端账号和数据落库都有额外限制。当前系统还在功能建设期，把它作为前置依赖会增加启动成本。

2. 当前策略粒度不匹配。

系统现有能力集中在日线、分钟线、因子、回测、策略报告和自选监控。Level2 的价值主要体现在 tick、盘口队列、逐笔委托成交、撤单、盘口压单/托单分析，这些不是当前主要策略闭环。

3. 筹码分布不等于 Level2。

筹码分布模型第一版可以用历史成交量、价格区间、换手率做估算。Level2 可以提高短周期资金行为识别，但不能直接提供真实投资者持仓成本。真实持仓结构本身并不公开。

4. 当前工程更需要先补基础数据。

比 Level2 更优先的是：

- 复权因子
- daily_basic：换手率、流通股本、市值
- 稳定实时快照
- 分钟线
- 资金流向
- 数据源状态和降级机制

## 什么时候 Level2 会变成必须

如果后续明确要做以下功能，Level2 就应纳入采购或券商权限申请：

- 逐笔成交和逐笔委托回放。
- 主动买入/主动卖出成交强度。
- 大单拆单、扫单、压单、托单识别。
- 撤单率、挂单生命周期、虚假挂单识别。
- 前五档/十档盘口不平衡因子。
- 竞价撮合阶段微观结构策略。
- 1 分钟以内的超短线或高频策略。
- 实盘自动交易中的盘口择价、排队、滑点控制。

若没有这些目标，Level2 对当前系统的边际收益不如把日线、分钟线、QMT 实时快照、Tushare daily_basic 和资金流向先做扎实。

## 推荐策略

### 当前阶段

不采购独立 Level2 数据权限。优先实现：

- QMT/miniQMT 普通实时行情接入。
- Tushare 日线、复权、daily_basic、moneyflow。
- AkShare/efinance 作为免费降级源。
- 前端展示当前行情来源和是否实时。
- 实时服务 provider 抽象，支持未来插入 Level2 provider。

### 中期阶段

保留 Level2 能力开关：

```text
REALTIME_PROVIDER=qmt
ENABLE_LEVEL2=false
LEVEL2_PROVIDER=
```

在数据模型中预留：

- bid/ask 五档或十档
- tick 序列
- 主动买卖方向
- order imbalance
- large order flow
- cancel ratio

但默认不启用，不影响系统启动。

### 后期阶段

当普通实时行情和日线策略闭环稳定后，再用小范围试点评估 Level2：

- 只选 50 到 100 只高流动性股票。
- 只保存聚合特征，不保存全量逐笔原始数据。
- 做盘口因子的离线验证，确认是否提升胜率、收益回撤比和信号提前量。
- 如果提升不明显，不扩大采购和存储规模。

## 最终判断

当前系统 **不必须具备 Level2 数据权限**。

最合理路径是：

```text
第一优先级：日线/复权/daily_basic/资金流 + QMT普通实时行情
第二优先级：分钟线、实时快照、五档盘口展示
第三优先级：Level2 逐笔/深度盘口增强
```

也就是说，Level2 是“未来做盘口主力行为和高频短线策略时的增强能力”，不是当前系统启动、选股、回测、筹码估算、报告和自选监控的必要条件。
