## 一、统一策略输出格式规范

所有策略引擎（量价/缠论/筹码/因子）的输出遵循以下统一结构：

### 1.1 信号结构

```json
{
  "signal_id": "uuid",
  "ts_code": "000001.SZ",
  "signal_date": "2026-06-10",
  "strategy_type": "volume_price | chanlun | chip | factor",
  "direction": "buy | sell | hold",
  "confidence": 0.0-1.0,
  "summary": "单行中文摘要",
  "detail": { /* 策略专属详细字段 */ },
  "metadata": {
    "model_version": "deepseek-chat-v4-flash",
    "computed_at": "2026-06-10T10:00:00"
  }
}
```

### 1.2 策略类型标识

| 策略 | type 值 | detail 主字段 |
|------|---------|--------------|
| 量价形态 | `volume_price` | patterns[], resonance_score |
| 缠论 | `chanlun` | bi[], duan[], zhongshu[], maidian[] |
| 筹码分布 | `chip` | chip_concentration, main_track |
| 因子评分 | `factor` | factor_scores[], composite_score |
| 共振信号 | `resonance` | contributing_strategies[], agreement |

### 1.3 信号总线对接

所有信号写入 `signal_records` 表，通过 WebSocket 广播至前端 AiSignalBus。

## 二、综合分析报告模板

### 2.1 结构

1. **概览** — 股票代码、名称、分析日期、总评分（0-100）
2. **技术面** — 趋势判断 + 关键支撑/阻力位 + 量价形态匹配
3. **缠论分析** — 当前笔/段/中枢状态 + 买卖点判定
4. **筹码分析** — 主力动向 + 集中度变化
5. **资金流** — 主力/散户净流向
6. **多策略共振** — 各策略信号一致度 + 置信度
7. **结论** — 综合建议 + 风险提示
8. **免责声明**

### 2.2 输出格式

- DeepSeek 生成的结构化 JSON（见 deepseek_analysis_service.py）
- 前端渲染为报告中心页面

## 三、量化评分标准

### 3.1 信号评分

| 分数区间 | 评级 | 含义 |
|---------|------|------|
| 80-100 | A | 高置信度，多策略共振确认 |
| 60-79  | B | 中等置信度，主策略明确 |
| 40-59  | C | 低置信度，信号冲突 |
| 0-39   | D | 无明确信号或数据不足 |

### 3.2 评分权重

| 维度 | 权重 | 说明 |
|------|------|------|
| 形态匹配度 | 25% | 量价规则匹配数量/质量 |
| 缠论信号 | 20% | 买卖点类型 + 中枢位置 |
| 筹码分布 | 15% | 主力集中度趋势 |
| 多策略共振 | 25% | 策略间一致性 |
| 环境过滤 | 15% | 大盘/板块环境 |

### 3.3 回测验证

- 所有信号需经历 T+5/T+10/T+20 验证
- 验证结果回写 `signal_records.verification_status`
- 每周输出策略健康度报告（StrategyHealthMonitor）
