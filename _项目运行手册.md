## 三、环境变量与 API 密钥配置

### 3.1 文件加载规则

```
.env          → Flask load_dotenv() 自动加载（无 Docker 主模式）
.env.local    → 本地开发覆盖，Flask 不会自动加载，仅 gitignore 中的手动配置
.env.example  → 模板参考，不会被代码读取
```

**关键：** `.env.local` 中的值不会被 Flask 或 Docker Compose 读取。所有生产配置写在 `.env` 中即可。

### 3.2 DeepSeek API 切换步骤

当前 `.env` 配置（已就绪）：

```
DEEPSEEK_API_KEY=sk-56efa3f976e9499cb645c3a0b4cdcf4c
DEEPSEEK_BASE_URL=https://api.deepseek.com/v1
```

## 五、部署策略评估与选择

### 5.1 三种方案对比

| 方案 | 适用场景 | 依赖 | 迁移成本 | 推荐度 |
|------|---------|------|---------|-------|
| **A: 裸跑（推荐）** | 本地开发 / 个人单机 | Python 3.10+  | 最低。复制目录 + 装 Python | ⭐⭐⭐⭐⭐ |
| **B: Docker** | 云服务器 / 需要环境隔离 | Docker Desktop | 中等。需解决 Docker Hub 网络问题 | ⭐⭐⭐ |
| **C: WSL + 裸跑** | Windows 上需要 Linux 兼容性 | WSL2 + Python | 中等。WSL 增加复杂度 | ⭐⭐ |

### 5.2 推荐方案 A：裸跑（无 Docker）

**Windows / macOS / Linux 通用**，只需 Python 3.10+。

#### 一键启动

```bash
# Windows
双击 start.bat

# macOS
双击 start.command    # 或命令行: bash start.command

# Linux
bash start.sh
```

自动完成：
1. 检查 Python 版本
2. 首次自动创建虚拟环境 + 安装依赖
3. 首次自动构建前端
4. 启动后端服务（端口 5001）
5. 打开浏览器

#### 手动启动（分段执行）

```bash
# 1. 构建前端（有 Node.js 时执行，仅首次或代码更新后）
cd frontend/vue-project
npm install
npm run build

# 2. 安装后端依赖（仅首次）
cd backend
python -m venv .venv
source .venv/bin/activate    # Windows: .venv\Scripts\activate
pip install -r requirements.txt

# 3. 启动
cd backend
python run.py --port 5001

# 4. 浏览器打开 http://localhost:5001
```

> **为什么端口是 5001 而不是 9000？**
> 无 Docker 模式下 Flask 直接托管前端 SPA 静态文件，不再需要 Nginx 反向代理。
> 只有一个进程、一个端口，简洁可靠。

#### 无 Docker 模式下的组件说明

| 组件 | 说明 |
|------|------|
| **数据库** | SQLite (`test.db`)，零配置，随代码一起复制 |
| **缓存** | DuckDB 本地文件，同上 |
| **Redis** | **不依赖**，缓存由 DuckDB 承担 |
| **前端** | Flask 内置托管 `frontend/vue-project/dist/` |
| **WebSocket** | Flask-SocketIO，原生支持 |

#### 停止

```bash
# Windows
双击 stop.bat

# macOS / Linux
./stop.sh
```

### 5.3 方案 B：Docker 部署（可选）

适用于云服务器、需要 PostgreSQL + Redis 的环境。

#### 前置条件
- Docker Desktop（需自行解决 Docker Hub 可访问性）
- 国内用户可能需要代理或镜像加速

#### 构建与启动

```bash
# Docker Hub 不可达时需先配置镜像加速（~/.docker/daemon.json）：
# {"registry-mirrors": ["https://docker.1ms.run", "https://docker.xuanyuan.me"]}

# 构建（国内用户加 pypi 镜像）
docker compose build --build-arg PIP_INDEX_URL=https://pypi.tuna.tsinghua.edu.cn/simple/

# 启动
docker compose up -d

# 访问 http://localhost:9000
```

> **Docker vs 裸跑对比**：Docker 提供 PostgreSQL + Redis + 环境隔离，但增加了网络依赖和复杂度。
> 对于个人分析系统，裸跑的 SQLite + DuckDB 完全够用，建议优先选方案 A。

### 5.4 设备迁移步骤

**携带全部数据复制到新机器：** 只需复制整个项目目录（含 `data/`、`frontend/vue-project/dist/`），新机器上安装 Python 3.10+ 后双击 `start.bat` 即可。

关键文件清单：

| 文件/目录 | 重要性 | 说明 |
|-----------|--------|------|
| `backend/` | 必需 | 后端全部代码 |
| `frontend/vue-project/dist/` | 必需 | 已构建的前端（丢失则重新 `npm run build`）|
| `data/` | 必需 | DuckDB 缓存 + SQLite 数据库 |
| `.env` | 必需 | 环境变量含 API Key |
| `002-方案存档/` | 可选 | 方案文档 |

## 十、日志管理

```
日志位置 (Docker 内):
  /app/logs/                     # 后端日志
  /var/log/nginx/                # Nginx 日志 (Docker 外映射)
  
日志轮转:
  后端: TimedRotatingFileHandler, 每日轮转, 保留 30 天
  Nginx: Docker 日志驱动自动管理

关键关键词:
  ERROR     - 系统错误, 需关注
  WARNING   - 资源不足或降级
  TRIGGERED - 策略触发事件
  SWITCH    - 数据源切换
```
