# A股分析系统 — 运维手册

> 版本: 1.0 | 更新日期: 2026-06-09

---

## 一、系统架构

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│   Nginx      │────▶│   Gunicorn   │────▶│  PostgreSQL  │
│   (前端静态)  │     │  (Flask API) │     │  (主数据库)   │
│   local:9000 │     │  local:5001  │     │  local:5432  │
└──────────────┘     └──────┬───────┘     └──────────────┘
                            │                     │
                            ▼                     ▼
                     ┌──────────────┐     ┌──────────────┐
                     │   Redis 7    │     │   DuckDB     │
                     │  (缓存/会话) │     │  (OLAP缓存)   │
                     └──────────────┘     └──────────────┘
```

## 二、启动与停止

```bash
# 启动
./start.sh

# 停止 (优雅关闭, 等待请求完成)
./stop.sh

# 查看日志
docker compose logs -f          # 全部
docker compose logs -f backend  # 仅后端
docker compose logs -f frontend # 仅前端
```

## 三、健康检查

| 端点 | 用途 | 预期响应 |
|------|------|---------|
| `/api/v3/health` | 综合健康检查 | `{"success":true,"status":"healthy"}` |
| `/api/v3/health/live` | 进程存活 (liveness) | `{"success":true,"status":"alive"}` |
| `/api/v3/health/ready` | 依赖就绪 (readiness) | `{"success":true,"status":"ready","dependencies":{...}}` |

## 四、数据备份

```bash
# 手动备份到默认目录
./scripts/backup.sh

# 手动备份到指定目录
./scripts/backup.sh /path/to/backup

# 定时备份 (添加 crontab)
crontab scripts/crontab.txt
```

备份内容:
- PostgreSQL: `pg_dump` 完整导出
- DuckDB: 缓存文件快照
- .env: 配置文件副本

## 五、常见故障排查

### 5.1 前端页面白屏

```bash
# 检查前端容器
docker compose logs frontend | tail -20
# 检查 Nginx 是否正常
curl -sI http://localhost:9000 | head -5
# 检查 API 代理
curl -s http://localhost:9000/api/v3/health
```

### 5.2 后端 API 超时

```bash
# 检查后端日志
docker compose logs backend | grep ERROR | tail -20
# 检查数据库连接
curl -s http://localhost:5001/api/v3/health/database
# 检查 Redis
docker compose exec redis redis-cli ping
```

### 5.3 数据不更新

```bash
# 检查数据源状态
curl -s http://localhost:5001/api/v3/data-source/status
# 强制同步
curl -s -X POST http://localhost:5001/api/v3/data/sync
# 检查缓存状态
curl -s http://localhost:5001/api/v3/cache/stats
```

### 5.4 数据库问题

```bash
# 检查 PostgreSQL
docker compose exec postgres pg_isready -U stock_user -d stock_analysis
# 检查表结构
docker compose exec postgres psql -U stock_user -d stock_analysis -c "\dt"
# 修复迁移
docker compose exec backend flask db upgrade
```

## 六、升级流程

```bash
# 1. 备份
./scripts/backup.sh

# 2. 拉取最新代码
git pull

# 3. 重建并启动
docker compose up -d --build --wait

# 4. 执行数据库迁移
docker compose exec backend flask db upgrade

# 5. 验证
curl -s http://localhost:5001/api/v3/health
```

## 七、安全配置

| 配置项 | 位置 | 说明 |
|--------|------|------|
| `AUTH_TOKEN` | `.env` | API 访问令牌 |
| `SECRET_KEY` | `.env` | Flask 密钥 (90天轮换) |
| `TUSHARE_TOKEN` | `.env` | Tushare Pro 数据源令牌 |
| `DEEPSEEK_API_KEY` | `.env` | DeepSeek AI 密钥 |
| CSP 策略 | `nginx.conf` | Content-Security-Policy 头 |

## 八、日志管理

```
日志位置 (Docker 内):
  /app/logs/                     # 后端日志
  /var/log/nginx/                # Nginx 日志 (Docker 外映射)
  
日志轮转:
  后端: 每日轮转, 保留 30 天
  Nginx: Docker 日志驱动自动管理

关键关键词:
  ERROR     - 系统错误, 需关注
  WARNING   - 资源不足或降级
  TRIGGERED - 策略触发事件
  SWITCH    - 数据源切换
```

## 九、恢复操作

```bash
# PostgreSQL 恢复
gunzip -c backups/postgres_20260609_030000.sql.gz | docker compose exec -T postgres \
  psql -U stock_user -d stock_analysis

# DuckDB 恢复
cp backups/duckdb_20260609_030000.duckdb data/stock_cache.duckdb

# 环境变量恢复
cp backups/env_20260609_030000.txt .env
```

## 十、联系与报警

- 所有异常日志包含 `request_id`，可在日志中追溯完整请求链
- 数据源切换时自动记录日志
- 超过 1 个交易日数据未更新时前端展示警告
